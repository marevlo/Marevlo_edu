Scaling Your Data

Raw features live on different scales. Without preprocessing, your clustering algorithm doesn't see patterns — it sees whichever feature has the biggest numbers.

Learn
01 — Why Scaling is Non-Negotiable

The Feature Scale Trap

Consider a dataset with Age (18–80) and Income (₹30,000–₹1,50,000). Without scaling, Euclidean distance between two customers is dominated almost entirely by income — age becomes invisible.

K-Means will form clusters based on income alone, completely ignoring age patterns. This isn't a subtle effect — it's catastrophic.

Rule of thumb: If features have different units (years, rupees, kilograms), you MUST scale before any distance-based clustering. No exceptions.

02 — Standard Scaling (Z-Score)

Center at Zero, Spread to One

Standard Scaling
x_scaled = (x − μ) / σ
Subtract the mean, divide by standard deviation. Result: mean = 0, std = 1 for every feature. Most common preprocessing for clustering.
📐 Math Workout — Standard Scaling by Hand
Problem: Feature values: [10, 20, 30, 40, 50]
Compute the standardized (z-score) version of each value.
Step 1 — Mean
μ = (10+20+30+40+50)/5 = 150/5 = 30
Step 2 — Standard Deviation
σ = √(((10-30)²+(20-30)²+(30-30)²+(40-30)²+(50-30)²)/5)
= √((400+100+0+100+400)/5) = √(1000/5) = √200 ≈ 14.14
Step 3 — Scale each value
(10−30)/14.14 = −1.414
(20−30)/14.14 = −0.707
(30−30)/14.14 = 0.000
(40−30)/14.14 = +0.707
(50−30)/14.14 = +1.414
✓ Mean of scaled: 0.000 ✓ Std of scaled: 1.000
Every feature now has equal weight in distance calculations.

Weakness: Standard scaling uses mean and std — both are sensitive to outliers. If your data has extreme values (e.g., income of ₹5 crore among ₹30K–₹1.5L), the mean shifts and std inflates, compressing all normal values into a tiny range.

03 — Min-Max Scaling

Squeeze Into [0, 1]

Min-Max Scaling
x_scaled = (x − x_min) / (x_max − x_min)
Maps every feature to [0, 1]. The minimum becomes 0, the maximum becomes 1. Simple and interpretable, but extremely sensitive to outliers.
📐 Math Workout — Outlier Destroys Min-Max
Problem: Ages: [22, 25, 28, 30, 35] — Min-Max scale them.
Now add one outlier: [22, 25, 28, 30, 35, 200]. Re-scale.
What happens to the original 5 values?
Without outlier (range 22–35)
(22−22)/(35−22) = 0.000
(25−22)/13 = 0.231
(28−22)/13 = 0.462
(30−22)/13 = 0.615
(35−22)/13 = 1.000
With outlier (range 22–200)
(22−22)/178 = 0.000
(25−22)/178 = 0.017
(28−22)/178 = 0.034
(30−22)/178 = 0.045
(35−22)/178 = 0.073
(200−22)/178 = 1.000
All original values collapsed into [0, 0.073]! The entire meaningful range is now 7.3% of the scale. One outlier destroyed the resolution for 5 out of 6 data points. This is why Min-Max scaling is dangerous with outliers.
04 — Robust Scaling

Outlier-Resistant Preprocessing

Robust Scaling
x_scaled = (x − median) / IQR
median = middle value (robust center), IQR = Q3 − Q1 (robust spread). Outliers barely affect either.
📐 Hard Workout — Comparing All Three Scalers
Problem: Incomes (₹K): [30, 35, 40, 45, 50, 55, 60, 500]
Compute the scaled value of 45 using: (a) Standard Scaling (b) Min-Max (c) Robust Scaling
Which scaler gives 45 the most "central" position?
(a) Standard Scaling
μ = (30+35+40+45+50+55+60+500)/8 = 101.875
σ = √(Σ(xᵢ−μ)²/8) ≈ 153.2
z(45) = (45−101.875)/153.2 = −0.371
45 looks negative (below mean) because the outlier 500 dragged the mean to 101.9!
(b) Min-Max
(45−30)/(500−30) = 15/470 = 0.032
45 is at 3.2% of the range — crushed into the bottom.
(c) Robust Scaling
Sorted: 30,35,40,45,50,55,60,500
Median = (45+50)/2 = 47.5
Q1 = (35+40)/2 = 37.5, Q3 = (55+60)/2 = 57.5
IQR = 57.5−37.5 = 20
(45−47.5)/20 = −0.125
45 is slightly below center — exactly where it should be!
✓ Robust scaling: 45 → −0.125 (correct: slightly below median)
✗ Standard: 45 → −0.371 (distorted by outlier shifting mean)
✗ Min-Max: 45 → 0.032 (crushed into bottom 3%)

Use Robust Scaling when outliers are present and you don't want to remove them.

🎯 Scenario: Hospital Patient Clustering

You're clustering patients by: age (0–100), blood pressure (80–200 mmHg), hospital bill (₹500–₹50,00,000), and number of visits (1–500). A few patients have bills >₹40L (organ transplants).

Think: Standard scaling will let the ₹40L+ bills inflate σ for the bill feature, compressing everyone else. Min-Max will map 99% of patients into [0, 0.1] on the bill axis. Robust scaling uses median and IQR — the transplant patients barely affect the IQR. Best approach: Robust scale first, then optionally clip extreme values (winsorize at 1st/99th percentile) before clustering.
Python — All Three Scalers
from sklearn.preprocessing import StandardScaler, MinMaxScaler, RobustScaler
import numpy as np

X = np.array([[25, 50000], [30, 60000], [35, 55000],
              [28, 48000], [60, 500000]])  # outlier!

print("Standard:", StandardScaler().fit_transform(X).round(2))
print("MinMax  :", MinMaxScaler().fit_transform(X).round(2))
print("Robust  :", RobustScaler().fit_transform(X).round(2))
Output
