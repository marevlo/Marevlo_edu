Reading Structure Through a Hierarchy

The earlier methods ask for one hard decision before they begin: how many clusters do you want? Hierarchical clustering refuses to lock itself into one answer too early. It builds a whole tree of possibilities, lets you see how points merge or split over time, and only then asks where you want to cut the structure.

Dendrogram
01 — Core Idea

Instead of one clustering, build the entire family of them

K-Means and K-Prototypes give you one final partition. Hierarchical clustering gives you something broader: a nested sequence of groupings. At the smallest scale, every point can stand alone. At the largest scale, every point can belong to one giant cluster. Everything between those extremes is recorded.

This is why hierarchical clustering feels more exploratory. It does not force you to decide the number of groups before the data has spoken. It builds the structure first and lets you decide later where the meaningful separation lies.

Mental Model
Folders inside folders, or branches inside branches
Think of organizing files. You first group very similar files into small folders, then those folders into broader folders, and then broader folders into larger categories. Hierarchical clustering does this automatically, but using similarity instead of human names.

The real output is not just a set of labels. It is the history of how relationships formed.

02 — The Dendrogram

The tree is where the interpretation lives

The signature object of hierarchical clustering is the dendrogram. This tree records each merge or split and the dissimilarity level at which it happened. It is not decorative. It is the main interpretive tool.

On the horizontal axis you usually see the individual data points. On the vertical axis you see the distance or dissimilarity level at which branches come together. A merge that happens low on the tree means the objects were very similar. A merge that happens high on the tree means the algorithm had to tolerate a larger difference to connect them.

03 — Two Opposite Directions

Bottom-up and top-down are mirror images of the same idea

There are two broad ways to build a hierarchy. One begins with every point alone and keeps merging. The other begins with everything together and keeps splitting.

TypeStarting PointMain ActionHow it feels
AgglomerativeEvery point is its own clusterMerge the closest clusters repeatedlyBottom-up construction
DivisiveEverything begins in one clusterSplit clusters recursivelyTop-down refinement

In practice, when people casually say “hierarchical clustering,” they usually mean the agglomerative version.

04 — Agglomerative Clustering

Start with fragments and keep merging the nearest pieces

Agglomerative clustering is the standard bottom-up version. If there are 100 observations, the process begins with 100 tiny clusters. At each step, the algorithm searches for the two clusters that are most similar and merges them.

This continues until every point has been absorbed into one final cluster. The whole record of those merges becomes the dendrogram.

The greedy nature

Agglomerative clustering is greedy. Once two groups are merged, that decision is not revisited. There is no undo button. This is one reason early bad merges can have long-lasting effects.

1

Initialize

Put every point in its own separate cluster.

2

Compute distances

Measure distances between the current clusters using the chosen linkage rule.

3

Merge the closest pair

Combine the two clusters with the smallest current dissimilarity.

4

Update distances

Recompute how far the new cluster is from all remaining clusters.

5

Repeat

Continue until only one cluster remains.

Why the cost grows so quickly

The algorithm keeps updating a distance matrix after each merge. That repeated bookkeeping is what makes hierarchical clustering expensive in both time and memory on large datasets.

05 — Linkage Methods

The meaning of “distance between clusters” is a design choice

Once individual points start forming groups, the algorithm has to answer a new question: what does it even mean for one cluster to be close to another cluster? This is where linkage methods enter.

LinkageHow distance is definedTypical behavior
SingleDistance between the nearest pair of points across two clustersTends to create chain-like structures and can react poorly to outliers
CompleteDistance between the farthest pair of points across two clustersPrefers tighter, more compact groups
AverageAverage of all pairwise distances between the two clustersBalanced compromise between single and complete
WardMerge that produces the smallest increase in within-cluster varianceOften yields neat, compact, fairly even clusters
Interpretation
Different linkage rules can change the tree dramatically
Hierarchical clustering is not one single algorithmic personality. Single linkage, complete linkage, average linkage, and Ward’s method can produce visibly different dendrograms on the same data.

Single linkage can connect points through a sequence of small local bridges, producing long chain-like clusters even when the overall shape is not truly compact.

06 — Distance and Variance

Euclidean distance is the baseline, Ward is about variance growth

Most hierarchical workflows begin with a base distance between individual data points, often Euclidean distance. The linkage rule then tells the algorithm how to extend that idea from points to clusters.

Ward’s method is slightly different in spirit. It is not asking only which two clusters are close in ordinary geometric space. It is asking which merge causes the smallest increase in total within-cluster spread.

This is why Ward’s method is often described as the most K-Means-like member of the hierarchical family. It cares strongly about compactness.

07 — Agglomerative Example

From linkage matrix to dendrogram to final labels

python
import numpy as np
import matplotlib.pyplot as plt
from scipy.cluster.hierarchy import dendrogram, linkage
from sklearn.cluster import AgglomerativeClustering

X = np.array([
    [5, 3], [10, 15], [15, 12], [24, 10], [30, 30],
    [85, 70], [71, 80], [60, 78], [70, 55], [80, 91]
])

Z = linkage(X, method='ward', metric='euclidean')

plt.figure(figsize=(10, 5))
plt.title('Hierarchical Clustering Dendrogram')
plt.xlabel('Data Point Index')
plt.ylabel('Distance (Ward)')
dendrogram(Z)
plt.show()

model = AgglomerativeClustering(n_clusters=2, metric='euclidean', linkage='ward')
cluster_labels = model.fit_predict(X)
print(cluster_labels)

The usual workflow is visual first, labeling second. First inspect the dendrogram. Then decide where the cut seems meaningful. Only after that do you request final labels.

08 — Limitations

Where hierarchical clustering begins to struggle

LimitationWhy it happensWhat it affects
High time complexityRepeated merge decisions and matrix updatesLarge datasets become slow
High space complexityDistance matrix storage grows quadraticallyMemory usage rises quickly
Greedy decisionsMerges are not revisited once madeEarly mistakes can distort the whole tree
Sensitivity to noiseOutliers can influence merge structureThe dendrogram may reflect noise rather than clean groups

Hierarchical clustering is at its best when the dataset is not too large and the structure is reasonably clean. When noise is heavy or separation is weak, the tree can become much harder to trust.

09 — Where It Is Used

Best when relationship structure matters more than raw scale

Hierarchical clustering is particularly useful when analysts care about the relationship map itself, not just the final labels.

Biology and genomics for grouping genes or building taxonomic relationships
Document organization for building broad-to-specific folder structures
Small-scale customer segmentation where analysts want to inspect relationship tiers
Exploratory work when the number of clusters is not known in advance

A good rule of thumb is to use hierarchical clustering when the dataset is modest and interpretability matters as much as the clustering itself.

10 — Divisive Clustering

Start with one giant cluster and keep cutting it apart

Divisive clustering moves in the opposite direction. Instead of beginning with many tiny clusters, it begins with one giant cluster containing every point. It then repeatedly finds a cluster to split into smaller pieces.

In its ideal form, divisive clustering would search for the mathematically best split at each stage. That is extremely expensive. So in practice, approximate splitting rules are often used, and one common shortcut is to apply K-Means with k = 2 inside the cluster being split.

Big Picture
Agglomerative builds upward. Divisive carves downward.
The philosophical difference is simple. One asks which small pieces belong together. The other asks which large piece should be broken next.

A DIANA-style intuition

Divisive strategies often look for the cluster that seems most internally spread out or most internally inconsistent, then split that cluster into two tighter parts.

1

Initialize

Place all points in one giant cluster.

2

Choose a cluster to split

Usually the cluster with the greatest spread or internal dissimilarity.

3

Split it into two parts

A practical shortcut is to use K-Means with k = 2 inside that cluster.

4

Repeat

Continue recursively until the desired granularity is reached.

python
from sklearn.cluster import KMeans
import numpy as np

X = np.array([[1, 2], [1, 4], [1, 0], [10, 2], [10, 4], [10, 0]])

def divisive_clustering(data, current_clusters):
    if len(current_clusters) == len(data):
        return current_clusters

    largest_cluster_idx = max(range(len(current_clusters)), key=lambda i: len(current_clusters[i]))
    cluster_to_split = current_clusters.pop(largest_cluster_idx)

    if len(cluster_to_split) == 1:
        current_clusters.append(cluster_to_split)
        return current_clusters

    kmeans = KMeans(n_clusters=2, n_init=10, random_state=42)
    labels = kmeans.fit_predict(cluster_to_split)

    group1 = cluster_to_split[labels == 0]
    group2 = cluster_to_split[labels == 1]

    current_clusters.append(group1)
    current_clusters.append(group2)

    return divisive_clustering(data, current_clusters)

Divisive clustering is elegant conceptually, but much rarer in practice because optimal splitting is computationally hard. A poor split high in the tree can influence every branch below it.

12 — When to Use It

Excellent for small, interpretable exploration; weak for scale and noise

Use it when the number of clusters is unknown
Use it when visual interpretation matters
Use it for small or moderate datasets
Avoid it for very large datasets
Avoid it for noisy, messy, real-time settings
Decision Rule
Choose hierarchical clustering when you want to understand structure before committing to a final cut.
If your priority is a fast answer on large data, this is usually not the first method to reach for. If your priority is seeing the relationships themselves, it becomes much more attractive.
