01 — Feature Vectors & Data Representation

Turning Reality into Numbers

Imagine you run an online store. You know your customers — some are young students who buy cheap gadgets, some are professionals who buy premium electronics, and some are bargain hunters who wait for sales. You feel these groups exist. But how do you prove it to a machine?

The answer: you describe each customer as a feature vector — a fixed-length list of numbers where each number captures one measurable property.

A customer who is 28 years old, earns ₹52,000/month, and has a spending score of 68/100 becomes:

Feature Vector
x₁ = [28, 52, 68]
28 = age  ·  52 = income (₹ thousands)  ·  68 = spending score (0–100)

Another customer becomes [35, 75, 22]. Stack all customers vertically and you get the feature matrix — the entire dataset as one mathematical object.

Feature Matrix Notation
X ∈ ℝn × d
n = number of data points (rows)  ·  d = number of features (columns)  ·  Each row xᵢ = [xᵢ₁, xᵢ₂, ..., xᵢd] is one observation

Without this uniform format, no distance calculation, no similarity measure, no clustering algorithm can even start. The feature matrix is the universal input.

Why this matters for clustering: Every clustering algorithm treats each row as a point in geometric space. Two customers with similar vectors will be nearby in this space — and "nearby" means "likely in the same cluster." The entire game of clustering is measuring distances in this space.

Python 
import numpy as np
import matplotlib.pyplot as plt

# 6 customers [age, income in thousands, spending score 0-100]
X = np.array([
    [25, 45, 35],
    [32, 60, 75],
    [40, 75, 20],
    [28, 50, 60],
    [35, 55, 45],
    [22, 38, 90]
])

print("Matrix shape (observations × features):", X.shape)
print("Vector for first customer:", X[0])

plt.figure(figsize=(6, 4))
plt.scatter(X[:, 0], X[:, 1], s=80, alpha=0.8)
plt.xlabel("Age")
plt.ylabel("Income (thousands)")
for i in range(len(X)):
    plt.text(X[i, 0] + 0.4, X[i, 1] + 0.8, f"P{i+1}")
plt.grid(alpha=0.3)
plt.show()
Output
Matrix shape (observations × features): (6, 3)
Vector for first customer: [25 45 35]

# A scatter plot appears showing 6 customers plotted
# in the Age × Income plane, labeled P1 through P6.
# You can already see natural groupings forming.

Watch out: Problems appear when features have wildly different ranges (age: 20–80 vs income: 10k–200k). The high-range feature will dominate all distance calculations. We'll fix this with scaling later — but always be aware of your ranges.

02 — Mean, Median, Mode, Variance & Standard Deviation

Understanding the Shape of Your Features

Before clustering, you need to understand each feature's personality. Is it centered around a value? How spread out is it? Is it easily fooled by outliers? These five statistics answer all of that.

Mean — The Balancing Point

The mean is the arithmetic average — the point where the data "balances." But it has a weakness: one extreme value can drag it far from where most data lives.

Mean (Arithmetic Average)
μ = (1/n) · Σ xᵢ
μ = mean  ·  xᵢ = each observation  ·  n = total number of observations

Median — The Robust Middle

Sort all values, pick the middle one. If an outlier appears, the median barely moves. That's why it's called robust.

Median
If n odd: x(n+1)/2   |   If n even: (xn/2 + xn/2+1) / 2
Sort first, then pick the center value. Completely ignores how far away extreme values are.

Mode — The Most Popular

The value that appears most frequently. Especially useful for categorical features — what's the most common city, brand, or label in a group?

Variance & Standard Deviation — The Spread

Variance measures how far values wander from the mean, on average. Standard deviation is its square root — same idea but in the original units, so it's easier to interpret.

Population Variance
σ² = (1/n) · Σ (xᵢ − μ)²
Squared deviations: positive and negative differences don't cancel out. σ² is in squared units of the original feature.
Sample Variance (Bessel's Correction)
s² = (1/(n−1)) · Σ (xᵢ − x̄)²
Dividing by n−1 instead of n corrects for underestimation when working with a sample instead of the whole population. This is what NumPy uses with ddof=1.
Standard Deviation
σ = √σ²     (population)     s = √s²     (sample)
Back in the original units. If ages have std = 5 years, most ages fall within ±5 years of the mean.

Why this is critical for clustering: If one feature has std = 5 and another has std = 5,000, then Euclidean distance is almost entirely controlled by the second feature. Clusters will form along that axis alone. You must check variance before clustering.

Python 
import numpy as np
from scipy import stats

ages = np.array([23, 25, 26, 27, 120, 24, 25])

print("Mean     :", np.mean(ages))
print("Median   :", np.median(ages))
print("Mode     :", stats.mode(ages).mode)
print("Var (pop):", np.var(ages, ddof=0))
print("Var (sam):", np.var(ages, ddof=1))
print("Std (pop):", np.std(ages, ddof=0))
print("Std (sam):", np.std(ages, ddof=1))
Output
Mean     : 38.571  ← dragged up by 120
Median   : 25.0    ← barely moved
Mode     : 25      ← appears twice
Var (pop): 1124.24 ← huge because of 120
Var (sam): 1311.62
Std (pop): 33.53   ← values vary ~34 years from mean
Std (sam): 36.22

03 — NumPy Essentials for Clustering

The Toolkit You Actually Need

NumPy is the engine beneath every clustering algorithm in Python. It provides fast, memory-efficient arrays and mathematical operations that work without Python loops. For clustering, you only need a handful of operations — but you need to understand them deeply.

Python — The Essential 5 
import numpy as np

X = np.array([
    [25, 45, 35], [32, 60, 75], [40, 75, 20],
    [28, 50, 60], [35, 55, 45], [22, 38, 90]
])

# 1. Shape of your data
print("Shape:", X.shape)

# 2. Per-feature statistics
print("Means:", np.mean(X, axis=0))
print("Stds :", np.std(X, axis=0))

# 3. Distance between two points
x1, x2 = X[0], X[1]
print("Distance P1↔P2:", np.linalg.norm(x1 - x2))

# 4. Standardization (zero mean, unit variance)
X_scaled = (X - np.mean(X, axis=0)) / np.std(X, axis=0)
print("Scaled means:", np.round(np.mean(X_scaled, axis=0), 2))
print("Scaled stds :", np.round(np.std(X_scaled, axis=0), 2))

# 5. Broadcasting — distance from every point to a centroid
centroid = np.mean(X, axis=0)
distances = np.linalg.norm(X - centroid, axis=1)
print("Distances to centroid:", np.round(distances, 2))

Dense vs. Sparse — Memory Matters at Scale

After preprocessing like one-hot encoding or TF-IDF, your feature matrix might be 99% zeros. Storing all those zeros explicitly as a dense matrix wastes enormous memory. A sparse matrix only records the non-zero entries plus their coordinates — cutting memory by 90%+ for high-dimensional, mostly-empty data.

Python — Sparse Matrix 
from scipy.sparse import csr_matrix

# Convert dense matrix with many zeros to sparse
X_sparse = csr_matrix(X_dense)

print("Dense memory  :", X_dense.nbytes, "bytes")
print("Sparse memory :", X_sparse.data.nbytes
      + X_sparse.indices.nbytes
      + X_sparse.indptr.nbytes, "bytes")

# Scikit-learn's KMeans accepts sparse input natively
from sklearn.cluster import KMeans
km = KMeans(n_clusters=5)
km.fit(X_sparse)  # ✓ works
Output
Dense memory  : 4000000 bytes  (3.81 MB)
Sparse memory :   60012 bytes  (0.06 MB) — 98.5% savings!
