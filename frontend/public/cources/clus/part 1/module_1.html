01 — The Core Idea

What Is Clustering?

Imagine 10,000 unlabeled animal photos. You start grouping: pointy ears with whiskers go together, beaks with feathers go together. Without any teacher, your brain discovers natural groups by similarity. That's clustering.

Formally: given a dataset where no "answer" is known, discover groups by measuring how similar or different data points are. This is unsupervised learning.

Formal Definition
Partition X = {x₁, ..., xₙ} into C₁, C₂, ..., Cₖ
such that points within each Cⱼ are maximally similar and minimally similar to points in other clusters.

02 — Four Families of Clustering

Each Algorithm Has a Different Assumption

There's no universal "best" algorithm. Each family makes a different mathematical assumption (inductive bias) about what a cluster looks like.

MOST COMMON
◉

Centroid-Based

Each cluster has a center. Every point joins nearest center. Assumes spherical, similarly-sized clusters.

🌳

Hierarchical

Builds a tree. Cut at any level for different granularity. No need to predefine K.

🌊

Density-Based

Dense regions separated by sparse gaps. Any shape works. Outliers become noise.

🕸️

Graph-Based

Points are nodes, edges connect similar pairs. Finds densely connected subgraphs.

03 — Cohesion & Separation

What Makes a "Good" Cluster?

Cohesion = how tight points are within a cluster. Separation = how far apart clusters are. Best clustering maximizes both.

Cluster Quality
Quality ∝ Separation / Cohesion
High separation + low intra-cluster distance = excellent clusters.

📐 Math Workout — Computing Cluster Quality
Problem: Three clusters:
C₁ center=(2,3), points: (1,2), (3,4), (2,2), (2,4)
C₂ center=(8,8), points: (7,7), (9,9), (8,8)
C₃ center=(8,2), points: (7,1), (9,3), (8,2)

Compute: (a) Average intra-cluster distance for C₁ (b) Average inter-cluster distance (c) Which cluster is tightest?
Step 1 — Cohesion of C₁
Distance from each point to center (2,3):
d(1,2)→center = √((1−2)²+(2−3)²) = √2 ≈ 1.41
d(3,4)→center = √((3−2)²+(4−3)²) = √2 ≈ 1.41
d(2,2)→center = √(0+(2−3)²) = 1.00
d(2,4)→center = √(0+(4−3)²) = 1.00

Avg C₁ = (1.41+1.41+1.00+1.00)/4 = 1.205
Step 2 — Cohesion of C₂ and C₃
C₂: d(7,7)=√2≈1.41, d(9,9)=√2≈1.41, d(8,8)=0 → Avg = 0.94
C₃: d(7,1)=√2≈1.41, d(9,3)=√2≈1.41, d(8,2)=0 → Avg = 0.94
Step 3 — Separation
d(C₁,C₂) = √(36+25) = √61 ≈ 7.81
d(C₁,C₃) = √(36+1) = √37 ≈ 6.08
d(C₂,C₃) = √(0+36) = 6.00
Avg separation = 6.63
✓ C₂ and C₃ are tightest (0.94 each). C₁ is loosest (1.205).
✓ Quality = 6.63/1.03 ≈ 6.44 — excellent clustering.
📐 Math Workout — Proving K-Means Fails on Crescents
Problem: Two crescents: Moon A: (1,3),(2,4),(3,3.5),(4,2) — Moon B: (2,1),(3,0),(4,0.5),(5,2)
K-Means centroids: μ_A=(2.5, 3.125), μ_B=(3.5, 0.875)
Show that point (4,2) from Moon A gets incorrectly assigned to Moon B.
Distance to μ_A
d = √((4−2.5)²+(2−3.125)²) = √(2.25+1.266) = √3.516 ≈ 1.875
Distance to μ_B
d = √((4−3.5)²+(2−0.875)²) = √(0.25+1.266) = √1.516 ≈ 1.231
1.231 < 1.875 → K-Means assigns (4,2) to B. Wrong! It belongs to Moon A.
K-Means fails because it uses straight-line distance to a center — it cannot follow curved manifold structure.
04 — The Shape Problem

Why Shape Determines Algorithm Choice

K-Means assumes spherical clusters. Real data: elongated sausages, interlocking crescents, concentric rings, blobs with wildly different densities.

The #1 clustering mistake: Using K-Means on non-spherical data. Always visualize first.

🎯 Scenario: E-Commerce Customer Clustering

50,000 customers: spend (₹500–₹5L), returns (0–50), session time (1–90 min), days since last purchase (0–365). You see: one massive blob (~45K regular), tight cluster (~2K high-spenders, zero returns), scattered long tail (~3K frequent returners).

Think: Why would K-Means K=3 fail? Hint: size imbalance (45K vs 2K vs 3K). K-Means splits the 45K blob rather than finding the small distinctive clusters. Also: spend ranges 500–500,000 vs returns 0–50. Without scaling, spend dominates all distances. Use density-based (DBSCAN) with standardized features.

🎯 Scenario: Network Intrusion Detection

99.8% normal connections (dense cloud), 0.2% attacks (scattered, sparse, unusual). A colleague suggests K-Means K=2.

Think: K-Means places centroids to minimize total variance. With ~200 attack points vs ~99,800 normal, one centroid goes in the normal cloud and the other drifts to some random region — it won't find the actual attack patterns. Density-based clustering naturally labels sparse points as noise, making DBSCAN ideal: the normal cloud forms one dense cluster, attack points become noise or small clusters.
📐 Hard Workout — Silhouette Score Computation
Problem: Given clustering with 2 clusters:
C₁: {(0,0), (1,0), (0,1)} — C₂: {(5,5), (6,5)}
Compute the Silhouette Score for point (1,0).

Recall: s(i) = (b(i) − a(i)) / max(a(i), b(i)) where a(i) = avg distance to same-cluster points, b(i) = avg distance to nearest other cluster.
Step 1 — a(i): avg distance to same cluster
d((1,0),(0,0)) = 1.00
d((1,0),(0,1)) = √(1+1) = √2 ≈ 1.414
a(i) = (1.00 + 1.414) / 2 = 1.207
Step 2 — b(i): avg distance to nearest other cluster (C₂)
d((1,0),(5,5)) = √(16+25) = √41 ≈ 6.403
d((1,0),(6,5)) = √(25+25) = √50 ≈ 7.071
b(i) = (6.403 + 7.071) / 2 = 6.737
Step 3 — Silhouette Score
s = (b − a) / max(a, b) = (6.737 − 1.207) / 6.737 = 5.530 / 6.737 ≈ 0.821
✓ s ≈ 0.821 — close to 1.0, meaning this point is well-placed in its cluster.
✓ s near 1 = good, s near 0 = on boundary, s negative = wrong cluster.
