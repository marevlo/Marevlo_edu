01 — The Algorithm

What Is K-Means Clustering?

K-Means is an unsupervised, centroid-based algorithm. It divides a dataset into K non-overlapping groups by iteratively assigning points to the nearest cluster center, then updating that center. The process is called Lloyd's Algorithm.

Objective Function (WCSS)
J = Σⱼ₌₁ᴷ Σᵢ∈Cⱼ ‖xᵢ − μⱼ‖²
Minimize the Within-Cluster Sum of Squares: the total squared Euclidean distance from every point to its assigned centroid. Lower J = tighter clusters.

Lloyd's Algorithm — The Four Steps

1

Initialize

Pick K data points at random as starting centroids. (Or use K-Means++ for smarter initialization — covered in Module 5.)

2

Assign

Compute Euclidean distance from every point to every centroid. Assign each point to the nearest centroid. This partitions space into Voronoi regions.

3

Update

Recompute each centroid as the mean of all points assigned to it: μⱼ = (1/|Cⱼ|) Σ xᵢ

4

Repeat

Go back to Step 2. Stop when centroids stop moving (convergence), or max iterations reached, or centroid shift < tolerance threshold.

02 — Evaluating Clusters

WCSS, Inertia & the Dunn Index

Inertia is sklearn's name for WCSS — the sum of squared distances from each point to its centroid. Lower = tighter clusters. But inertia alone isn't enough: it always decreases as K increases (K=N gives inertia=0).

Dunn Index
DI = min(inter-cluster distance) / max(intra-cluster diameter)
Higher = better. Numerator wants clusters far apart. Denominator wants clusters tight. A Dunn Index > 1 means the smallest gap between any two clusters is larger than the largest cluster's diameter.

The Elbow Method

Run K-Means for K=1,2,...,10. Plot WCSS vs K. The curve always decreases, but at some point the rate of decrease slows drastically — creating an "elbow." That K is your best choice.

📐 Math Workout — Computing WCSS by Hand
Problem: K=2 clustering result:
C₁ centroid=(2,2), points: (1,1), (3,3), (2,1), (2,3)
C₂ centroid=(7,7), points: (6,6), (8,8), (7,6)
Compute total WCSS.
C₁ contributions
‖(1,1)−(2,2)‖² = 1+1 = 2
‖(3,3)−(2,2)‖² = 1+1 = 2
‖(2,1)−(2,2)‖² = 0+1 = 1
‖(2,3)−(2,2)‖² = 0+1 = 1
WCSS₁ = 2+2+1+1 = 6
C₂ contributions
‖(6,6)−(7,7)‖² = 1+1 = 2
‖(8,8)−(7,7)‖² = 1+1 = 2
‖(7,6)−(7,7)‖² = 0+1 = 1
WCSS₂ = 2+2+1 = 5
Total WCSS = 6 + 5 = 11
This is the value K-Means minimizes. Each iteration either decreases or maintains WCSS — never increases.
📐 Hard Workout — Dunn Index Computation
Problem: Same clustering as above. Compute the Dunn Index.
You need: (a) minimum inter-cluster distance (closest pair between C₁ and C₂), (b) maximum intra-cluster diameter (farthest pair within any single cluster).
Min inter-cluster distance
Closest pair between C₁ and C₂:
d((3,3),(6,6)) = √(9+9) = √18 ≈ 4.24 ← closest!
d((3,3),(7,6)) = √(16+9) = 5.00
(checking all: smallest = 4.24)
Max intra-cluster diameter
Farthest pair within same cluster:
C₁: d((1,1),(3,3)) = √8 ≈ 2.83 ← max in C₁
C₂: d((6,6),(8,8)) = √8 ≈ 2.83 ← max in C₂
Max diameter = 2.83
Dunn Index = 4.24 / 2.83 ≈ 1.50
DI > 1 means the gap between clusters is wider than any cluster's own spread — excellent separation!
03 — Strengths, Weaknesses & K-Medians

When K-Means Works — and When It Doesn't

✓ Advantages✗ Disadvantages
Fast — O(nKd) per iterationMust choose K manually
Simple, transparent mathAssumes spherical, similarly-sized clusters
Guaranteed convergenceSensitive to outliers (mean shifts toward extremes)
Scales to millions of pointsRandom init → different results each run
Works well on compact blobsFails on non-convex shapes (moons, rings)

K-Medians: The Outlier-Robust Variant

When outliers are present and can't be removed, K-Medians swaps two components of Lloyd's Algorithm:

ComponentK-MeansK-Medians
Distance metricEuclidean (L₂)Manhattan (L₁)
Centroid updateMean of pointsMedian of points
Objective functionWCSS (sum of squared distances)SAD (sum of absolute deviations)
Outlier sensitivityHigh (mean shifts)Low (median ignores extremes)
K-Medians Objective (SAD)
J = Σⱼ₌₁ᴷ Σᵢ∈Cⱼ |xᵢ − medianⱼ|
Absolute differences instead of squared. For [1,2,3,4,100]: mean=22 (pulled by outlier), median=3 (stable).
📐 Math Workout — Mean vs Median Under Outlier
Problem: Cluster with 1D points: [10, 12, 11, 13, 10, 200].
Compute (a) K-Means centroid (mean) (b) K-Medians centroid (median) (c) WCSS for K-Means (d) SAD for K-Medians. Which is more stable?
(a) Mean
μ = (10+12+11+13+10+200)/6 = 256/6 = 42.67
The mean is pulled to 42.67 — far from the actual data cluster at ~11!
(b) Median
Sorted: [10, 10, 11, 12, 13, 200]
Median = (11+12)/2 = 11.5
The median sits right in the cluster. The outlier 200 has zero effect.
(c) WCSS (mean-based)
Σ(xᵢ−42.67)² = (−32.67)²+(−30.67)²+(−31.67)²+(−29.67)²+(−32.67)²+(157.33)²
= 1067.3+940.6+1003.0+880.3+1067.3+24752.7 = 29,711.2
(d) SAD (median-based)
Σ|xᵢ−11.5| = 1.5+0.5+0.5+1.5+1.5+188.5 = 194.0
WCSS = 29,711 (the squared outlier term is 24,753 — 83% of total!)
SAD = 194 (outlier contributes 188.5 but doesn't distort the center)
K-Medians keeps the centroid at 11.5; K-Means drags it to 42.67. Use K-Medians for outlier-heavy data.

🎯 Scenario: Smartphone Photo Gallery Clustering

You're building an auto-album feature for a phone gallery app. Features: GPS coordinates, timestamp, face embeddings (128-dim), color histogram (256-dim). 10,000 photos total.

Think: GPS ranges from -180 to 180 (degrees) while face embeddings are unit-normalized vectors ~0.01–1.0. Without scaling, GPS dominates. But even after scaling, K-Means on 384 dimensions hits the curse of dimensionality. Better approach: cluster GPS separately (K-Means works — lat/lng are roughly Euclidean), cluster faces with cosine similarity (direction matters, not magnitude), then combine using a multi-modal assignment.

🎯 Scenario: OTT Content Recommendation

A streaming platform wants to group 5 million users into viewing profiles: "reality binger," "action fan," "documentary nerd," etc. Features: watch time per genre (10 genres), time-of-day patterns (24 hourly bins), device type (3 categories).

Think: 5M users × 37 features = too large for standard K-Means in memory. Use Mini-Batch K-Means (Module 5). The 3 device-type columns are categorical — K-Means can't handle categorical data natively. Options: one-hot encode (increases dimensions), use K-Modes for categoricals, or cluster only on the 34 numerical features and add device as a post-hoc label. Also: the Elbow Method at this scale may be slow — consider the Silhouette Score on a random subsample instead.
04 — Python Implementation

Full K-Means Pipeline

Python — Complete K-Means Pipeline
import numpy as np
import matplotlib.pyplot as plt
from sklearn.cluster import KMeans
from sklearn.datasets import make_blobs
from sklearn.preprocessing import StandardScaler

# 1. Generate synthetic data (or load real CSV)
X, y_true = make_blobs(n_samples=300, centers=3,
                        cluster_std=0.80, random_state=42)

# 2. Scale features (critical for real data)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 3. Elbow Method — find optimal K
wcss = []
for k in range(1, 11):
    km = KMeans(n_clusters=k, init='k-means++',
                n_init=10, max_iter=300, random_state=42)
    km.fit(X_scaled)
    wcss.append(km.inertia_)

# 4. Train final model with optimal K
model = KMeans(n_clusters=3, init='k-means++',
               n_init=10, random_state=42)
labels = model.fit_predict(X_scaled)
print("Centroids:\n", model.cluster_centers_)
print("Inertia (WCSS):", model.inertia_)

Python — K-Medians (outlier-robust)
from pyclustering.cluster.kmedians import kmedians

# Data with outlier
data = [[1.2,1.5],[1.1,1.3],[1.4,1.2],[1.3,1.5],
        [5.1,5.2],[5.3,5.4],[5.2,5.1],[5.5,5.3],
        [10.0,10.0]]  # ← outlier

initial = [[1.0,1.0], [5.0,5.0]]
inst = kmedians(data, initial)
inst.process()
print("Medians:", inst.get_medians())
