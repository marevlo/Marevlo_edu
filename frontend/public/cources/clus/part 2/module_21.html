01 — Why Graph Neural Networks

Clustering Must Use Both Features and Connections

Most basic clustering methods treat every data point as isolated. They only see the feature vector. But real-world datasets often come with important relational structure: social connections, citation links, co-purchases, or biological interactions. In these cases, ignoring the graph means ignoring some of the strongest clustering signal available.

Graph Representation Learning
A GNN turns each node into a useful embedding by exchanging information along edges.
Cluster Assignment
The learned embeddings are then grouped with K-Means or a neural clustering loss.
Practical Benefit
Feature similarity and graph structure are fused instead of being handled separately.

02 — How We Represent a Graph

Nodes, Edges, Features, and the Adjacency Matrix

A graph is written as G = (V, E), where V is the set of nodes and E is the set of edges. Each node also has a feature vector, stored in the feature matrix X, while the connections are stored in the adjacency matrix A.

03 — The GCN Layer

Graph Convolution and Message Passing

The most common building block for GNN clustering is the Graph Convolutional Network layer. A GCN layer transforms node features, aggregates information from neighbors, and then applies an activation function.

GCN Layer FormulaFORMULA
H^(l+1) = sigma(D~^-1/2 A~ D~^-1/2 H^l W^l)
  • Hl: node embeddings at layer l.
  • Wl: trainable weight matrix.
  • σ: activation function, usually ReLU.
  • Ã = A + I: adjacency matrix with self-loops added.
  • D̃: degree matrix of Ã.
  • D̃-1/2: normalization so high-degree nodes do not dominate the messages.
Message Passing IntuitionANIMATED
Final Embedding Matrix
Z = H^L

After the final graph-convolution layer, the rows of Z become the node embeddings used for clustering.

04 — End-to-End GNN Clustering Pipeline

From Graph Construction to Final Cluster Labels

  1. Build the graph G(V,E) and load the feature matrix X.
  2. Initialize the node embeddings.
  3. Run multiple graph convolution layers.
  4. At each layer, aggregate information from neighbors.
  5. Produce final node embeddings Z.
  6. Run K-Means or a clustering head on the rows of Z.
  7. Assign each node to a cluster.

05 — Code, Limits, and Real-World Example

GCN Implementation and Product-Graph Clustering

PyTorch
GCN + KMEANS
import torch
import torch.nn as nn
import torch.nn.functional as F
from sklearn.cluster import KMeans

class GCNLayer(nn.Module):
    def __init__(self, in_features, out_features):
        super().__init__()
        self.weight = nn.Parameter(torch.randn(in_features, out_features))

    def forward(self, x, adj):
        h = torch.mm(x, self.weight)
        h = torch.mm(adj, h)
        return F.relu(h)

class GCN(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim):
        super().__init__()
        self.gcn1 = GCNLayer(input_dim, hidden_dim)
        self.gcn2 = GCNLayer(hidden_dim, output_dim)

    def forward(self, x, adj):
        h = self.gcn1(x, adj)
        z = self.gcn2(h, adj)
        return z

def cluster_nodes(embeddings, k):
    kmeans = KMeans(n_clusters=k)
    labels = kmeans.fit_predict(embeddings.detach().numpy())
    return labels

Scalability Issue
Huge graphs may require sampling methods such as GraphSAGE.
Over-Smoothing
Too many layers make node embeddings look too similar because messages keep averaging.
Graph Dependency
Bad edges produce bad clusters no matter how good the GNN is.
Limits to Remember

Large graphs can be memory-heavy, full adjacency can be too expensive, and weakly constructed graphs will poison clustering quality. Graph quality and sampling strategy matter just as much as model choice.
