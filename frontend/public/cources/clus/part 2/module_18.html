Self-Organizing Maps (SOM)

SOM is the first neural-network step in the course: it learns a low-dimensional map that preserves neighborhood structure. It does not directly optimize cluster boundaries, but it reshapes data so clusters become visible and interpretable.

01 — What SOM Solves

Topology-Preserving Representation Learning

Earlier methods such as K-Means, hierarchical clustering, and density-based methods operate directly on raw features. In high-dimensional or structurally complex data, raw distances often stop reflecting meaningful similarity. Self-Organizing Maps solve this by learning a structured low-dimensional map where nearby neurons represent similar inputs.

What SOM Is Not
Not a direct clustering algorithm and not an optimizer of explicit cluster boundaries.
What SOM Does
Learns a spatial representation where similar samples land near each other and clusters become visible.
Why It Matters
Improving representation is often more effective than endlessly swapping clustering algorithms.

The method was introduced in “Self-Organized Formation of Topologically Correct Feature Maps” by Teuvo Kohonen in 1982.

Intuition

Think of SOM as a flexible sheet laid over the data. The sheet moves to fit the structure while preserving neighborhood order.

02 — Mathematical Formulation

Best Matching Unit and Neighborhood Update

Let the dataset be X = {x₁, x₂, …, xₙ} with each xᵢ ∈ ℝᵈ. A SOM contains a grid of M neurons, each with a weight vector wⱼ ∈ ℝᵈ. For every input sample, the model finds the neuron whose weight is closest to the input. This is the Best Matching Unit, or BMU.

BMU RuleFORMULA
Competition Step
BMU(x) = argmin_j ||x - w_j||

The BMU becomes the anchor point for that input. Unlike K-Means, SOM then updates not only the winning neuron but also its spatial neighbors.

Neighborhood Update RuleFORMULA
Cooperation Step
w_j(t+1) = w_j(t) + alpha(t) h_(b,j)(t) (x - w_j(t))
  • α(t): learning rate.
  • hb,j(t): neighborhood function measuring how strongly neuron j should follow the BMU b.
  • b: BMU index.

Key Takeaway

Competition finds the best representative neuron. Cooperation spreads the update into the grid. That combination is what makes SOM representation learning fundamentally different from K-Means.

03 — Training Dynamics

Global Shape First, Local Detail Later

SOM training compares every input with every neuron at each iteration, making it more expensive than simple centroid methods, but the result is a structured map rather than a flat partition.

Complexity
O(n · M · T) for n samples, M neurons, and T iterations.
Early Training
Large neighborhoods capture global organization of the data.
Late Training
Small neighborhoods refine local structure and improve map detail.
  1. Initialize neuron weights randomly.
  2. Select an input vector x.
  3. Find the BMU using Euclidean distance.
  4. Update the BMU and its neighbors.
  5. Decrease learning rate and neighborhood radius.
  6. Repeat until the map stabilizes.

Core Idea

Competition selects the BMU, cooperation updates nearby neurons, and adaptation comes from learning-rate decay and neighborhood shrinkage.

04 — Practical Usage

Implementation, Limitations, and Real-World Role

Python
CUSTOM SOM
import numpy as np

class SOM:
    def __init__(self, m, n, dim, lr=0.5, sigma=None, epochs=100):
        self.m = m
        self.n = n
        self.dim = dim
        self.lr = lr
        self.epochs = epochs
        self.sigma = sigma if sigma else max(m, n) / 2
        self.weights = np.random.rand(m * n, dim)

    def _bmu(self, x):
        distances = np.linalg.norm(self.weights - x, axis=1)
        return np.argmin(distances)

    def _neighborhood(self, bmu_idx, step):
        coords = np.array([[i//self.n, i%self.n] for i in range(self.m * self.n)])
        bmu_coord = coords[bmu_idx]
        dist = np.linalg.norm(coords - bmu_coord, axis=1)
        sigma = self.sigma * np.exp(-step / self.epochs)
        return np.exp(-(dist**2) / (2 * sigma**2))

    def fit(self, X):
        for step in range(self.epochs):
            for x in X:
                bmu = self._bmu(x)
                h = self._neighborhood(bmu, step)
                lr = self.lr * np.exp(-step / self.epochs)
                self.weights += lr * h[:, None] * (x - self.weights)

    def map_vectors(self, X):
        return np.array([self._bmu(x) for x in X])

Strength
Better than K-Means when visualization and topology preservation matter.
Weakness
Grid size is fixed in advance, so clusters can merge or fragment if the grid is poorly chosen.
No Uncertainty Model
Each input maps to one neuron; probabilistic cluster assignment is not modeled.
Limitations

SOM is sensitive to learning-rate decay, neighborhood radius, and grid design. It also becomes expensive for large datasets because every input is compared with all neurons repeatedly.

Modern Extension

Recent systems combine CNN or transformer feature extractors with SOM-like topological organization. The neural encoder learns stronger features, then SOM organizes them for visualization and interpretable structure.

05 — Where SOM Fits

A Bridge Between Classical and Deep Clustering

SOM organizes data through competitive neural learning and topology preservation, but it still relies on a predefined grid and does not directly optimize a clustering objective. That makes it a powerful visualization and representation tool, but less flexible than later deep clustering methods.

Position in the Course

SOM is better than K-Means for visualization, less flexible than modern deep clustering, and especially useful when interpretability matters. The next step is learning embeddings and cluster assignments simultaneously.
