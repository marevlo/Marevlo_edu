01 — Community Detection and Louvain

The Louvain Method

Large graphs such as social networks, recommendation systems, citation graphs, and knowledge graphs can contain millions of nodes and edges. Traditional clustering becomes expensive in this regime, so community detection focuses on identifying densely connected subgraphs with sparse inter-community links.

Concept Overview

Louvain is one of the most widely used large-scale community detection algorithms because it uses greedy hierarchical optimization rather than searching the full partition space.

Community Structure in a Large GraphANIMATED

Modularity Objective

Louvain optimizes modularity, which compares the observed edge density inside communities against the density expected from a randomized degree-preserving baseline.

Modularity FormulaFORMULA
Q = (1 / 2m) sum[i,j] [Aij - (ki kj / 2m)] delta(ci, cj)
Total Edge Weight
m = (1 / 2) sum[i,j] Aij

Each undirected edge appears twice in the adjacency matrix, so modularity uses this normalization to keep scale correct.

Interpretation

If actual within-community connectivity exceeds the expected random connectivity, modularity rises. Louvain repeatedly searches for moves that increase this value.

02 — Louvain Optimization Phases

Local Modularity Optimization and Graph Aggregation

Louvain alternates between two phases. It first lets each node move into neighboring communities if the modularity gain is positive. Once no better local moves remain, each community is collapsed into a super-node and the process repeats on the compressed graph.

Modularity GainFORMULA
\Delta Q = [(Sigma_in + k_i,in)/2m - ((Sigma_tot + k_i)/2m)^2] - [Sigma_in/2m - (Sigma_tot/2m)^2]
  • Initialize with one node per community.
  • Evaluate moving each node into each neighboring community.
  • Take the move with the largest positive ΔQ.
  • Stop the first phase when no positive gain remains.
  • Compress communities into super-nodes and repeat.
Louvain Hierarchical FlowANIMATED
Key Takeaway

Louvain is greedy, not exhaustive. That is why it scales so well, but it also explains why the final partition can depend on update order and local structure.

04 — Leiden Algorithm

Leiden Improves Community Quality

Leiden preserves the modularity objective but improves the optimization strategy. Its key contribution is a refinement stage between local node movement and graph aggregation, ensuring that each detected community remains internally connected.

Connectivity Constraint
\forall u,v in C, there exists a path from u to v entirely inside C

Unlike Louvain, Leiden guarantees that a community is a connected subgraph rather than a loose or fragmented collection of nodes.

Leiden RefinementANIMATED
  1. Local node movement using modularity gain, just like Louvain.
  2. Community refinement to split weak or disconnected groups into valid connected subsets.
  3. Graph aggregation on the refined communities.
Key Takeaway

Leiden does not change the objective. It changes the path the optimizer takes, which is why it usually produces more stable and higher-quality communities than Louvain.

05 — Leiden Math and Practical Use

Refinement, Resolution, and Implementation

Leiden still uses the same modularity objective and the same local ΔQ rule, but it adds subset-level refinement so that communities are not accepted unless they remain structurally coherent.

Refinement and AggregationFORMULA
C -> {C1, C2, ..., Ck} with each Ci connected
\Delta Q(S -> C') >= 0
G' = (C, E')
Resolution ParameterFORMULA
Q(gamma) = (1 / 2m) sum[i,j] [Aij - gamma(ki kj / 2m)] delta(ci,cj)

06 — Markov Clustering

Flow-Based Community Detection with MCL

Markov Clustering takes a very different route. Instead of optimizing modularity, it simulates random walks on the graph. The idea is that flow tends to remain inside dense regions and rarely crosses weak bridges, so cluster structure emerges naturally from repeated diffusion and sharpening.

Core Operations

MCL alternates between expansion, which explores multi-step connectivity, and inflation, which strengthens strong transitions while suppressing weak ones.

Expansion and InflationANIMATED
MCL Mathematical CoreFORMULA
G = (V,E)
A in R^(n x n)
Mij = Aij / sum_k Akj
\sum_i Mij = 1
Expansion
M^(t+1) = M^e, usually e = 2

Expansion simulates multi-step random walks and reveals indirect connectivity.

Inflation
Mij = Mij^r / sum_k Mkj^r

Inflation sharpens strong transitions and weakens weak ones, which is what creates community separation.

Pruning and Convergence
Mij < epsilon => Mij = 0
M^(t+1) ~= M^(t)

Pruning improves stability and speed; convergence means the flow has concentrated into near block-diagonal communities.

Key Takeaway

Louvain and Leiden ask which partition improves modularity. MCL asks where random walks naturally get trapped. That makes it especially intuitive for strongly modular networks.

07 — MCL in Practice

Implementation, Cost, and Biological Relevance

Python
NUMPY
import numpy as np

def normalize(M):
    return M / M.sum(axis=0, keepdims=True)

def expand(M, power=2):
    return np.linalg.matrix_power(M, power)

def inflate(M, r=2):
    M = np.power(M, r)
    return normalize(M)

def mcl(M, expand_power=2, inflate_power=2, max_iter=20, tol=1e-5):
    M = normalize(M)
    for _ in range(max_iter):
        M_prev = M.copy()
        M = expand(M, expand_power)
        M = inflate(M, inflate_power)
        if np.allclose(M, M_prev, atol=tol):
            break
    return M
Inflation Parameter
As r approaches 1, clusters become larger; larger r creates tighter, smaller clusters.
Time Complexity
O(n^3) in the dense-matrix view because matrix multiplication dominates.
Typical Domain
Widely used in biological protein-interaction networks and other strongly modular graphs.
