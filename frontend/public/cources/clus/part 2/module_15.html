Graph-Based & Spectral Clustering

Full graph representation generalizes Euclidean and tree-based clustering by keeping pairwise relationships explicit. Spectral methods then turn graph partitioning into an eigenvalue problem, revealing non-convex structure that geometry alone misses.

01 — Transition

From Euclidean Clustering to Graph Learning

Euclidean clustering methods such as K-Means and DBSCAN collapse once data violate convexity, uniform density, or metric assumptions. K-Means forces spherical Voronoi partitions, DBSCAN becomes extremely parameter-sensitive, and MST clustering improves global connectivity but remains a tree that is brittle to single noisy bridges.

Motivation

Full graph representation generalizes all of these approaches by retaining pairwise relationships explicitly. This turns clustering into a global optimization problem over connectivity rather than geometry or density alone.

02 — Graph Representation of Data

Graph Representation of Data

Define the data graph G = (V, E), where V = {x1, …, xn} are the data points and E encodes pairwise relationships. For clustering we use undirected weighted graphs, and edge weight wij ≥ 0 quantifies similarity, where larger means stronger connection.

Similarity Functions

The choice of wij is the single most consequential design decision. In practice, this usually means choosing an affinity kernel and its parameterization carefully, after feature normalization.

Practical Implication

Always normalize features first. Otherwise, σ tuning becomes meaningless because the similarity kernel is dominated by scale rather than structure.

03 — Graph Matrices and Laplacians

Graph Matrices and Normalization

  • Adjacency matrix A in ℝn×n, where Aij = wij.
  • Degree matrix D = diag(d1, …, dn), where di = Σj Aij.
  • Unnormalized Graph Laplacian: L = D - A.
Laplacian DefinitionFORMULA
Core Matrix
L = D - A

Normalized Laplacians

Normalized FormsFORMULA
Lsym = D^-1/2 L D^-1/2
Lrw = D^-1 L
Why Normalization Matters

Unnormalized L tends to cut off low-degree nodes, which is undesirable. Normalized variants remove degree bias, align with random-walk diffusion distances, and produce eigenvectors that behave more like cluster indicators when cluster sizes differ strongly.

04 — Spectral Clustering Intuition

Spectral Clustering: From Graph Cuts to Eigenvectors

Treat clustering as graph partitioning: cut the graph into k subgraphs so that intra-cluster edges are strong and inter-cluster edges are weak. Naive min-cut fails because it prefers cutting off tiny low-degree regions, so balanced objectives such as RatioCut and NormalizedCut are used instead.

Critical Point

The discrete cluster-indicator formulation is NP-hard. Spectral clustering works by relaxing the discrete cut problem into a continuous eigen-problem over the Laplacian.

Optimization to EigenvectorsFORMULA
Relaxed Solution
Take the top k smallest eigenvectors of the Laplacian
  • The multiplicity of the zero eigenvalue equals the number of connected components.
  • The Fiedler vector, the second smallest eigenvector, gives the optimal 2-way cut.
  • Larger eigenvalues correspond to higher-frequency modes and finer cuts.

Spectral Embedding

Embedding MatrixFORMULA
U = [v1, v2, ..., vk] in R^(n x k)

Each row ui becomes the new coordinate of node i. In this embedding space, clusters that were non-convex in the original space often become linearly separable because the eigenvectors encode diffusion structure rather than Euclidean shape.

Uses Lsym and row-normalizes U. Best empirical stability in practice.

Uses Lrw. Interpretable in terms of diffusion and stationary random walks.
Default Guidance

Use normalized variants almost always. Unnormalized spectral clustering is only reasonable when cluster sizes are perfectly balanced and degree structure is benign.

GRAPH BUILD
import numpy as np
from scipy.spatial.distance import pdist, squareform

def build_gaussian_graph(X, sigma=1.0):
    dist = squareform(pdist(X))
    A = np.exp(-dist**2 / (2 * sigma**2))
    np.fill_diagonal(A, 0)
    return A
Python
SPECTRAL
from scipy.sparse.linalg import eigsh
from sklearn.cluster import KMeans

def spectral_clustering(A, k, norm='sym'):
    D = np.diag(A.sum(axis=1))
    if norm == 'sym':
        L = np.diag(D.diagonal()**(-0.5)) @ (D - A) @ np.diag(D.diagonal()**(-0.5))
    else:
        L = np.linalg.inv(D) @ (D - A)
    vals, vecs = eigsh(L, k=k, which='SM')
    U = vecs[:, :k]
    if norm == 'sym':
        U = U / np.linalg.norm(U, axis=1)[:, None]
    return KMeans(n_clusters=k).fit_predict(U)
Engineering Note

scikit-learn's SpectralClustering uses the normalized NJW-style pipeline by default. For sparse graphs and large inputs, use ARPACK or Lanczos-style partial solvers.

06 — Complexity, Failure Cases, and Scaling

Complexity, Failure Cases, and Practical Issues

Complexity AnalysisTABLE
StepComplexityMain Bottleneck

Eigen-decompositionO(n³) dense or O(n²) sparse iterativeDominant scaling limit
  • Wrong σ can disconnect the graph or make it too dense.
  • Sparse noisy graphs can contaminate eigenvectors and collapse the embedding.
  • Multiscale data often cannot be handled by a single σ.
  • For very large n, full eigen-decomposition becomes intractable.
Failure ModesANIMATED
Eigengap Heuristic
Choose k at the largest gap λk+1 - λk.
Out-of-Sample Limitation
Spectral clustering is transductive and has no direct mapping for new points.
Feature Sensitivity
Garbage features create garbage similarities and a meaningless spectrum.
Practical Issues

Near-zero degrees create numerical instability in D-1/2; disconnected graphs create multiple zero eigenvalues; and repeated runs can flip eigenvector signs even when the cluster geometry is unchanged.

Scaling Strategies

  • Nyström approximation for landmark-based low-rank extension.
  • Sparse graphs plus Lanczos or ARPACK for partial eigen-decomposition.
  • Approximate nearest-neighbor libraries such as HNSW or FAISS for graph construction.

Key Insights

Spectral clustering can be understood as diffusion-map-style embedding followed by K-Means. Graph construction is 80% of success, normalization is non-negotiable, and eigengap is usually more reliable than elbow-style heuristics for choosing k.
