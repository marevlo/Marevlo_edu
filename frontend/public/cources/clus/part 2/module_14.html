Minimum Spanning Tree (MST) Based Clustering

Graph-based clustering removes geometric shape assumptions by representing data as a graph, then extracting its essential backbone. The MST keeps the minimum-cost connectivity structure, and clustering emerges by cutting weak links.

01 — Overview

Why MST-Based Clustering?

Previous methods such as K-Means and DBSCAN rely on assumptions about cluster shape or density. K-Means assumes spherical clusters, while DBSCAN depends on carefully chosen density parameters and can struggle with varying densities.

Graph-based clustering removes these assumptions by representing the dataset as a graph: nodes represent data points, edges represent relationships between points, and clustering is performed by analyzing connectivity rather than centroids or density thresholds.

Key Takeaway

To focus on essential connections, the graph is reduced to a Minimum Spanning Tree, which keeps only the cheapest links needed to connect all nodes. Clustering is then achieved by cutting weak edges in that tree.

02 — Graph-Based View of Data

Graph-Based View of Data

Notation Summary X = {x1, x2, …, xn} is a dataset of n data points, each xi in ℝd. G = (V, E, W) is an undirected graph, V = {1, 2, …, n} is the set of nodes, E is the set of edges, and w(i,j) is the weight of the edge between nodes i and j.

  • A connected component is a maximal subset of nodes in which every pair is linked by a path.
  • A cluster is a group of data points that form a connected component after processing the graph.
  • Each connected component in the graph corresponds to a cluster.
Problem Definition
Input: dataset X of n points in ℝd. Output: a partition of the n points into k clusters, where k can be discovered automatically.
  • Pairwise distances or similarities between points can be computed in a chosen metric.
  • Natural clusters appear as densely connected groups separated by relatively weak connections.
  • The graph construction produces a connected graph, or disconnected cases are handled explicitly.

03 — Minimum Spanning Tree

Minimum Spanning Tree

Given a connected undirected graph G = (V, E, W), a Minimum Spanning Tree (MST) is a subset of edges T ⊆ E that connects all nodes, contains no cycles, and minimizes the total weight.

MST ObjectiveFORMULA
Minimum Total Connection Cost
min_T Σ(i,j in T) w(i,j)
  • Exactly n - 1 edges for n nodes.
  • Connects all nodes without cycles.
  • Retains the most essential low-cost connections.
Cut Property
For any partition of V into two nonempty sets S and V∖S, the edge of smallest weight that crosses the cut belongs to some MST.
Clustering Interpretation

The MST preserves the cheapest backbone of connectivity. Removing a few expensive edges from that backbone naturally separates the graph into connected components, which become the clusters.

04 — MST Construction

Kruskal's and Prim's Algorithms

Kruskal's Algorithm

  • Sort all edges in non-decreasing order of weight.
  • Initialize each node as its own connected component.
  • If an edge connects different components, add it to the MST and merge them.
  • Stop when the MST contains n - 1 edges.

Complexity O(E log E), dominated by sorting.

Python
IMPLEMENTATION
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))

    def find(self, i):
        if self.parent[i] == i:
            return i
        self.parent[i] = self.find(self.parent[i])
        return self.parent[i]

    def union(self, i, j):
        root_i = self.find(i)
        root_j = self.find(j)
        if root_i != root_j:
            self.parent[root_i] = root_j
            return True
        return False

def kruskal_mst(vertices, edges):
    # edges = [(weight, u, v)]
    edges.sort()
    uf = UnionFind(vertices)
    mst = []
    for weight, u, v in edges:
        if uf.union(u, v):
            mst.append((u, v, weight))
            if len(mst) == vertices - 1:
                break
    return mst
Code Explanation

Union-Find prevents cycles, edge sorting ensures global order, and the output mst is the structural backbone of the data.

Prim's Algorithm

  • Start with an arbitrary node and place it in the MST set.
  • Maintain a priority queue of edges connecting the tree to outside nodes.
  • Repeatedly add the minimum-weight edge that reaches a new node.
  • Stop when all nodes belong to the tree.

Complexity O(E log V) with a binary heap.

Python
IMPLEMENTATION
import heapq

def prim_mst(vertices, adjacency_list):
    mst = []
    visited = set([0])
    edges = [(weight, 0, neighbor) for weight, neighbor in adjacency_list[0]]
    heapq.heapify(edges)

    while len(visited) < vertices:
        weight, u, v = heapq.heappop(edges)
        if v not in visited:
            visited.add(v)
            mst.append((u, v, weight))
            for next_weight, neighbor in adjacency_list[v]:
                if neighbor not in visited:
                    heapq.heappush(edges, (next_weight, v, neighbor))
    return mst
Intuition

Kruskal adds globally cheapest safe edges. Prim grows one connected tree outward by always attaching the closest external node.

05 — MST Clustering

MST Clustering by Edge Removal

Core Idea Build the MST of the data graph, then remove a few high-weight edges. The resulting connected components become the clusters.

Mathematical FormulationFORMULA
After Sorting MST Edges
w1 ≤ w2 ≤ ... ≤ w(n-1)
Remove the k - 1 largest edges to obtain k connected components
  • Construct graph G from dataset X.
  • Compute the MST of G.
  • Sort MST edges by weight.
  • Choose k or a threshold.
  • Remove the k - 1 largest edges or all edges above threshold.
  • Identify connected components in the remaining forest.

Hierarchical Link

MST clustering is exactly equivalent to single-linkage hierarchical clustering. Removing edges in decreasing weight order reproduces the single-link dendrogram.

Edge Cut Strategies

Largest Gap
Sort MST edges, find the biggest jump, then cut edges above that jump.
Threshold R
Remove any edge whose weight strictly exceeds a predefined distance parameter R.
Inconsistency
Compare an edge to its local neighborhood and remove it if it is abnormally long.

Failure Cases

  • Chaining: a thin bridge can merge two distant clusters.
  • Noise: isolated points create many tiny components unless filtered.
  • Varying density: a valid low-density cluster may be split.
  • Uniform edge weights: no meaningful gap exists, so clustering degenerates.
Stability Analysis

MST-based clustering can be highly unstable. A single strategically placed noise point can create a shortcut that changes the whole tree structure.

Preprocessing
z = (x - μ) / σ is recommended before distance computation. Outliers should be removed or down-weighted because they can artificially lengthen MST edges and distort cuts.

MST vs K-Means
MST handles arbitrary shapes and does not need pre-specified k, while K-Means is faster for spherical clusters.
MST vs DBSCAN
MST is deterministic given the graph and less tied to local density parameters, but DBSCAN explicitly models noise.
When Not to Use MST

Do not use MST clustering when the dataset contains heavy noise, clusters have highly varying internal densities, soft assignments are required, or computational resources are extremely limited and n is huge.

Worked Example

Consider four 2D points: A(0,0), B(0,1), C(4,0), D(4,1). Visually, A/B form one group and C/D form another.

  • Distance(A,B) = 1 and Distance(C,D) = 1.
  • Distance(A,C) = 4 and Distance(B,C) ≈ 4.12.
  • Kruskal selects (A,B), then (C,D), then (A,C).
  • To get k = 2 clusters, remove the longest remaining edge, (A,C).
  • Component 1 = {A,B}, Component 2 = {C,D}.

Sanity Checks

After obtaining clusters, verify that intra-cluster MST edges are shorter than removed inter-cluster edges, inspect the number of connected components, and use silhouette or 2D visualization where available.

07 — Similarity Graph Construction

Similarity Graph Construction

Why Graph Construction Matters MST-based clustering relies only on the final tree and discards most pairwise information. Spectral clustering needs a richer graph that preserves local neighborhood structure more faithfully.

  • Fully connected graph: every pair is linked.
  • k-NN graph: each node connects to its k nearest neighbors.
  • ε-neighborhood graph: edge if distance < ε.
  • Mutual k-NN graph: edge only if both nodes appear in each other's k nearest neighbors.

Disconnected graphs, overly dense graphs, and high parameter sensitivity can drastically change connectivity. This graph construction step directly shapes the later Laplacian matrices used in spectral clustering.
