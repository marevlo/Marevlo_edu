Affinity Propagation

Affinity Propagation discovers exemplars directly from pairwise similarities. Instead of specifying k, it lets representative data points emerge through message passing, balancing local preference against global support.

01 — Motivation and Positioning

Why Affinity Propagation Exists

Many clustering methods need the number of clusters in advance. K-Means requires k and assumes roughly spherical structure. Spectral clustering still requires k after graph embedding, and its quality depends heavily on how the graph is built. In real datasets, the number of clusters is often unknown and the shapes are rarely simple.

Core Goal

Affinity Propagation solves a different problem: discover cluster representatives directly from similarity relationships, without graph embedding and without predefining the cluster count.

02 — Message Passing Intuition

Responsibility, Availability, and Stable Exemplars

Affinity Propagation treats every point as a potential cluster representative. Instead of assigning labels immediately, points exchange two kinds of messages until a stable set of exemplars appears.

Responsibility
How strongly point i prefers candidate k over the alternatives.
Availability
How strongly the rest of the dataset supports candidate k as a reliable exemplar.
Decision Rule
A point becomes an exemplar when it strongly prefers itself and also receives strong support from others.
Message-Passing FlowANIMATED
Iteration Dynamics

Early iterations contain many plausible exemplar candidates. In the middle phase, candidates compete. By the final phase, only stable exemplars remain and assignments stop changing.

Key Takeaway

Responsibility is a local “I want this representative” score. Availability is a global “this representative is genuinely supported” score. Affinity Propagation succeeds only when both agree.

03 — Mathematical Formulation

Similarity Matrix and Message Updates

Let the dataset contain n points and define a similarity matrix S(i,k), where i is the data point and k is the candidate exemplar. In the standard Euclidean form, similarity is the negative squared distance, so close points receive higher values because they are less negative.

Similarity DefinitionFORMULA
Negative Squared Distance
S(i,k) = - ||x_i - x_k||^2

Responsibility Update

Responsibility measures whether candidate k is better for point i than the strongest alternative.

Responsibility FormulaFORMULA
r(i,k) = s(i,k) - max_{k' != k}(a(i,k') + s(i,k'))
Responsibility CompetitionANIMATED

Availability Update

Availability collects positive evidence from other points. Self-availability measures whether candidate k deserves to become an exemplar at all, while non-self availability controls whether point i should choose it.

Availability FormulasFORMULA
a(k,k) = sum_{i' != k} max(0, r(i',k))
a(i,k) = min(0, r(k,k) + sum_{i' notin {i,k}} max(0, r(i',k)))
Interpretation

Only positive support contributes. Negative evidence is not allowed to endlessly suppress a candidate; instead, availability is capped so too many exemplars do not appear.

Assignment Score, Preference, and Exemplar Criterion

Cluster Assignment Rule
\hat{k}(i) = argmax_k (a(i,k) + r(i,k))

The chosen exemplar is the candidate with the highest combined global support and local preference.

Default Preference Setting
s(k,k) = median(s(i,k))

Preference is the self-similarity value that indirectly controls how many exemplars survive.

Exemplar Identification
a(k,k) + r(k,k) > 0

If the self-support plus self-preference becomes positive, point k is declared an exemplar.

Key Takeaway

Affinity Propagation does not “choose k clusters.” It shapes a competition in which self-similarity, local evidence, and global support determine how many exemplars are viable.

04 — Preference and Cluster Count

Preference Replaces the Role of k

Each point has a self-similarity value S(k,k), called the preference. A higher preference makes that point more likely to become an exemplar. A lower preference suppresses candidacy. This is how Affinity Propagation indirectly controls cluster count instead of asking for k explicitly.

Preference EffectANIMATED
High Preference
Many points qualify as exemplars, often creating too many small clusters.
Median Preference
Balanced behavior and the most common practical starting point.
Low Preference
Very few exemplars survive, causing under-segmentation.
Failure Mode

Affinity Propagation is very sensitive to preference. Small changes can drastically change the number of clusters, so cluster stability should be inspected across nearby values.

Practical Guideline

Start at the median similarity, then move preference upward for finer granularity or downward for coarser clusters. This is the Affinity Propagation analogue of controlling model complexity.

05 — Full Algorithm

Step-by-Step Affinity Propagation

  1. Compute the similarity matrix S(i,k) for all point pairs.
  2. Initialize the responsibility and availability matrices to zero.
  3. Update responsibility using the best competing candidate for each point.
  4. Update availability using positive support received by each candidate exemplar.
  5. Apply damping to avoid oscillation.
  6. Repeat until messages stabilize and assignments stop changing.
  7. Identify exemplars using a(k,k) + r(k,k) > 0.
  8. Assign each point to the exemplar that maximizes a(i,k) + r(i,k).

Key Takeaway

The algorithm is simple to state, but its behavior is governed by a delicate interaction between similarity choice, preference setting, and damping. That is why practical tuning matters so much.

06 — Complexity and Practical Usage

Implementation, Parameters, and When to Use It

Time Complexity
O(n^2) per iteration because every point interacts with every candidate exemplar.
Memory Complexity
O(n^2) for similarity, responsibility, and availability matrices.
Scalability
Good for small-to-medium datasets, but expensive for very large collections.

SCIKIT-LEARN
import numpy as np
from sklearn.cluster import AffinityPropagation

def affinity_propagation_clustering(X):
    model = AffinityPropagation(random_state=0)
    model.fit(X)

    labels = model.labels_
    exemplars = model.cluster_centers_

    return labels, exemplars
