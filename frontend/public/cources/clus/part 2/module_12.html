Gaussian Mixture Models (GMM)

Real-world data is rarely cleanly separated. Clusters often overlap, and rigid boundaries fail to capture uncertainty. To model such data, we move from deterministic clustering to probabilistic modeling.

01 — Why K-Means Fails

Why K-Means Fails

  • Hard boundaries break when clusters overlap
  • Cannot represent uncertainty
  • Assumes spherical clusters
02 — Gaussian Mixture Model

Gaussian Mixture Model

A single Gaussian distribution forms one bell curve. It represents one group of data. Real datasets contain multiple groups. To model the entire dataset, we must combine multiple Gaussians together. This creates a Gaussian Mixture Model (GMM).

A GMM assumes the dataset was generated by a mixture of several hidden Gaussian distributions.

03 — The Mixture Intuition

The Mixture Intuition

If you have three distinct groups in your data, a GMM will place three separate Gaussian bell curves on the graph. By adding the curves together, the model maps the complex, overlapping shapes of the complete dataset.

04 — Soft Clustering Formulation

Soft Clustering Formulation

This overlap is what makes probabilistic assignment necessary, as a single point can be explained by multiple clusters simultaneously.

Because Gaussians stretch out to infinity, they overlap. A data point located in an overlapping region belongs to multiple distributions at the same time. The GMM calculates the exact percentage of ownership each cluster has over that specific point.

05 — Model Components

Model Components

06 — Core GMM Parameters

Core GMM Parameters

Formula / Core Expression
Means (μ): The center coordinate of the cluster.
Formula / Core Expression
Covariances (Σ): The geometric shape, spread, and diagonal tilt of the cluster.
Formula / Core Expression
Mixing Coefficients (π): The overall size or weight of the cluster.

Some clusters contain more data points than others. The mixing coefficient (πk) is the probability that a randomly selected data point belongs to cluster k, before you even look at its coordinates. If Cluster 1 contains 80% of the total dataset, its mixing coefficient is 0.8. The sum of all mixing coefficients in the model must equal exactly 1.0.

These coefficients act as prior probabilities, indicating how likely a point is to come from a particular cluster before observing its features.

The GMM Formula

To find the probability of a data point x existing in the entire dataset, we add up its probabilities from all the individual clusters:

Single-Point Mixture FormulaFORMULA
Probability of One Data Point Under the Mixture
p(x) = Σ(k=1→K) πk N(x | μk, Σk)

Full Likelihood of Dataset

For a dataset of N points, the likelihood is:

Dataset LikelihoodFORMULA
Probability of the Entire Dataset
L(X) = Π(i=1→N) Σ(k=1→K) πk N(xi | μk, Σk)
07 — Log-Likelihood Objective

Log-Likelihood Objective

This represents the probability of observing the entire dataset under the model.

To simplify optimization, we take the logarithm:

Log-LikelihoodFORMULA
Objective Maximized by EM
log L(X) = Σ(i=1→N) log[Σ(k=1→K) πk N(xi | μk, Σk)]

This is the objective function that EM maximizes.

Expectation-Maximization (EM) Algorithm

We cannot directly optimize the likelihood because cluster assignments are unknown.

The EM algorithm solves this by alternating between two steps:

  • Initialize parameters
  • E-Step: compute responsibilities (soft assignments)
  • M-Step: update parameters using weighted averages
  • Repeat until convergence

The E-step does not move the means yet. It only asks: given the current means, what is the posterior probability that each cluster generated this point?

Latent Variable (z)

This hidden variable represents which cluster generated each data point.

09 — Bayesian Interpretation

Bayesian Interpretation

This equation applies Bayes’ rule by combining prior probabilities (mixing coefficients) and likelihoods (Gaussian densities) to compute posterior probabilities (responsibilities).

11 — The M-Step (Maximization)

The M-Step (Maximization)

Now we have the responsibilities (γ). Every point is fractionally assigned to every cluster. We use the "Expectation Concept" (weighted averages) from Part 2 to update the rules for cluster k.

We calculate the effective number of points assigned to cluster k:

Effective Count VisualANIMATED
Formula / Core Expression
We simply add up all the responsibility percentages for cluster k. If 10 points have a 0.5 responsibility to Cluster A, Cluster A effectively has Nk = 5 full points.
Key Takeaway

The M-Step converts soft ownership into weighted parameter updates, so strongly assigned points influence the cluster most.

12 — Update the Mean (μk)

Update the Mean (μk):

Formula / Core Expression
μknew = (1 / Nk) Σi γik xi

We multiply every data point xi by its responsibility to cluster k, add them all together, and divide by Nk. Points with high responsibility pull the center closer to them. Points with near-zero responsibility do not move the center.

13 — Update the Covariance (Σk)

Update the Covariance (Σk):

Formula / Core Expression
Σknew = (1 / Nk) Σi γik (xi - μknew)(xi - μknew)T

We calculate the distance of every point from the new mean, square it, multiply it by the point's responsibility, and average it. This adjusts the tilt and spread of the cluster to fit the points assigned to it.

14 — Update the Mixing Coefficient (πk)

Update the Mixing Coefficient (πk):

Formula / Core Expression
πknew = Nk / N

Numerical Example of EM

Formula / Core Expression
Imagine a 1D dataset with two points: x₁ = 2 and x₂ = 5. We initialize two clusters with means μ₁ = 3 and μ₂ = 6, equal variances, and mixing coefficients π₁ = π₂ = 0.5.
  • E-Step (Responsibilities): We calculate how much each cluster "claims" each point. Since x₁ is closer to μ₁, its responsibility might be γ₁₁ = 0.8 and γ₁₂ = 0.2. For x₂, which is closer to μ₂, responsibilities might be γ₂₁ = 0.3 and γ₂₂ = 0.7.
  • M-Step (Parameter Update): We update the center of Cluster 1 using a weighted average: μ₁_new = (0.8 * 2 + 0.3 * 5) / (0.8 + 0.3) ≈ 2.82. The clusters shift to better fit the points they are most responsible for.
  • Effective Number of Points (Nₖ): Calculate the total responsibility for each cluster: N₁ = γ₁₁ + γ₂₁ = 0.8 + 0.3 = 1.1 and N₂ = γ₁₂ + γ₂₂ = 0.2 + 0.7 = 0.9.
  • Update the Second Mean (μ₂_new): Weighted average for Cluster 2: μ₂_new = (0.2 * 2 + 0.7 * 5) / 0.9 ≈ 4.33.
  • Update Mixing Coefficients (πₖ_new): Ratio of effective points to total points (N=2): π₁_new = 1.1 / 2 = 0.55 and π₂_new = 0.9 / 2 = 0.45.

16 — Convergence

Convergence

  • Log-likelihood increases after each EM iteration
  • The increase slows over time
  • Eventually, it stabilizes

The algorithm stops when the improvement becomes negligible.

17 — Local Optima Trap

Local Optima Trap

The EM algorithm is powerful, but it is blind. It relies entirely on where you place the random starting clusters in Step 1.

Critical Point

If you initialize the starting clusters in a terrible location, the EM algorithm will optimize them into the nearest available mathematical valley. This is a "local optimum." It is the best answer in that specific area, but it is not the correct answer for the whole dataset.

To fix this, we never use completely random initialization. We run the K-Means algorithm first. K-Means quickly finds rough, hard boundaries. We use the K-Means centers as the starting positions for the EM algorithm, ensuring it begins near the correct answer.

The EM algorithm transforms clustering into a probabilistic optimization process. Instead of assigning points directly, it models how data is generated and iteratively refines both assignments and parameters.

18 — Likelihood Stabilization

Likelihood Stabilization

After every single M-Step, the algorithm calculates the total probability of the entire dataset based on the new cluster positions. Because the M-Step is mathematically designed to maximize likelihood, this total probability number will increase every single loop.

When the clusters settle into their final positions, the total probability number stops growing. It becomes flat. This is called convergence. The algorithm stops.

19 — Covariance Types

Covariance Types

  • Full: each cluster has its own covariance matrix
  • Diagonal: no feature correlation
  • Spherical: equal variance in all directions

This affects flexibility and computational cost.

Covariance Types Explained

Spherical keeps every direction equally spread, Diagonal lets each feature scale differently but assumes no cross-feature correlation, and Full allows rotated ellipses so the cluster can model correlation between features.

Key Takeaway

More expressive covariance types capture richer ellipse shapes, but they also increase the number of parameters the model must learn.

23 — PYTHON IMPLEMENTATION

PYTHON IMPLEMENTATION

Python
IMPLEMENTATION
from sklearn.mixture import GaussianMixture
from sklearn.datasets import make_blobs
from sklearn.preprocessing import StandardScaler
import matplotlib.pyplot as plt
import numpy as np
# 1. Generate Synthetic Data
# Data points are generated from 3 distinct, but slightly overlapping, clusters
X, y_true = make_blobs(n_samples=300, centers=3, cluster_std=1.5, random_state=42)
# 2. Standardize Data (Critical for scaling features)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
# 3. Initialize and Train the GMM (EM Algorithm in action)
# n_components=3 (K=3 clusters)
# covariance_type='full' allows for elliptical clusters
gmm = GaussianMixture(n_components=3, covariance_type='full', random_state=42, n_init=10)
gmm.fit(X_scaled)
# 4. Perform Soft Assignment (The E-Step: Responsibilities)
# predict_proba returns the responsibility matrix (γik)
responsibilities = gmm.predict_proba(X_scaled)
# 5. Perform Hard Assignment (Prediction)
# predict() performs argmax_k P(zi = k | xi) to get the final label
cluster_labels = gmm.predict(X_scaled)
# 6. Print Model Parameters (The M-Step Results)
print("--- GMM Model Parameters (M-Step Results) ---")
print(f"Cluster Means (μk):\n{gmm.means_}")
print(f"\nMixing Coefficients (πk):\n{gmm.weights_}")
print(f"\nLog-Likelihood at Convergence: {gmm.score(X_scaled):.2f}")
# 7. Simple Visualization of Hard Clustering
plt.scatter(X_scaled[:, 0], X_scaled[:, 1], c=cluster_labels, s=40, cmap='viridis', alpha=0.7)
plt.title("GMM Hard Clustering Result")
plt.xlabel("Feature 1 (Scaled)")
plt.ylabel("Feature 2 (Scaled)")
plt.show()
