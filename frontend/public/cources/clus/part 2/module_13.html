Clustering in High-Dimensional Spaces

We have seen how powerful algorithms like K-Means and GMMs are in low dimensions, but as the number of features scales into the hundreds or thousands, the very laws of geometry break down. We must abandon standard Euclidean distance and shift to grid-based, subspace, and projected clustering techniques.

01 — Overview

Overview

As the dimensionality increases, the behavior of distance and volume becomes highly counterintuitive. Classical clustering methods fail because both distance and density lose meaning in high dimensions.

Key Takeaway

High-dimensional clustering is not just "harder clustering." It requires different assumptions, different geometry, and often completely different algorithms.

02 — 5.1 The Curse of Dimensionality

5.1 The Curse of Dimensionality (The Geometric Problem)

Key Idea As the dimensionality d increases, the behavior of distance and volume becomes highly counterintuitive. This causes traditional distance-based clustering methods to break down.

Distance Concentration (Loss of Meaningful Distance)

Distance ConcentrationFORMULA
As Dimensionality Grows
lim(d→∞) (distmax - distmin) / distmin = 0
Where
d = dimensionality of the space, distmax = maximum pairwise distance, distmin = minimum pairwise distance.

Interpretation As d increases, distmax - distmin → 0 and all pairwise distances converge to the same value. Thus, distmin ≈ distmax.

  • Nearest neighbor ≈ farthest neighbor.
  • Distance loses discriminative power.
  • Algorithms such as K-Means and DBSCAN become ineffective.
Mental Model

In high-dimensional space, all points are almost equally distant.

Near vs Far CollapseANIMATED

Example (Intuition) In low dimensions (2D or 3D), there is clear separation between near and far points. In high dimensions (100+), distmin ≈ distmax and distance becomes almost constant.

Conclusion Distance is no longer a useful signal.

Volume Explosion and Sparsity

Hypersphere VolumeFORMULA
Volume of a d-Dimensional Hypersphere
Vd(r) = [π^(d/2) / Γ(d/2 + 1)] r^d
Special Case
Γ(n) = (n - 1)! for integer n.

As d increases, the denominator Γ(d/2 + 1) grows faster than exponential, the ratio shrinks rapidly, and therefore Vd(r) → 0 relative to the enclosing space.

  • Most of the volume concentrates near the boundary.
  • The interior becomes nearly empty.
  • Data becomes extremely sparse.
  • Density-based reasoning becomes unreliable.
  • Local neighborhoods lose meaning.
Mental Model

High-dimensional space is mostly empty, and data lives near the edges.

Boundary ConcentrationANIMATED
Final Takeaway

Distance behavior: distmin ≈ distmax means no separation. Volume behavior: Vd(r) → 0 means extreme sparsity. Both together break classical clustering intuition.

03 — 5.2 Breakdown of Traditional Models

5.2 Breakdown of Traditional Models (The Impact)

K-Means Degradation in High Dimensions

Core Principle K-Means assigns each point to the nearest centroid using Euclidean distance.

K-Means ObjectiveFORMULA
Objective Function
J = Σ || xi - μk ||^2

From Section 5.1, distmin ≈ distmax. This implies ||x - μ1|| ≈ ||x - μ2|| ≈ ... ≈ ||x - μk||.

  • All centroids appear equally distant.
  • The assignment step becomes ambiguous.
  • Cluster assignments become effectively random.
  • Centroids become unstable across iterations.
  • Objective function J loses discriminative meaning.
Mental Model

K-Means relies on distance contrast. High dimensions eliminate that contrast.

Failure Example Consider d = 1000 dimensions with randomly distributed data. Pairwise distances concentrate and K-Means cannot form meaningful clusters.

Gaussian Mixture Model (GMM) Breakdown

Core Principle Each cluster is modeled as a Gaussian distribution with mean μ and covariance Σ.

Covariance Matrix ComplexityFORMULA
Number of Parameters in Σ
P = d(d + 1) / 2

Parameter count grows quadratically with d, so model complexity increases rapidly. When d >> sqrt(2N), the covariance matrix Σ becomes singular, |Σ| → 0, and Σ-1 cannot be computed.

Undefined Gaussian LikelihoodFORMULA
Likelihood Becomes Ill-Conditioned
p(x) ∝ exp(-(1/2)(x - μ)^T Σ^-1 (x - μ))
  • EM algorithm fails to converge.
  • Model overfits noise.
  • Clusters lose statistical meaning.
Mental Model

Too many parameters, too little data leads to model collapse.

Failure FlowANIMATED
Final Takeaway

K-Means failure: distance loses contrast and assignments become random. GMM failure: covariance estimation breaks and the model becomes unstable. Traditional clustering models cannot scale reliably to high-dimensional data.

Python
IMPLEMENTATION
# K-Means Failure Demo
import numpy as np
from sklearn.cluster import KMeans

# high-dimensional random data
X = np.random.rand(1000, 200)
kmeans = KMeans(n_clusters=3, n_init=10)
labels = kmeans.fit_predict(X)

print("Cluster distribution:")
unique, counts = np.unique(labels, return_counts=True)
print(dict(zip(unique, counts)))

from sklearn.mixture import GaussianMixture

except Exception as e:
    print("Failure:", e)
04 — 5.3 Grid-Based Spatial Clustering (STING)

5.3 Grid-Based Spatial Clustering (STING)

Core Idea STING (Statistical Information Grid) partitions the data space into a hierarchical grid structure and stores summary statistics in each cell instead of raw data.

STING WorkflowANIMATED

Grid Structure

  • The data space is divided into rectangular cells.
  • Cells are organized in multiple levels from coarse to fine.
  • Each higher-level cell aggregates information from its child cells.

Cell Statistics

Each Cell Stores
n = number of data points, m = mean, s = standard deviation, min = minimum value, max = maximum value.

For a parent cell with child cells indexed by i:

Aggregation EquationsFORMULA
Bottom-Up Aggregation
n = Σ ni
m = (Σ ni mi) / n
s = sqrt([(Σ ni(si^2 + mi^2)) / n] - m^2)
min = min(mini), max = max(maxi)
  • Statistics are computed bottom-up.
  • Raw data is not needed after aggregation.
  • Each level provides a summarized view of the data.

Query Processing Queries operate on grid cells instead of raw data, reducing time complexity from O(N) to O(K), where K is the number of cells.

  • Avoids expensive distance computations.
  • Uses summarized statistics for fast decisions.
  • Efficient for large datasets with spatial structure.
Limitations

Fixed grid boundaries can split natural clusters, the number of cells grows exponentially with dimension d, and performance degrades in very high-dimensional spaces.

Mental Model

Divide space into cells, summarize each cell, and reason using summaries instead of raw data.

When to Use STING

Use STING when the data has spatial or geometric structure, dimensionality is moderate, and fast query processing is required.

Python
IMPLEMENTATION
# Simple Grid-Based Clustering (STING-like)
import numpy as np

def grid_clustering(X, bins=10):
    grid = np.floor(X * bins).astype(int)
    unique_cells, counts = np.unique(grid, axis=0, return_counts=True)
    return unique_cells, counts

X = np.random.rand(1000, 2)
cells, counts = grid_clustering(X)

print("Number of grid cells:", len(cells))
print("Sample cell densities:", counts[:10])
05 — 5.4 Subspace Clustering (CLIQUE)

5.4 Subspace Clustering (CLIQUE)

Core Idea CLIQUE (Clustering In QUEst) identifies clusters that exist in subspaces of the full feature space rather than across all dimensions.

  • Many dimensions are irrelevant.
  • Clusters may only appear in specific subsets of features.
  • Clustering must be performed in subspaces instead of the full space.
Dense Unit DefinitionFORMULA
A Unit u is Dense If
|{x in u}| ≥ τ
Interpretation
Only regions with sufficient data points are retained, sparse regions are ignored, and clusters are formed by connecting adjacent dense units.

Subspace Search Strategy CLIQUE uses a bottom-up approach. It starts with 1-dimensional subspaces, identifies dense units, and combines them to form higher-dimensional subspaces.

Apriori Principle
If a k-dimensional unit is dense, then all its (k−1)-dimensional projections must also be dense.
CLIQUE WorkflowANIMATED
  • Dense adjacent units are grouped together.
  • Each group represents a cluster in a specific subspace.
  • CLIQUE eliminates irrelevant dimensions automatically.
  • It does not require the number of clusters in advance.
Limitations

The number of candidate subspaces grows exponentially with dimension, threshold τ matters heavily, and memory usage can become high.

Mental Model

Search for dense regions only in the dimensions where structure actually exists.

When to Use CLIQUE

Use CLIQUE for very high-dimensional datasets, unknown cluster counts, and cases where clusters exist in different feature subsets.

Python
IMPLEMENTATION
# CLIQUE Implementation
# install if needed:
# pip install pyclustering
from pyclustering.cluster.clique import clique
from pyclustering.utils import read_sample
from pyclustering.samples.definitions import FCPS_SAMPLES

data = read_sample(FCPS_SAMPLES.SAMPLE_TARGET)
clique_instance = clique(data, intervals=10, threshold=0)
clique_instance.process()
clusters = clique_instance.get_clusters()

print("Number of clusters:", len(clusters))
06 — 5.5 Projected Clustering (PROCLUS)

5.5 Projected Clustering (PROCLUS)

Core Idea PROCLUS (Projected Clustering) assumes that each cluster exists in its own subset of dimensions and explicitly identifies both clusters and their relevant subspaces.

Motivation Unlike CLIQUE, which searches for all dense subspaces, PROCLUS assigns a specific subspace to each cluster. Each cluster is defined by its own set of relevant features.

Projected DistanceFORMULA
Distance in Cluster-Specific Subspace S
DS(x,m) = Σ(j in S) |xj - mj|
Where
S = selected subset of dimensions, x = data point, m = medoid.
  • Distance is computed only on selected dimensions.
  • Irrelevant features are ignored.
  • Each cluster uses its own feature subset.

Algorithm Overview Select a set of candidate medoids, choose a subset of dimensions for each medoid, assign points based on projected distance, and iteratively refine medoids and subspaces.

PROCLUS WorkflowANIMATED
  • Reduces noise from irrelevant dimensions.
  • Captures clusters that exist in different subspaces.
  • Provides interpretable feature subsets for each cluster.
Limitations

PROCLUS requires the number of clusters k, requires the average subspace dimensionality l, and is sensitive to initialization of medoids.

Mental Model

Each cluster lives in its own feature space, and distance is measured only within that space.

CLIQUE vs PROCLUS

CLIQUE finds all dense subspaces and does not assign a fixed subspace per cluster. PROCLUS assigns one subspace per cluster and produces explicit cluster partitions.

Python
IMPLEMENTATION
# Simplified PROCLUS Distance Demo
import numpy as np

def projected_distance(X, medoid, dims):
    return np.sum(np.abs(X[:, dims] - medoid[dims]), axis=1)

X = np.random.rand(100, 10)
medoid = X[0]
dims = [0, 2, 4]  # selected subspace
dist = projected_distance(X, medoid, dims)

print("Projected distances:", dist[:10])
07 — 5.6 Summary: Method Selection

5.6 Summary: Method Selection (Synthesis)

Core Idea In high-dimensional clustering, no single algorithm works universally. The choice of method depends on data characteristics, dimensionality, and the objective of analysis.

Step 1: Identify the Problem Type

  • Is the data high-dimensional (d >> 10)?
  • Are all features equally relevant?
  • Is the number of clusters known?
  • Is interpretability required?

Step 2: Understand Method Behavior

STING
Grid-based summarization, fast and query-efficient, works well for spatial data.
CLIQUE
Discovers clusters in subspaces, does not require number of clusters, suitable for exploratory analysis.
PROCLUS
Assigns a specific subspace to each cluster, requires k, produces interpretable feature subsets.

Step 3: Practical Decision Rules

  • Use STING when data has spatial structure, fast query processing is required, and dimensionality is moderate.
  • Use CLIQUE when dimensionality is very high, number of clusters is unknown, and clusters exist in different subspaces.
  • Use PROCLUS when number of clusters is approximately known, feature interpretability is important, and each cluster depends on different dimensions.

Step 4: Trade-Off Summary

  • STING: fast but limited by grid boundaries.
  • CLIQUE: flexible but computationally expensive.
  • PROCLUS: interpretable but requires parameter tuning.
Unified Insight

High-dimensional clustering requires abandoning full-space assumptions: distance is unreliable, density is misleading, and features are not equally important.

Choosing the Right MethodANIMATED

Final Conclusion Effective clustering in high-dimensional spaces is not about choosing the best algorithm, but about choosing the right assumption for the data.
