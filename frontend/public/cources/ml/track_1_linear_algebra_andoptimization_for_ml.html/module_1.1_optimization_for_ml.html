Walking downhill,
without falling off

"Training a model" is, almost without exception, minimizing a loss. The toolkit for that is small but deep: gradients tell you which way is down, Hessians tell you how the slope itself bends, and a handful of update rules — vanilla GD, momentum, Newton — turn those facts into an algorithm. Build the geometric picture once and the rest of the course is just instances.

01 / Interactive · the cleanest functions in the world
What makes a loss convex?
A function is convex if every chord — every straight line between two points on the graph — lies on or above the graph. The payoff: every local minimum is the global minimum. OLS, ridge, logistic+L², linear SVM — all convex. Many deep networks are not. Pick a function and slide the endpoints to see chords above (✓ convex) or below (✗ violation).

02 / Derivation
Definition & tests — convexity, formally definition · proof

The "chords lie above" picture is exactly one inequality. A function f is convex on a convex set if, for all x, y and every θ ∈ [0,1]:

f(θx + (1−θ)y) ≤ θ f(x) + (1−θ) f(y) └ the point on the graph ─┘ └─ the point on the chord ─┘

Three escalating tests, each needing more smoothness:

0th order · the chord inequality above (always valid) 1st order · f(y) ≥ f(x) + ∇f(x)ᵀ(y−x) — the tangent lies below f 2nd order · ∇²f(x) ⪰ 0 for all x — the Hessian is PSD everywhere

The 2nd-order test is the one you actually use: compute the Hessian, check it is positive semidefinite. Strictly convex (∇²f ≻ 0) ⟹ at most one minimizer.

Why ML obsesses over it

If f is convex, every local minimum is global and any stationary point (∇f = 0) solves the problem. Quick proof: suppose x* is a local min but some y has f(y) < f(x*). Convexity says f(x* + θ(y−x*)) ≤ f(x*) + θ(f(y)−f(x*)) < f(x*) for all small θ > 0 — points arbitrarily close to x* that beat it, contradicting "local min." ∎

This is why OLS, ridge, lasso, logistic regression, and the SVM are "tame" — their losses are convex, so gradient descent cannot get stuck in a bad local minimum. Deep nets are non-convex, which is why their training is an art.

04 / Derivation
Derivation — GD and Newton from the Taylor model, and the exact step-size limit proof · numeric · code
1 · Gradient descent = minimize the linear model with a leash

The 1st-order Taylor model f(w+Δ) ≈ f(w) + ∇f(w)ᵀΔ has no minimum — push Δ infinitely far along −∇f and it drops forever. So we add a quadratic trust term that penalizes big steps, then minimize:

minₐ f(w) + ∇f(w)ᵀΔ + (1/2η)‖Δ‖² set gradient wrt Δ to 0: ∇f(w) + (1/η)Δ = 0 ⟹ Δ = −η ∇f(w) ⟹ w ← w − η ∇f(w)

So the learning rate η is literally how far you trust the linear approximation. Larger η = looser leash.

2 · Newton = jump to the bottom of the quadratic model

Keep the 2nd-order term and minimize the full quadratic Taylor model:

minₐ f(w) + ∇f(w)ᵀΔ + ½ Δᵀ H Δ set gradient wrt Δ to 0: ∇f(w) + H Δ = 0 ⟹ Δ = −H⁻¹ ∇f(w) ⟹ w ← w − H(w)⁻¹ ∇f(w)

On a quadratic the model is the function, so Newton lands at the exact minimum in one step — what you see in the demo. The price is forming and inverting H: O(d³) per step, infeasible for large d.

3 · Exactly when does GD diverge? (the bound the demo uses)

Take the demo's bowl L = w₁² + 5w₂², so ∇L = (2w₁, 10w₂) and the Hessian eigenvalues are 2 and λ_max = 10. Along the steep axis the GD update is a simple recursion:

w₂ ← w₂ − η·10 w₂ = (1 − 10η) w₂ (a geometric sequence) converges ⟺ |1 − 10η| < 1 ⟺ 0 < η < 0.2 in general: 0 < η < 2/λ_max for a quadratic with Hessian H

That is the corrected limit: 2/λ_max = 2/10 = 0.20, not 1/λ_max. Note it is set by the steepest direction while progress along the flat direction crawls — the ratio λ_max/λ_min = 5 (the condition number) is exactly why vanilla GD zigzags, and why momentum and preconditioning help.

4 · All three update rules in NumPy
import numpy as np A = np.array([[1., 0.], [0., 5.]]) # L = wᵀAw (the demo bowl) grad = lambda w: 2 * A @ w H = 2 * A # constant Hessian here def run(method, w, eta=0.08, beta=0.9, steps=50): v = np.zeros_like(w) for _ in range(steps): g = grad(w) if method == "gd": w = w - eta * g elif method == "momentum": v = beta*v + g; w = w - eta * v elif method == "newton": w = w - np.linalg.solve(H, g) # 1 step! return w run("newton", np.array([-2.6, 1.8]), steps=1) # ≈ [0, 0] already # eta >= 0.2 with method="gd" -> blows up, exactly as derived

06 / First-order zoology
The optimizer menagerie
Once you're committed to first-order methods, the question becomes: which gradients do you use, and how do you blend them across iterations? Five ideas you'll meet everywhere.

Batch GD

∇L on all n samples

The truest gradient. Smooth descent, but every step costs O(n) — painful for large datasets and impossible for streaming.

Stochastic GD (SGD)

∇L on one sample

Noisy but cheap. The noise itself can help escape shallow minima. In practice, used in the mini-batch form below.

Mini-batch GD

∇L on a batch of B samples

The sweet spot: vectorized hardware loves it, the noise is just enough, and B becomes a knob you tune. The default for deep learning.

Momentum / Nesterov

v ← βv + ∇L  ·  w ← w − ηv

A heavy ball with inertia. Damps oscillations in narrow valleys and accelerates along consistent directions. Nesterov peeks ahead before stepping.

RMSprop / Adam

per-parameter η, divided by recent ‖∇‖

Adaptive learning rates: dimensions with big gradients shrink their step; quiet dimensions grow theirs. Adam adds momentum on top — the workhorse of modern deep learning.

LR schedules

η decays: step · exponential · cosine · warmup

Start aggressive, settle gentle. Almost every modern training run uses a schedule — a constant η is leaving performance on the table.

08 / Derivation
Derivation — Lagrange multipliers and the full KKT conditions proof
1 · Equality constraints: why ∇f = λ∇g

At a constrained optimum on g(x)=0, you cannot decrease f by moving along the constraint. Any feasible direction d is tangent to the curve, i.e. ∇gᵀd = 0. Optimality means ∇fᵀd = 0 for all such d too. So ∇f is orthogonal to the same tangent space as ∇g — they must be parallel:

∇f(x) = λ ∇g(x) define L(x,λ) = f(x) − λ g(x) stationary point of L ⟺ ∇f = λ∇g and g(x) = 0

The multiplier λ is not bookkeeping — it is the shadow price: λ = ∂(optimal f)/∂(constraint level), how much the objective improves if you relax the constraint by one unit.

2 · Inequalities: the four KKT conditions

For min f(x) s.t. gᵢ(x) ≤ 0 and hⱼ(x) = 0, an optimum (under mild regularity) satisfies:

1. stationarity ∇f + Σ μᵢ∇gᵢ + Σ νⱼ∇hⱼ = 0 2. primal feasibility gᵢ(x) ≤ 0, hⱼ(x) = 0 3. dual feasibility μᵢ ≥ 0 4. complementary slack μᵢ · gᵢ(x) = 0 for every i

Condition 4 is the powerful one: either a constraint is active (gᵢ = 0) or its multiplier is zero (μᵢ = 0). In the SVM this says only points on the margin get μᵢ > 0 — those are the support vectors, and every other training point can vanish without changing the boundary. → Track 4.5

Practice — gradients, steps, and stability exercises + solutions
  1. For L(w) = ½‖Xw − y‖², derive ∇L and the one Newton step from any w₀.

    ∇L = Xᵀ(Xw − y), Hessian H = XᵀX. Newton step: w₁ = w₀ − (XᵀX)⁻¹Xᵀ(Xw₀ − y) = (XᵀX)⁻¹Xᵀy — the OLS solution, regardless of w₀. Least squares is a quadratic, so Newton solves it in one step.

  2. On L = w₁² + 5w₂², what is the largest η for which GD converges, and which direction sets the limit?

    Hessian eigenvalues are 2 and 10. Stability needs η < 2/λ_max = 2/10 = 0.2. The steep w₂ direction (λ=10) sets the cap; the flat w₁ direction would tolerate up to η < 1.0 but is forced to crawl.

  3. Is f(x) = |x| differentiable at 0? How does gradient descent cope?

    No — there is a kink at 0. But it is convex, with a subdifferential ∂f(0) = [−1, 1]. Subgradient methods pick any element of that set; 0 itself is a valid subgradient, which is why x = 0 is correctly identified as the minimum. This is the same machinery behind Lasso's sparsity.

  4. Minimize f(x,y) = x + y subject to x² + y² = 1 with a Lagrange multiplier.

    ∇f = (1,1), ∇g = (2x,2y). Setting ∇f = λ∇g: 1 = 2λx, 1 = 2λy ⟹ x = y. With x²+y²=1: x = y = ±1/√2. Minimum at (−1/√2, −1/√2), f = −√2; maximum at (1/√2, 1/√2), f = +√2.
