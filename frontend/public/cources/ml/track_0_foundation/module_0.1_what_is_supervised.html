What is supervised
learning, really?

Strip away the hype and it's one clean idea: you're handed examples that already carry the answer, and you learn a function that reproduces that answer on examples it has never seen. Everything in this course is a different way of choosing and fitting that function.

𝒟 = {(xᵢ, yᵢ)}ᵢ₌₁ⁿ ~ⁱⁱᵈ P(x, y) → learn f : 𝒳 → 𝒴 minimizing 𝔼[ L(y, f(x)) ]
01 / The core inversion
ML vs traditional programming
In classical programming you write the rules by hand. In machine learning you hand over the data and the answers, and the rules are what comes out. That flip is the whole game.
Traditional programming

You write the rules

rules+data →output

A human encodes the logic. If the world shifts, a human rewrites the logic.

Machine learning

The machine writes the rules

data+answers (y) →f̂ (the rules)

You supply labeled examples; the algorithm searches for the f that best maps inputs to those labels.

02 / Interactive · the only thing that changed is 𝒴
Regression vs classification
Same inputs, same goal — predict y from x. The two tasks are distinguished by one thing only: the shape of the output space 𝒴. Continuous (a number on the real line) → regression. A finite set of labels → classification. Flip the switch and watch the identical data get a different lens.

03 / Derivation
Derivation — what is the best possible f? (the Bayes predictor) proof

Before choosing any model, ask: if we knew the true distribution P(x,y) exactly, what f would minimize expected loss? The answer is the target every algorithm is chasing — and it depends only on the loss.

Squared loss ⟹ the conditional mean

Minimize E[(y − f(x))²] pointwise at each x. Let c = f(x) and minimize over the scalar c:

d/dc E[(y − c)² | x] = −2 E[y − c | x] = 0 ⟹ c = E[y | x] ⟹ f*(x) = E[Y | x] (the regression function)

So regression is, ideally, estimating the conditional mean. The leftover error at the optimum is exactly the irreducible noise σ² from the bias–variance story (Module 0.4).

0–1 loss ⟹ the most probable class

For classification with L = 1[y ≠ f(x)], the expected loss at x is 1 − P(f(x) | x), minimized by picking the most probable label:

f*(x) = argmax_k P(y = k | x) (the Bayes classifier)

Its error rate — the Bayes error — is the floor no classifier can beat, the classification analogue of σ². This is why every classifier in the course ultimately tries to estimate P(y|x): the two routes to it (model it directly vs. via Bayes' rule) are exactly the discriminative/generative split below.

04 / Interactive · two routes to the same prediction
Discriminative vs generative
Both predict a class for each point — but they get there through opposite philosophies. A discriminative model carves the boundary directly: it learns P(y|x) and never bothers to describe what each class looks like. A generative model first learns what each class looks like — P(x|y) and P(y) — then flips it with Bayes' rule. Toggle the lens; notice they land on the same boundary here, but the generative view also draws the class "clouds."

07 / Derivation
Derivation — how a generative model turns into a classifier proof · code

A discriminative model estimates P(y|x) directly. A generative model takes the long way: model what each class looks like, P(x|y), plus the base rates P(y), then invert with Bayes' rule.

P(y=k | x) = P(x | y=k) · P(y=k) / P(x) prediction: argmax_k P(x|y=k)·P(y=k) (P(x) is the same for all k — drop it)

For Gaussian class-conditionals with a shared covariance the quadratic terms cancel and the boundary is linear (that's LDA — and why the demo's generative and discriminative boundaries coincide). With per-class covariances the boundary is quadratic (QDA).

import numpy as np # Gaussian generative classifier (shared covariance ≈ LDA) — by hand def fit_gaussian(X, y): classes = np.unique(y) means = {k: X[y==k].mean(0) for k in classes} priors= {k: np.mean(y==k) for k in classes} cov = np.cov(X.T) # pooled P(x|y) return means, priors, np.linalg.inv(cov) def predict(X, means, priors, Cinv): # score_k(x) = log P(x|k) + log P(k), Gaussian -> linear discriminant scores = {k: -0.5*np.sum((X-m)@Cinv*(X-m), 1) + np.log(priors[k]) for k, m in means.items()} keys = list(scores) return np.array(keys)[np.argmax([scores[k] for k in keys], 0)]

The trade: generative models can use unlabeled data and handle missing features (they have a full P(x|y)), but they pay for modeling more than they need. Discriminative models usually win on raw accuracy because they spend all their capacity on the one thing that matters — the boundary.

Practice — place the model, name the target exercises + solutions
  1. "Predict tomorrow's temperature" vs "predict whether it rains tomorrow" — classify each as regression or classification, and name the ideal f*.

    Temperature is continuous ⟹ regression; ideal f*(x)=E[temp|x]. Rain is a label ⟹ binary classification; ideal f*(x)=argmax_k P(rain=k|x), i.e. predict "rain" iff P(rain|x) > 0.5 under 0–1 loss.

  2. Is logistic regression discriminative or generative? Naive Bayes? Place both on the four dichotomies.

    Logistic regression is discriminative (models P(y|x) directly), parametric, model-based, and can be batch or online (SGD). Naive Bayes is generative (models P(x|y)P(y)), parametric, model-based, and naturally online (counts update incrementally).

  3. Why is KNN called non-parametric and instance-based at once?

    Non-parametric: it has no fixed-size weight vector — its effective complexity grows with n (more data = finer decision regions). Instance-based: it stores the training set and defers all work to prediction time (find neighbors, vote), rather than distilling data into parameters and discarding it.
