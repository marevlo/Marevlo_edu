Bias, variance &
the generalization story

Every prediction error breaks into three pieces. Two of them you can trade against each other; one you can never beat. Understanding this decomposition is what lets you look at a model and say "it's too simple" or "it's memorizing" — and know exactly which lever to pull.

01 / Read the equation
Three sources of error
Imagine refitting your model on many different training sets drawn from the same world. Each term measures something different about that family of fits.

Bias²

How far the average fit lands from the truth. High bias = the model is too rigid to capture the real pattern → underfitting.

Variance

How much the fit wiggles as the training set changes. High variance = the model chases noise → overfitting.

Noise σ²

Irreducible randomness in the labels themselves. No model, however clever, can predict below this floor. Accept it.

02 / Derivation
Derivation — the bias–variance decomposition, line by line full proof

This is the proof the module points to "Stats 5.7" for — here in full. Fix a test point x. The label is y = f*(x) + ε with noise E[ε]=0, Var(ε)=σ². The estimator f̂ is random because the training set is random; write f̄(x) = E[f̂(x)] for its average over training sets. We want the expected squared error E[(y − f̂(x))²].

Step 1 — split off the noise

Insert ± f*(x) and expand. Because ε is independent of f̂ and has mean 0, the cross term dies:

E[(y − f̂)²] = E[(f* + ε − f̂)²] = E[(f* − f̂)²] + 2·E[ε]·E[(f* − f̂)] + E[ε²] = E[(f* − f̂)²] + σ² ← noise floor, irreducible
Step 2 — split the rest at the average fit f̄

Now insert ± f̄(x) inside the remaining term and expand again:

E[(f* − f̂)²] = E[((f* − f̄) + (f̄ − f̂))²] = (f* − f̄)² ← constant, comes out of E + E[(f̄ − f̂)²] + 2(f* − f̄)·E[f̄ − f̂]
Step 3 — the cross term vanishes
E[f̄ − f̂] = f̄ − E[f̂] = f̄ − f̄ = 0 (by definition of f̄)
Step 4 — name the survivors
E[(y − f̂(x))²] = (f̄ − f*)² + E[(f̂ − f̄)²] + σ² = Bias² + Variance + Noise

Bias² = how far the average fit sits from the truth (rigidity). Variance = how much the fit jitters around its own average as the training set changes (sensitivity). σ² = a floor no model can cross. Capacity trades the first two: more flexibility → less bias, more variance. The interactive lab below is literally this expectation estimated over 36 resampled training sets.

05 / The one lever, many names
The regularization lever
Almost every "hyperparameter that prevents overfitting" is the same move in disguise: deliberately add a little bias to buy a large cut in variance. Turn the dial up and the model gets simpler and steadier; turn it down and it gets flexible and twitchy.
L1 / L2 penalty

Shrink the weights

Adding λ‖w‖ to the loss pulls coefficients toward zero — less freedom to chase noise. → Track 3.3–3.5

max_depth · min_samples_leaf

Limit the tree

Shallower trees and bigger leaves can't carve noise-fitting micro-regions. → Track 3.8 · 5

k · dropout · early stop

Soften capacity

Bigger k in KNN, dropout in nets, stopping training early — all trade bias for lower variance. → Track 3.7

The curse of dimensionality
In high dimensions, geometry betrays intuition. Scatter points in a d-dimensional cube and measure, from a query point, the distance to its nearest and farthest neighbor. As d grows, those two distances converge — everything becomes roughly equidistant, and "nearest neighbor" loses its meaning.

07 / Two ideas that complicate the tidy story
Double descent & no free lunch
The U-shaped curve is the classical picture — but it isn't the whole story. And no decomposition changes the deepest fact: learning only works because real data has structure.

🍽️ No Free Lunch

Averaged over all possible problems, every learning algorithm performs exactly the same — including random guessing. There is no universally best model.

What rescues us is that real-world data isn't drawn from "all possible problems." It has structure: smoothness, locality, sparsity. A model wins by matching its assumptions to that structure.

The corollary: choosing a model is choosing a bias. The art is choosing one the data rewards.

08 / Derivation
From scratch — estimate bias and variance by bootstrap in NumPy code

The decomposition needs an expectation over training sets, which you never have. The bootstrap fakes it: resample the training data many times, fit a model on each, and look at the spread of predictions on a fixed test point. This is exactly what the lab above does.

import numpy as np from sklearn.base import clone def bias_variance(model, X_tr, y_tr, X_te, y_te, n_boot=200, seed=0): rng = np.random.default_rng(seed) n = X_tr.shape[0] preds = np.empty((n_boot, X_te.shape[0])) for b in range(n_boot): idx = rng.integers(0, n, n) # sample WITH replacement preds[b] = clone(model).fit(X_tr[idx], y_tr[idx]).predict(X_te) mean_pred = preds.mean(axis=0) # f̄(x), the average fit var = preds.var(axis=0).mean() # E[(f̂ − f̄)²] bias2 = ((mean_pred - y_te) ** 2).mean() # (f̄ − f*)² (y_te ≈ f*) err = ((preds - y_te) ** 2).mean() # total expected error return bias2, var, err # err ≈ bias2 + var + σ² # swap a deep tree (high var) for a stump (high bias) and watch the split move

Try it: a single deep decision tree gives tiny bias, large variance; a depth-1 stump flips it. Wrapping the tree in BaggingRegressor averages many bootstrapped fits — driving the variance term down while leaving bias roughly fixed. That single fact is the entire reason random forests work (Track 5).

Practice — reason about the three terms exercises + solutions
  1. You collect 10× more training data but keep the same model. Which of bias², variance, σ² changes, and how?

    More data mainly reduces variance (each fit is more stable, so f̂ jitters less around f̄). Bias² is largely unchanged — it is a property of the model class, not the sample size. σ² is irreducible and never moves. This is why "high variance / overfit" learning curves close their gap with more data, while "high bias" ones don't.

  2. Averaging B independent fits divides variance by roughly B. Why doesn't it also reduce bias?

    The average of unbiased-noise fits is (1/B)Σf̂ᵦ; its expectation is still f̄ (averaging doesn't move the mean), so bias = f̄ − f* is unchanged. But the variance of an average of B iid terms is Var/B. In practice fits are correlated, so the reduction is ρ·Var + (1−ρ)Var/B — which is exactly why bagging decorrelates trees (random feature subsets) to push ρ down.

  3. A model achieves 0 training error. What does the decomposition predict about its test error?

    Zero training error signals very high capacity → typically very low bias but high variance, plus it cannot beat the σ² floor on test data. Expect test error ≈ variance + σ² to be large. Memorizing the noise is fitting ε, which by definition does not generalize.
