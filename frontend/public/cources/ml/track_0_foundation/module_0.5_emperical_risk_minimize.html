Empirical risk &
what we actually minimize

Every learning algorithm in this course is, underneath, the same move: define a loss, average it over your data to get a risk, and search for the function that makes that risk small. The catch — the risk you can compute is not the risk you care about.

01 / Interactive · the gap ERM can't see
Minimizing the wrong risk
ERM minimizes empirical risk R̂ — error on the training sample. But the goal is true risk R — error on the whole population. Push capacity up and R̂ marches toward zero while R turns around and climbs. With little data the gap is a canyon; with lots of data, the sample looks more like the world (the law of large numbers) and R̂ ≈ R.

03 / Derivation
Derivation — every loss is a negative log-likelihood proof

Assume the data is iid from a model p(y | x; θ). Maximum likelihood maximizes the product of probabilities; taking −log turns the product into a sum and the max into a min:

θ̂ = argmax Πᵢ p(yᵢ|xᵢ) = argmin Σᵢ −log p(yᵢ|xᵢ) so: L(y, f) ≜ −log p(y|f) + const ⟹ ERM = MLE
Gaussian noise ⟹ squared loss
y = f(x) + ε, ε ~ 𝒩(0, σ²) −log p(y|f) = −log[ (1/√(2πσ²)) exp(−(y−f)²/2σ²) ] = (y−f)²/(2σ²) + const ⟹ L ∝ (y − f)²

So least squares is the MLE of the mean under Gaussian noise. Its thin tails make it sensitive to outliers — one far point contributes a huge squared term.

Laplace noise ⟹ absolute loss
p(y|f) = (1/2b) exp(−|y−f|/b) −log p = |y−f|/b + const ⟹ L ∝ |y − f|

MAE is the MLE under Laplace (double-exponential) noise; its fat tails make big errors "cheaper," so it is robust and recovers the conditional median.

Bernoulli ⟹ cross-entropy (log loss)
y ∈ {0,1}, p = P(y=1|x) p(y|x) = pʸ (1−p)¹⁻ʸ −log p(y|x) = −[ y log p + (1−y) log(1−p) ] ⟹ cross-entropy

This is the natural loss for classification, and it generalizes to softmax/categorical for K classes. Choosing a loss is therefore choosing a noise model — never an arbitrary decision.

04 / Interactive · taming ERM with a penalty
Structural risk minimization
If pure ERM overfits, leash it. Structural Risk Minimization adds a complexity penalty: minimize R̂(f) + λ·Ω(f). The penalty pulls the chosen model toward simplicity; λ sets how hard. This is the single principle behind every regularizer in the course. Slide λ and watch the objective's minimum slide with it.

05 / Putting Track 0 together
Train, then validate
We can't see true risk R — it needs the unknown distribution P. So we do the next best thing: minimize R̂ on training data, but judge on held-out data that stands in for the population. That's why the whole machinery of Module 0.3 exists.
FIT

Minimize R̂ on train

ERM does the optimization — find the f that best fits the examples you have.

CONTROL

Add λ·Ω (SRM)

Regularize so ERM doesn't chase noise. λ is chosen, not minimized away.

JUDGE

Estimate R on held-out

Validation / CV is your proxy for true risk — the only honest read on generalization.

07 / Derivation
Derivation — a regularizer is a prior (SRM = MAP) proof

Where does the penalty λΩ(w) come from? Put a prior p(w) on the parameters and do maximum-a-posteriori instead of maximum-likelihood. Bayes: p(w|data) ∝ p(data|w) p(w). Take −log:

ŵ_MAP = argmin [ −log p(data|w) − log p(w) ] = argmin [ Σ L(yᵢ, f(xᵢ)) + (−log p(w)) ] └ empirical risk ┘ └─ the penalty ─┘
Gaussian prior ⟹ Ridge (L²)
p(w) = 𝒩(0, τ²I) ⟹ −log p(w) = ‖w‖²/(2τ²) + const ⟹ penalty = λ‖w‖² with λ = 1/(2τ²)
Laplace prior ⟹ Lasso (L¹)
p(w) ∝ exp(−‖w‖₁/b) ⟹ −log p(w) = ‖w‖₁/b + const ⟹ penalty = λ‖w‖₁

So SRM's R̂ + λΩ is just MAP, and λ encodes how strong your prior belief in "small/simple weights" is. A tighter prior (smaller τ) ⟹ bigger λ ⟹ more shrinkage. λ is still chosen by validation — the data picks how much to trust the prior.

From scratch — ERM by gradient descent in NumPy code

"Define a loss, average it, minimize it" — coded directly. Here is regularized linear regression (ridge) trained by the gradient descent of Module 1.1, no library solver:

import numpy as np def fit_erm(X, y, lam=0.0, eta=0.05, steps=2000): n, d = X.shape w = np.zeros(d) for _ in range(steps): resid = X @ w - y # r = ŷ − y grad = (X.T @ resid) / n + lam * w # ∇[ R̂ + λ‖w‖² ] w -= eta * grad return w # empirical risk we are minimizing (mean squared error + penalty): def risk(X, y, w, lam): return ((X @ w - y) ** 2).mean() + lam * (w @ w) # lam = 0 -> pure ERM (can overfit); lam > 0 -> SRM / MAP with a Gaussian prior
Practice — risk, losses, and priors exercises + solutions
  1. Why can we never compute the true risk R(f), and what do we substitute for it?

    R(f) = E_{(x,y)∼P}[L] requires the unknown population distribution P. We substitute the empirical risk R̂ on training data for optimization, and a held-out / cross-validation estimate as an unbiased proxy for R when judging. By the law of large numbers R̂ → R as n → ∞ for a fixed model.

  2. You have heavy outliers in your regression target. Which loss should you prefer and why, in likelihood terms?

    Prefer absolute / MAE (or Huber). It is the MLE under Laplace noise, whose fat tails make large residuals far less costly than the Gaussian's squared penalty — so a few outliers don't dominate the fit. MAE recovers the conditional median, which is robust.

  3. Show that pure ERM (λ = 0) prefers the most complex model in a nested family. What fixes it?

    A larger family contains the smaller one, so it can always achieve R̂ at least as low — empirical risk is monotonically non-increasing in capacity, bottoming out at 0 (interpolation). ERM therefore picks maximum complexity and overfits. SRM adds λΩ(w) (equivalently a prior), bending the objective into a bowl whose minimum tracks true risk.
