MODULE 3.10 · GENERALIZED LINEAR MODELS Track 3 Module 3.10 11 / 12

Don't change the model.
Change the link.

Module 3.1 fit a line to a Gaussian target. Module 4.1 will squash the same line through a sigmoid to fit a Bernoulli one. Those aren't two tricks — they're one recipe run with different ingredients. A Generalized Linear Model is the recipe itself: pick a distribution from the exponential family (the random component), keep the familiar linear predictor η = wᵀx (the systematic component), and choose a link g so that g(𝔼[y|x]) = η. Counts get a log link, probabilities a logit, positive-skewed amounts a log or inverse — and one fitter, IRLS, handles them all. Deviance replaces squared error, and "regression" stops being one model and becomes a family.

02 / The trinity
Random + systematic + link — one framework
Every GLM is three decisions. Random: which exponential-family distribution does y | x follow? Systematic: the linear predictor η = wᵀx — unchanged since Module 3.1. Link: the function g that connects the distribution's mean to the predictor via g(𝔼[y|x]) = η. The link is the whole story geometrically: it takes the one straight line you know how to fit and reshapes it so it lives in the right space — probabilities for logit, positive rates for log. Move the sliders below and watch the same line become four different models.
the trinity : 1) RANDOM y | x ~ ExponentialFamily( μ(x) ) ← Gaussian, Bernoulli, Poisson, Gamma, … 2) SYSTEMATIC η(x) = wᵀx + b ← the linear predictor, same as always 3) LINK g( μ(x) ) = η(x) ⇔ μ(x) = g⁻¹( η(x) ) ← connects mean to predictor special cases : Gaussian + identity link ⇒ linear regression (Module 3.1) Bernoulli + logit link ⇒ logistic regression (Module 4.1) Poisson + log link ⇒ count regression — new today estimation : maximize the exponential-family log-likelihood ℓ(w) ⇒ IRLS (Newton's method, specialized — §04)
🎲

Random component — what kind of noise is y?

Pick the distribution that matches the target's support: real line → Gaussian, {0,1} → Bernoulli, counts → Poisson, positive skewed → Gamma. The family also fixes the variance function V(μ) — how spread scales with the mean.

Systematic component — the linear predictor

η = wᵀx + b. Everything you learned about features, interactions, regularization and collinearity transfers untouched — the linear machinery is shared by every member of the family.

Link function — the shape adapter

g(μ) = η. The link maps the mean's natural range onto the whole real line so a line can model it: logit maps (0,1) → ℝ, log maps (0,∞) → ℝ. Its inverse g⁻¹ bends the line back into the right space.

The exponential family — why these distributions, & canonical links math · code

The families a GLM accepts aren't arbitrary — they all share one algebraic shape, and that shape is what makes the likelihood concave and the fitting algorithm universal.

exponential family : p(y; θ, φ) = exp{ [ y·θ − b(θ) ] / a(φ) + c(y, φ) } mean 𝔼[y] = b′(θ) variance Var[y] = b″(θ) · a(φ) ← variance is a FUNCTION of the mean canonical link : the g that sets θ = η directly Gaussian → identity Bernoulli → logit Poisson → log Gamma → inverse why it matters : canonical link ⇒ ∇ℓ(w) = Xᵀ(y − μ)/a(φ) ← gradient identical in form to OLS & logistic variance functions : Gaussian V(μ)=1 Poisson V(μ)=μ Gamma V(μ)=μ² Tweedie V(μ)=μᵖ, 1<p<2

Two consequences worth memorizing. (1) The gradient Xᵀ(y − μ) you derived for linear regression and will re-derive for logistic is not a coincidence — it's the exponential family's signature, the same expression with a different μ. (2) The variance function is a modeling assumption you are silently making: Poisson asserts spread grows like the mean, Gamma asserts the coefficient of variation is constant. When data violates it, the diagnostics of §05 will tell you.

# statsmodels — full-inference GLMs: pick a family + link, get SEs, deviance, AIC import statsmodels.api as sm X = sm.add_constant(X_raw) m = sm.GLM(y, X, family=sm.families.Poisson(link=sm.families.links.Log())).fit() print(m.summary()) # coefficients, std errors, z-scores, CIs print(m.deviance, m.aic) # deviance + AIC for nested-model comparison # sklearn — prediction-focused, regularized; link is built into each estimator from sklearn.linear_model import PoissonRegressor, GammaRegressor, TweedieRegressor pr = PoissonRegressor(alpha=1e-4).fit(X_raw, y) # counts · log link gr = GammaRegressor().fit(X_raw, y_pos) # positive-skewed · log link tw = TweedieRegressor(power=1.5, link='log').fit(X_raw, y) # compound Poisson-Gamma

Check yourself — three field decisions quiz
  1. Your target is the number of insurance claims per policy-year, and policies have different exposure lengths. Which family, which link — and what handles the exposure?
    ▸ reveal answer

    Poisson with a log link. Exposure enters as an offset: log 𝔼[y] = log(exposure) + wᵀx, a coefficient fixed at 1 — so you model the rate while fitting counts. (Code in §05's deep dive.)

  2. An OLS model of hospital length-of-stay predicts −1.4 days for some young patients, and residuals fan out for long stays. Diagnosis and fix?
    ▸ reveal answer

    The target is positive and right-skewed, so identity-link Gaussian is wrong twice: support (negative predictions) and variance (spread grows with the mean). A Gamma GLM with a log link fixes both — predictions stay positive and V(μ) = μ² absorbs the fanning.

  3. Your Poisson fit reports Pearson dispersion φ̂ = 4.3. What does that mean and what do you do?
    ▸ reveal answer

    The data is overdispersed: variance ≈ 4.3× the mean, while Poisson asserts they're equal. Coefficients stay roughly fine; standard errors are badly optimistic. Switch to quasi-Poisson (inflate SEs by √φ̂) or a Negative Binomial family (§05 deep dive).

04 / Fitting GLMs
IRLS — every Newton step is a weighted least squares solve
There's no closed form for most GLMs, but there's something nearly as good: the Newton update on the exponential-family likelihood rearranges into a weighted OLS solve on a "working response" — recompute weights, re-solve, repeat. That loop is Iteratively Reweighted Least Squares, and it usually converges in a handful of steps. Click through the iterations below and watch a flat initial guess snap onto the data.
score / gradient : ∇ℓ(w) = Xᵀ (y − μ) / a(φ) ← canonical link: same form as OLS & logistic Hessian : −∇²ℓ(w) = Xᵀ W X W = diag( 1 / [ V(μᵢ) · g′(μᵢ)² ] ) Newton step : w ← w + (XᵀWX)⁻¹ Xᵀ(y − μ) rewrite : w ← (XᵀWX)⁻¹ XᵀW z z = η + (y − μ) · g′(μ) ← the "working response" = weighted least squares on z, with weights W — recomputed every step hence the name : Iteratively Reweighted Least Squares — Newton's method (→ Track 1.1) specialized to GLMs Poisson, log link: μ = e^η W = diag(μ) z = η + (y − μ)/μ

IRLS from scratch — twelve lines of NumPy, then production math · code

For the canonical link the algebra collapses beautifully: the Fisher information equals the observed Hessian, W is just the variance function evaluated at μ, and each Newton step is literally one call to a weighted least-squares solver.

why W = diag(μ) for Poisson : V(μ) = μ and g′(μ) = 1/μ (log link) W = 1 / [ V(μ) · g′(μ)² ] = 1 / [ μ · 1/μ² ] = μ working response : z = η + (y − μ)·g′(μ) = η + (y − μ)/μ the loop : η → μ → (W, z) → weighted OLS → new w → repeat
# IRLS for Poisson regression — tinyml style, every step is a weighted OLS solve import numpy as np def irls_poisson(X, y, iters=25, tol=1e-8): w = np.zeros(X.shape[1]); w[0] = np.log(y.mean() + 1e-3) for _ in range(iters): eta = np.clip(X @ w, -15, 15) mu = np.exp(eta) # inverse log link W = mu # Poisson weights (see derivation above) z = eta + (y - mu) / mu # working response WX = X * W[:, None] w_new = np.linalg.solve(X.T @ WX, X.T @ (W * z)) # ← one weighted OLS if np.max(np.abs(w_new - w)) < tol: return w_new w = w_new return w # production — sklearn (regularized, predictive) & statsmodels (inference) from sklearn.linear_model import PoissonRegressor model = PoissonRegressor(alpha=1e-3, max_iter=300).fit(X_raw, y) print(model.coef_, model.intercept_) print(model.score(X_raw, y)) # D² — the deviance analogue of R²

One nuance. sklearn's GLMs minimize penalized deviance with an L2 term (alpha) and solve by L-BFGS rather than vanilla IRLS — same optimum family, more robust at scale. statsmodels' GLM.fit() is classic IRLS and reports the inferential extras (standard errors, deviance, AIC) that sklearn deliberately omits.

05 / Deviance & diagnostics
Deviance, dispersion & which family when
Squared error was never fundamental — it was the Gaussian's deviance all along. The deviance generalizes it to every family: twice the log-likelihood gap between your model and the saturated model that fits each point perfectly. Everything you did with residuals and R² has a deviance-flavored analogue, plus one new failure mode to watch: overdispersion.
deviance : D = 2 [ ℓ(saturated) − ℓ(model) ] saturated = one parameter per point, perfect fit Gaussian ⇒ D = Σᵢ (yᵢ − μ̂ᵢ)² ← deviance literally generalizes RSS Poisson ⇒ D = 2 Σᵢ [ yᵢ log(yᵢ/μ̂ᵢ) − (yᵢ − μ̂ᵢ) ] deviance residual : dᵢ = sign(yᵢ − μ̂ᵢ) · √Dᵢ ← plot THESE, not raw residuals Pearson dispersion: φ̂ = (1/(n−p)) Σᵢ (yᵢ − μ̂ᵢ)² / V(μ̂ᵢ) φ̂ ≫ 1 ⇒ overdispersion → quasi / NegBin pseudo-R² : McFadden = 1 − ℓ_model / ℓ_null Nagelkerke = rescaled Cox–Snell model choice : AIC = D + 2p BIC = D + p·log n ← nested-family comparison

Which family, when

GAUSSIAN · IDENTITY

Symmetric continuous targets

  • Target lives on all of ℝ, roughly constant spread
  • This is Modules 3.1–3.6 — OLS, Ridge, Lasso unchanged
  • Default until the residuals object
POISSON · LOG

Counts & rates

  • Events per unit: claims, clicks, defects, arrivals
  • Use offset = log(exposure) to model rates
  • Check dispersion — real counts are usually overdispersed → quasi-Poisson / Negative Binomial
GAMMA · LOG

Positive, right-skewed continuous

  • Costs, durations, claim severity — strictly positive
  • V(μ) = μ²: constant coefficient of variation
  • Canonical inverse link exists but log wins in practice
TWEEDIE · POWER

Zero-inflated positive mass

  • Exact zeros + continuous positive tail (insurance totals)
  • Compound Poisson-Gamma for power p ∈ (1, 2)
  • TweedieRegressor(power=1.5); tune p by CV on deviance
🧰 REACH FOR A GLM WHEN…

The target is counts, rates, or positive-skewed continuous — that's the entire decision. You get the right support, a variance that scales correctly with the mean, and interpretable multiplicative effects under log links. Watch for: link/variance misspecification can mislead quietly (always look at deviance residuals and dispersion), and never fall back to OLS on counts because "it still gives a number." From here, Module 4.1 revisits the Bernoulli row in depth, and Module 3.11 (Gaussian Processes) shows the other axis of generalization — keeping the Gaussian but letting the function go nonlinear.

Overdispersion & offsets — the two field skills code

Two patterns cover most production count-modeling: exposure offsets (so coefficients describe rates) and overdispersion handling (so your standard errors aren't fantasy).

# OFFSETS — model rates with counts: log E[y] = log(exposure) + Xw import statsmodels.api as sm m = sm.GLM(claims, X, family=sm.families.Poisson(), offset=np.log(exposure)).fit() # OVERDISPERSION CHECK — Pearson chi² / dof should be ≈ 1 for a healthy Poisson fit print(m.pearson_chi2 / m.df_resid) # ≫ 1 ⇒ variance exceeds the mean # fix 1 — quasi-Poisson: identical coefficients, SEs inflated by √φ̂ m_q = sm.GLM(claims, X, family=sm.families.Poisson()).fit(scale='X2') # fix 2 — Negative Binomial: an explicit extra variance parameter m_nb = sm.GLM(claims, X, family=sm.families.NegativeBinomial(alpha=0.7)).fit() # compare nested models on AIC / deviance, NOT on R² print(m.aic, m_nb.aic)

Rule of thumb. Overdispersion changes uncertainty, not point predictions — if you only need ŷ, plain Poisson is usually fine; the moment you report intervals or p-values, dispersion is the first thing to check.
