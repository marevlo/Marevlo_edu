MODULE 3.5 · ELASTIC NET Track 3 Module 3.5 6 / 12

Mix the two penalties.
Keep what works from each.

Ridge gives you stability but never zeros. Lasso gives you sparsity but flips its mind on correlated features. Elastic Net is the convex combination — α·‖w‖₁ + ½(1−α)·‖w‖² — and that single line of math fixes both problems. The constraint ball acquires rounded corners: still sparsity (zeros possible at the corners), but now stable on correlated features (the L² component shares variance instead of arbitrarily picking one). Two hyperparameters now — α for the mix and λ for the strength — but with that comes a model that's the right default for messy real-world predictors.

01 / Learning outcomes
What you should be able to do after this lesson
Treat Elastic Net as Ridge ∪ Lasso, with one parameter controlling where on the spectrum you sit. By the end you should be able to read the rounded-polytope geometry, recognize the grouping effect on correlated features, choose α and λ by cross-validation, and know when EN strictly beats both endpoints.
01 · INTERPOLATE

α slides between L1 and L2

Read α=1 as Lasso, α=0 as Ridge, and intermediate α as Elastic Net — the constraint shape and behavior all change continuously.

02 · GROUP

Correlated features stay together

See the grouping effect made literal: where Lasso picks one of a correlated pair arbitrarily, Elastic Net assigns them comparable coefficients.

02 / A convex combination of two penalties
The loss & the two hyperparameters
Take Ridge's ½λ‖w‖² and Lasso's λ‖w‖₁, multiply each by a weight, and add them up. The L¹ side preserves the kinks at the axes that produce sparsity; the L² side adds curvature that stabilizes the answer when features are correlated. The same coordinate-descent algorithm as Lasso still works — only the per-coordinate update changes slightly.
loss : L(w) = ‖y − Xw‖² + λ ( α ‖w‖₁ + ½ (1 − α) ‖w‖² ) α ∈ [0, 1] α = 1 ⇒ Lasso α = 0 ⇒ Ridge coord-descent : ŵⱼ ← S(ρⱼ, α λ) / ( zⱼ + (1 − α) λ ) ↑ same soft-threshold numerator (sparsity) ↑ extra (1 − α) λ in the denominator (L² shrinkage) hyperparameters: α (l1_ratio) — the mix λ (alpha in sklearn) — the strength
Deriving the Elastic Net coordinate update — the L1 sparsity, augmented by L2 math · code

Fix every coefficient except wⱼ and write the EN loss as a function of wⱼ alone. You get a quadratic-plus-absolute-value, the same as Lasso, plus an extra quadratic from the L² piece. The L² term changes the curvature of the bowl but doesn't affect the kink — so you still get a piecewise solution with three cases that collapse into one formula. The only change from pure Lasso is the denominator.

1D sub-problem : min_w ½ (w − u)² + (α λ / zⱼ) |w| + ½ ((1 − α) λ / zⱼ) w² where u = ρⱼ / zⱼ collapsed : ŵⱼ = S(u · zⱼ, α λ) / (zⱼ + (1 − α) λ) = S(ρⱼ, α λ) / (zⱼ + (1 − α) λ) notice : numerator = Lasso's soft-threshold (the sparsity mechanism) denominator = Ridge-like shrinkage on what survives
# Elastic Net coordinate descent — exactly Lasso's loop, one line different import numpy as np def soft_threshold(z, gamma): return np.sign(z) * np.maximum(np.abs(z) - gamma, 0.0) def elastic_net_cd(X, y, lam, alpha=0.5, n_iter=200, tol=1e-6): """Coordinate descent for Elastic Net (assumes pre-standardized X, centered y).""" n, p = X.shape w = np.zeros(p) z = (X**2).sum(axis=0) # column norms squared r = y - X @ w for _ in range(n_iter): w_old = w.copy() for j in range(p): rho_j = X[:, j] @ (r + X[:, j] * w[j]) num = soft_threshold(rho_j, alpha * lam) w_new = num / (z[j] + (1 - alpha) * lam) # <-- the EN denominator r += X[:, j] * (w[j] - w_new) w[j] = w_new if np.abs(w - w_old).max() < tol: break return w # sklearn — same algorithm, with cross-validation over (alpha=l1_ratio, lambda=alpha) from sklearn.linear_model import ElasticNetCV from sklearn.preprocessing import StandardScaler from sklearn.pipeline import make_pipeline cv = make_pipeline(StandardScaler(), ElasticNetCV(l1_ratio=[0.1, 0.3, 0.5, 0.7, 0.9, 1.0], cv=5) ).fit(X, y) print("chosen (α, λ):", cv[-1].l1_ratio_, cv[-1].alpha_)

One naming gotcha worth flagging: scikit-learn's alpha is the penalty strength (our λ), while their l1_ratio is the L1/L2 mix (our α). It's easy to confuse them — especially since the same letter α shows up on both sides of the literature.

03 / 📐 The constraint shape, between diamond and ball
The rounded polytope — sparsity with curved walls
Same OLS loss ellipses and same ŵ_OLS as Modules 3.3 and 3.4. Now the constraint is a shape that's part diamond, part circle: the corners on the axes are preserved (you can still hit them and zero a coordinate) but the walls between them are curved (less likely to be hit by sheer alignment of corrupted data). Slide α from 1 to 0 and watch the diamond inflate smoothly into a circle — and watch the touch point slide off the corner accordingly.

Why the corners survive but the edges round — and what that means for behavior geometry · proof

The Elastic Net constraint region {β : α‖β‖₁ + ½(1−α)‖β‖² ≤ t} is the level set of a convex function that mixes L¹ and L² pieces. Where does the L¹ contribute kinks and where does the L² smooth them out?

on an axis (one coord = 0): gradient of ‖β‖₁ has a discontinuity (sign jump) gradient of ½‖β‖² is smooth ⇒ kink remains, just softened between axes (both coords nonzero): gradient of ‖β‖₁ is smooth (just sign of each coord) gradient of ½‖β‖² is smooth ⇒ totally smooth boundary

So Elastic Net's region inherits its kinks from L¹ at exactly the axes (and only those axes) — that's why sparsity is preserved. Everywhere else, the boundary is smooth, and the L² piece dominates the local curvature. Geometrically, sparsity is concentrated where it should be (on individual feature axes) while stability is distributed where it should be (between correlated columns). This is the precise reason EN can both zero coefficients and group correlated ones.

04 / The unique payoff — stability on correlated features
The grouping effect
Two features that carry almost the same information (correlation ρ = 0.95) — let's call them income and income_v2. Lasso will pick one and zero the other, but which one is essentially random — small data shifts flip the choice. Ridge keeps both, with similar smallish coefficients. Elastic Net does the right thing: keeps both, with comparable coefficients, but lets the L¹ side still discard genuine noise features. Bootstrap many resamples and the difference is dramatic — Lasso's coefficient cloud splits in two; Ridge's and EN's stay tight together.

Why L² alone yields equal coefficients on duplicates — and the grouping theorem math · intuition

For two perfectly identical columns x = xᵢ = xⱼ, the loss ‖y − βᵢx − βⱼx‖² depends only on the sum βᵢ + βⱼ — there's an entire line of equally-good OLS solutions. The L¹ penalty |βᵢ| + |βⱼ| is minimized at the corners of that line; the L² penalty βᵢ² + βⱼ² is minimized at the symmetric point βᵢ = βⱼ. So Ridge picks the symmetric (equal-coefficient) solution, and Elastic Net, which adds a bit of L² to the L¹, inherits that preference.

two identical columns, OLS slack: βᵢ + βⱼ = s (any s ∈ ℝ) L¹ argmin on the slack line : any (s, 0), (0, s), or interior with all same sign L² argmin : (s/2, s/2) ← strictly equal EN argmin (any α > 0) : (s/2, s/2) on duplicates, even tiny α suffices grouping theorem (Zou-Hastie): | β̂ᵢ - β̂ⱼ | ≤ (‖y‖ · √(2(1-ρ))) / λ (1-α) as ρ → 1, bound → 0 ⇒ coefficients forced toward equality as α → 1, bound → ∞ ⇒ Lasso allows arbitrary splits

That second inequality — the grouping inequality from Zou & Hastie's original 2005 paper — is the precise statement. The L² strength λ(1−α) appears in the denominator: more L², tighter grouping. The data correlation ρ appears in the numerator factor √(2(1−ρ)): more correlation, tighter grouping. The conclusion is a clean prescription: when features come in correlated groups and you want all members of a group to enter or leave together, ensure α < 1 — any non-trivial L² component is enough.

The right protocol — and the l1_ratio = 0 gotcha code · pitfall

Two practical points that bite real codebases.

First, the grid layout. Don't run a uniform 2-D grid; instead, give ElasticNetCV a small list of l1_ratio values it should try (typically [0.1, 0.5, 0.7, 0.9, 0.95, 0.99, 1.0], biased toward Lasso) and let it pick alphas=None for the strength grid (sklearn auto-generates a 100-point log-spaced grid spanning the meaningful range). For each l1_ratio it warm-starts down the path, so the total cost is roughly the cost of len(l1_ratio) Lasso fits.

Second, the l1_ratio = 0 trap. sklearn refuses ElasticNet(l1_ratio=0) with an error — but if you compute the grid and your "best" l1_ratio comes back as the smallest value you tried, that's a signal you've effectively chosen Ridge and should use RidgeCV directly (which is faster). Symmetrically, l1_ratio = 1 is Lasso, and LassoCV is the cleaner API.

from sklearn.linear_model import ElasticNetCV from sklearn.preprocessing import StandardScaler from sklearn.pipeline import make_pipeline cv = make_pipeline( StandardScaler(), # always scale first ElasticNetCV( l1_ratio=[0.1, 0.5, 0.7, 0.9, 0.95, 0.99, 1.0], # the α grid alphas=None, # let sklearn pick the λ grid per ratio cv=5, n_jobs=-1, max_iter=10_000, ), ).fit(X, y) en = cv[-1] print(f"chosen: α={en.l1_ratio_}, λ={en.alpha_:.4g}, nonzeros={(en.coef_ != 0).sum()}") # sanity: if best l1_ratio == smallest tried, you've effectively chosen Ridge if en.l1_ratio_ == min([0.1, 0.5, 0.7, 0.9, 0.95, 0.99, 1.0]): print("⚠ EN chose its most-Ridge option — RidgeCV might be a simpler/faster fit.")

Stability selection remains a complement here, just like in Lasso: refit on bootstrap resamples and inspect which features survive consistently. The grouping effect makes EN's stability selection more reliable than Lasso's — it doesn't flip between correlated members on every bootstrap.

06 / The right tool for the right situation
Ridge · Lasso · Elastic Net — which when
All three solve the same OLS-plus-penalty template; the question is what shape the penalty should take. Read the data first, then pick.
RIDGE (α = 0)

All features matter, none should be zero

  • Want every column to contribute, shrunk for stability
  • Features moderately correlated, no need for selection
  • p comparable to n; closed form is fast
LASSO (α = 1)

Many features irrelevant, want sparsity

  • p ≫ n; expect most coefficients to be zero
  • Features mostly uncorrelated
  • Need an interpretable subset for downstream use
ELASTIC NET (0 < α < 1)

Sparsity AND correlated features

  • Want a sparse model, but features come in correlated groups
  • p ≫ n with messy real-world predictors
  • Default for genomics, text, anywhere features cluster
PRACTICAL DEFAULT

Start with EN, l1_ratio ∈ [0.5, 0.9]

  • CV often picks an l1_ratio above 0.7 — Lasso-leaning
  • Falls back to Ridge or Lasso if appropriate
  • Robust default when you can't characterize features upfront
🧰

Reach for Elastic Net when…

You want sparsity but your features include correlated groups — genomics (correlated gene clusters), text features (synonyms), sensor arrays (spatially adjacent channels), time-shifted features. Default l1_ratio around 0.5–0.9 is a reasonable starting point; ElasticNetCV with a small list of ratios is the standard production recipe. Skip it when features are roughly uncorrelated and Lasso suffices (simpler API, faster CV), or when you don't need sparsity at all and Ridge is enough (also simpler).
