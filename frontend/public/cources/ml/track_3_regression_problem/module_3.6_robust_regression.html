When outliers can't
be removed — bound them.

Module 3.1 showed it on day one: a single bad point can drag OLS halfway across the room. The fix isn't always to delete the outlier — sometimes it's a real measurement you can't justify removing, and sometimes you don't know which point is bad. Robust regression is the family of methods that limit the damage a contaminated row can do, without you having to identify it first. Three classic recipes — Huber caps the loss in the tails, RANSAC randomly excludes outliers from the fit, Theil-Sen takes the median of pairwise slopes — each with its own breakdown point and its own use case.

02 / Outliers in OLS, the recap
Why squared loss amplifies outliers
OLS minimizes Σ rᵢ². A point ten units off the line contributes 100 times the loss of a point one unit off — so the optimizer will distort the entire fit to reduce that single squared residual. The fix at the loss level is to stop squaring residuals once they pass a threshold: keep MSE's smooth optimum near zero, but switch to MAE's bounded influence in the tails. That's exactly the Huber loss.
Huber loss with threshold δ: L_δ(r) = ½ r² if |r| ≤ δ ← Gaussian core, smooth = δ (|r| − ½δ) if |r| > δ ← Laplace tails, linear gradient is bounded: |∇L| ≤ δ everywhere ⇒ no single point can dominate the gradient direction IRLS view: weighted least squares with per-point weight wᵢ = 1 if |rᵢ/σ̂| ≤ δ wᵢ = δ · σ̂ / |rᵢ| if |rᵢ/σ̂| > δ ← outliers downweighted

Huber regression in practice uses Iteratively Reweighted Least Squares (IRLS) — each iteration fits a weighted OLS where outliers from the previous step get downweighted, and the cycle repeats until convergence. Below, every point is colored by its current Huber weight: the bright orange points have full influence (within the δ-band), the dim points are being capped. Slide ε and watch the dim points proliferate.

Huber's M-estimator family, breakdown point & the 1.35 convention theory · code

Huber is a member of the M-estimator family — estimators defined by minimizing Σ ρ(rᵢ) for some function ρ. Different ρ give different robustness/efficiency trade-offs:

OLS ρ(r) = ½ r² efficiency 100% (Gaussian), breakdown point 0% Huber ρ(r) = above formula efficiency 95%, breakdown point 0% Tukey biwt ρ(r) ∝ 1 − (1 − (r/c)²)³ efficiency 95%, breakdown point ~30% (zero weight for |r| > c — redescending) Cauchy ρ(r) = log(1 + (r/c)²) efficiency 86%, breakdown point ~50% breakdown point: max fraction of data that can be arbitrarily corrupted before the estimator returns arbitrary values

The standard epsilon=1.35 comes from a calibration result: when residuals are truly Gaussian and you set the cutoff at 1.35 σ̂, Huber's M-estimator is 95% as efficient as OLS — you give up almost no precision for a lot of outlier protection. Smaller ε increases robustness but lowers efficiency; larger ε approaches OLS.

from sklearn.linear_model import HuberRegressor import numpy as np # Huber is an M-estimator — IRLS under the hood huber = HuberRegressor( epsilon=1.35, # cap-point in σ̂-units; 1.35 is the standard alpha=1e-4, # a tiny L2 penalty for numerical stability max_iter=100, ).fit(X, y) # IRLS from scratch — one loop, weighted least squares inside def huber_irls(X, y, eps=1.35, n_iter=30): w = np.ones(len(y)) for _ in range(n_iter): # weighted OLS XW = X * np.sqrt(w)[:, None] yw = y * np.sqrt(w) beta = np.linalg.lstsq(XW, yw, rcond=None)[0] r = y - X @ beta sigma = np.median(np.abs(r - np.median(r))) / 0.6745 # MAD-based scale rs = r / max(sigma, 1e-9) w = np.where(np.abs(rs) <= eps, 1.0, eps / np.abs(rs)) # Huber weights return beta

Huber's breakdown point is technically 0% — a single outlier far enough out can still pull the estimator if ε is not adapted to it — but its influence function is bounded, so in practice it tolerates moderate contamination well. For data with potentially 30%+ corrupted rows, switch to a redescending M-estimator (Tukey biweight) or RANSAC.

03 / Random consensus — the computer-vision favorite
RANSAC — sample, fit, count, repeat
RANSAC takes a different tack from M-estimators: instead of bounding the influence of outliers, it excludes them entirely. The trick is randomized — repeatedly sample the smallest viable subset (2 points for a line), fit through them, count how many of the rest agree (the inliers), keep the model with the most consensus, and refit cleanly on those inliers only. The trial-count formula & why RANSAC tolerates 50%+ outliers probability · code

RANSAC's robustness comes from sampling theory. A trial succeeds when the sampled subset contains no outliers. The probability that any given trial draws only inliers is (1 − ε)^s where ε is the outlier fraction and s is the sample size. Run N independent trials and the probability that at least one is clean is 1 − (1 − (1 − ε)^s)^N. Set that to your target confidence p and solve for N:

N ≥ log(1 − p) / log(1 − (1 − ε)^s) example, line fitting (s=2): ε = 0.30, p = 0.99 → N ≈ 7 trials ε = 0.50, p = 0.99 → N ≈ 17 trials ε = 0.70, p = 0.99 → N ≈ 49 trials breakdown point of RANSAC: theoretically up to 100% — give it enough trials, you eventually sample a clean pair. In practice the limit is computational.

That's why RANSAC dominates in computer vision: image features routinely have 50% or more mismatches (outliers), and RANSAC just turns up the trial count and keeps going. For regression with up to ~30% contamination, Huber is faster; beyond that, RANSAC starts to be the better choice.

from sklearn.linear_model import RANSACRegressor, LinearRegression import numpy as np ransac = RANSACRegressor( LinearRegression(), residual_threshold=0.5, # in units of y; smaller = stricter inlier definition max_trials=100, min_samples=2, # minimal sample for the base estimator (line: 2 points) random_state=0, ).fit(X, y) inlier_mask = ransac.inlier_mask_ # True for the inliers used in the final fit print(f"RANSAC inliers: {inlier_mask.sum()} / {len(y)}") print(f"slope: {ransac.estimator_.coef_[0]:.3f}") # for nonlinear or higher-dim base estimators, swap in a Pipeline or any sklearn regressor from sklearn.pipeline import make_pipeline from sklearn.preprocessing import PolynomialFeatures poly_ransac = RANSACRegressor( make_pipeline(PolynomialFeatures(2), LinearRegression()), residual_threshold=0.5, ).fit(X, y)

One subtlety: residual_threshold needs tuning to the scale of your y — sklearn's default uses MAD of y, but it's worth setting explicitly. Too tight (small t) and many clean points fall outside; too loose and outliers slip in as inliers. The right cutoff is typically 1–2 standard deviations of the noise, which you may have to estimate separately on known-clean data.

04 / The simplest robust estimator
Theil-Sen — the median of every pairwise slope
For a single feature, Theil-Sen's algorithm is one line: compute the slope between every pair of points, then take the median. Outliers contaminate some slopes; non-outliers contaminate none; the median ignores both extremes and reports the right answer. The intercept follows the same recipe. It's slow (O(n²) slopes) but blissfully simple, has a high breakdown point (29.3% asymptotically), and needs no hyperparameters.
SLOPE

median of C(n, 2) pairs

For every pair (i, j) with xᵢ ≠ xⱼ, compute slope (yⱼ − yᵢ) / (xⱼ − xᵢ). The median is the estimate.

β̂₁ = median { (yⱼ − yᵢ) / (xⱼ − xᵢ) }
INTERCEPT

median of point-line offsets

Once the slope is fixed, β̂₀ is the median of { yᵢ − β̂₁ xᵢ } — every point produces its own intercept guess; we trust the middle one.

β̂₀ = median { yᵢ − β̂₁ xᵢ }
BREAKDOWN POINT

29.3% asymptotically

The median of pairs can be corrupted by replacing one element of each pair, but with two coords needing to be flipped per pair, only ~29% of the data can be corrupted before the estimator fails.

1 − 1/√2 ≈ 0.293
multivariate generalization: for each pair of points, fit the unique linear model through them the estimator is the (spatial) median of all such fits sklearn : TheilSenRegressor(max_subpopulation=10000) ← caps the n² blow-up cost : O(n²) for univariate, O(n^(p+1)) naively for p features — sklearn uses random subsets no hyperparameters ! unlike Huber's ε or RANSAC's residual_threshold

06 / Which robust estimator, when
Pick by contamination level
All four estimators are convex and computable; the choice is dictated by what you know about your contamination and your data scale.
OLS (α = 0)

Use if you trust every point

  • Residuals plausibly Gaussian, no known outliers
  • Fastest, most efficient under correct model
  • Single bad row tanks the fit
HUBER

Light contamination < 10%

  • 95% Gaussian efficiency at ε = 1.35
  • Smooth gradient — plays well with downstream optimization
  • Breakdown point asymptotically 0%, but bounded influence
RANSAC

Heavy contamination 30–80%

  • Theoretical breakdown up to 100%
  • Tune residual_threshold to noise scale
  • Computationally heavy; stochastic outputs
THEIL-SEN

Small n, want simplicity

  • No hyperparameters at all
  • 29.3% breakdown point, deterministic output
  • O(n²) — careful with large data
⚠️

Robust regression is for the model, not the data

The goal is a fit that describes the bulk, not a fit that uses every point. After fitting, always plot residuals to verify that the points the robust estimator excluded or downweighted are actually outliers — typos, sensor failures, data-entry errors — and not legitimate observations from a heavy-tailed distribution that you should have modeled directly (Student-t regression, quantile regression, GLMs in 3.10). Robust regression hides distributional misspecification rather than fixing it.

🧰

Reach for robust regression when…

You have known outliers you can't remove (legitimate but extreme observations), sensor data with occasional failures, vision/SLAM-style data where 30%+ of matches are wrong (RANSAC's home turf), or diagnostic fits where you want a baseline that ignores leverage points. Skip in favor of OLS when residuals are clearly clean and Gaussian (Module 3.1 diagnostics confirm); reach for a GLM (3.10) when residuals are heavy-tailed but not contaminated — e.g., Student-t errors are better modeled directly than bounded.
