02 / One added term, one cleaner closed form
The loss & the L² closed form
Take the OLS loss from Module 3.1 and add a quadratic penalty on the coefficients. The penalty is also linear in its gradient, so the same "set ∇L = 0" recipe still works — out the other end comes a closed form that differs from OLS by exactly one matrix: a λI added inside the inverse.
loss : L(w) = ‖y − Xw‖² + λ ‖w‖² gradient : ∇L = −2 Xᵀ(y − Xw) + 2 λ w ∇L = 0 : (XᵀX + λI) w = Xᵀy ← always invertible for λ > 0 closed form : ŵ = (XᵀX + λI)⁻¹ Xᵀy limit : λ → 0 ⇒ OLS λ → ∞ ⇒ ŵ → 0
Why λI guarantees invertibility — and what it costs math · numerics · code

Let XᵀX have eigenvalues d₁² ≥ d₂² ≥ … ≥ dₚ² ≥ 0. Adding λI shifts every eigenvalue by exactly λ, so the new matrix XᵀX + λI has eigenvalues d₁² + λ, …, dₚ² + λ. As long as λ > 0, every eigenvalue is strictly positive — the matrix is positive-definite, the inverse always exists, and the solve is stable.

eigen(XᵀX) = d₁² ≥ d₂² ≥ … ≥ dₚ² ≥ 0 eigen(XᵀX + λI) = d₁²+λ ≥ d₂²+λ ≥ … ≥ dₚ²+λ > 0 condition number κ(XᵀX + λI) ≤ (d₁² + λ) / (dₚ² + λ) ≤ κ(XᵀX), monotonically with λ ↑

The trade. The penalty isn't free — it biases ŵ toward zero. The bias-variance analysis from Module 3.0 says that's exactly what you want: trade a little bias for a lot of variance reduction. The directions where OLS is most uncertain (small dⱼ²) are also the ones Ridge shrinks the hardest, which we'll prove in the spectral view below.

# Ridge from scratch — one line different from OLS import numpy as np def ridge(X, y, lam): p = X.shape[1] return np.linalg.solve(X.T @ X + lam * np.eye(p), X.T @ y) # sklearn equivalent with multiple solvers + cross-validated lambda from sklearn.linear_model import Ridge, RidgeCV from sklearn.preprocessing import StandardScaler from sklearn.pipeline import make_pipeline # scaling MATTERS — L2 penalizes raw coefficients pipe = make_pipeline( StandardScaler(), RidgeCV(alphas=np.logspace(-4, 4, 50), cv=5), # picks the best alpha ) pipe.fit(X, y) print("chosen λ:", pipe[-1].alpha_)

Solver choice in Ridge matters for big problems. cholesky uses the direct closed form (fastest for small p); lsqr and sparse_cg are iterative (memory-light, for huge sparse X); sag/saga are stochastic (best when n is enormous). Default 'auto' picks one for you.

03 / 📐 The picture that explains everything
The L² ball constraint — smooth shrinkage, no zeros
Ridge has an equivalent constrained formulation: min ‖y − Xβ‖² subject to ‖β‖₂ ≤ t. In coefficient space, the level sets of the OLS loss are ellipses centered at ŵ_OLS, and the constraint is a ball centered at the origin. The Ridge solution is exactly where the innermost ellipse touches the ball — and because the ball is smooth, that touch happens generically at an interior point. No corner, no exact zero. Slide λ and watch the touch point slide smoothly toward the origin.

The Lagrangian bridge & why "smooth ball = no sparsity" is a theorem, not a hope geometry · math

The constrained form min ‖y − Xβ‖² s.t. ‖β‖₂² ≤ t and the penalized form min ‖y − Xβ‖² + λ‖β‖² are dual: for every λ > 0 there exists a t(λ) such that the two problems share the same solution. Sliding λ up is identical to sliding t down — the ball shrinks, the touch point retreats toward the origin.

at the optimum, KKT conditions: ∇‖y − Xβ‖² + λ ∇‖β‖² = 0 −2 Xᵀ(y − Xβ) + 2 λ β = 0 ⇒ (XᵀX + λI) β = Xᵀy ← same closed form, derived geometrically the L² ball ‖β‖₂ ≤ t has a SMOOTH boundary (sphere) ⇒ tangent direction at the touch point is generic (no special axis-alignment) ⇒ β̂ rarely has any exact-zero coordinate (probability zero, for generic data)

This is the precise statement of "Ridge does not select features." It always uses every column — possibly with tiny coefficients, but never exactly zero. That's the right behavior when you believe all features carry some signal but want to share variance among them. When you want a few strong features and the rest off, switch to the L¹ ball (Lasso, next module) where the boundary has corners that pin coefficients to exactly zero. The geometry of the constraint set is the regularization behavior.

04 / Ridge in PCA coordinates — soft dimensionality reduction
Spectral shrinkage
Rotate into the principal-axis basis of XᵀX and the Ridge solution decouples completely: each principal direction j gets shrunk by a factor dⱼ²/(dⱼ² + λ), where dⱼ² is its eigenvalue. Well-determined directions (large dⱼ²) barely shrink. Poorly-determined directions (small dⱼ²) shrink to nearly zero. That's Ridge as soft PCA — the directions you can't measure precisely are the ones it suppresses.

Why this is the cleanest view of what Ridge does math · code · intuition

The SVD X = U Σ Vᵀ gives an orthonormal coordinate system in which the regression problem decouples: each principal direction is fit independently. In that basis, OLS divides by dⱼ — so when dⱼ is small, the OLS coefficient blows up (high variance). Ridge replaces the divisor with dⱼ + λ/dⱼ, which for small dⱼ is dominated by λ/dⱼ — so the coefficient gets pulled toward zero instead of exploding.

OLS : ŵⱼ = (Uᵀy)ⱼ / dⱼ ← small dⱼ ⇒ huge ŵⱼ ⇒ unstable Ridge: ŵⱼ = (Uᵀy)ⱼ · dⱼ / (dⱼ² + λ) = OLS · [ dⱼ² / (dⱼ² + λ) ] ← shrinkage factor ∈ [0, 1] high dⱼ² (well-determined): factor ≈ 1, barely shrinks low dⱼ² (poorly-determined): factor → 0, shrunk to nearly zero

Connection to PCA. Ridge with very large λ behaves like PCA truncation — it effectively zeros out small-eigenvalue directions. But unlike hard PCA truncation, the cutoff is smooth: each direction gets weight proportional to how well the data determines it. This is one of the deepest properties of Ridge and the reason it's so well-behaved on real-world correlated data.

# Ridge via SVD — explicit, numerically robust import numpy as np def ridge_svd(X, y, lam): U, d, Vt = np.linalg.svd(X, full_matrices=False) shrink = d / (d**2 + lam) # the per-direction factor return Vt.T @ (shrink * (U.T @ y)) # compare: same answer as the normal-equation form, but stable when X is ill-conditioned w_ne = np.linalg.solve(X.T @ X + lam * np.eye(p), X.T @ y) w_svd = ridge_svd(X, y, lam) assert np.allclose(w_ne, w_svd) # effective degrees of freedom: how many "effective parameters" you used df_eff = (d**2 / (d**2 + lam)).sum() print(f"with lam={lam}, df_eff ≈ {df_eff:.2f} (out of p = {p})")

The df_eff formula is genuinely useful: it's the right notion of "model complexity" to plug into AIC / BIC for Ridge, and it's what scikit-learn's RidgeCV internally tracks when picking λ. It also explains why Ridge can handle p > n — even though OLS doesn't exist there, df_eff stays sensible and the fit is well-defined.

05 / The Bayesian view — Ridge as a posterior mode
Ridge = MAP under a Gaussian prior
Same equation, completely different interpretation. Place a Gaussian prior on the coefficients — "I expect w to be small but nonzero" — and ask for the maximum-a-posteriori (MAP) estimate under Gaussian noise. The negative log-posterior turns out to be exactly the Ridge loss, with λ set by the ratio of noise variance to prior variance. The penalty isn't an arbitrary regularizer — it's a prior belief made explicit.
prior : w ~ 𝒩(0, τ² I) ← "small coefficients are more probable" likelihood : y | X, w ~ 𝒩(Xw, σ² I) log-posterior: log p(w | X, y) = −‖y − Xw‖²/(2σ²) − ‖w‖²/(2τ²) + const MAP : argmax log p = argmin ‖y − Xw‖² + λ ‖w‖² where λ = σ²/τ² ∴ Ridge solution = posterior mode under Gaussian-Gaussian conjugacy
SMALL τ²

Strong prior → large λ

You believe coefficients should be tiny. The prior pulls hard, Ridge shrinks aggressively. Good when p ≫ n and most features are weak.

LARGE τ²

Weak prior → small λ

You're nearly agnostic about coefficient magnitudes. Ridge barely shrinks; the solution approaches OLS. Default to this when data is abundant.

FULL POSTERIOR

BayesianRidge / Bayesian LR

Instead of just the mode, compute the entire posterior 𝒩(μ_post, Σ_post) — that's Bayesian linear regression (Module 3.9), which gives uncertainty bands for free.

Choosing λ honestly — RidgeCV & stability selection code · protocol

Picking λ on training error is the same mistake as picking polynomial degree on training error — train RSS is monotone in λ⁻¹, so smaller is always better. The honest signal is held-out performance, ideally cross-validated. RidgeCV sweeps a grid of λ values, picks the one with best CV score, and refits.

from sklearn.linear_model import RidgeCV from sklearn.preprocessing import StandardScaler from sklearn.pipeline import make_pipeline import numpy as np # the right pipeline — scaling FIRST (penalty is on raw coefficients) ridge_cv = make_pipeline( StandardScaler(), RidgeCV(alphas=np.logspace(-4, 4, 50), cv=5, scoring="r2"), ).fit(X, y) print("chosen λ:", ridge_cv[-1].alpha_) print("coefficients:", ridge_cv[-1].coef_)

Stability selection is a complement: re-fit Ridge on many bootstrap resamples and ask which features consistently get large coefficients. Features that survive across resamples are the ones the model is confidently using; those whose coefficients drift across resamples are sample-dependent noise. The same procedure flags non-robust features whether you're using Ridge, Lasso, or trees.

07 / The same closed form, in feature space
Kernel Ridge — nonlinear without leaving the linear toolkit
Module 3.2 lifted inputs through φ(x) = [1, x, x², …] to make linear OLS handle curves. Kernel Ridge takes that idea to its limit: it works in a feature space that can be infinite-dimensional, by computing dot products with a kernel K(x, x′) = φ(x) · φ(x′) instead of materializing φ. The Ridge closed form rewrites cleanly in terms of the n × n Gram matrix K — and suddenly you have a nonlinear regressor with the same one-line training.
primal (p × p) : ŵ = (XᵀX + λI)⁻¹ Xᵀy dual (n × n) : ŷ(x) = k(x)ᵀ (K + λI)⁻¹ y where Kᵢⱼ = K(xᵢ, xⱼ), k(x)ᵢ = K(x, xᵢ) RBF kernel : K(x, x′) = exp(−‖x − x′‖² / (2 ℓ²)) ← infinite-dim feature space polynomial kernel : K(x, x′) = (1 + xᵀx′)ᵈ ← degree-d feature space, no expansion cost : O(n³) — switch to a different method if n is huge
DUAL FORM

n × n instead of p × p

When p ≫ n (lifted features explode), invert the smaller matrix. Ridge is one of the few losses where the dual is also a closed form.

RBF KERNEL

Infinite-dim feature space

Locality-based regression with one length-scale parameter ℓ. Captures arbitrary smooth nonlinearities without choosing a polynomial degree.

RKHS & GPs

Same math as Gaussian Process

The posterior mean of a GP with kernel K and noise σ² is literally Kernel Ridge with λ = σ². The bridge to Module 3.11.

Why the dual exists for Ridge — & one-line Kernel Ridge in code math · code

The primal-dual identity for Ridge comes from a small matrix-inversion lemma. Starting from ŵ = (XᵀX + λI)⁻¹Xᵀy and using the push-through identity (XᵀX + λI)⁻¹Xᵀ = Xᵀ(XXᵀ + λI)⁻¹, we get an equivalent expression involving only the Gram matrix XXᵀ:

ŵ = Xᵀ α where α = (XXᵀ + λI)⁻¹ y ← dual coefficients ŷ(x*) = w · x* = α · (X x*) = Σᵢ αᵢ ⟨xᵢ, x*⟩ ← prediction is a weighted sum of inner products replace ⟨xᵢ, x*⟩ with K(xᵢ, x*) ⇒ Kernel Ridge in any RKHS
from sklearn.kernel_ridge import KernelRidge # nonlinear regression with the same training-time complexity as linear regression — for moderate n krr = KernelRidge( alpha=1.0, # lambda kernel="rbf", gamma=0.5, # gamma = 1/(2*ell^2) ).fit(X, y) # from scratch — three lines import numpy as np from scipy.spatial.distance import cdist K_train = np.exp(-0.5 * cdist(X, X)**2 / ell**2) alpha = np.linalg.solve(K_train + lam * np.eye(len(X)), y) K_pred = np.exp(-0.5 * cdist(X_star, X)**2 / ell**2) y_pred = K_pred @ alpha # nonlinear regression, one matmul

The catch is the O(n³) training cost from inverting the n × n kernel matrix — fine for n < 10⁴, painful beyond. For larger problems, switch to sparse approximations (Nyström, random features) or move to Gaussian Process inducing-point methods (Module 3.11).

🧰

Reach for Ridge when…

p is comparable to or greater than n, features are correlated, you want all features in the model (with shared variance), or you need a stable closed form for downstream inference. It's the right default when OLS feels unstable and you don't yet need sparsity. Skip it when you want a small interpretable subset of features (use Lasso → 3.4), correlated features should be grouped (Elastic Net → 3.5), or the target has heavy-tailed errors (robust regression → 3.6). And remember: always scale features first, and pick λ with RidgeCV, never by eye.
