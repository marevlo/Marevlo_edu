Predict a number —
but which number?

The "right" answer for a regression at X = x depends on the question you're asking: the conditional mean (squared loss), the median (absolute loss), a quantile (pinball loss), or a robust middle (Huber). Each loss is a noise model in disguise, and choosing it is the first modeling decision. This module is the why before the how — the foundation every algorithm in Track 3 builds on.

02 / The optimal point predictor, derived
Regression as estimating 𝔼[Y | X = x]
For any candidate predictor m(x), the expected squared error decomposes as variance around the conditional mean + squared distance to it. The cross-term vanishes, so the minimum is reached when m(x) = 𝔼[Y | X = x] exactly. Every regression in this track is some way of estimating that curve. Drag the slider and watch the local mean of Y in a window around x trace it out.

Proof that 𝔼[Y|X] minimizes squared loss — in three lines proof · code

For any predictor m(x), add and subtract the conditional mean μ(x) = 𝔼[Y|X=x] inside the squared error and take expectations conditional on X = x:

𝔼[(Y − m(x))² | X = x] = 𝔼[(Y − μ(x))² | X = x] + (μ(x) − m(x))² + 2(μ(x) − m(x)) · 𝔼[Y − μ(x) | X = x] └──── = 0 by definition of μ ────┘ = Var(Y | X = x) + (μ(x) − m(x))²

The first term is irreducible noise — nothing you can model away. The second is non-negative and equals zero iff m(x) = μ(x). So at every x the best squared-error predictor is the conditional mean. Integrate over the distribution of X and the same conclusion holds globally.

# empirical conditional expectation — a window-average over X import numpy as np def cond_mean(x, X, y, h=0.08): """Estimate E[Y | X=x] as the mean of Y over points with |X - x| <= h.""" mask = np.abs(X - x) <= h return y[mask].mean() if mask.any() else np.nan grid = np.linspace(X.min(), X.max(), 200) m_hat = np.array([cond_mean(x, X, y) for x in grid]) # m_hat is the Nadaraya-Watson estimator with a uniform kernel — a non-parametric regression.

Every algorithm in Track 3 is a different way of estimating μ(x): linear regression assumes μ is affine, polynomial regression that it's polynomial, KNN takes a local average like the snippet above, trees use axis-aligned step functions. The target never changes — only the assumption about its shape.

03 / The choice that picks which statistic you estimate
Five loss functions, five statistics
Each loss minimizes to a different summary of the conditional distribution. Squared loss recovers the mean; absolute loss recovers the median; pinball-τ recovers the τ-quantile; Huber gives a robust-mean that resists outliers. Toggle the loss and watch the optimal constant predictor jump from one statistic to another — and watch the mean (only) get yanked toward an outlier.

Every loss is a noise model in disguise — the MLE bridge math · code

Maximum-likelihood estimation under a noise distribution p(ε) is equivalent to minimizing −log p(ε) as a loss. Plug in different densities and you get the famous losses out the other end — same recipe, different ingredient.

Gaussian ε ~ 𝒩(0, σ²) → −log p = ½r²/σ² + const ⇒ squared loss Laplace ε ~ Laplace(0, b) → −log p = |r|/b + const ⇒ absolute loss Asymm. Lap (τ-skewed) → −log p = pinball_τ(r)/b ⇒ quantile loss Huber mix Gaussian centre, Laplace tails → −log p ∝ Huber_δ(r) ⇒ Huber loss

So the choice of loss is the choice of which noise model you believe. If errors are symmetric and light-tailed, MSE/Gaussian is the MLE — and the conditional mean is recovered. If errors are heavy-tailed, MAE/Laplace is the MLE — and the conditional median is recovered. The geometry follows: a fat-tailed noise model can't let a single far point dominate, which is exactly why MAE and Huber resist outliers while MSE does not.

from sklearn.linear_model import QuantileRegressor, HuberRegressor, LinearRegression # OLS: Gaussian-noise MLE -> predicts E[Y|X] m_mean = LinearRegression().fit(X, y) # Quantile regression at the median -> Laplace MLE -> predicts median(Y|X) m_med = QuantileRegressor(quantile=0.5, alpha=0).fit(X, y) # 5th and 95th conditional quantiles -> a prediction interval, model-free m_lo = QuantileRegressor(quantile=0.05, alpha=0).fit(X, y) m_hi = QuantileRegressor(quantile=0.95, alpha=0).fit(X, y) # robust mean — bounded outlier gradients m_rob = HuberRegressor(epsilon=1.35).fit(X, y) # epsilon = delta scaled by sigma

That last block is the practical payoff: by changing one estimator, you're switching the entire question — from "what's the average outcome?" to "what's the 95th-percentile worst case?" — without rewriting any features.

04 / The map you keep in your head
Loss ↔ noise ↔ statistic
A compact summary of the bridge above — the right cheat sheet to consult when picking a loss.
SQUARED · MSE

𝒩 noise → mean

The default and the MLE if residuals are symmetric and light-tailed. Sensitive to outliers — every error is squared, so one extreme point dominates.

ABSOLUTE · MAE

Laplace noise → median

Equal weight to every error. Robust to outliers; predicts the median rather than the mean. Non-smooth at zero — needs sub-gradient methods.

HUBER

Mixed → robust mean

Gaussian inside |r| ≤ δ, Laplace outside. Gradients are bounded in the tails, so a single outlier can't drag the fit. The practitioner's compromise.

QUANTILE · PINBALL

Asymm. Laplace → quantile

Predicts a chosen percentile, not the mean. Fit τ=0.05 and τ=0.95 separately to get a prediction interval without any distributional assumption.

LOG-COSH

Smooth ≈ MAE

Behaves like MSE near zero, like MAE in the tails — twice-differentiable everywhere, which gradient-based optimizers love. Common in deep regressors.

RULE OF THUMB

Match loss to data

Clean tails → squared. Outlier-prone → Huber or MAE. Need an interval → quantile pair. Count / rate / positive-skewed target → GLM (→ Module 3.10).

05 / Two philosophies of "what shape is 𝔼[Y|X]?"
Parametric vs non-parametric
Once you've chosen a loss, you still have to choose a function class for m(x). The split is fundamental: commit to a fixed-size family, or let the model grow with the data.
parametric

Fixed-size function family

  • A finite parameter vector w ∈ ℝᵖ summarizes the whole model
  • Examples: linear (3.1), polynomial (3.2), Ridge/Lasso/Elastic Net (3.3–3.5), GLMs (3.10)
  • Cheap inference, strong assumptions, interpretable
  • Fails when the assumption is wrong (linearity, distribution)
non-parametric

Model size grows with the data

  • No fixed parameter count — predictions reference the data itself
  • Examples: KNN (3.7), trees (3.8), Gaussian processes (3.11)
  • Flexible, weaker assumptions, locally adaptive
  • Hungry for data; vulnerable to the curse of dimensionality
🔗

Linear & logistic regression are special GLMs

Both estimate 𝔼[Y|X] via a linear predictor η = Xw passed through a link: identity for linear (Gaussian noise), logit for logistic (Bernoulli). The full unifying framework — Poisson, Gamma, Tweedie — arrives in Module 3.10, so we can talk about count regression and rate models the same way we talk about OLS.

06 / The engineer's view of generalization
Bias² · variance · irreducible σ²
Held-out error has three sources, and they trade against each other through model complexity. The same equation explains why a deep tree memorizes and a constant predictor underfits. The decomposition derivation, and how to estimate it from data proof · code

Let m̂_𝒟 be the model trained on dataset 𝒟, and μ(x) = 𝔼[Y|X=x]. Treat both the dataset and the noise as random. The expected test error at x splits as:

𝔼[(Y − m̂(x))²] = 𝔼[(Y − μ(x))²] + 𝔼[(μ(x) − m̂(x))²] = σ²(x) + 𝔼[(μ(x) − 𝔼[m̂(x)] + 𝔼[m̂(x)] − m̂(x))²] = σ²(x) + (μ(x) − 𝔼[m̂(x)])² + Var(m̂(x)) └ irreducible ─┘ └────── bias² ─────┘ └ variance ┘

The cross-term vanishes by exactly the same trick as in the conditional-expectation proof above. Each piece has a different lever: bias shrinks with model capacity, variance grows with it, noise is fixed by the data-generating process. You cannot beat σ² with better modeling, only with better data.

# empirical bias-variance decomposition via repeated resampling import numpy as np def bias_var(make_model, gen_data, x_grid, B=200): preds = np.zeros((B, len(x_grid))) for b in range(B): Xb, yb = gen_data() # a fresh draw of the training set preds[b] = make_model().fit(Xb, yb).predict(x_grid.reshape(-1, 1)) mean_pred = preds.mean(axis=0) bias2 = ((mean_pred - true_mu(x_grid))**2).mean() var = preds.var(axis=0).mean() return bias2, var # noise sigma^2 is set by the simulator

This recipe powers the demo above: 20 datasets, 20 polynomial fits, average them to see bias, spread them to see variance. The slider proves the trade-off — there's a single complexity that minimizes the sum, and it's not at either extreme.
