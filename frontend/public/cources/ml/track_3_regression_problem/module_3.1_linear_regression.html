One equation,
five centuries of method.

Linear regression is the simplest algorithm in this entire track — and it's also where most of the deep ideas first appear: a loss derived from a noise model, a closed-form solution by setting a gradient to zero, an interpretation as an orthogonal projection, and a host of diagnostics that reveal whether the assumptions you made are honest. Master it here and every later algorithm becomes a variation on the same theme.

02 / The model, the loss, the closed-form solution
Setup & the normal equation
Assume the conditional mean is linear: y = Xw + ε with ε ~ 𝒩(0, σ²I). The maximum-likelihood estimator under that noise model is exactly the least-squares estimator. Differentiating the loss and setting it to zero yields a single linear system — and the closed form. Below, watch how dragging one outlier tilts the entire fitted line: OLS gives every error equal squared weight, so one bad point has outsized influence.

model : y = Xw + ε, ε ~ 𝒩(0, σ²I) loss : L(w) = ‖y − Xw‖² = (y − Xw)ᵀ(y − Xw) ∇L = 0 ⇒ −2 Xᵀ(y − Xw) = 0 normal eqn : XᵀX w = Xᵀy closed form: ŵ = (XᵀX)⁻¹ Xᵀy (= MLE under Gaussian noise)
The derivation in three lines + the MLE bridge that justifies the loss proof · math · code

Expand the loss and take the gradient with respect to w. Two identities do all the work: ∂(aᵀw)/∂w = a and ∂(wᵀAw)/∂w = (A + Aᵀ) w. Since XᵀX is symmetric, the second collapses to 2 XᵀX w.

L(w) = yᵀy − 2 yᵀX w + wᵀ XᵀX w ∇L = − 2 Xᵀy + 2 XᵀX w ∇L = 0 ⇒ XᵀX w = Xᵀy (the normal equation) ⇒ ŵ = (XᵀX)⁻¹ Xᵀy (assuming XᵀX is invertible)

Why squared loss? Under ε ~ 𝒩(0, σ²I), the log-likelihood of w is const − ‖y − Xw‖² / (2σ²). Maximizing the likelihood is therefore literally identical to minimizing the squared loss — OLS is the Gaussian-noise MLE. Pick a different noise distribution and you get a different loss (recall Module 3.0).

Classical assumptions. The closed form needs five things to hold for the inference to be honest: linearity (𝔼[Y|X] is actually linear in X), independence of errors across rows, homoscedasticity (constant σ²), normality of errors, and no perfect multicollinearity (so XᵀX is invertible). The residual diagnostics later in this module are exactly checks for each of these.

# OLS from scratch via the normal equation import numpy as np X = np.c_[np.ones(len(x)), x] # prepend intercept column w = np.linalg.solve(X.T @ X, X.T @ y) # solve, don't invert # sklearn equivalent — predictions only, no inference from sklearn.linear_model import LinearRegression ols = LinearRegression().fit(x.reshape(-1, 1), y) # statsmodels for FULL inference (p-values, CIs, F-test, diagnostics) import statsmodels.api as sm ols_sm = sm.OLS(y, sm.add_constant(x)).fit() print(ols_sm.summary())

Use np.linalg.solve rather than np.linalg.inv(X.T @ X) @ X.T @ y — solving the system directly is numerically more stable than forming the inverse explicitly, especially when columns are nearly collinear.

03 / 📐 The geometric heart of OLS
ŷ is the orthogonal projection of y onto col(X)
The normal equation isn't just an algebraic trick — it's a statement of perpendicularity. Xᵀ(y − Xŵ) = 0 literally says "the residual is perpendicular to every column of X." Slide the slope away from the OLS solution and watch two things happen in lock-step: the residual stops being orthogonal to x, and the total squared error grows. The minimum of RSS is the orthogonal projection — the same fact, viewed two ways.

normal equation: Xᵀ(y − Xŵ) = 0 ← residual ⟂ every column of X component-wise : Σᵢ rᵢ = 0 (perpendicular to the intercept column) Σᵢ rᵢ · xᵢⱼ = 0 (perpendicular to feature j) the hat matrix H = X(XᵀX)⁻¹Xᵀ projects: ŷ = H y, r = (I − H) y, Hᵀ(I − H) = 0
The hat matrix & why projection is the right mental picture geometry · math

Treat y ∈ ℝⁿ as a single point in n-dimensional space. The fitted values ŷ = Xŵ can only live in the column space of X — a (p+1)-dimensional subspace, where p is the number of features. The closest point in that subspace to y, in Euclidean distance, is the foot of a perpendicular dropped from y onto the subspace. That perpendicular drop is exactly OLS.

ŷ = X ŵ = X(XᵀX)⁻¹ Xᵀ y = H y (H = the "hat matrix" / projection operator) H is symmetric and idempotent : H² = H, Hᵀ = H ← that's what makes it a projection trace(H) = p + 1 ← degrees of freedom used by the model r = (I − H) y ← residual = projection onto the orthogonal complement Hᵀ(I − H) = 0 ← ŷ ⟂ r, always

The orthogonality conditions you see in the demo are a direct consequence: r ⟂ col(X) means r is perpendicular to every column of X, which is the same as Xᵀr = 0, which is the same as the normal equation. Same fact, three notations.

Why this matters practically. Two consequences fall out immediately. First, the closest point in a subspace is unique only if the columns of X are linearly independent — otherwise the projection is well-defined but the coordinates ŵ are not (that's multicollinearity, next section). Second, the geometry gives you the variance of ŵ for free: Var(ŵ) = σ² (XᵀX)⁻¹, so directions in feature space where the column cloud is narrow get amplified inverse-quadratically — small data shifts cause huge coefficient swings.

04 / 🔨 Three ways to solve the same equation
Normal equation · GD · SVD pseudo-inverse
The math gives one closed form; numerically, you implement it three different ways depending on the conditioning of X, the size of n and p, and whether you want incremental updates. This is also the first contribution to your tinyml/ from-scratch library.
A · NORMAL EQUATION

Direct solve via np.linalg.solve

Fastest for small/medium p, well-conditioned X. Forms the p × p Gram matrix XᵀX — squares the condition number, which is the catch.

cost ≈ O(np² + p³) accuracy: great if κ(X) small
B · GRADIENT DESCENT

Iterate ŵ ← ŵ − η Xᵀ(Xŵ − y)

Memory-light, streaming-friendly, the only option when n is enormous or you want SGD. Needs a learning-rate schedule; convergence depends on the spectrum of XᵀX.

cost / iter ≈ O(np) accuracy: depends on η, iters
C · SVD PSEUDO-INVERSE

ŵ = V Σ⁺ Uᵀ y

The robust choice when X is ill-conditioned or rank-deficient. Handles non-invertible XᵀX by truncating tiny singular values — gives the minimum-norm solution.

cost ≈ O(np² ) accuracy: best κ(X) ≫ 1
All three solvers in NumPy, and which to reach for when code · numerics

Behind all of them is the same closed form; the differences are entirely numerical. The condition number κ(X) — the ratio of the largest to smallest singular value of X — decides which is safe. Forming XᵀX squares κ; the SVD avoids it entirely.

solve(XᵀX, Xᵀy) works well when κ(X) is small squares κ(X) to κ(XᵀX) = κ(X)² SVD : ŵ = V Σ⁺ Uᵀ y, where X = U Σ Vᵀ Σ⁺ inverts non-zero singular values, sets zeros to zero gives the minimum-norm ŵ even when rank(X) < p
import numpy as np def ols_normal(X, y): """Normal equation. Fast, fails when X^T X is near-singular.""" return np.linalg.solve(X.T @ X, X.T @ y) def ols_gd(X, y, lr=0.05, n_iter=1000): """Gradient descent. Memory-light, streaming-friendly.""" n, p = X.shape w = np.zeros(p) for _ in range(n_iter): w -= (lr / n) * X.T @ (X @ w - y) # gradient step return w def ols_svd(X, y, tol=1e-10): """SVD pseudo-inverse. Robust to ill-conditioning and rank-deficiency.""" U, s, Vt = np.linalg.svd(X, full_matrices=False) s_inv = np.where(s > tol * s.max(), 1 / s, 0.0) # truncate tiny sv's return Vt.T @ (s_inv * (U.T @ y)) # condition number — your guide to which solver to pick kappa = np.linalg.cond(X) if kappa < 1e6: w = ols_normal(X, y) # well-conditioned: fastest path else: w = ols_svd(X, y) # ill-conditioned: trust SVD

Two rules in production: (1) never form an explicit inverse — always np.linalg.solve or np.linalg.lstsq (which uses SVD internally). (2) check np.linalg.cond(X) before trusting the coefficients — if it's above ~10⁶, the columns are nearly dependent and the fit is unstable. The next section is the diagnostic for exactly that case.

05 / When two columns mean almost the same thing
Multicollinearity & the VIF
If two columns of X are nearly proportional, XᵀX is nearly singular, its inverse blows up, and so does the variance of ŵ. The fitted line is still fine — but the coefficients are wild. The Variance Inflation Factor quantifies exactly how much: VIF_j = 1 / (1 − R²_j), where R²_j is the R² from regressing column j on the other columns. Crank up the correlation and watch the coefficient cloud explode.

Why the variance explodes, and the three things you can do about it math · code · fix

Write X in standardized form. For two features with correlation ρ, the off-diagonal of XᵀX/n is ρ, and the diagonal of its inverse — which appears in Var(ŵ) — is exactly 1/(1 − ρ²). That denominator is the VIF, derived geometrically: as the columns of X swing toward parallel, the data cloud collapses onto a line, and OLS has to amplify the noise to keep distinguishing them.

2-feature case (standardized): XᵀX/n = [[1, ρ], [ρ, 1]] (XᵀX/n)⁻¹ = 1/(1−ρ²) · [[1, −ρ], [−ρ, 1]] Var(ŵⱼ) ∝ 1/(1 − ρ²) ← the VIF ρ → 1 ⇒ Var(ŵ) → ∞, fit is still fine, but ŵ is meaningless

Three responses, in increasing aggression:

1. Diagnose, then drop. Compute VIFs; if one column has VIF > 10 and is conceptually redundant with another (e.g., height_cm and height_m), remove it.

2. Regularize. Add an L2 penalty to make the system invertible by construction — that's Ridge regression (Module 3.3). The closed form becomes ŵ = (XᵀX + λI)⁻¹Xᵀy; the λI shifts the spectrum away from zero, killing the variance inflation at the cost of a little bias.

3. Reparameterize. Replace correlated columns with principal components, or combine them into a single hand-built feature (BMI = weight / height²) that the domain blesses.

# diagnose collinearity before trusting the coefficients from statsmodels.stats.outliers_influence import variance_inflation_factor import numpy as np, pandas as pd Xc = sm.add_constant(X) vif = pd.Series( [variance_inflation_factor(Xc.values, i) for i in range(Xc.shape[1])], index=Xc.columns ) print(vif.sort_values(ascending=False)) # rule of thumb: VIF > 10 -> investigate; VIF > 100 -> the columns are essentially duplicates # condition number gives the same warning in one number: print("cond(X) =", np.linalg.cond(Xc))

VIF is a per-column number; condition number is a single global one. In practice you check both. If condition is high, your inference is unstable regardless of which column you blame.

06 / Is OLS actually a good fit for this data?
The four residual plots
Five assumptions, four plots, one fast verdict. Each panel is targeted at a specific assumption: linearity, normality, constant variance, and influence. Switch the scenario and watch the diagnostic scream at the broken assumption while the others stay quiet.

How to read each panel — and what to do when one screams code · interpretation

① Residuals vs fitted — should be a featureless cloud around zero. A curve (smile or frown) means the linearity assumption is broken: 𝔼[Y|X] is not actually linear. Fix: add a polynomial or interaction term (Module 3.2), bin the feature (Module 2.5), or move to a nonlinear model (KNN / trees / GP).

② Q-Q plot — quantiles of standardized residuals against quantiles of a standard normal. Should be a straight diagonal. Tails curving away mean residuals are heavy-tailed; coefficient estimates are still unbiased but their p-values and CIs are off. Fix: switch loss (Huber/robust → 3.6) or transform y (log, Box-Cox).

③ Scale-location — √|standardized residual| vs fitted. Should be flat. A rising trend is heteroscedasticity (variance grows with ŷ); your standard errors lie. Fix: weighted least squares, sandwich (HC0–HC3) standard errors, or a GLM with a non-constant variance link (Module 3.10).

④ Residuals vs leverage — leverage h_ii = diagonal of the hat matrix, measuring how much each point's own fit pulls itself. High leverage + large residual = a point with disproportionate influence (Cook's distance > 1 is the classic threshold).

leverage h_ii = (H)_ii = (X(XᵀX)⁻¹Xᵀ)_ii Σ h_ii = p + 1 (degrees of freedom) Cook's D_i = (rᵢ² / (p+1)) · (h_ii / (1 − h_ii)²) / σ̂² ← influence on the whole fit
# the four plots in statsmodels — one of the best reasons to use it over sklearn import statsmodels.api as sm import matplotlib.pyplot as plt fit = sm.OLS(y, sm.add_constant(X)).fit() infl = fit.get_influence() resid = fit.resid # raw residuals std_res = infl.resid_studentized_internal # standardized (panel 2 & 3) lev = infl.hat_matrix_diag # leverage h_ii (panel 4) cooks_d = infl.cooks_distance[0] # influence # 1) residuals vs fitted — linearity # 2) Q-Q — normality (scipy.stats.probplot) # 3) scale-location — homoscedasticity # 4) residuals vs leverage — influence print(sm.stats.diagnostic.het_breuschpagan(resid, fit.model.exog)) # formal hetero. test

The visual is the fast triage; the formal tests (Breusch-Pagan for heteroscedasticity, Shapiro-Wilk for normality, Durbin-Watson for autocorrelation) come in when you need a number for a report. But almost every issue you'll find first appears in the four plots above — which is why "always plot the residuals" is the oldest rule in regression.

07 / Reporting the fit
R² · adjusted R² · MAE · RMSE
Each metric answers a slightly different question. Use them together — never one alone.
R²

Variance explained

Fraction of Var(y) the model captures. Useful as a sanity score; can be made arbitrarily high by adding noise features in-sample.

R² = 1 − SSE / SST
ADJ. R²

R² penalised for p

Subtracts a complexity term so adding useless features can lower it. The right metric for comparing models with different feature counts.

R²_adj = 1 − (1−R²)·(n−1)/(n−p−1)
MAE

Mean absolute error

Same units as y, robust to outliers. Use when individual error magnitudes matter and you don't want squared-error to dominate.

MAE = (1/n) Σ |yᵢ − ŷᵢ|
RMSE

Root mean squared error

Same units as y, penalises large errors quadratically. Native to OLS (it's the MLE loss). Always pair with MAE to spot heavy-tailed errors.

RMSE = √( (1/n) Σ (yᵢ − ŷᵢ)² )
🧰

Reach for OLS when…

The relationship is plausibly linear, you need interpretable coefficients, you want a fast baseline, or you need statistical inference (p-values, CIs, F-tests). It has effectively no hyperparameters (fit_intercept, positive) and is the right first model in any tabular regression. Skip it when relationships are clearly nonlinear (use 3.2/3.7/3.8), columns are highly collinear (Ridge → 3.3), or the target is counts / rates / heavy-tailed (GLM → 3.10).
