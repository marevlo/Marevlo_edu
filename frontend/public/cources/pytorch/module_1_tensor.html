SECTION 01

Tensor Creation, dtype, device, shape

What is a Tensor?

A tensor is just a multi-dimensional array — a generalisation of scalars, vectors, and matrices into any number of dimensions. Think of it as NumPy's ndarray, but with two superpowers:

  1. It can live on a GPU and do millions of operations in parallel.
  2. It knows how to track gradients for automatic differentiation (we cover this in Module 2).
NameDimensionsExample
Scalar0-DA single temperature reading: 38.5
Vector1-DDaily temperatures for a week: [38, 37, 39, 40, 36, 35, 38]
Matrix2-DTemperatures for 7 days × 5 cities
TensorN-DTemperatures for 7 days × 5 cities × 24 hours

The Three Properties You Must Always Know

Every tensor has three properties you should check immediately when debugging:

  • shape — how many elements in each dimension (e.g., torch.Size([32, 128]))
  • dtype — what kind of numbers are stored (float32, int64, bool, etc.)
  • device — where the tensor lives (cpu or cuda:0)

Real-World Analogy

Imagine you manage a chain of coffee shops. You track daily sales for 5 items across 3 branches for 30 days. That's a 3-D tensor of shape (30, 3, 5). The dtype tells you whether those numbers are whole units sold (int64) or revenue in dollars (float32). The device tells you whether that data is sitting in your laptop's RAM (CPU) or being processed at lightning speed on a GPU.

dtype: Choosing the Right Number Type

# Default dtypes
f32 = torch.tensor([1.0, 2.0])               # float32 — default for floats
i64 = torch.tensor([1, 2])                   # int64 — default for ints

# Explicit dtype
f16 = torch.tensor([1.0, 2.0], dtype=torch.float16)  # Half precision (saves memory)
f64 = torch.tensor([1.0, 2.0], dtype=torch.float64)  # Double precision
i32 = torch.tensor([1, 2],     dtype=torch.int32)
boo = torch.tensor([True, False, True], dtype=torch.bool)

x = torch.tensor([1.7, 2.9, 3.1])

print(x.float())           # Stays float32
print(x.half())            # Converts to float16

# Why does dtype matter?
# float32 → standard for training neural networks
# float16 → half the memory; used in mixed-precision training
# int64   → used for class labels, indices
# bool    → used for masks (which elements to keep/ignore)

TIP — Rule of Thumb
Use float32 for model weights and activations. Use int64 for class labels. Use bool for masks and conditions.

SECTION 02

Broadcasting Rules

What is Broadcasting?

Broadcasting lets PyTorch do arithmetic on tensors of different shapes by virtually stretching the smaller tensor — without actually copying data in memory. It's what makes vectorised code both fast and readable.

The Two Broadcasting Rules

PyTorch checks shapes right-to-left, dimension by dimension. Two dimensions are compatible if:

  1. They are equal, OR
  2. One of them is exactly 1 (that dimension gets stretched)

If the tensors have different numbers of dimensions, the shape with fewer dims is padded with 1s on the left.

Real-World Analogy: Student Bonuses

You have 3 students. Each student completed 4 exercises. You want to add a subject-specific bonus of [10, 20, 30, 40] to every student's scores.

  • Matrix A is (3, 4) — 3 students, 4 exercises.
  • Bonus vector b is (4,) — one bonus per exercise.

PyTorch checks: right dimension 4 == 4. PyTorch virtually stretches b to (3, 4) by repeating it.

Broadcasting in Practice: Feature Normalisation

python
# Normalise a dataset — the most common real-world use
X = torch.randn(100, 5)        # 100 samples, 5 features

mu    = X.mean(dim=0)             # Shape (5,)   — mean per feature
sigma = X.std(dim=0)              # Shape (5,)   — std per feature

X_norm = (X - mu) / sigma         # (100,5) - (5,) = broadcast over rows

print("Mean after normalise (should ~0):", X_norm.mean(dim=0).round(decimals=2))
print("Std after normalise  (should ~1):", X_norm.std(dim=0).round(decimals=2))

Shape Incompatibility: When Broadcasting Fails

python
# Shape alignment — read RIGHT to LEFT
# (3, 4) vs (3,)
#      4 vs 3    ← NOT equal, neither is 1 → ERROR

try:
    a = torch.ones(3, 4)
    b = torch.ones(3)         # Wrong! Should be (4,) or (1,4) or (3,1)
    result = a + b
except RuntimeError as e:
    print("Error:", e)

# Fix 1: reshape b to a column vector
b_col = torch.ones(3).unsqueeze(1)  # Shape (3, 1)
result = a + b_col                  # (3,4) + (3,1) → broadcasts over columns
print("Fixed shape:", result.shape)
Error: The size of tensor a (4) must match the size of tensor b (3) at non-singleton dimension 1 Fixed shape: torch.Size([3, 4])
NOTE — The Broadcasting Mental Model
Shape A:    ( 3,    4)
Shape b:    (     4 )    ← pad left with 1s: (1, 4)
            ---------
Result:     ( 3,    4)   ← 1 stretches to 3

Common Broadcasting Patterns in Data Science

python
# Pattern 1: Add a bias term to a batch of vectors
batch = torch.randn(32, 128)       # 32 samples, 128 features
bias  = torch.randn(128)            # one bias per feature
result = batch + bias              # (32,128) + (128,)

# Pattern 2: Scale each sample differently
scale = torch.rand(32, 1)          # one scale factor per sample
result = batch * scale             # (32,128) * (32,1)

# Pattern 3: Pairwise distances (outer subtraction)
A = torch.randn(5, 3)              # 5 points in 3D space
B = torch.randn(7, 3)              # 7 points in 3D space
diff = A.unsqueeze(1) - B.unsqueeze(0)    # (5,1,3) - (1,7,3) = (5,7,3)
dist = diff.pow(2).sum(dim=-1).sqrt()     # (5,7) — pairwise distances
print("Pairwise distance matrix shape:", dist.shape)
Pairwise distance matrix shape: torch.Size([5, 7])

Tensor indexing follows NumPy conventions closely. The golden rule: each comma represents one dimension.

Reshape, View, Permute, Transpose

The Shape-Changing Family

Four operations change how a tensor's data is "seen" without (usually) copying it:

OperationWhat it doesRequires contiguous?
.view()Reinterpret shape — shares memoryYes
.reshape()Reinterpret shape — copies if neededNo
.permute()Reorder ALL dimensionsNo
.transpose()Swap exactly TWO dimensionsNo

Real-World Analogy: A Spreadsheet Reorganisation

Imagine you have 24 hourly temperature readings in a flat list. You want to view them as: 4 rows × 6 columns (4 groups of 6), 6 rows × 4 columns (6 groups of 4), or a 3-D tensor of shape (2, 3, 4) (2 days × 3 periods × 4 hours). The underlying 24 numbers don't change — only how you slice them up.

.view() vs .reshape() — The Critical Difference

python
x = torch.arange(12).float().reshape(3, 4)

# .view() — zero-copy reshape, ONLY works on contiguous tensors
y = x.view(4, 3)         # Fine — x is contiguous
print("view:    ", y.shape)

# .reshape() — works regardless, copies if needed
z = x.reshape(4, 3)      # Always safe
print("reshape: ", z.shape)

# WHEN VIEW FAILS: after permute/transpose, tensor becomes non-contiguous
x_t = x.permute(1, 0)   # Transpose: (3,4) → (4,3), non-contiguous now
print("Is contiguous after permute?", x_t.is_contiguous())

try:
    x_t.view(-1)         # This will CRASH
except RuntimeError as e:
    print("Error:", e)

# FIX: call .contiguous() first
x_t.contiguous().view(-1)    # Works
# OR: just use reshape() which handles this automatically
x_t.reshape(-1)              # Always safe

IMPORTANT
Rule: Default to .reshape() for safety. Use .view() only when you've verified contiguity and care about zero-copy guarantees.

SECTION 05

Memory: Contiguous Tensors, .clone() vs .view()

What is Memory Contiguity?

When a tensor is contiguous, its elements are stored sequentially in memory, row by row (C-order). Operations like .permute() and .transpose() return a view with a different stride — they don't rearrange memory. After these operations, the tensor is no longer contiguous.

python
x = torch.arange(12).reshape(3, 4)
print("Original strides:", x.stride())                 # (4, 1) — row stride=4, col stride=1
print("Contiguous?", x.is_contiguous())                # True

x_t = x.T                                              # Transpose
print("Transposed strides:", x_t.stride())             # (1, 4) — reversed!
print("Contiguous?", x_t.is_contiguous())              # False
Original strides: (4, 1) Contiguous? True Transposed strides: (1, 4) Contiguous? False

.clone() vs View — When to Copy

OperationCopies memory?Shares gradients?Use when
y = xNoYes (same tensor)Just aliasing
y = x.view(...)NoYes (shared storage)Cheap reshape
y = x.detach()NoNo (breaks gradient)Inference / stopping grad

SECTION 06

NumPy & Torch Interop

The Bridge Between Worlds

PyTorch and NumPy share memory when a tensor is on CPU. Conversions are (usually) zero-copy.

python
import numpy as np

# NumPy → Torch (shares memory on CPU!)
arr = np.array([1.0, 2.0, 3.0], dtype=np.float32)
tensor = torch.from_numpy(arr)
print("NumPy to Torch:", tensor)

# Proof they share memory
arr[0] = 99.0
print("After modifying arr, tensor:", tensor)       # tensor also changes!

# Torch → NumPy (shares memory)
t = torch.tensor([10.0, 20.0, 30.0])
n = t.numpy()
print("Torch to NumPy:", n)

# CRITICAL: GPU tensors cannot directly convert to NumPy
# t_gpu = t.cuda()
# t_gpu.numpy()         # ← ERROR!
# Fix:
# t_gpu.cpu().numpy()   # Move to CPU first
NumPy to Torch: tensor([1., 2., 3.]) After modifying arr, tensor: tensor([99., 2., 3.]) Torch to NumPy: [10. 20. 30.]

Safe Conversion Pattern

python
def to_numpy_safe(tensor: torch.Tensor) -> np.ndarray:
    """Safely convert any tensor to NumPy."""
    return tensor.detach().cpu().numpy()

# Why each step:
# .detach() — removes from computation graph (safe if requires_grad=True)
# .cpu()    — moves to RAM if on GPU
# .numpy()  — zero-copy bridge to NumPy

# Usage
x = torch.randn(3, 4, requires_grad=True)
arr = to_numpy_safe(x)
print("Converted safely:", arr.shape, type(arr))
Converted safely: (3, 4) <class 'numpy.ndarray'>

def make_windows(series: torch.Tensor, window: int = 24) -> tuple:
    """
    Create overlapping windows from a 1-D time series.

    Args:
        series: 1-D tensor of shape (T,)
        window: number of past steps per window

    Returns:
        X: (T - window, window)    — input windows
        y: (T - window,)           — next-step targets
    """
    # .unfold creates a view — no data copy!
    windows = series.unfold(0, window + 1, 1)     # (T-window, window+1)
    X = windows[:, :-1]                          # All but last column → inputs
    y = windows[:, -1]                            # Last column → target
    return X, y


window_size = 24
