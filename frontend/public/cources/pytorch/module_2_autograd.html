MODULE GOAL

Understand how PyTorch automatically computes gradients through any computation. You will build the intuition for computation graphs, learn how to control gradient flow, and implement linear regression from scratch using only tensors and autograd — no nn.Module, no shortcuts.

SECTION 01

Computation Graph Intuition

What is a Computation Graph?

Every time you do math with PyTorch tensors that have requires_grad=True, PyTorch silently builds a directed acyclic graph (DAG) in the background. Each node in this graph is either:

  • A tensor (the data), or
  • A function (an operation like +, *, sin, matmul)

When you call .backward(), PyTorch traverses this graph in reverse — from the output back to the inputs — applying the chain rule of calculus at each step to compute gradients.

Real-World Analogy: A Factory Assembly Line (in Reverse)

Imagine a factory: raw materials go in, a product comes out. To figure out how much each raw material contributed to the final product's quality, you trace backwards through the assembly steps — that's exactly what backprop does.

Computation Graph Visualiser

Backward pass: right to left (compute "how much does each input affect the loss?").

python
import torch

# Simple computation graph
x = torch.tensor(3.0, requires_grad=True)          # Leaf node
w = torch.tensor(2.0, requires_grad=True)          # Leaf node
b = torch.tensor(1.0, requires_grad=True)          # Leaf node

# Forward pass — graph is built as we compute
y = x ** 2          # y = x²      = 9
z = w * y + b       # z = 2x² + 1 = 19

print("z =", z.item())
print("z.grad_fn:", z.grad_fn)        # Shows the last operation

# Backward pass — traverses graph, accumulates gradients
z.backward()

# Gradients: dz/dx, dz/dw, dz/db
print("\ndz/dx =", x.grad.item())         # d(2x²+1)/dx = 4x = 12
print("dz/dw =", w.grad.item())         # d(2x²+1)/dw = x²  = 9
print("dz/db =", b.grad.item())         # d(2x²+1)/db = 1
z = 19.0 z.grad_fn: <AddBackward0 object at 0x000002AB5363B7C0> dz/dx = 12.0 dz/dw = 9.0 dz/db = 1.0

SECTION 02

requires_grad, .grad, .detach(), .no_grad()

requires_grad: Opt-In to Gradient Tracking

Not every tensor needs a gradient. Only trainable parameters (model weights) should have requires_grad=True. Data tensors should not.

python
# Tensors without gradient tracking (default for most data)
data = torch.randn(5, 3)                           # requires_grad=False (default)
print("Data requires_grad:", data.requires_grad)

# Tensors WITH gradient tracking (model parameters)
weight = torch.randn(3, 2, requires_grad=True)
print("Weight requires_grad:", weight.requires_grad)

# Enable/disable after creation
data.requires_grad_(True)                          # In-place toggle
print("Data after toggle:", data.requires_grad)
Data requires_grad: False Weight requires_grad: True Data after toggle: True

.grad: Where Gradients Live

After .backward(), gradients are stored in the .grad attribute of each leaf tensor (tensors created directly, not computed from others).

python
x = torch.tensor([1.0, 2.0, 3.0], requires_grad=True)

# Scalar output required for .backward() without arguments
loss = (x ** 2).sum()                 # sum gives scalar: 1 + 4 + 9 = 14
loss.backward()

print("x.grad:", x.grad)                    # d(sum(x²))/dx = 2x = [2, 4, 6]
x.grad: tensor([2., 4., 6.])

The Classic Gradient Accumulation Bug

This is one of the most common PyTorch bugs — gradients accumulate across .backward() calls unless you explicitly zero them.

Gradient Accumulation Demo
Run a training loop with or without zero_grad() and watch what happens to x.grad.
Mode
Steps 5
Click "Run loop" to see how x.grad behaves over multiple backward() calls.
python
x = torch.tensor(2.0, requires_grad=True)

# Simulating a training loop WITHOUT zeroing gradients
for step in range(3):
    loss = x ** 2
    loss.backward()
    print(f"Step {step}: x.grad = {x.grad.item()}")   # 4, 8, 12 — keeps adding!

# The correct pattern: zero gradients before each backward pass
x.grad.zero_()                          # or use optimizer.zero_grad() in training

for step in range(3):
    if x.grad is not None:
        x.grad.zero_()                  # ← CRITICAL: clear before each step
    loss = x ** 2
    loss.backward()
    print(f"Step {step}: x.grad = {x.grad.item()}")   # 4, 4, 4 — correct!
Step 0: x.grad = 4.0 Step 1: x.grad = 8.0 Step 2: x.grad = 12.0 Step 0: x.grad = 4.0 Step 1: x.grad = 4.0 Step 2: x.grad = 4.0
WARNING — Classic Bug
Forgetting to call optimizer.zero_grad() at the start of each training step. Gradients accumulate silently, leading to exploding updates. Always zero gradients before loss.backward().

.detach(): Break the Graph

.detach() creates a new tensor that shares the same data but is removed from the computation graph. No gradients flow through it.

python
x = torch.tensor(3.0, requires_grad=True)
y = x ** 2                             # y is part of the graph

# Detach — same values, no gradient connection
y_detached = y.detach()
print("y value:", y.item())
print("y_detached value:", y_detached.item())
print("y_detached.requires_grad:", y_detached.requires_grad)

# Common use: logging metrics without affecting training
loss = x ** 3
loss_value = loss.detach().item()      # Safe to log, convert to Python float
print("Loss for logging:", loss_value)
y value: 9.0 y_detached value: 9.0 y_detached.requires_grad: False Loss for logging: 27.0

torch.no_grad(): Disable Graph Building

Use torch.no_grad() whenever you don't need gradients — inference, evaluation, metric calculation. It's faster and uses less memory.

python
x = torch.randn(1000, 1000, requires_grad=True)

# With gradient tracking (slower, more memory)
import time
start = time.time()
for _ in range(100):
    y = (x * x).sum()
print(f"With grad:    {time.time()-start:.3f}s")

# Without gradient tracking (faster, less memory)
start = time.time()
with torch.no_grad():
    for _ in range(100):
        y = (x * x).sum()
print(f"Without grad: {time.time()-start:.3f}s")
With grad: 0.009s Without grad: 0.007s
TIP — Rule of Thumb
During training: gradients on. During evaluation/inference: always wrap in torch.no_grad(). When logging a loss value: use .detach().item(), never .item() directly on a tensor in the graph.
SECTION 03

Custom Autograd Function

Sometimes you need to define a forward pass whose gradient isn't automatically derivable — for example, a non-standard activation or a domain-specific operation.

Real-World Analogy: Teaching PyTorch a New Math Operation

PyTorch knows how to differentiate sin, exp, matmul etc. But if you invent your own function (say, a clamped activation), you need to tell it: "Here's the forward formula, and here's the derivative."

ClampedReLU Visualiser
A clamped activation: f(x) = clamp(x, 0, cap). The gradient is 1 inside the active zone, 0 outside.
Cap value 1.5
Forward f(x) Gradient df/dx
python
class ClampedReLU(torch.autograd.Function):
    """
    A ReLU that clamps output to [0, cap].
    Forward: f(x) = clamp(x, 0, cap)
    Backward: gradient = 1 if 0 < x < cap, else 0
    """

    @staticmethod
    def forward(ctx, x, cap=1.0):
        ctx.save_for_backward(x)        # Save input for use in backward
        ctx.cap = cap
        return x.clamp(0, cap)

    @staticmethod
    def backward(ctx, grad_output):
        x, = ctx.saved_tensors
        cap = ctx.cap
        # Gradient passes through only where 0 < x < cap
        grad_input = grad_output * ((x > 0) & (x < cap)).float()
        return grad_input, None      # None for cap (not a tensor)


# Test it
x = torch.tensor([-1.0, 0.5, 1.5, 2.0], requires_grad=True)
y = ClampedReLU.apply(x, 1.0)
print("Forward:", y)                   # [0, 0.5, 1.0, 1.0]

loss = y.sum()
loss.backward()
print("Gradients:", x.grad)            # [0, 1, 0, 0] — only middle value passes grad

SECTION 04

Numerical Gradient Checking

How do you know your custom backward is correct? Numerical gradient checking computes the gradient by finite differences and compares it to your analytical gradient.

The finite difference formula: df/dx ≈ (f(x+ε) - f(x-ε)) / (2ε)

python
def numerical_gradient(f, x, eps=1e-5):
    """Compute gradient of scalar function f at tensor x using finite differences."""
    grad = torch.zeros_like(x)
    x_flat = x.view(-1)
    grad_flat = grad.view(-1)

    for i in range(len(x_flat)):
        x_plus = x.clone(); x_plus.view(-1)[i] += eps
        x_minus = x.clone(); x_minus.view(-1)[i] -= eps
        grad_flat[i] = (f(x_plus) - f(x_minus)) / (2 * eps)

    return grad


# Test function: f(x) = sum(x^3)
def f(x):
    return (x ** 3).sum()


x = torch.tensor([1.0, 2.0, 3.0], requires_grad=True)

# Analytical gradient
y = f(x)
y.backward()
analytical = x.grad.clone()

# Numerical gradient
numerical = numerical_gradient(f, x.detach())

print("Analytical gradient:", analytical)
print("Numerical gradient: ", numerical)
print("Max difference:     ", (analytical - numerical).abs().max().item())
# Should be very small (< 1e-5) — confirms our backward is correct
Analytical gradient: tensor([ 3., 12., 27.]) Numerical gradient: tensor([ 3.0518, 12.2070, 27.0844]) Max difference: 0.20703125

Linear Regression from Scratch

The Challenge

Implement full linear regression using only tensors and autograd. No nn.Linear, no nn.MSELoss, no torch.optim. Just raw math and .backward().

This exercise builds the muscle memory that makes every higher-level PyTorch API click into place later.

X_train = torch.linspace(-5, 5, 100).unsqueeze(1)                # (100, 1)

The Model and Training Loop

python
# Initialise parameters randomly
w = torch.randn(1, 1, requires_grad=True)            # Weight
b = torch.zeros(1,     requires_grad=True)            # Bias

learning_rate = 0.01
n_epochs      = 200

loss_history = []

# Training loop
for epoch in range(n_epochs):
    # 1. Forward pass: compute predictions
    y_pred = X_train @ w + b              # Matrix multiply + broadcast bias

    # 2. Compute loss (MSE)
    loss = ((y_pred - y_train) ** 2).mean()

    # 3. Backward pass: compute gradients
    loss.backward()

    # 4. Manual gradient descent step
    with torch.no_grad():                  # ← Don't track these updates in the graph
        w -= learning_rate * w.grad
        b -= learning_rate * b.grad

    # 5. Zero gradients (MUST do before next backward)
    w.grad.zero_()
    b.grad.zero_()

    loss_history.append(loss.item())

    if epoch % 40 == 0:
        print(f"Epoch {epoch:3d}: loss={loss.item():.4f}  w={w.item():.3f}  b={b.item():.3f}")

print(f"\nFinal: w={w.item():.3f} (true=3.0),  b={b.item():.3f} (true=7.0)")
