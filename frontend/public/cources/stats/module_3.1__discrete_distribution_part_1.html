Why Distributions Exist
The pattern in randomness

Random Is Not Chaos — It Has Structure

In Modules 2.4–2.5, we built the language of random variables and their summary statistics. Now we ask a deeper question:

If randomness appears over and over again, does it follow any pattern?

Consider these real-world streams:

  • Coin tosses → H, T, H, H, T, …
  • Exam outcomes → Pass, Fail, Pass, Pass, Fail, …
  • Website clicks per minute → 0, 2, 5, 1, 3, 4, 2, …
  • Daily defective items → 0, 1, 0, 0, 2, 1, 0, …

These aren't chaos. They follow recognizable statistical patterns — patterns that mathematicians have catalogued into a small set of probability distributions. Each distribution is a "story" — a model for a particular kind of randomness.

Knowing which distribution fits your data is one of the most valuable skills in statistics. It lets you:

  • Make precise predictions
  • Compute exact probabilities
  • Quantify uncertainty
  • Train and evaluate ML models
  • Design experiments correctly
The five core stories

The Five Discrete Distributions of This Module

Real-world discrete randomness collapses into a few archetypal scenarios. Each is modeled by one of these five distributions:

Distribution Models Asks
BernoulliA single yes/no trial"Did success happen?"
Binomialn independent yes/no trials"How many successes?"
GeometricTrials until first success"When does success arrive?"
PoissonRare event counts in an interval"How many events occur?"
HypergeometricSampling without replacement"What changes when items are not put back?"

Part 1 covers the first two — Bernoulli and Binomial. They are the easiest to understand and form the foundation for the others. Once these click, the rest become much easier to learn.

A quick reminder

Discrete vs Continuous — Why This Distinction Matters

Recall from Module 2.4: a discrete random variable takes countable values (you can list them). A continuous one takes any value in an interval.

Discrete

  • Outcomes: 0, 1, 2, … (countable)
  • P(X = x) makes sense and is positive
  • Described by a PMF
  • Examples: coin flips, dice rolls, customer counts, defects

Continuous

  • Outcomes: any real number in an interval
  • P(X=x) is always 0; probability comes from ranges/intervals like P(a≤X≤b)
  • Described by a PDF
  • Examples: heights, weights, times, temperatures

Module 3.1 lives entirely in the discrete world. Module 3.2 will explore continuous distributions (Uniform, Exponential, Normal, etc.).

2
Bernoulli Distribution
The atom of probability

What is the Bernoulli Distribution?

The Bernoulli distribution is the simplest non-trivial distribution in all of probability. It models a single experiment with exactly two outcomes:

  • Success → labeled 1, occurring with probability p
  • Failure → labeled 0, occurring with probability 1 − p

That's it. One trial. Two outcomes. One number p to define everything.

Why do we call it the "building block"? Almost every other discrete distribution can be built from Bernoulli pieces. Binomial = adding up n Bernoullis. Geometric = waiting for the first Bernoulli success. The Bernoulli is the simplest yes/no building block — you can't break it into anything simpler.
Real-world examples

Where Bernoulli Lives

🪙 Single Coin Toss

X = 1 if heads, 0 if tails. Fair coin: p = 0.5. Biased coin: p ≠ 0.5. The textbook example.

🩺 Single Medical Test

X = 1 if positive, 0 if negative. The diagnostic test on one patient is a single Bernoulli trial.

📧 One Email Classification

X = 1 if the email is spam, 0 if it is not. Classifying a single email as spam or not spam is a Bernoulli trial.

🛒 One Purchase Decision

X = 1 if customer buys, 0 if not. E-commerce A/B tests measure conversion rate p.

⚙️ One Defect Check

X = 1 if defective, 0 if not. A single quality inspection on one item.

🤖 One Binary Prediction

X = 1 if model predicts positive, 0 otherwise. Every binary classifier outputs Bernoulli probabilities.

⟁ Bernoulli PMF
P(X = 1) = p
P(X = 0) = 1 − p

Or written as a single formula: P(X = x) = px(1−p)1−x for x ∈ {0, 1}. Let's plug in both values to see why this works:

xpx(1−p)1−xResult
1p¹ = p(1−p)⁰ = 1p × 1 = p ✅
0p⁰ = 1(1−p)¹ = 1−p1 × (1−p) = 1−p ✅

So the single formula gives the same answers as the two separate rules above — it's just a shorthand.

X — Bernoulli random variable (0 or 1)
p — probability of success (must be in [0, 1])
1 − p — probability of failure
⟁ Bernoulli Mean & Variance
E[X] = p
Var(X) = p(1 − p)

Mean — step by step:

E[X] = (value₁ × its probability) + (value₂ × its probability)
E[X] = 1 × p + 0 × (1−p) = p
So the expected value simply equals the success probability.

Variance — step by step:

Step 1: Find E[X²] = 1² × p + 0² × (1−p) = p
Step 2: We already know E[X] = p, so (E[X])² = p²
Step 3: Var(X) = E[X²] − (E[X])² = p − p² = p(1−p)

Key insight: Variance reaches its maximum of 0.25 when p = 0.5 — that's when the outcome is most unpredictable (like a fair coin). When p = 0 or p = 1, variance is 0 because the result is guaranteed.

When to use Bernoulli

Decision Guide

  • You have exactly one experiment (a single trial).
  • The experiment has exactly two outcomes (binary).
  • You know (or want to estimate) the probability p of "success."
  • Common framing: yes/no, success/failure, on/off, 1/0, hit/miss.

Don't use Bernoulli when: you have more than two outcomes (use Categorical), multiple trials (use Binomial), or the trial is repeated until success (use Geometric).

Practical applications

Where Bernoulli Shows Up in Data Science

  • Logistic regression: predicts the probability that something is "yes" or "no" — that probability is Bernoulli.
  • Binary classification: every spam filter, fraud detector, or churn predictor is built on Bernoulli thinking — each prediction is either correct (1) or wrong (0).
  • Dropout in neural networks: during training, each neuron is randomly "kept" or "dropped" — a Bernoulli coin flip decides.

From One Trial to Many — Why Binomial?
The natural next question

What if We Repeat the Experiment?

Bernoulli handles one trial. But real life rarely stops at one. A coin gets tossed 10 times. An ad gets shown to 1,000 users. A factory inspects 50 items per shift. A student takes 5 quizzes.

The natural question becomes: among n trials, how many successes occurred?

This is exactly what the Binomial distribution answers. It is what you get when you add up n independent Bernoulli trials, each with the same success probability p, and count how many came up "1."

Definition shortcut: If X₁, X₂, …, Xₙ are independent Bernoulli(p) random variables, then their sum  S = X₁ + X₂ + … + Xₙ  follows a Binomial(n, p) distribution.
The four conditions

When is the Binomial Model Valid?

For Binomial to apply, the experiment must satisfy all four of these conditions:

  • 1. Fixed number of trials n. You decide n in advance — it doesn't depend on the results.
  • 2. Independent trials. Outcome of one trial does not affect any other.
  • 3. Two outcomes per trial. Each trial is success or failure (Bernoulli).
  • 4. Same probability p across trials. The probability of success is constant.

If any one of these breaks, Binomial is the wrong model. For example: sampling 10 cards without replacement violates independence — that's a job for Hypergeometric (Part 3). Counting events in a time window has no fixed n — that's Poisson (Part 2).

4
Binomial Distribution
⟁ Binomial PMF
P(X = k) = C(n, k) · pk · (1 − p)n − k

Where X = number of successes in n trials, and C(n, k) = n! / [k!(n−k)!] is the binomial coefficient — "n choose k." The formula has three pieces that each capture a key idea:

C(n, k) — number of ways to arrange k successes among n trials
pk — probability of getting k successes in their chosen positions
(1 − p)n − k — probability of getting n−k failures in the remaining positions

Intuition: One specific sequence of k successes and n−k failures has probability pk(1−p)n−k. But there are C(n, k) different sequences that all give exactly k successes (just rearranged). Multiplying gives the total probability of getting k successes regardless of order.

⟁ Binomial Mean & Variance
E[X] = n · p
Var(X) = n · p · (1 − p)

Why E[X] = n·p? Think of it this way: X is the total number of successes across n trials. Each trial contributes an average of p successes. So across n trials, the average total = n × p.

Why Var(X) = n·p(1−p)? Because the trials are independent, we can add up their individual variances. Each single Bernoulli trial has variance p(1−p). There are n of them, so the total variance = n × p(1−p).

Quick check: Toss 10 fair coins (n=10, p=0.5). Expected heads = 10 × 0.5 = 5. Variance = 10 × 0.5 × 0.5 = 2.5. Standard deviation ≈ 1.58 — so typical results fall roughly between 3 and 7 heads (within 1 standard deviation of the mean).

Visualizing the shape

How Binomial Shape Changes with n and p

Shape by p (with n fixed)

  • p = 0.5 → perfectly symmetric (centered at n/2)
  • p < 0.5 → right-skewed (long tail to the right)
  • p > 0.5 → left-skewed (long tail to the left)
  • p near 0 → mass piles up at 0
  • p near 1 → mass piles up at n

Shape by n (with p fixed)

  • Small n → discrete, spiky, possibly skewed
  • Larger n → smoother, more bell-shaped
  • n very large → approximated by Normal(np, np(1−p))
  • You'll learn the formal name for this pattern later — for now, just notice how the shape becomes a smooth bell curve as n gets bigger

Real-world applications

Where Binomial Appears in the Wild

⚙️ Quality Control

Inspect 100 widgets; each is defective with probability p. The number of defects X follows Binomial(100, p). Used to design sampling plans and acceptance tests.

📊 A/B Testing

Show variant B to 5,000 users; clicks per user are Bernoulli(p_B). Total clicks ∼ Binomial(5000, p_B). The basis for every online experiment.

🎯 Ad Click-Through

An ad is shown 10,000 times with click probability p. Total clicks ∼ Binomial(10000, p). Advertising spend models rest on this.

📚 Multiple-Choice Tests

20 questions, each with 4 options, guessed randomly. Correct answers ∼ Binomial(20, 0.25). Used in evaluating exam difficulty.

🧬 Genetic Inheritance

10 offspring, each inheriting a trait with probability 0.75. Inheriting count ∼ Binomial(10, 0.75). Classical Mendelian genetics.

🤖 ML Model Accuracy

Test a classifier on 1,000 examples. If true accuracy is p, correct predictions ∼ Binomial(1000, p). Used for confidence intervals on model performance.

💼 Sales Conversion

Salesperson contacts 50 leads, each converts with probability p. Conversions ∼ Binomial(50, p). Used to forecast revenue.

📩 Email Open Rate

Send a campaign to 100K subscribers, open probability p. Total opens ∼ Binomial(100000, p). Marketing analytics 101.

Common Mistakes
Mistake 1 — Confusing Bernoulli and Binomial. Bernoulli describes ONE trial. Binomial describes the COUNT of successes across n trials. They are not interchangeable: Bernoulli is the special case n = 1 of Binomial.
Mistake 2 — Using Binomial when trials aren't independent. Drawing cards without replacement, polling a small finite population, or sampling defective items from a batch breaks independence. Use Hypergeometric (Part 3) instead.
Mistake 3 — Using Binomial when p isn't constant. If success probability changes across trials (different ad creatives, varying lighting, learning agents), Binomial doesn't apply.
Mistake 4 — Forgetting the combination C(n, k) coefficient. A common error is writing P(X = k) = pk(1−p)n−k without C(n, k). That formula gives probability of ONE specific sequence — not of getting k successes in any order.
Mistake 5 — Confusing "probability of success" p with "number of successes" k. p is a parameter (in [0, 1]); k is an outcome (an integer between 0 and n). Always check units.
Mistake 6 — Applying Binomial to events in a time window. "How many emails arrive per hour?" has no fixed n, so Binomial doesn't fit. Use Poisson (Part 2).

Practice Questions
Test yourself — Easy to Moderate

🧪 Check Your Understanding

Try each question on your own first, then click "Show Answer" to check.

Q1 — Easy

A coin has P(Heads) = 0.7. What is the probability of getting Tails in a single toss?

Answer: P(Tails) = 1 − P(Heads) = 1 − 0.7 = 0.3.
This is a Bernoulli trial. The two probabilities always add up to 1.
Q2 — Easy

For a Bernoulli random variable with p = 0.4, calculate the mean E[X] and the variance Var(X).

Answer:
E[X] = p = 0.4
Var(X) = p(1−p) = 0.4 × 0.6 = 0.24
Q3 — Easy

A fair coin is tossed 8 times. What distribution does "number of heads" follow? Name the distribution and state its parameters.

Answer: Number of heads follows a Binomial(n = 8, p = 0.5) distribution.
Why? Fixed n = 8 trials, independent tosses, two outcomes per toss, same p = 0.5 every time — all four conditions are met.
Q4 — Moderate

A student guesses randomly on a 10-question true/false quiz. What is the probability of getting exactly 7 correct?

Answer: X ~ Binomial(10, 0.5). Want P(X = 7).
P(X = 7) = C(10,7) × (0.5)7 × (0.5)3 = C(10,7) × (0.5)10
C(10,7) = 120. (0.5)10 = 1/1024 ≈ 0.000977.
P(X = 7) = 120 × 0.000977 ≈ 0.1172 (about 11.7%)
Q5 — Moderate

A factory has a 3% defect rate. In a batch of 15 items, what is the expected number of defective items and the standard deviation?

Answer: X ~ Binomial(15, 0.03).
E[X] = n × p = 15 × 0.03 = 0.45
Var(X) = n × p × (1−p) = 15 × 0.03 × 0.97 = 0.4365
σ = √0.4365 ≈ 0.66
On average, you'd expect less than 1 defective item per batch.
Q6 — Moderate

Which of the four Binomial conditions is violated in this scenario: "Drawing 5 cards from a deck of 52 without replacement, and counting how many are hearts." What distribution should be used instead?

Answer: The independence condition is violated.
When you draw without replacement, each draw changes the deck — so the probability of drawing a heart changes after each card. You should use the Hypergeometric distribution instead (covered in Part 3).
