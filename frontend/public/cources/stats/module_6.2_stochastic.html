Stochastic Processes & Markov Models

Some systems don't sit still — they evolve. A stochastic process is the mathematical language for systems whose state changes randomly over time.

This module covers six core ideas: random processes (the umbrella concept), the Markov property (when the past doesn't matter beyond the present), transition probabilities (the rules of motion), random walks (the simplest non-trivial process), the Poisson process (counting random events in time), and continuous-time randomness (when time itself flows smoothly instead of in steps). Together they form the toolkit behind everything from stock prices to disease spread to AI search.

Topic 1
Random Processes
What it is

Randomness Indexed by Time

A random process (also called a stochastic process) is a collection of random variables, one for each point in time. Instead of a single uncertain number, we have an uncertain trajectory — a value at time 1, a value at time 2, and so on.

You can think of a random variable as one snapshot. A stochastic process is the whole movie: the snapshot keeps changing, and each new frame is uncertain.

One-line definition: A stochastic process is a sequence (or continuum) of random variables Xₜ, where t is a time index. Each Xₜ is a random quantity, and together they describe how the system evolves randomly.
Why we need it

Why a Single Random Variable Is Not Enough

Many real systems are not "one shot." Stock prices change every second. Server requests arrive throughout the day. A patient's blood-sugar level rises and falls. A robot's position updates every frame. To describe any of these, you need a model of how the value at time t depends on the values at earlier times.

What a Process Captures

  • The set of possible values (the state space)
  • How time is measured (discrete steps or continuous)
  • The probability rules linking one time to the next

Two Main Time Types

  • Discrete-time: X₀, X₁, X₂, … (one value per step)
  • Continuous-time: X(t) defined for every real t
  • Discrete is easier to reason about; continuous models real-world flow

Two Main State Types

  • Discrete state: finite or countable values (sunny, cloudy, rainy)
  • Continuous state: real numbers (temperature, price)
  • Process type = (time type, state type) combo
Formal Notation
{ Xₜ : t ∈ T } — family of random variables indexed by time T
Each Xₜ is a random variable. T is the time set: T = {0, 1, 2, …} for discrete time, T = [0, ∞) for continuous time. The full process is the collection across all t.
Xₜ — random variable at time t
T — time index set
State space — set of all possible values Xₜ can take
Sample path — one realised trajectory over time
⟁ Real-Life Example

The Weather Process

Let Xₜ describe the weather on day t with values from {sunny, cloudy, rainy}. This is a discrete-time, discrete-state stochastic process. You know today's weather but tomorrow's is uncertain. The forecast model gives you the probabilities of each next state.

A different example: let X(t) be the temperature in your city at time t (continuous, measured every second). That's a continuous-time, continuous-state process. Same idea, different flavour of time and values.

Where it is used

Stochastic Processes in the Real World

Finance

Stock prices, exchange rates, and option valuations are all modelled as stochastic processes. The famous Black-Scholes model is built on one.

Epidemiology

The number of people infected with a virus over time is a stochastic process. Public health models predict future cases from current ones.

Reinforcement Learning

Agent-environment interactions in RL are modelled as Markov decision processes — a stochastic process with an action layer on top.

Network Traffic

Packets arriving at a router, requests hitting a web server — random arrivals form a stochastic process that determines capacity planning.

Topic 2
The Markov Property
What it is

The Past Doesn't Matter Beyond the Present

A process has the Markov property if the future depends only on the current state — not on how we got there. The history before the present moment carries no extra information once we know where we are now.

Processes with this property are called Markov processes, or Markov chains when the state space is discrete. They are the most widely used family of stochastic processes because they are simple to specify and powerful enough to capture many real systems.

One-line definition: A process is Markov if knowing the current state makes the past irrelevant for predicting the future.
The Markov Property — Formal
P( Xₜ₊₁ = x | Xₜ, Xₜ₋₁, …, X₀ ) = P( Xₜ₊₁ = x | Xₜ )
Read: the probability of the next state given the entire history equals the probability given only the most recent state. The conditioning on older states collapses to just the present.
Xₜ — current state
Xₜ₋₁, …, X₀ — past states (irrelevant if Markov)
Xₜ₊₁ — next state to predict
= — equality is the entire content of the property
Why it matters

What the Markov Property Buys Us

Drastically Simpler Models

  • No need to remember full history
  • The current state is a sufficient summary
  • Specifying the model takes far fewer numbers

Tractable Computation

  • Multi-step predictions reduce to matrix powers
  • Long-run behaviour can be computed exactly
  • Sampling and inference are well understood

Real-World Fit

  • Many physical systems are approximately Markov
  • Position + velocity often summarises history well
  • If not Markov, expanding state can fix it
Watch out: the Markov property is a property of the state representation, not the world itself. A non-Markov process can be made Markov by expanding the state to include the relevant past. Picking the right state is the modelling skill.
⟁ Real-Life Example

Board Games and Snakes & Ladders

In Snakes and Ladders, your next position depends only on your current square and the die roll. It doesn't matter how you arrived at square 47 — whether you climbed a ladder or just walked there. The current square is enough. This is a clean Markov chain.

Contrast with poker, where future betting depends not just on your current hand but on what you have shown earlier in the game. Naive "current hand" state is not Markov for poker — you'd need to include betting history in the state to recover the property.

Topic 3
Transition Probabilities
What it is

The Rules of Motion

For a Markov chain with a finite set of states, the entire dynamics is captured by transition probabilities — one number for each pair of states (i, j), giving the probability of moving from i to j in one step.

Pack all these numbers into a square table and you get the transition matrix P. Row i of P is the distribution over next states when you are currently in state i. Rows must sum to 1.

One-line definition: P[i, j] = probability of being in state j next step, given you are in state i now.
Transition Probability & Matrix
P[i, j] = P( Xₜ₊₁ = j | Xₜ = i )
Each row of P is a probability distribution over the next state. The full matrix P is the entire definition of the chain (along with a starting state or starting distribution).
P[i, j] — one-step transition probability i → j
Row sum — Σⱼ P[i, j] = 1 for every i
Pⁿ[i, j] — probability of i → j in exactly n steps
π — stationary distribution (long-run state probabilities)
How to use it

Using the Transition Matrix

  1. Write down the states. Label them 1, 2, …, n.
  2. Build the transition matrix P. Estimate P[i, j] from data, expert knowledge, or model assumptions. Make sure each row sums to 1.
  3. One-step prediction. If your current state distribution is row vector p, then one step later you are at p · P.
  4. n-step prediction. The probability of being in state j after n steps starting from i is the (i, j) entry of Pⁿ. Just take matrix powers.
  5. Long-run behaviour. If you keep multiplying, the distribution often converges to a fixed vector π — the stationary distribution. It satisfies π · P = π.
⟁ Worked Example

A Two-State Weather Chain

States: Sunny (S), Rainy (R). Transition matrix:

P =  [ S → S = 0.8,  S → R = 0.2 ]
    [ R → S = 0.4,  R → R = 0.6 ]

If today is Sunny, what is the probability that it's Rainy two days later?

  • ▸ Two-day path Sunny → Sunny → Rainy: 0.8 × 0.2 = 0.16
  • ▸ Two-day path Sunny → Rainy → Rainy: 0.2 × 0.6 = 0.12
  • ▸ Total: 0.16 + 0.12 = 0.28

This is exactly the (S, R) entry of P² — matrix multiplication automates this kind of bookkeeping.

Topic 4
Random Walks
What it is

One Step Left, One Step Right

A random walk is the simplest interesting stochastic process. Start at 0 on a number line. At each step, flip a fair coin: move +1 if heads, −1 if tails. Repeat. The sequence of positions is a random walk.

Despite its simplicity, the random walk shows up everywhere — in stock prices, diffusion of molecules, search algorithms, animal foraging, neuron firing patterns. It is the discrete-time, discrete-state ancestor of Brownian motion.

One-line definition: A random walk is the running sum of independent random steps. Position after n steps = sum of n step variables.
Simple Symmetric Random Walk
Xₙ = X₀ + Z₁ + Z₂ + … + Zₙ   where Zᵢ = +1 or −1 with prob 1/2 each
The position at step n is the starting point plus the sum of n independent ±1 jumps. The expected position stays at X₀ (the steps cancel on average), but the spread grows with √n.
Xₙ — position after n steps
Zᵢ — i-th step, ±1 with equal probability
E[Xₙ] — expected position = X₀ (no drift)
Var(Xₙ) — variance grows linearly: n × Var(Z)
Why it matters

What Makes Random Walks Important

  • Spread grows like √n — after n steps, the typical distance from the origin is √n. To get twice as far, you need four times the steps.
  • It's a Markov chain — the next position depends only on the current position. Past trajectory doesn't matter.
  • In 1D and 2D it returns to the origin infinitely often (with probability 1). In 3D and higher, it might wander off forever.
  • It's the bridge to Brownian motion — take smaller and smaller steps in shorter and shorter time and you get continuous random motion in the limit.
⟁ Real-Life Example

Stock Prices and the Random Walk Hypothesis

One of the foundational ideas in finance is that short-term stock price changes look like a random walk — the next move is hard to predict from the past. Eugene Fama's efficient market hypothesis (1970s) builds directly on this. Forecasting stock prices well is hard precisely because they behave so much like random walks.

Topic 5
Poisson Process
What it is

Counting Random Events in Time

The Poisson process is the model for counting how many random events occur over time, when the events happen independently at a constant average rate. It's the natural model for "things that arrive one at a time, at random moments."

Three key features define it: events occur one at a time; the average rate λ (events per unit time) is constant; and counts in non-overlapping time intervals are independent.

One-line definition: A Poisson process counts random arrivals occurring independently at a constant average rate λ per unit time.
Poisson Probability of k Events in Time t
P( N(t) = k ) = e−λt · (λt)k / k!
The number of events in an interval of length t follows a Poisson distribution with mean λt. The bigger λ or t, the more events we expect — but each specific count k has the probability above.
N(t) — number of events by time t
λ — rate: average events per unit time
λt — expected number of events in time t
k! — k factorial
How to use it

When to Reach for the Poisson Process

  1. Check the three conditions. Events occur one at a time, the rate is roughly constant, and events don't influence each other.
  2. Estimate the rate λ. If you observed N events in T time units, λ ≈ N / T.
  3. Compute probabilities. Use the Poisson formula with mean λt to get the chance of seeing exactly k events in any time window of length t.
  4. Use the gap property. The time between consecutive events is exponentially distributed with mean 1/λ. This is useful for simulating arrivals.
⟁ Worked Example

Customer Arrivals at a Café

Customers arrive at a small café at an average rate of λ = 6 per hour. Assume arrivals are independent and the rate is steady during opening hours.

  • ▸ Expected arrivals in 30 minutes: λt = 6 × 0.5 = 3
  • ▸ Probability of exactly 4 customers in the next 30 min: e⁻³ × 3⁴ / 4! ≈ 0.168
  • ▸ Expected wait until next customer: 1/λ = 1/6 hour = 10 minutes
Where it is used

Poisson in the Real World

Web Traffic

Requests hitting a server in normal hours follow a Poisson process. This drives queueing theory and capacity planning.

Earthquakes

The occurrence of major earthquakes in a region (above some threshold) is modelled as Poisson. The rate varies by region.

Click Streams

Clicks on a low-traffic ad over time roughly follow a Poisson process — handy for predicting daily click volume.

Neuron Spikes

Many simple neuron models treat spike timings as a Poisson process with rate depending on input — basic computational neuroscience.

Topic 6
Continuous-Time Randomness
What it is

When Time Doesn't Tick

Continuous-time stochastic processes let the state change at any moment, not just at fixed steps. Time is a real number flowing smoothly, and the process is defined for every t ≥ 0.

The most famous example is Brownian motion (also called the Wiener process). It is the continuous-time limit of a random walk: as you take smaller and smaller steps in shorter and shorter time, the path approaches a fractal-like, continuous, nowhere-smooth curve.

One-line definition: A continuous-time stochastic process has a random value X(t) defined for every real time t, with transitions allowed at any instant.
Brownian Motion — Key Properties
X(t) − X(s) ~ Normal( 0, t − s ) for any s < t
Increments are normally distributed with variance equal to the time elapsed. Non-overlapping increments are independent. The path is continuous, but it is not differentiable anywhere — it wiggles at every scale.
X(t) — position at continuous time t
X(0) — starts at 0 by convention
Variance — grows linearly with time
Paths — continuous but jagged at every zoom level
Why it matters

Where Continuous-Time Processes Win

  • Physical realism. Stock prices, particle motion, and temperature don't change only at neat time steps — they evolve continuously. Continuous-time models capture this.
  • Clean math. Differential equations driven by random noise (stochastic differential equations) are a powerful modelling tool — but they require continuous time.
  • Bridging discrete and continuous. Many discrete-time models, when scaled correctly, converge to continuous-time processes. This connects theory and practice.
⟁ Real-Life Example

Pollen Grains and the History of Brownian Motion

In 1827, botanist Robert Brown observed pollen grains in water jiggling under a microscope. Albert Einstein, in 1905, gave a precise statistical explanation: the grains were being hit by water molecules at random. The mathematical model — Brownian motion — is now the foundation of stochastic calculus, used everywhere from option pricing (Black-Scholes) to molecular dynamics.
