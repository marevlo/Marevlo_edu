Topic 1
Multiple Predictors
What it is

What is Multiple Linear Regression?

In Module 5.1 we used one input (say, square footage) to predict one output (price). That works only if the input we picked tells almost the whole story. Real datasets are not that kind. A house's price depends on size, age, neighbourhood, number of rooms, condition, distance from a metro station, and several other things at the same time.

Multiple linear regression extends the single-input line to handle many inputs at once. Instead of fitting a line ŷ = β₀ + β₁x, we fit:

ŷ = β₀ + β₁x₁ + β₂x₂ + β₃x₃ + … + βₚxₚ

Each input variable x₁, x₂, …, xₚ has its own coefficient β₁, β₂, …, βₚ. The model still adds them up linearly (no x squared, no products of two x's), but now the prediction blends information from many sources.

Simple meaning: Multiple linear regression takes several pieces of information about an item and combines them — each with its own weight — to predict one outcome. Same shape of equation, just more terms.

The visual picture

With one predictor, the model is a line drawn through a 2D scatter plot. With two predictors, the model becomes a flat plane floating through a 3D cloud of dots. With three or more predictors, the model is a flat hyperplane we cannot draw, but the math behaves the same way — find the flat surface that sits as close to all the data points as possible, using the least-squares rule.

Why we need it

Why One Predictor Is Often Not Enough

Three solid reasons to move from one predictor to many:

1. Real outcomes have many causes

  • Marks depend on study hours, sleep, prior knowledge
  • Salary depends on experience, education, location
  • Sales depend on price, ads, season, competitor activity

2. Better predictions

  • One predictor explains part of y
  • Adding relevant predictors often improves the model fit,
  • reduces unexplained variation (residual error), and can increase R²

3. Removing confusion (control)

  • If salary depends on both experience and education
  • A single-predictor model mixes them up
  • Multiple regression estimates the effect of each predictor while accounting for the others
Big idea: Adding more relevant predictors lets us isolate each one's effect after the others are already accounted for. That ability to isolate effects is one major reason multiple regression is useful.
Why it's used in practice

The Two Main Jobs of Multiple Regression

Practitioners use multiple regression for two slightly different goals — and the right choice depends on which one matters:

🎯 Prediction

  • Goal: forecast y as accurately as possible for new cases
  • Don't care if coefficients are interpretable — only that ŷ is close to y
  • Example: predicting next month's revenue from many factors
  • Used in ML, forecasting, recommender baselines

🔍 Inference / Explanation

  • Goal: understand the relationship between each predictor and y
  • Care strongly about the value & uncertainty of each coefficient
  • Example: estimating the effect of a drug while controlling for age, sex, BMI
  • Used in science, policy, economics, medicine
How it works

How Each Coefficient Should Be Read

This is the single most important interpretation rule in multiple regression — and the most common place beginners trip up.

In single regression (one predictor), the slope β₁ meant: "when x changes by 1, y changes by β₁ on average." Simple.

In multiple regression, each coefficient must be read with an extra phrase:

Reading rule for βj: "When xj increases by 1 unit, while all other predictors are held fixed, the predicted y changes by βj on average."

That little phrase — "while all other predictors are held fixed" — changes everything. It means each coefficient measures the unique contribution of its variable, after the model has already accounted for the others. The same data with different predictors included can produce different βj values for the same variable, because the meaning is "controlled for whatever else is in the model."

How the model is fit

  • Step 1: Pick predictor variables x₁, x₂, …, xₚ that you believe influence y.
  • Step 2: Collect data — every observation has values for all predictors plus the response y.
  • Step 3: Apply the least-squares rule (same idea as Module 5.1): find β₀, β₁, …, βₚ that minimise the sum of squared residuals.
  • Step 4: Read each coefficient with the "holding others fixed" interpretation.
  • Step 5: Evaluate the fit (R², residual checks) and decide whether to add/drop predictors.

Software does step 3 for you. Behind the scenes, it solves a system of equations called the normal equations (the same idea as in single regression, just with more unknowns). For one predictor we had clean closed-form formulas. For many predictors the formulas are easier to write in matrix form — but the principle is identical: minimise the sum of squared residuals.

⟁ Multiple Linear Regression Equation
ŷ = β₀ + β₁x₁ + β₂x₂ + β₃x₃ + … + βₚxₚ

What every symbol means:

ŷ — predicted value of the response
x₁, x₂, …, xₚ — the p predictor variables (inputs)
β₀ — intercept; predicted ŷ when ALL predictors are 0
β₁, β₂, …, βₚ — coefficients; one per predictor
p — total number of predictors
βⱼ — change in ŷ per 1-unit increase in xⱼ, others fixed

Why this form? Each predictor contributes a piece — its value times its weight — and we add up all pieces (plus the intercept). The model is still linear in the coefficients, which is why it is called linear regression even with many inputs. (We do not square, multiply, or take logs of the betas.)

How to compute one prediction: For any new observation with predictor values (x₁, x₂, …, xₚ), multiply each xⱼ by its coefficient βⱼ, add them all up, then add the intercept β₀. The result is the model's predicted y for that observation.

Finding the coefficients: Same rule as Module 5.1 — least squares. Software computes the (β₀, β₁, …, βₚ) that make the total sum of squared residuals as small as possible across the entire dataset.

⟁ Worked Example

Example: Predicting House Prices in a Small City

A real-estate firm fits a regression with three predictors. After running the fit on 200 homes, it gets the equation:

Price (lakh ₹) = 12.5 + 0.040·Size + 3.2·Rooms − 0.18·Age

Where Size is in square feet, Rooms is the number of bedrooms, and Age is in years.

Reading every coefficient correctly:

  • β₀ = 12.5 lakh: the predicted price for a house with Size = 0, Rooms = 0, Age = 0. Obviously not a real house — the intercept just anchors the prediction surface mathematically.
  • β₁ = 0.040 lakh per sqft: for every extra sqft, price rises about ₹4,000, holding rooms and age fixed.
  • β₂ = 3.2 lakh per room: for every extra bedroom, price rises about ₹3.2 lakh, holding size and age fixed.
  • β₃ = −0.18 lakh per year: for every extra year of age, price drops about ₹18,000, holding size and rooms fixed.

Predicting one specific house: a 1,200 sqft, 3-bedroom, 10-year-old apartment.

Price = 12.5 + 0.040×1200 + 3.2×3 − 0.18×10
= 12.5 + 48 + 9.6 − 1.8
= 68.3 lakh ₹
The lesson: Each coefficient is the marginal effect of its variable, with the other variables held still. The total prediction is just the weighted sum plus the intercept. This is the entire engine of multi-input prediction.
When to use it

When Multiple Linear Regression Fits the Job

  • You have a numerical outcome y and at least two related predictors x₁, x₂, …
  • You want to predict y from the predictors for new observations.
  • You want to isolate each predictor's effect after controlling for the others.
  • The relationship is approximately additive and linear — each predictor's contribution stacks on top of the others.
  • You have more observations than predictors — a rough rule is at least 10–20 observations per predictor for stable estimates.
Watch out: Multiple regression cannot magically extract truth from data. If two predictors carry the same information (highly correlated), or if a predictor causes both y and another predictor, the coefficients can be misleading. Part 2 covers multicollinearity — exactly this issue.
Real-life examples

Where Multiple Regression Powers Real Work

🏠 Real-Estate Pricing

Real-estate models often use many predictors… — area, school score, crime rate, transit, age, condition — to estimate every listed home's price.

💊 Clinical Risk Scores

Hospitals fit models for outcomes like ICU length-of-stay using age, BMI, blood pressure, comorbidities, lab values. Each coefficient guides clinical decisions.

💼 Salary Studies

Pay-equity audits use regression of salary on experience, education, role, location, gender — to check whether salary differences remain after controlling for other factors.

📈 Marketing Attribution

Brands fit weekly sales against TV, digital, search, OOH ad spend. The coefficients estimate which channels drive sales most efficiently.

🚗 Insurance Premiums

Auto insurers price policies using regression of historical claims against driver age, car model, mileage, region, prior accidents.

🌾 Crop Yield Prediction

Agritech firms model yield against rainfall, soil pH, fertilizer use, sunlight hours and seed variety. Helps farmers and planners.

Topic 2
Regression Assumptions
What they are

The Four Conditions Linear Regression Quietly Expects

Linear regression is not "no-questions-asked." Behind every coefficient and every confidence interval are four background assumptions about the data. When they hold, the model's outputs are trustworthy. When they break, the coefficients still get computed — but the numbers can mislead.

The four assumptions go by the memorable shortcut LINE:

L · Linearity

  • The true relationship between each predictor and y is approximately straight
  • Residuals should not show a curve when plotted against any predictor

I · Independence

  • Each observation is independent of the others
  • Knowing one residual gives no information about another
  • Time-series or grouped data often violates this

N · Normality (of residuals)

  • Residuals follow a roughly bell-shaped distribution
  • Needed for confidence intervals and p-values, not for the slope estimate itself

E · Equal Variance (homoscedasticity)

  • The spread of residuals is roughly the same across all predictor values
  • A widening fan in the residual plot signals trouble
Simple meaning: LINE = Linearity, Independence, Normality of residuals, Equal variance. Four boxes to mentally tick before trusting a regression's coefficients, p-values, and confidence intervals.
Why they matter

What Goes Wrong When Each Assumption Breaks

Linearity broken

  • The straight model systematically misses parts of the data
  • Coefficients become a poor summary of the real relationship
  • Predictions are biased — too high in some regions, too low in others
  • Fix: transform the predictor (log, square root) or use a non-linear model

Independence broken

  • Standard errors get smaller than they should be
  • p-values look significant when the effect is uncertain
  • You trust the model more than you should
  • Fix: time-series methods or mixed-effects models

Normality broken

  • Confidence intervals and p-values become unreliable
  • The slope estimates themselves are still fine in large samples
  • Fix: larger sample size, or robust methods

Equal variance broken

  • Standard errors are biased — some too small, some too large
  • Confidence intervals become misleading
  • Fix: weighted least squares, log transformation, robust SE
Big idea: The coefficient estimates themselves are usually still reasonable even when assumptions bend. What truly suffers is the uncertainty reported around them — standard errors, p-values, and confidence intervals. That is why the assumptions matter most when you want to draw scientific conclusions, not just predictions.
Why they are checked

Why Every Serious Analysis Checks LINE

  • Honest reporting: Coefficients with broken assumptions are like a thermometer that hasn't been calibrated — the number still shows, but you can't trust it.
  • Scientific publication: Reviewers expect residual plots, QQ plots, and assumption discussions before accepting a paper's findings.
  • Business decisions: An ad-attribution model with hidden assumption breaks can mislead a company into burning budget on the wrong channels.
  • Regulation: Models used in lending, insurance, and medicine are audited for these very assumptions before being approved.
How to check them

Practical Tests for Each LINE Assumption

  1. Linearity — Residuals vs Fitted plot. Plot residuals (vertical) against fitted ŷ (horizontal). A healthy fit shows a shapeless cloud around zero. A clear curve = linearity broken.
  2. Independence — Domain knowledge first, then tests. Was each row collected independently? Time-series data is rarely independent. Statistical tests like Durbin-Watson check for autocorrelation between consecutive residuals.
  3. Normality — Q-Q plot of residuals. Plot the residuals' sorted values against the theoretical bell-curve values. Points should lie close to a straight diagonal line. Strong S-shapes or heavy curves at the tails signal non-normality.
  4. Equal variance — Residuals vs Fitted plot (again). Same plot as for linearity, but now look at the vertical spread. A constant spread across the plot is good. A widening or narrowing fan = unequal variance.

In Part 2 we'll build the actual residual plots and Q-Q plots and look at them on real fitted data. For now, the rule is: fit your model, then look at the residuals before trusting anything.

⟁ The Mathematical Statement of Assumptions

The full linear model with error term:

yᵢ = β₀ + β₁x₁ᵢ + β₂x₂ᵢ + … + βₚxₚᵢ + εᵢ

What every symbol means:

yᵢ — actual response for observation i
xⱼᵢ — value of predictor j for observation i
βⱼ — true coefficient for predictor j
εᵢ — random error term for observation i

The LINE assumptions in formal language — they are all about the error terms εᵢ:

  • L (Linearity): the true relationship is yᵢ = β₀ + Σ βⱼxⱼᵢ + εᵢ — the model has no missing curves or interactions.
  • I (Independence): the εᵢ are independent of each other. Knowing one error tells you nothing about another.
  • N (Normality): each εᵢ follows a normal (bell-shaped) distribution centred at 0.
  • E (Equal variance): all εᵢ share the same spread σ² — Var(εᵢ) = σ² for every i, regardless of the predictor values.

Put together in one line: εᵢ ~ Normal(0, σ²) independently — the famous "iid normal errors" assumption that underlies classical regression theory.

In plain words: "The leftover noise after the line has done its job is unrelated point-to-point, has the same size everywhere, and follows a bell curve." That is the whole technical statement.

⟁ Worked Example

Example: A Real Diagnostic Walkthrough

An analyst fits the house-price model from Topic 1 and prepares to publish the coefficients. Before sharing the report, the four LINE checks are run:

  • Linearity check. Residuals are plotted against fitted prices. The cloud looks shapeless — no curve. ✅ Linearity holds.
  • Independence check. Each home is a separate, independently sampled property. There is no time order or repeated visit. ✅ Independence holds.
  • Equal variance check. Looking at the same residual plot, the vertical spread of residuals grows with fitted price — expensive homes have much bigger residuals than cheap ones. ❌ Equal variance broken.
  • Normality check. A Q-Q plot of residuals shows most points on the diagonal but a few extreme high-price homes far above the line. ❌ Normality slightly broken at the upper tail.

The interpretation: The coefficients (₹4,000 per sqft, etc.) are likely still in the right ballpark — least squares is robust enough for the point estimates. But the confidence intervals and p-values reported by the software will under-state the uncertainty on expensive homes. The analyst chooses to: (a) apply a log-transform to price to stabilise variance, (b) refit, and (c) re-check.

The lesson: Checking the LINE assumptions is not optional bureaucracy. It is the difference between a number you can defend and a number that quietly misleads. Always check, especially before publishing or shipping a model.
When checking is strict vs. relaxed

How Hard to Insist on the Assumptions

Strict (every assumption checked)

  • Scientific publication where p-values matter
  • Clinical trials and drug studies
  • Regulated industries (finance, insurance, lending)
  • Policy & economic research

Relaxed (focus on predictions)

  • Pure prediction problems in ML
  • Cases where you care about ŷ accuracy more than coefficient interpretation
  • Quick exploratory analyses
  • You can still benefit from diagnostics but slight violations matter less
Real-life examples

When Each Assumption Tends To Break In Practice

📉 Stock Returns (Independence)

Daily returns are autocorrelated — today's return is mildly linked to yesterday's. A naive regression that ignores this will look more confident than it should.

💰 Salaries (Equal Variance)

The variance of salary grows with experience — junior salaries are tightly clustered, senior salaries vary widely. Equal-variance assumption breaks; analysts often log-transform salary.

🏠 Real-Estate (Linearity)

House price vs lot size is often non-linear — doubling lot size doesn't double price. Analysts add quadratic terms or use log(price) to recover linearity.

🔬 Lab Repeated Measures (Independence)

Multiple measurements per patient are clearly not independent. Mixed-effects models handle this; standard linear regression would over-state precision.

📊 Survey Data (Normality)

Likert-scale responses are bounded and skewed. Normality of residuals breaks at the boundaries. Robust regression or ordinal models are used instead.

🌡️ Climate Models (All four)

Temperature data has trend, seasonality and growing variance. Climate scientists carefully account for all four LINE issues before estimating warming rates.
