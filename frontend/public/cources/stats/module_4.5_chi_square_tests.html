◆ 01
Categorical Data & the Chi-Square Statistic
⟁ WHAT IS CATEGORICAL DATA

Categories instead of numbers

Categorical data consists of observations that fall into discrete labels or categories rather than numerical values. Examples include eye color (blue / brown / green), customer segment (A / B / C), survey response (yes / no / undecided), or product type (electronics / clothing / books).

Because categorical data has no natural numerical scale, we cannot compute means, standard deviations, or t-statistics for them in the usual way. Instead, we work with counts — the number of observations in each category.

Why a new test is needed

The tests in Modules 4.3 and 4.4 work with continuous measurements or proportions. For multiple categories or for testing relationships between two categorical variables, the chi-square test provides the right tool. It compares the counts we actually observed in the data against the counts we would expect if a specific claim were true.

Pronunciation. "Chi" is pronounced like "kai" (rhymes with "sky"). The symbol χ² (chi-squared) refers to a family of probability distributions used as the reference for these tests.
⟁ THE CHI-SQUARE STATISTIC (GENERAL FORM)
χ² = Σ ( O − E )² / E
The chi-square statistic measures the total squared difference between observed counts and expected counts, scaled by the expected counts. Large values of χ² indicate that the observed data differs substantially from what was expected.
⟁ SYMBOL EXPLANATION
χ²
Pronounced "chi-squared." The test statistic. Always non-negative because each term in the sum is squared.
Σ
Pronounced "sigma" (capital). The summation symbol — add up the expression that follows over all categories or cells.
O
The observed count in a category — the number of observations actually seen in the data.
E
The expected count in a category — the number that would be expected if the null hypothesis were true.
(O − E)
The deviation between observed and expected counts in that category.
(O − E)²
The squared deviation. Squaring removes the sign, so positive and negative deviations both contribute positively to the total.
(O − E)²/E
Each squared deviation is divided by the expected count, so smaller expected categories are weighted more heavily.
Assumption. The chi-square test relies on a normal approximation that works well only when each expected count is at least 5. If any expected count is too small, results can be unreliable and an alternative test (such as Fisher's exact test) should be used.
◆ 02
Chi-Square Goodness-of-Fit Test
⟁ WHAT IS IT

Definition

The goodness-of-fit test checks whether the observed frequencies across categories of a single variable match an expected distribution stated by H₀. It answers the question: "Does the data fit the claimed distribution?"

Example. A die manufacturer claims its six-sided die is fair. We roll it 600 times and count how often each face appears. If the die is fair, each face should appear about 100 times. A chi-square goodness-of-fit test checks whether the observed counts match this expectation.

Hypotheses

H₀: The observed data follow the claimed distribution (proportions p₁, p₂, ..., p_k for the k categories).
H₁: At least one observed proportion differs from the claimed value.

Computing expected counts

The expected count for category i is the total sample size multiplied by the probability claimed under H₀:

⟁ EXPECTED COUNTS
E_i = n × p_i
The expected count in category i is the sample size n multiplied by the claimed probability p_i for that category.
⟁ SYMBOL EXPLANATION
E_i
Expected count for category i — the number of observations we would expect in this category under H₀.
n
Total sample size — the total number of observations across all categories.
p_i
The probability claimed under H₀ for category i. All p_i values must sum to 1.
i
An index that ranges from 1 to k, where k is the number of categories.
⟁ GOODNESS-OF-FIT TEST STATISTIC
χ² = Σᵢ (Oᵢ − Eᵢ)² / Eᵢ   with df = k − 1
Sum the term (O − E)²/E across all k categories. Compare the result to a chi-square distribution with k − 1 degrees of freedom.
⟁ SYMBOL EXPLANATION
Oᵢ
Observed count in category i.
Eᵢ
Expected count in category i.
k
Total number of categories.
df
Degrees of freedom = k − 1. One degree of freedom is lost because the total count must equal n.
Σᵢ
Sum over all categories from i = 1 to i = k.
⟁ WORKED EXAMPLE — IS THE DIE FAIR?

Problem. A six-sided die is rolled n = 120 times. The observed counts are: face 1: 18, face 2: 22, face 3: 16, face 4: 24, face 5: 17, face 6: 23. Test at α = 0.05 whether the die is fair.

Step 1 — Hypotheses

H₀: Each face has probability 1/6 (the die is fair).
H₁: At least one face has a different probability.

Step 2 — Expected counts
E_i = 120 × (1/6) = 20 for every face
Step 3 — Compute each contribution
(18−20)²/20 = 0.20    (22−20)²/20 = 0.20    (16−20)²/20 = 0.80
(24−20)²/20 = 0.80    (17−20)²/20 = 0.45    (23−20)²/20 = 0.45
Step 4 — Sum to get χ²
χ² = 0.20 + 0.20 + 0.80 + 0.80 + 0.45 + 0.45 = 2.90
Step 5 — Degrees of freedom and p-value

df = 6 − 1 = 5. For χ² = 2.90 with df = 5, p-value ≈ 0.715.

Step 6 — Decision

Since 0.715 > 0.05, fail to reject H₀. The data is consistent with a fair die.

◆ 03
Contingency Tables
⟁ TWO-WAY CATEGORICAL DATA

What is a contingency table?

A contingency table (also called a cross-tabulation or two-way table) organizes counts of observations classified by two categorical variables. Rows represent the categories of one variable; columns represent the categories of the other. Each cell holds the number of observations falling into that specific combination.

Example layout

Suppose a survey records 200 customers' preferences for a product type (A, B, or C) along with their region (North or South).

Product A Product B Product C Row Total
North 40 30 30 100
South 20 50 30 100
Column Total 60 80 60 200

Marginal totals

The row totals and column totals along the edges of the table are called marginal totals. They summarize each variable individually, ignoring the other.

Joint vs marginal distributions

The cells in the table represent the joint distribution — how the two variables behave together. The row and column totals represent the marginal distributions — each variable on its own.

◆ 04
Chi-Square Test of Independence
⟁ TESTING RELATIONSHIPS

What does independence mean?

Two categorical variables are independent if knowing the value of one provides no information about the value of the other. The test of independence asks: "Is there a statistical association between the two variables, or are they unrelated?"

Hypotheses.
H₀: The two variables are independent (no association).
H₁: The two variables are not independent (there is an association).

Computing expected counts under independence

If H₀ is true, the expected count for each cell can be computed from the row total, column total, and grand total:

⟁ EXPECTED COUNTS UNDER INDEPENDENCE
E_ij = ( row_i total × column_j total ) / n
The expected count in the cell at row i, column j is the product of its row total and column total divided by the grand total.
⟁ SYMBOL EXPLANATION
E_ij
Expected count in the cell at row i and column j, assuming the two variables are independent.
row_i total
The total of all counts in row i (the marginal total for that row).
column_j total
The total of all counts in column j.
n
The grand total — the sum of all counts in the entire table.
i, j
Indices labeling rows and columns. i ranges from 1 to r (number of rows), j from 1 to c (number of columns).
⟁ TEST OF INDEPENDENCE STATISTIC
χ² = Σᵢ Σⱼ ( Oᵢⱼ − Eᵢⱼ )² / Eᵢⱼ   with df = (r − 1)(c − 1)
Sum the term (O − E)²/E across every cell in the table. Compare the result to a chi-square distribution with (r − 1)(c − 1) degrees of freedom.
⟁ SYMBOL EXPLANATION
Oᵢⱼ
Observed count in the cell at row i, column j.
Eᵢⱼ
Expected count for the same cell under H₀.
r
Number of rows in the contingency table.
c
Number of columns in the contingency table.
df
Degrees of freedom for the test: (r − 1) × (c − 1). For example, a 2 × 3 table has df = 1 × 2 = 2.
ΣᵢΣⱼ
Double sum — add the term across all rows and all columns of the table.
⟁ WORKED EXAMPLE — REGION AND PRODUCT PREFERENCE

Problem. Using the contingency table above (200 customers, two regions, three products), test at α = 0.05 whether region and product preference are independent.

Step 1 — Hypotheses

H₀: Region and product preference are independent.
H₁: Region and product preference are associated.

Step 2 — Expected counts
E(North, A) = 100 × 60 / 200 = 30    E(North, B) = 100 × 80 / 200 = 40    E(North, C) = 100 × 60 / 200 = 30
E(South, A) = 100 × 60 / 200 = 30    E(South, B) = 100 × 80 / 200 = 40    E(South, C) = 100 × 60 / 200 = 30
Step 3 — Cell contributions
(40−30)²/30 = 3.333    (30−40)²/40 = 2.500    (30−30)²/30 = 0
(20−30)²/30 = 3.333    (50−40)²/40 = 2.500    (30−30)²/30 = 0
Step 4 — Sum to get χ²
χ² = 3.333 + 2.500 + 0 + 3.333 + 2.500 + 0 = 11.667
Step 5 — Degrees of freedom and p-value

df = (2−1)(3−1) = 2. For χ² = 11.667 with df = 2, p-value ≈ 0.0029.

Step 6 — Decision

Since 0.0029 ≤ 0.05, reject H₀. There is strong evidence that region and product preference are associated.

◆ 05
Real-World Applications

Genetics — Mendel's Inheritance

The goodness-of-fit test compares observed phenotype counts against Mendel's predicted ratios (e.g., 9:3:3:1) to verify genetic models.

Marketing & Customer Segmentation

The test of independence detects whether customer demographics (age group, region) are associated with product or brand preferences.

Medical Research

Contingency tables compare treatment outcomes (recovered / not recovered) against treatment type to evaluate treatment effectiveness across categories.

Quality Control

Goodness-of-fit tests verify whether the distribution of defect types observed in production matches the expected pattern from past data.

Machine Learning Feature Selection

Chi-square scores are used to rank categorical features by their association with a target class, helping identify the most informative inputs.

Survey Analysis

Independence tests determine whether responses to one question are related to responses to another, revealing patterns across the dataset.

// Module 4.5 · Compiled Summary

Chi-square tests provide the inferential framework for categorical data. They compare observed counts to expected counts under a null hypothesis, summed across all cells, and compare the result to a chi-square distribution. The goodness-of-fit test works on a single categorical variable; the test of independence works on the cells of a two-way contingency table. Both rely on the same statistic χ² = Σ (O − E)² / E, but differ in how expected counts are computed and in degrees of freedom.

02
Goodness-of-fit. Tests whether observed counts match a claimed distribution. df = k − 1.
03
Test of independence. Tests whether two categorical variables are associated. df = (r − 1)(c − 1).
04
Expected counts (independence). E_ij = (row_i total × column_j total) / n.
05
Decision rule. Reject H₀ when χ² is large enough that the p-value is at or below α.
06
Assumption. Each expected count should be at least 5 for the chi-square approximation to be reliable.
07
Always non-negative. Because every term in the sum is squared, χ² ≥ 0.
