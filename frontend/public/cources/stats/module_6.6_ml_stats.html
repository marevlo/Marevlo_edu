Statistics for Machine Learning

Modern machine learning is built on a handful of statistical ideas that show up everywhere — in cross-entropy loss, in regularisation, in maximum likelihood training, and in the information-theoretic glue holding it all together. This module ties those threads into one coherent picture.

Seven core ideas: maximum likelihood estimation (the foundational training principle), entropy (how uncertain a distribution is), cross-entropy (the classification loss every neural net uses), KL divergence (how different two distributions are), loss functions as likelihood (why MSE and cross-entropy are not arbitrary), regularisation as priors (the Bayesian view of L1 and L2), and the statistical foundations of ML (overfitting, bias-variance, generalisation). This is the capstone bridge from statistics to ML.

Topic 1
Maximum Likelihood Estimation (MLE)
What it is

The Most-Used Idea in Machine Learning

Maximum likelihood estimation is the principle behind how almost every ML model learns its parameters. You have a model with unknown parameters θ, and you have observed data. MLE picks the θ that makes the observed data as likely as possible under the model.

If you have ever fit a regression, trained a neural network, or estimated probabilities from counts, you have used MLE — usually without knowing it. It's the unifying mathematical principle.

One-line definition: Maximum likelihood estimation picks the parameter values that make the observed data the most probable, according to the model.
The Likelihood Function
L(θ; data) = Πᵢ P( xᵢ | θ )
The likelihood is the joint probability of all observed data points, treated as a function of the parameter θ. We find θ_MLE = argmax_θ L(θ; data). In practice we work with the log-likelihood (sum of log-probabilities) because products of many small numbers are numerically unstable.
L(θ; data) — likelihood: probability of data as a function of θ
xᵢ — i-th observation
P(xᵢ | θ) — model's probability for observation i
argmax — the θ value that maximises the likelihood
How to use it

The MLE Workflow

  1. Specify a model. Write down P(x | θ) — how the model thinks data is generated for each θ.
  2. Write the likelihood. Multiply P(xᵢ | θ) across all observations.
  3. Take the log. Sum of log-probabilities — easier to maximise and numerically stable.
  4. Maximise. Either analytically (for simple models) or with gradient descent (for everything else). Gradient descent on negative log-likelihood is what neural network training actually does.
  5. Report θ_MLE. That's your trained model.
The connection to ML: training a classifier by minimising cross-entropy is exactly MLE on a categorical model. Training regression by minimising MSE is exactly MLE on a Gaussian noise model. We will see this in Topic 5.
⟁ Worked Example

MLE for a Coin's Bias

You flip a coin 10 times and see 7 heads. The model: each flip is Bernoulli(p). The likelihood is L(p) = p⁷ · (1−p)³. Take the log: ℓ(p) = 7·log(p) + 3·log(1−p). Set the derivative to zero: 7/p − 3/(1−p) = 0. Solving gives p = 7/10 = 0.7. That's the maximum likelihood estimate — the value of p that makes "7 heads in 10 flips" the most probable outcome.

Topic 2
Entropy
What it is

A Measure of Uncertainty in a Distribution

Entropy measures how uncertain a probability distribution is. A distribution that puts all its mass on one outcome has zero entropy — there's no uncertainty. A uniform distribution that spreads probability evenly across many outcomes has high entropy — maximum uncertainty.

The concept comes from Claude Shannon's 1948 paper that founded information theory. The same number that quantifies physical entropy in thermodynamics turns up as the average number of bits needed to encode a message — and as the loss function that classifiers minimise.

One-line definition: Entropy H(P) is the expected number of bits needed to describe a sample from distribution P. High entropy = surprising; low entropy = predictable.
Shannon Entropy
H(P) = − Σᵢ P(xᵢ) · log P(xᵢ)
Sum over all possible outcomes xᵢ. For each outcome, multiply its probability by the log of its probability, sum these up, and flip the sign (because logs of probabilities are negative). The unit depends on the log's base — log₂ gives bits, ln gives nats. ML usually uses ln.
H(P) — entropy of distribution P
P(xᵢ) — probability of outcome i
log — natural log in ML, log₂ in info theory
H = 0 — distribution is certain (one outcome)
Why it matters

What Entropy Tells You

Low Entropy

  • Distribution is sharp — one outcome dominates
  • Easy to predict the next sample
  • Example: a 99% biased coin

High Entropy

  • Distribution is spread — no clear winner
  • Hard to predict the next sample
  • Example: a fair die roll

Maximum Entropy

  • Uniform distribution over k outcomes
  • Entropy = log(k)
  • Used when you know nothing else

Topic 3
Cross-Entropy
What it is

The Classification Loss Every Neural Network Uses

Cross-entropy measures the average number of bits needed to describe samples from one distribution P when you use a code optimised for a different distribution Q. If Q matches P, cross-entropy equals entropy. If Q is far from P, cross-entropy is much higher.

In ML: P is the true distribution over labels (often "all probability mass on the correct class"), and Q is the model's predicted distribution. Minimising cross-entropy is what classification networks do at every gradient step.

One-line definition: Cross-entropy H(P, Q) is the cost of describing P using Q — minimised when Q = P. It's the standard loss for classification.
Cross-Entropy
H(P, Q) = − Σᵢ P(xᵢ) · log Q(xᵢ)
Identical to entropy but with P inside the sum and Q inside the log. When P is "one-hot" (all mass on the true class, zero elsewhere), the sum collapses to −log Q(true class). That's why classification loss for a single example is just "negative log of the predicted probability of the correct class."
P — true distribution (often one-hot label)
Q — model's predicted distribution
H(P, Q) — cross-entropy from P to Q
H(P, Q) ≥ H(P) — always; equal only when Q = P
How it's used

Cross-Entropy in a Neural Network Training Loop

  1. Input a batch of examples. Each example has a true label.
  2. Forward pass. Model outputs predicted probabilities Q for each class.
  3. Compute cross-entropy per example. For label class c: loss = −log Q(c).
  4. Average across the batch. This is the batch loss.
  5. Backpropagate and update weights to reduce the loss. Repeat.
Equivalent statement: minimising cross-entropy is equivalent to maximum likelihood estimation on a categorical model. The two phrases describe the same procedure from different angles — information theory and statistics meet at this loss function.
⟁ Worked Example

Cross-Entropy on a Single Classification Step

True label: class "cat" (one-hot: [1, 0, 0]). Model's prediction: [0.7, 0.2, 0.1] over (cat, dog, fish).

  • ▸ Cross-entropy = −(1 · log 0.7 + 0 · log 0.2 + 0 · log 0.1) = −log 0.7 ≈ 0.357

If the model had been more confident (say 0.95 for cat), the loss would be −log 0.95 ≈ 0.051. If it had been completely wrong (0.05 for cat), the loss would be −log 0.05 ≈ 2.996. Cross-entropy punishes confident wrong predictions very harshly — which is what drives the model to learn.

Topic 4
KL Divergence
What it is

A Distance Between Two Distributions

Kullback–Leibler divergence (KL divergence) measures how different one distribution is from another. KL(P || Q) is zero when P and Q are identical and grows larger as they differ. It is not symmetric — KL(P || Q) is not the same as KL(Q || P).

KL divergence shows up in many corners of ML: as the objective in variational inference, in the loss for variational autoencoders (VAEs), in reinforcement learning policy updates, and as the natural way to ask "how much information do I lose by approximating P with Q?"

One-line definition: KL divergence KL(P || Q) is the extra cost (in bits or nats) of using Q to encode samples actually drawn from P. Always ≥ 0; zero only when P = Q.
KL Divergence
KL(P || Q) = Σᵢ P(xᵢ) · log [ P(xᵢ) / Q(xᵢ) ]  =  H(P, Q) − H(P)
KL divergence equals cross-entropy minus entropy. That equality is the key: minimising cross-entropy is the same as minimising KL divergence, because H(P) is a constant (it doesn't depend on the model Q). All three quantities — entropy, cross-entropy, KL — are different views of the same information-theoretic picture.
KL(P || Q) — divergence from P to Q
P — true / reference distribution
Q — approximating / model distribution
KL ≥ 0 — always non-negative; zero iff P = Q
Why it matters

Where KL Divergence Shows Up in ML

Variational Autoencoders

The VAE objective includes a KL term that pulls the encoder's latent distribution toward a standard Gaussian prior. This is what lets VAEs generate new samples.

Policy Optimisation

Reinforcement learning methods like PPO and TRPO use a KL constraint to prevent policy updates from being too aggressive — keeping the new policy close to the old one.

Knowledge Distillation

A student model is trained to match a teacher's predicted distribution using KL divergence — letting smaller models learn from larger ones.

Bayesian Inference

Variational inference approximates a posterior by finding the distribution in a chosen family that minimises KL to the true posterior. Massive in modern Bayesian ML.

⟁ Worked Example

KL Between Two Coin Distributions

P (true coin): P(heads) = 0.7, P(tails) = 0.3
Q (model coin): P(heads) = 0.5, P(tails) = 0.5

  • ▸ KL(P || Q) = 0.7 · log(0.7/0.5) + 0.3 · log(0.3/0.5)
  • ▸ = 0.7 · 0.336 + 0.3 · (−0.511)
  • ▸ ≈ 0.235 − 0.153 = 0.082 nats

The reverse KL(Q || P) = 0.5 · log(0.5/0.7) + 0.5 · log(0.5/0.3) ≈ 0.087 nats. Different value — that's the asymmetry.

Topic 5
Loss Functions as Likelihood
What it is

Why the Common Losses Are Not Arbitrary

It looks like every ML textbook just picks loss functions out of a hat. Mean squared error for regression. Cross-entropy for classification. Why those two? The answer: both are derived from maximum likelihood on different probabilistic models. Choosing a loss function = choosing a noise model for your data.

One-line definition: Every standard ML loss function corresponds to maximum likelihood under some assumed noise distribution. Pick the noise model, get the loss for free.
MSE = Maximum Likelihood under Gaussian Noise
If y = f(x; θ) + ε, ε ~ Normal(0, σ²)
then − log L(θ) ∝ Σ ( yᵢ − f(xᵢ; θ) )²
Assume each observation differs from the model by Gaussian noise. The negative log-likelihood is proportional to the sum of squared errors. Minimising MSE = maximising Gaussian likelihood. That's the entire derivation.
Cross-Entropy = Maximum Likelihood under Bernoulli / Categorical Noise
If y is a class label, P(y = c | x; θ) = Q_c(x; θ)
then − log L(θ) = − Σ log Q_{y_i}(xᵢ; θ)
Treat the output as a probability distribution over classes. The negative log-likelihood for each example is the negative log probability the model assigned to the true class. Sum across examples → cross-entropy loss.
Why it matters

Practical Consequences

Choose a Different Noise Model → Get a Different Loss

  • Laplace noise → MAE (L1 loss)
  • Poisson noise → Poisson loss
  • Heavy-tailed noise → Huber loss

When MSE Is Wrong

  • Outliers → Gaussian assumption breaks → use MAE or Huber
  • Counts → Poisson better than Gaussian
  • Bounded output → consider Beta or Logit-Normal

The Mental Shift

  • Don't ask "what loss should I use?"
  • Ask "what noise model fits my data?"
  • The loss follows automatically
Topic 6
Regularisation as Priors
What it is

L1 and L2 Penalties Are Bayesian Priors in Disguise

Adding an L2 penalty (sum of squared weights) to a loss function is called ridge regression or weight decay. Adding an L1 penalty (sum of absolute values) is lasso. In a frequentist view these are just "shrink the weights toward zero to prevent overfitting." But there's a deeper interpretation.

In a Bayesian view, regularisation corresponds to placing a prior on the parameters (Module 6.1). Then maximum likelihood becomes MAP — maximum a posteriori — which adds a prior term to the log-likelihood objective.

One-line definition: L2 regularisation is MAP estimation under a Gaussian prior on the weights. L1 regularisation is MAP under a Laplace prior. Regularisation isn't an ad-hoc shortcut — it's Bayesian inference with a flat noise model and a thoughtful prior.
Regularised Objective as MAP
θ_MAP = argmaxθ [ log L(θ; data) + log P(θ) ]
MAP estimation maximises log-likelihood plus log-prior. With a Gaussian prior N(0, σ²) on the weights, log P(θ) = −λ · Σθᵢ² (plus constant). With a Laplace prior, log P(θ) = −λ · Σ|θᵢ|. The two correspond exactly to L2 and L1 penalties.
θ_MAP — maximum a posteriori estimate
L(θ; data) — likelihood from Topic 1
P(θ) — prior on parameters
λ — regularisation strength (also the prior's tightness)
Why it matters

What This Unification Buys You

  • Principled tuning. The regularisation strength λ isn't arbitrary — it has a meaning in terms of how strongly you believe weights should be near zero.
  • Choosing a regulariser. L1 produces sparse solutions because the Laplace prior has a sharp peak at zero — most weights snap to zero. L2 produces small but non-zero weights because the Gaussian prior shrinks smoothly.
  • Generalisation guarantees. Bayesian treatment ties regularisation to model complexity bounds, giving theoretical backing to practical methods.
Big picture: ML methods that look like ad-hoc engineering solutions usually turn out to have clean Bayesian interpretations. Regularisation is the cleanest example.
Topic 7
Statistical Foundations of ML
What it is

Three Ideas Every ML Practitioner Must Internalise

The last piece of this module pulls together three statistical concepts that govern how machine learning models behave: overfitting, the bias-variance trade-off, and generalisation. These are not three separate topics — they are three faces of the same problem.

Concept 1

Overfitting

Overfitting means a model has learned not just the underlying signal but also the random noise in the training data. Training accuracy looks great. Test accuracy looks awful. The model has memorised, not generalised.

Symptoms: large gap between training and validation loss. Predictions that look unreasonable on slightly perturbed inputs. Performance that collapses on new data the model has never seen.

Fixes: more data, simpler models, regularisation, early stopping, dropout, data augmentation. Every one of these is a way to prevent the model from clinging too tightly to training-set quirks.

Bias–Variance Decomposition
Expected Error = Bias² + Variance + Irreducible Noise
The expected prediction error of a model breaks into three parts. Bias is how far the model's average prediction is from the truth — high when the model is too simple. Variance is how much the model's predictions wobble across different training sets — high when the model is too flexible. Irreducible noise is the part you can never remove.
Bias² — error from oversimplified model
Variance — error from over-flexible model
Noise — inherent randomness in the data
Trade-off — reducing one usually increases the other
Concept 2

The Bias–Variance Trade-off

Underfitting (High Bias)

  • Model too simple to capture the pattern
  • Bad on both training and test data
  • Example: linear model on curved data
  • Fix: more flexible model, more features

Overfitting (High Variance)

  • Model too flexible — memorises training noise
  • Great on training, bad on test
  • Example: 50-degree polynomial on 100 points
  • Fix: regularisation, more data, simpler model

Sweet Spot

  • Model complex enough to capture real patterns
  • Simple enough not to chase noise
  • Found by validation-set tuning
  • Modern deep learning bends some of these rules — but the intuition still helps
⟁ Bias–Variance Curve
Slide model complexity from very simple to very flexible. Training error keeps falling, but test error U-shapes — first decreasing (less bias), then rising (more variance). The minimum of test error is the sweet spot.

Concept 3

Generalisation

Generalisation is the goal — the model performs well on data it has never seen during training. Without generalisation, no model is useful. Every other concept in this module is in service of generalisation.

  1. Honest evaluation. Always hold out a test set you never touch during training or tuning.
  2. Cross-validation. Use multiple train/validation splits to get stable performance estimates.
  3. Look at gap between train and test loss. A wide gap = overfitting. A close, both-bad pair = underfitting.
  4. Regularise. Add priors / penalties to push the model toward simpler explanations.
  5. Get more data when possible. The single most reliable way to improve generalisation.
