Geometric & Poisson

Two distributions that answer fundamentally different questions — when does success arrive, and how many rare events occur in a fixed window. The waiting-time distribution and the rare-event distribution.

Part 2 moves beyond fixed trials. We now ask two new types of questions: "How long until success happens?" and "How many events occur in a given time period?" The Geometric distribution answers the first — it models how many tries you need before your first success. The Poisson distribution answers the second — it counts how many random events happen in a fixed window of time or space.

From Counting to Waiting
A new kind of question

The Question Binomial Cannot Answer

In Part 1, Binomial answered: "In n trials, how many successes occurred?"

But often we don't care about the count. We care about when success arrives. Different question, different distribution.

  • "How many sales calls before the first 'yes'?"
  • "How many ad impressions before the first click?"
  • "How many retries before the network packet gets through?"
  • "How many free-throw attempts before the basketball player scores?"
  • "How many job applications before the first offer?"

The number of trials isn't fixed — it depends on when success happens. The experiment continues until success, then stops. This is the territory of the Geometric distribution.

The framing flip: Binomial fixes the number of trials and asks "how many successes?" Geometric fixes "first success" as the stopping rule and asks "how many trials did that take?" Same Bernoulli Bernoulli trials — they just look at the problem differently.
7
Geometric Distribution
What it models

The Waiting-Time Distribution

The Geometric distribution models the number of independent Bernoulli(p) trials required to obtain the first success.

Specifically, let X = the trial number on which the first success occurs. Then X is geometrically distributed. Possible values: X ∈ {1, 2, 3, …} — infinitely many integers.

The story: we keep performing trials, one after another. Each trial succeeds with probability p (independently). Eventually success happens. X records the trial number where it first happened. It is always possible — though sometimes unlikely — that you wait a very long time.

Two conventions exist: Some textbooks define X as the number of failures before the first success (X ∈ {0, 1, 2, …}). We use the more common version: X = trial number of first success (X ∈ {1, 2, 3, …}). The formulas differ slightly; check before applying.
Why does this matter?

Why Do We Need the Geometric Distribution?

In real life, we often don't know how many attempts something will take. Think about these situations:

  • A salesperson doesn't know how many calls they'll make before closing a deal.
  • A student doesn't know how many job applications they'll send before getting an offer.
  • A network doesn't know how many retries a packet will need before getting through.

In all these cases, the number of trials is not fixed in advance — it depends on when success happens. That's exactly what Geometric models. Without it, we'd have no mathematical way to answer "how long should I expect to wait?"

The key difference from Binomial:
  • Binomial: "I'll do exactly 10 trials — how many successes will I get?"
  • Geometric: "I'll keep going until I succeed — how many trials will that take?"
⟁ Geometric PMF (Probability Formula)
P(X = x) = (1 − p)x − 1 · p

What this means in plain English: This formula tells you the probability that your first success happens on exactly trial number x.

Breaking it down:

  • (1 − p)x − 1 = the probability of failing (x − 1) times in a row. If p = 0.3, then (1 − p) = 0.7. So (0.7)² means two failures in a row.
  • p = the probability that trial x finally succeeds.
  • Multiply them together = because each trial is independent, you multiply the probabilities.

Example: If p = 0.5 (fair coin) and you want the first heads on trial 3: (0.5)² × 0.5 = 0.25 × 0.5 = 0.125 (12.5% chance).

Sanity check: If you add up all possible probabilities P(X=1) + P(X=2) + P(X=3) + ... forever, they add up to exactly 1. ✓ This proves it's a valid probability distribution.

⟁ Geometric Mean & Variance (Average & Spread)
E[X] = 1 / p
Var(X) = (1 − p) / p²

E[X] = 1/p — The Average Wait Time:

  • If p = 0.1 (10% success rate): E[X] = 1/0.1 = 10 trials on average
  • If p = 0.5 (50% success rate): E[X] = 1/0.5 = 2 trials on average
  • If p = 0.01 (1% success rate): E[X] = 1/0.01 = 100 trials on average

The pattern: The smaller p is, the longer you wait on average. This makes intuitive sense — rare successes take longer.

Var(X) = (1−p)/p² — How Spread Out Are the Results?

  • For p = 0.1: Var = 0.9/0.01 = 90, so σ ≈ 9.5 trials
  • For p = 0.5: Var = 0.5/0.25 = 2, so σ ≈ 1.4 trials

What this means: When success is rare (small p), waiting times are longer and less predictable. When success is common (large p), waiting times are shorter and more consistent.

A worked walkthrough

Concrete Computation — Simple Example

Setup: A fair coin (p = 0.5). What is the probability the first head appears on the third toss?

Solution: We need Tails, then Tails, then Heads (TT then H). Since each flip is independent:

P(X = 3) = (1 − 0.5)² · 0.5 = 0.25 · 0.5 = 0.125

What this means: About 12.5% chance. Look at the pattern: P(X=1)=0.5, P(X=2)=0.25, P(X=3)=0.125, P(X=4)=0.0625. Each step, the probability cuts in half — that's the geometric decay pattern.

Expected trials before head: E[X] = 1/0.5 = 2. On average, you'll see the first head on the second flip.

The signature property

The Memoryless Property — Geometric's Unique Superpower

The Geometric distribution has a remarkable and counterintuitive property: it is memoryless. Formally:

⟁ Memoryless Property (The Coin Doesn't Remember)
P(X > m + n | X > m) = P(X > n)

What this means in simple terms: If you've already failed m times, the probability that you'll need to wait at least n MORE trials is exactly the same as if you were starting fresh. The past doesn't matter.

Real-world example: You flip a coin 10 times and get tails every time. What's the probability the next flip is heads? Still 50%. The coin doesn't "remember" the 10 tails. Each flip is independent.

Why this matters: This is why the Gambler's Fallacy is wrong. People think "I've lost 10 times, so I'm due to win." But that's not how probability works. Each trial is a fresh start.

Mathematical proof (if you're curious): P(X > m+n | X > m) = P(X > m+n) / P(X > m) = (1−p)m+n / (1−p)m = (1−p)n = P(X > n). ✓

Fun fact: Geometric is the ONLY discrete distribution with this memoryless property.
The Gambler's Fallacy Trap. Many people incorrectly believe that after a streak of failures, success becomes "due." This is the gambler's fallacy. For independent Bernoulli trials, it's literally untrue: the coin doesn't remember previous flips. If a fair coin has come up tails 10 times in a row, the probability of heads on the next toss is still 0.5 — not higher.
Why memoryless matters in practice

Implications in Real Systems

  • Reliability engineering: A memoryless device has the same chance of failing in the next moment, no matter how long it has already been running.
  • Retry policies: If each retry has the same success chance (independent), then past failures don't help or hurt your next attempt.
  • Customer wait modeling: If waiting times are memoryless, waiting for 10 minutes does not make service happen sooner. The chance of being served in the next moment stays the same no matter how long you have already waited.
When to use Geometric

Decision Guide

  • Trials are independent and identically distributed Bernoulli(p).
  • You're counting trials until the first success (or, in the alt convention, failures before first success).
  • Trials continue until success — no fixed n.
  • You care about the waiting time, not the number of successes.

Don't use Geometric when: trials aren't independent (sampling without replacement), success probability changes between trials, or you're counting events in a fixed time window (Poisson, next!).

Real-world applications

Where Geometric Appears

🛒 Customer Conversion

How many website visits until a user makes their first purchase? With conversion rate p, expected visits = 1/p. Marketing funnels use this to forecast acquisition costs.

📞 Sales Calls

How many cold calls until the first deal? If close rate is 5%, expect 20 calls on average. Sales pipeline planning uses Geometric directly.

📡 Network Retries

Number of transmission attempts before successful packet delivery. With success probability p per attempt, retry counts follow Geometric.

🎯 Sports — Free Throws

Number of free-throw attempts before a player makes one. With 80% accuracy: expected attempts = 1.25. Sports analytics uses this for performance modeling.

🧬 PCR Cycles

Number of PCR cycles until detectable amplification. The cycle threshold (Ct) value is essentially a Geometric quantity in many lab assays.

🎰 Reinforcement Learning

Number of episodes until an RL agent first reaches a goal state. Used to characterize exploration efficiency.

🔬 A/B Test First Win

Number of users exposed to a variant before the first "conversion event." Useful for early stopping decisions.

💼 Job Search

Number of applications before the first offer. With response rate p, expected applications = 1/p. The math behind "law of averages" job-search strategies.

From Trials to Continuous Time
The third kind of question

The Question Bernoulli, Binomial, and Geometric Cannot Answer

So far we've assumed discrete trials. Coin flips. Ad impressions. Sales calls. Each trial is a distinct event.

But often, events don't come in discrete trials. They arrive in continuous time:

  • Emails arrive at random moments throughout the day.
  • Customer support tickets are submitted at any time.
  • Earthquakes strike unpredictably.
  • Photons hit a detector at random instants.
  • Cars pass a checkpoint at irregular intervals.

For these, we don't ask "how many successes in 10 trials?" There are no trials. Instead we ask: "how many events in a fixed interval?"

This is the Poisson distribution's domain. It models the count of rare, randomly-timed events in a fixed window of time (or space, or any continuous medium). It is one of the most important and most used distributions in all of applied probability.
9
Poisson Distribution
The rare-event count

What Poisson Models

The Poisson distribution models the number of events occurring in a fixed interval, given the events:

  • Happen independently of each other.
  • Happen at a constant average rate.
  • Do not happen simultaneously (one at a time).

The interval can be:

  • Time: emails per hour, calls per minute, deaths per year.
  • Space: raindrops per square meter, stars per galaxy region, defects per square inch.
  • Distance: potholes per kilometer of road, splice errors per yard of cable.
  • Volume: bacteria per milliliter, dust particles per cubic meter.

The single parameter λ (lambda) is the mean rate — the average number of events per interval.

Why does this matter?

Why Do We Need the Poisson Distribution?

Many real-world events don't happen in "trials" — they happen at random moments in time. You can't say "trial 1: did an email arrive? trial 2: did an email arrive?" because emails can arrive at any instant.

Poisson gives us a way to answer practical questions like:

  • "What's the chance we get exactly 3 customer complaints today?"
  • "How likely is it that zero accidents happen on this road this week?"
  • "What's the probability our server gets more than 100 requests this second?"

Without Poisson, we'd have no model for random events in continuous time or space. It fills a gap that Bernoulli, Binomial, and Geometric can't cover.

The key difference: Binomial and Geometric count events in discrete trials. Poisson counts events in continuous time/space with no fixed number of "trials."
⟁ Poisson PMF (Probability Formula)
P(X = x) = (e−λ · λx) / x!

What this means: This formula tells you the probability of getting exactly x events in your time window.

Breaking down each piece:

  • λ (lambda) = the average number of events you expect. If you get 5 emails per hour on average, λ = 5.
  • λx = lambda raised to the power of x. If λ = 5 and x = 3, then λ³ = 125.
  • e−λ = a special number (e ≈ 2.718) raised to the power of negative lambda. This is a "normalization factor" that makes sure all probabilities add up to 1.
  • x! = x factorial (x × (x-1) × (x-2) × ... × 1). For example, 3! = 3 × 2 × 1 = 6.

Example: If λ = 3 (average 3 events) and you want P(X = 2):

P(X = 2) = (e−3 · 3²) / 2! = (0.0498 · 9) / 2 ≈ 0.224 (22.4% chance)

Sanity check: If you add up all possible probabilities P(X=0) + P(X=1) + P(X=2) + ... forever, they add up to exactly 1. ✓

⟁ Poisson Mean & Variance (Average & Spread)
E[X] = λ
Var(X) = λ

The Unique Feature of Poisson: The mean and variance are EQUAL. Both equal λ. This is Poisson's signature — no other major distribution does this.

What this means:

  • If λ = 5 (average 5 events), then the variance is also 5.
  • The standard deviation σ = √λ = √5 ≈ 2.24
  • This is a quick diagnostic: if your data's average equals its variance, Poisson might be the right model.

Example: A call center gets λ = 10 calls per hour on average. The variance is also 10, so σ ≈ 3.16. This means most hours will have between 7 and 13 calls (roughly mean ± σ).

Important note: As λ gets bigger, the distribution becomes more predictable relative to the mean. When events are common, results are more stable.

A concrete example

Worked Example: Call Center

Setup: A call center receives an average of 5 calls per hour. What is the probability of receiving exactly 3 calls in the next hour?

Step 1: Identify λ = 5 (the mean rate) and x = 3 (the count we're asking about).

Step 2: Plug into the formula:

P(X = 3) = (e−5 · 5³) / 3! = (0.00674 · 125) / 6 ≈ 0.1404

Step 3: Interpretation: about 14% chance of exactly 3 calls. The most likely outcome is around 5 calls (the mean). Receiving 0 calls has probability e−5 ≈ 0.67% — very rare.

Practical use: Staffing decisions. If we need to handle peak loads with high probability, we don't staff for the average — we staff for the upper tail of the Poisson distribution.

A deep connection

The Binomial → Poisson Limit

Poisson is intimately related to Binomial. In fact:

When n is large and p is small, with np = λ constant, Binomial(n, p) ≈ Poisson(λ).

This is the "law of rare events." The intuition: imagine slicing one hour into n tiny sub-intervals. In each sub-interval, either an event occurs (with small probability p) or it doesn't. The total count over the hour is Binomial(n, p). As n → ∞ and p → 0 with np fixed at the mean rate λ, the Binomial converges to Poisson(λ).

This is why Poisson naturally arises whenever you have many opportunities for a rare event — it is the mathematical limit of "many trials, low success rate."

When to use Poisson

Decision Guide

  • You're counting events in a fixed window (time, space, volume, distance).
  • Events are independent and randomly timed.
  • The mean rate λ is constant over the window.
  • Events are rare relative to the granularity of the underlying process — no simultaneous events.

Don't use Poisson when: events come in clusters (e.g., earthquakes with aftershocks), the rate changes over time (e.g., more calls during lunch hour), or one event triggers another (events aren't independent).

Real-world Poisson

Where Poisson Lives

📞 Call Centers

Calls per hour. Managers use Poisson to decide how many staff to schedule for each shift — making sure enough people are available even during busy periods.

🚗 Traffic Engineering

Cars passing a checkpoint per minute. Highway design, traffic signal timing, and accident-rate modeling all use Poisson.

🏥 Hospital Arrivals

Emergency room arrivals per shift. ICU bed planning, ambulance dispatch, and pandemic modeling lean on Poisson.

🌐 Web Server Requests

Requests per second. Autoscaling policies and load balancers are tuned to Poisson-modeled traffic surges.

⚠️ Cyber Attacks

Intrusion attempts per day. Security operations centers (SOC) use Poisson baselines to detect anomalies (rates above λ).

🔬 Radioactive Decay

Decays per second. Geiger counter clicks are Poisson — this is one of the original applications, by Rutherford and Bateman.

📷 Photon Counts

Photons hitting an astronomy detector per exposure. Image-noise models in low-light photography (shot noise) are Poisson.

⚙️ Manufacturing Defects

Defects per square meter of fabric or per yard of wire. Quality control limits are set using Poisson tail probabilities.

Common Mistakes
Mistake 1 — Confusing Geometric conventions. Some books define X = number of failures before first success (X ∈ {0, 1, 2, …}), others define X = trial number of first success (X ∈ {1, 2, 3, …}). Mean = (1−p)/p in the first; 1/p in the second. Always check.
Mistake 2 — Gambler's Fallacy. "I've flipped 9 tails in a row — heads is due!" No. Geometric is memoryless. Each flip is independent. Previous failures don't increase future success probability.
Mistake 3 — Using Poisson when trials are fixed. "10 free throws, p = 0.7, how many made?" That's Binomial(10, 0.7), not Poisson. Poisson is for events in continuous time/space with no fixed trial count.
Mistake 4 — Forgetting Poisson's mean=variance signature. If your data has variance much bigger than the mean, Poisson is probably the wrong model. This is a quick diagnostic check you can always do before choosing Poisson.
Mistake 5 — Wrong interval scaling. If λ = 5 per hour and you want probability in 30 minutes, scale λ to 2.5 (not 5). Always match λ to the interval you're studying.
Mistake 6 — Using Poisson for clustered events. Earthquakes have aftershocks. Buses arrive in clusters after delays. When events come in bunches instead of randomly, Poisson doesn't work well — it will undercount the extreme values.

Q1 — Easy

A basketball player makes free throws with 80% accuracy (p = 0.8). What is the probability the first miss happens on the 4th attempt?

Answer: Here "success" = miss, so p_miss = 1 − 0.8 = 0.2. X ~ Geometric(0.2).
P(X = 4) = (1 − 0.2)³ × 0.2 = (0.8)³ × 0.2 = 0.512 × 0.2 = 0.1024 (about 10.2%)
Q2 — Easy

For a Geometric distribution with p = 0.25, what is the expected number of trials until the first success?

Answer: E[X] = 1/p = 1/0.25 = 4 trials.
On average, you'd need 4 attempts before your first success.
Q3 — Easy

A coffee shop gets an average of 3 customer complaints per week. What distribution models the number of complaints? State its parameter.

Answer: Poisson(λ = 3).
Why? Complaints arrive at random times during the week, independently of each other, at a constant average rate. No fixed number of "trials" — this is Poisson territory.
Q4 — Moderate

A website gets an average of 2 spam comments per day. What is the probability of getting exactly 0 spam comments tomorrow?

Answer: X ~ Poisson(λ = 2). Want P(X = 0).
P(X = 0) = e−2 × 20 / 0! = e−2 × 1 / 1 = e−2 ≈ 0.1353 (about 13.5%)
So there's roughly a 13.5% chance of a spam-free day.
Q5 — Moderate

A coin has P(Heads) = 0.3. You've already flipped 5 tails in a row. What is the probability that the next flip is heads?

Answer: Still 0.3 (30%).
This is the memoryless property! Past failures don't change future probabilities. The coin doesn't "remember" previous flips. This is also why the Gambler's Fallacy is wrong.
Q6 — Moderate

If a call center gets λ = 8 calls per hour, what is the correct value of λ to use when calculating the probability of exactly 2 calls in 15 minutes?

Answer: 15 minutes = 1/4 of an hour. Scale λ: 8 × (1/4) = λ = 2.
Always match λ to the time window you're studying. Don't use λ = 8 for a 15-minute window!
