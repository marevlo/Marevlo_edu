Multiple Testing
& Corrections

When you run many statistical tests at once, random chance will produce false positives — even if nothing real is happening. This module shows you why that happens and how two powerful correction methods fix it.

You already know how a single hypothesis test works. But in real research and data science, you rarely run just one test — you might run dozens or hundreds. This creates a specific problem: the more tests you run, the more likely you are to see a "significant" result purely by chance. Understanding this problem, and knowing how to correct for it, is essential for doing reliable statistical analysis.

02
The Multiple Comparisons Problem
What this topic is about

Running Many Tests at Once

In a standard hypothesis test, you set a significance level — usually α = 0.05. This means: if the null hypothesis is actually true, there is a 5% chance you will incorrectly reject it. That single incorrect rejection is called a false positive or a Type I error.

Now imagine you run not one test, but 20 tests. Even if absolutely nothing is happening in your data — no real effects, pure noise — the probability of getting at least one false positive across all 20 tests is no longer 5%. It is much higher.

Definition — Multiple Comparisons Problem: When you conduct multiple hypothesis tests simultaneously, the probability of observing at least one statistically significant result by pure chance increases with the number of tests. This is called the multiple comparisons problem, or the multiple testing problem.
Why this happens

The Probability Math Behind It

Let's think through the logic carefully. For a single test at α = 0.05:

  • Probability of not getting a false positive = 1 − 0.05 = 0.95

If you run m independent tests, the probability of getting no false positives across all of them is:

P(no false positive in m tests) = (1 − α)ᵐ = 0.95ᵐ

So the probability of getting at least one false positive is:

P(at least one false positive) = 1 − (1 − α)ᵐ

Let's see what this looks like as m grows:

False Positive Risk by Number of Tests (α = 0.05)

  • m = 1 test → risk = 5%
  • m = 5 tests → risk ≈ 23%
  • m = 10 tests → risk ≈ 40%
  • m = 20 tests → risk ≈ 64%
  • m = 50 tests → risk ≈ 92%
  • m = 100 tests → risk ≈ 99.4%

What This Means

  • With 20 tests, there's nearly a 2-in-3 chance of at least one fake "significant" result
  • With 100 tests, a false positive is almost certain
  • This happens even when all your data is pure noise
  • This is the core danger of data dredging or p-hacking
⚠ The Danger in Practice: A researcher tests 50 different genetic markers for a disease. At α = 0.05, even if no marker is truly associated with the disease, they can expect around 2–3 false positives just from random chance. Without correction, those would look like real discoveries.
Key terminology

Family-Wise Error Rate (FWER)

When you run multiple tests together, statisticians call the whole collection a family of tests. The probability of making at least one false positive anywhere in that family is called the Family-Wise Error Rate (FWER).

FWER = P(at least one false positive across all m tests)
Without correction: FWER = 1 − (1 − α)ᵐ
With correction: the goal is to keep FWER ≤ α

The two main correction methods — Bonferroni and FDR — take different approaches to controlling this risk.

Bonferroni Correction
What this is

The Simplest Fix: Divide Alpha by the Number of Tests

The Bonferroni correction is the most straightforward approach to the multiple comparisons problem. The idea is this: if you want the overall probability of making any false positive (the FWER) to stay at or below α, then each individual test must use a much smaller threshold.

Instead of using α = 0.05 for each test, you divide α by the number of tests m. A result is only called significant if its p-value is below this new, tighter threshold.

Core idea: Make each individual test harder to pass, so that even across many tests, the total risk of a false positive stays controlled.
◈ Bonferroni Corrected Significance Threshold
α_bonferroni = α / m
Reject the null hypothesis for test i only if: p_i < α / m
α — The desired overall family-wise error rate (e.g. 0.05)
m — Total number of tests being performed
α_bonferroni — The new, corrected significance threshold for each individual test
p_i — The p-value of the i-th individual test
◈ Why this controls FWER (Bonferroni Inequality)
FWER ≤ m × (α / m) = α
This is based on the Boole/Union-Bound inequality: P(A₁ ∪ A₂ ∪ … ∪ Aₘ) ≤ P(A₁) + P(A₂) + … + P(Aₘ). Each test contributes at most α/m risk, and there are m tests, so the total FWER ≤ m × (α/m) = α. This bound is guaranteed regardless of how correlated the tests are — making Bonferroni very robust.
Worked example

Applying Bonferroni — Step by Step

Scenario: A nutrition researcher runs 15 tests comparing a new diet drug against a placebo on different health markers (cholesterol, blood pressure, weight, blood sugar, etc.). They want an overall significance level of α = 0.05.

Step 1 — Calculate the corrected threshold

α_bonferroni = 0.05 / 15 = 0.00333

Step 2 — Compare each test's p-value to 0.00333

Only tests with p < 0.00333 are called significant. If a test has p = 0.02 (which would normally pass at 0.05), it is not significant after Bonferroni correction.

Step 3 — Interpret

If any test survives Bonferroni correction, you can be confident the overall FWER is still ≤ 0.05. The result is genuinely unlikely to be a false positive.

✓ Bonferroni is conservative: It may miss some real effects (lower statistical power), but it will almost never give you false positives. Use it when false positives are very costly — clinical trials, safety-critical systems, legal decisions.

Limitations

When Bonferroni Is Too Strict

Bonferroni is very conservative. When m is large (say, 10,000 genes), the threshold becomes 0.05 / 10,000 = 0.000005. Most real effects might have p-values of 0.001 — which is strong evidence but would still fail Bonferroni. This is called low statistical power — many real effects get missed.

⚠ The Power Problem: In genomics studies with 500,000 SNPs tested, Bonferroni requires p < 10⁻⁷. Many true genetic associations exist but don't reach that bar. Using Bonferroni here would miss many real discoveries. This is exactly where FDR (next section) is used instead.

Good Times to Use Bonferroni

  • A small number of pre-planned tests (m < 20–30)
  • Each false positive has a serious cost (medicine, law)
  • You want a very strong guarantee against any false discovery
  • Tests are planned before seeing the data (confirmatory analysis)

When to Avoid Bonferroni

  • Large m (hundreds or thousands of tests)
  • Exploratory analysis where some false positives are acceptable
  • Tests are highly correlated (Bonferroni overcorrects)
  • You need to detect as many real effects as possible

False Discovery Rate (FDR)
A smarter approach

What Is the False Discovery Rate?

Instead of asking "can we guarantee zero false positives?" (FWER), the FDR approach asks a more realistic question:

Among all the tests we call significant, what fraction are false positives?
That fraction is the False Discovery Rate.

For example, if FDR = 0.05, it means that among all the results you call "significant," you are willing to accept that about 5% of them might be false positives. You don't try to eliminate all false positives — you control how many slip through, as a proportion of all your discoveries.

This is far more lenient than Bonferroni, which tries to prevent any false positive at all. But in large-scale exploratory research, some false positives are acceptable — as long as you know roughly how many to expect.

Formal definition

The FDR Formula

To understand FDR formally, think about what can happen when you run m tests. Every test either has a true null (no real effect) or a false null (real effect). Every test is either rejected (called significant) or not. This gives four combinations:

Called NOT Significant Called Significant Total
Null is TRUE (no effect) U (True Negatives) V (False Positives) m₀
Null is FALSE (real effect) T (Missed effects) S (True Positives) m − m₀
Total m − R R (total rejections) m
◈ False Discovery Rate — Definition
FDR = E[ V / R ] (when R > 0, else 0)
V = number of false positives among all rejected tests
R = total number of rejected tests (all "significant" calls)
V/R = the proportion of your discoveries that are false
E[·] = expected value (since V and R are random)

The goal of FDR control is to keep this expected proportion at or below a chosen level q (e.g. q = 0.05 or q = 0.10).

The procedure

Benjamini-Hochberg (BH) Procedure

The most widely used method for controlling FDR was proposed by Benjamini and Hochberg in 1995. It is straightforward to apply:

Step-by-Step

  • Step 1: Run all m hypothesis tests and collect all m p-values.
  • Step 2: Sort the p-values in ascending order: p₍₁₎ ≤ p₍₂₎ ≤ … ≤ p₍ₘ₎. The subscript (i) means the i-th smallest.
  • Step 3: Choose your desired FDR level q (e.g. 0.05).
  • Step 4: Find the largest rank k such that p₍ₖ₎ ≤ (k/m) × q
  • Step 5: Reject the null hypothesis for all tests with rank 1 through k (all tests up to and including rank k).
◈ Benjamini-Hochberg Critical Value
Reject H₀ for all i ≤ k, where k = max{ i : p₍ᵢ₎ ≤ (i/m) × q }
p₍ᵢ₎ — The i-th smallest p-value (sorted ascending)
m — Total number of tests
q — The desired FDR level (e.g. 0.05)
k — The largest rank where the condition holds
(i/m) × q — The BH threshold for the i-th test
✓ Proven result: When tests are independent (or positively correlated), the BH procedure guarantees FDR ≤ q. This was mathematically proven by Benjamini and Hochberg. It is a weaker guarantee than FWER control, but it allows far more discoveries.

Monthly Spend 0.004 2 2/8 × 0.05 = 0.01250 ✓ YES
Login Frequency 0.013 3 3/8 × 0.05 = 0.01875 ✓ YES

The largest rank k where p₍ₖ₎ ≤ (k/m) × q is k = 4 (Support Calls at rank 4: 0.022 ≤ 0.025). So we reject tests 1 through 4. Under Bonferroni (α/m = 0.05/8 = 0.00625), only 1 feature would pass. BH found 4 discoveries while still controlling FDR at 5%.

FWER vs FDR — The Core Tradeoff

Two Different Goals, Two Different Guarantees

The fundamental difference between the two correction approaches is what they are trying to control:

Bonferroni / FWER Control

  • Controls: P(at least 1 false positive)
  • Guarantee: Family-wise error rate ≤ α
  • Method: Make each test's threshold stricter by factor m
  • Cost: Low power — many real effects missed
  • Used when: Even 1 false positive is unacceptable

Benjamini-Hochberg / FDR Control

  • Controls: E[ false positives / total positives ]
  • Guarantee: Expected proportion of false discoveries ≤ q
  • Method: Adaptive threshold based on sorted p-values
  • Benefit: Higher power — more real effects found
  • Used when: Some false positives okay if rate is known

Real-World Applications

🧬 Genomics — The Original Use Case

Genome-wide association studies test 500,000 to 10,000,000 genetic variants. Without correction, thousands of false associations would appear. BH (FDR control at q = 0.05–0.20) is the standard method. Bonferroni is also used but requires p < 5 × 10⁻⁸.

📊 A/B Testing at Scale

Large tech companies run dozens of simultaneous A/B tests. Each test uses Bonferroni correction or sequential testing frameworks to avoid the false positive inflation that would otherwise corrupt feature decisions.

🧠 fMRI Brain Imaging

A single brain scan tests ~100,000 voxels (spatial units). Without correction, brain regions would appear "active" by pure chance. Both Bonferroni and FDR methods are applied, depending on the study's tolerance for false discoveries.

💊 Drug Discovery

Pharmaceutical screening tests hundreds or thousands of candidate molecules. FDR control allows researchers to identify a list of "promising" compounds, knowing that perhaps 10% may be false positives — which is fine, since they all go to further lab testing.

📈 Quantitative Finance

Backtesting trading strategies on historical data with many parameter combinations requires correction. Without it, a strategy might look profitable purely because it was optimized over many random trials — called "overfitting to noise."

🤖 ML Model Comparison

When comparing many model architectures or hyperparameter configs in a benchmark, Bonferroni or FDR corrections prevent falsely concluding that one model beats another. Frameworks like MLflow often flag this risk.

T1
The Multiple Comparisons Problem is inevitable at scale. Every time you run m independent tests at α = 0.05, you expect about m × 0.05 false positives — even with zero real effects. With 100 tests, roughly 5 false positives appear by chance alone. This is a mathematical certainty, not a flaw in your data.
T2
FWER = 1 − (1−α)ᵐ grows fast. At m = 20, it exceeds 64%. At m = 50, over 92%. The family-wise error rate is not a theoretical concern — it makes uncorrected multiple testing essentially unreliable in any large-scale analysis.
T3
Bonferroni: α_corrected = α / m. Simple, conservative, and guaranteed to keep FWER ≤ α regardless of test correlation. Best for small m where false positives are costly. The weakness is low statistical power when m is large.
T4
Benjamini-Hochberg controls FDR, not FWER. It asks: "of everything I call significant, what fraction is false?" Sort p-values, compare each to (i/m)×q, and reject all tests up to the largest passing rank. It finds more true effects at the cost of allowing a controlled fraction of false positives.
T5
The choice is about context. Confirmatory analysis with few planned comparisons → Bonferroni. Large-scale exploratory analysis (genomics, drug screening, feature selection) → BH/FDR. The core tradeoff is always: strict error control (fewer discoveries, more confidence) vs. more discoveries (some false positives, lower power cost).
T6
Formula Recap: FWER uncorrected: 1−(1−α)ᵐ — Bonferroni threshold: α/m — BH rejection rule: p₍ᵢ₎ ≤ (i/m)×q, reject all ranks 1 to k — FDR definition: E[V/R] where V = false positives among R total rejections.
