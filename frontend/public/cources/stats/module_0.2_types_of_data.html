CONCEPT 01 // LABELS VS. NUMBERS

Categorical vs. Numerical

Categorical data describes qualities or labels — things you put in buckets. You can't average a color or sum a city name. Examples: blood type, country, payment method, customer segment.

Numerical data describes quantities — things you can do math on. Examples: temperature, revenue, latency, age.

CONCEPT 02 // COUNT VS. MEASURE

Discrete vs. Continuous

Within numerical data, the second fork. Discrete data is countable — finite integer steps with gaps between values. There is no "3.7 students."

Continuous data flows along a number line — between any two values, infinite more exist. Height could be 1.7m, 1.71m, 1.711m, ...

// 03
The Four Measurement Scales

Statistician Stanley S. Stevens proved that not all numbers are equal. Each scale unlocks more powerful operations than the one before it — and using the wrong operation on the wrong scale produces meaningless results.

SCALE // 01 — WEAKEST
Nominal LABELS
Pure names. No order, no math. Categories that just are. Two values are either the same or different — nothing more.
examples:
→ blood types (A, B, AB, O)
→ gender, country, brand
allowed ops: = ≠ , mode, count
SCALE // 02
Ordinal RANKED
Order matters, but gaps between values are unknown. "1st place" beat "2nd place," but we don't know by how much.
examples:
→ movie ratings (1–5 stars)
→ education (HS, BS, MS, PhD)
allowed ops: < > , median
SCALE // 03
Interval EQUAL GAPS
Order plus equal intervals between values. But the zero point is arbitrary — so ratios don't work. 20°C is not "twice as hot" as 10°C.
examples:
→ temperature (°C, °F)
→ calendar years, IQ scores
allowed ops: + − , mean, σ
SCALE // 04 — STRONGEST
Ratio TRUE ZERO
All the power of interval, plus a true zero meaning "none at all." Ratios make sense: 200 kg is twice 100 kg.
examples:
→ weight, height, age, income
→ Kelvin temperature, distance
allowed ops: × ÷ , all stats

Each step right adds a mathematical property: identity → order → equal intervals → true zero. A higher scale can always be "downgraded" to a lower one. The reverse is impossible.

Unstructured Data
// SCHEMA-FREE // ~80% OF WORLD DATA
Data with no fixed format — free text, images, audio, video. Traditional databases can't index its meaning. This is where deep learning and modern AI dominate.

// MIDDLE GROUND
SEMI-STRUCTURED = tags + flexible schema (JSON, XML, logs)

Real-world data is rarely cleanly one or the other. JSON documents, log files, and API responses sit between — partially organized but flexible enough to hold messy content.

// 06
Real-World Applications
🏥
HEALTHCARE AI
Patient Triage Systems
Blood type (nominal), pain level 1–10 (ordinal), body temperature (interval), and heart rate (ratio) all coexist in a single ER form. Each requires different statistical handling.
🛒
E-COMMERCE ML
Recommendation Features
Product category (nominal), star ratings (ordinal), price (ratio), review text (unstructured). Feature engineering depends entirely on knowing each type.
📡
IOT ENGINEERING
Sensor Data Pipelines
A single smart meter streams device-id (nominal), signal quality (ordinal), timestamp (interval), and power draw (ratio). Pipelines must respect each type.
🧠
NLP / DEEP LEARNING
Text Embeddings
LLMs convert unstructured text into structured high-dimensional vectors — the most important type transformation in modern AI.
💰
QUANTITATIVE FINANCE
Risk Feature Encoding
Credit ratings (AAA→D ordinal), income (ratio), employment type (nominal). Encoding ordinal as nominal — or vice versa — silently destroys model accuracy.
🛰️
CLIMATE SCIENCE
Satellite Telemetry
Image pixels (continuous ratio), cloud-type labels (nominal), surface temperature (interval °C → ratio K). Wrong scale = wrong climate model.

Neural Challenge
QUERY 01An AI model uses "T-shirt size" (S, M, L, XL) as a feature. Which scale is this?
Nominal — they are just labels.
Ordinal — they have a meaningful order but unequal gaps.
Interval — the differences between sizes are equal.
Ratio — XL is exactly 4× the size of S.
QUERY 02"Today's temperature is 20°C, twice as hot as yesterday's 10°C." Why is this statement statistically wrong?
Because Celsius is not a real unit of temperature.
Because Celsius is an interval scale with arbitrary zero — ratios are not meaningful.
Because temperature must always be measured in Fahrenheit.
Because the temperatures are discrete, not continuous.
QUERY 03"Number of customer support tickets received per day" is best described as which type?
Categorical — ordinal scale.
Numerical — continuous on a ratio scale.
Numerical — discrete on a ratio scale.
Unstructured — text data.
QUERY 04A startup stores: a customers table in PostgreSQL, scanned PDF invoices, and a folder of voice support recordings. How would you classify these three datasets?
All three are structured because they're all stored digitally.
All three are unstructured because they contain mixed media.
PostgreSQL is structured; PDFs and voice recordings are unstructured.
PostgreSQL is semi-structured; the rest is ordinal.

// CLASSIFICATION SCHEMA LOADED
Before any statistic, model, or algorithm — comes classification. Every datapoint lives somewhere on the data taxonomy: categorical (labels) or numerical (quantities), discrete (countable) or continuous (measurable). Each variable also lives on one of four measurement scales — nominal, ordinal, interval, ratio — and the scale alone determines which operations are mathematically legal. Add structured vs. unstructured, and you now know the language every modern data system speaks. Misclassify your data, and even the best model will quietly lie to you.
KEY // 01
Two Forks
Categorical vs. numerical, then discrete vs. continuous. Every variable in your dataset lands somewhere on this tree.
KEY // 02
Scale = Power
Nominal → Ordinal → Interval → Ratio. Each level unlocks a new class of operations and statistics.
KEY // 03
True Zero
Ratios only make sense when zero means "absence." That's what separates ratio from interval.
KEY // 04
80% Unstructured
Most real-world data has no schema. Modern AI exists largely to make sense of it.
