① GLM unifies the linear boxes

Linear reg = Gaussian + identity. Logistic = Bernoulli + logit. Poisson = Poisson + log. One framework, many faces. (Track 3.10)

② Bias–variance positions every box

Single trees & small-k KNN = high variance. Heavy regularization & large-k KNN = high bias. Ensembles exist to fix one or the other. Try it below ↓

Derivation — where the demo's ŵ = (XᵀX)⁻¹Xᵀy comes from proof · code

The playground above fits a polynomial by least squares. Here is the one equation it solves, derived from scratch — a taste of the "math + code" pairing every algorithm module uses.

Minimize the squared error

Stack the data as a matrix X (one row per point, columns 1, x, x², …) and minimize L(w) = ‖y − Xw‖². Expand and take the gradient:

L(w) = (y − Xw)ᵀ(y − Xw) = yᵀy − 2wᵀXᵀy + wᵀXᵀXw ∇L(w) = −2Xᵀy + 2XᵀXw = 0 ⟹ XᵀX w = Xᵀy (the normal equations) ⟹ ŵ = (XᵀX)⁻¹ Xᵀy when XᵀX is invertible

Because XᵀX is positive semidefinite (Track 1.0), this stationary point is a global minimum — least squares is convex. When XᵀX is near-singular the inverse explodes; that fragility is what ridge (adding λI) and the SVD-based pseudo-inverse fix later.

import numpy as np def polyfit_ls(x, y, degree): X = np.vander(x, degree + 1, increasing=True) # [1, x, x², …] w = np.linalg.solve(X.T @ X, X.T @ y) # normal equations return w # this is exactly what the demo computes # np.linalg.lstsq(X, y) is the stabler SVD-based version (Track 1.0)

This is the first brick of the tinyml/ library you'll build — by the end of Track 3 you'll have written linear, ridge, and lasso solvers yourself.
