// 02
Why Relationships Matter
// BEYOND SINGLE VARIABLES

Statistics Isn't Only About One Variable at a Time

So far, we have analyzed single variables — their average, spread, shape, position. But one of the biggest goals of data analysis is bigger than that: understanding relationships between variables. We want to answer real-world questions like:

  • Does studying more increase exam scores?
  • Does regular exercise improve health?
  • Does advertising spending increase sales?
  • Does temperature affect electricity usage?
  • Do neural network parameters affect model performance?

To study these connections, statistics uses a powerful tool called correlation. But one of the most dangerous mistakes in all of analytics is confusing correlation with causation. This module is about understanding what relationships actually mean, when variables truly move together, and why "moving together" does NOT always mean "causing."

⚡ THE MISTAKE THAT BREAKS DECISIONS

A company sees that the time users spend on its app correlates with revenue. The CEO says: "Force users to stay longer — revenue will go up!" A senior data scientist says: "Stop. Time-on-app doesn't cause spending. People who already want to buy spend more time browsing AND buy more — the same hidden motivation drives both." Forcing low-intent users to stay longer would create resentment, not revenue. Correlation could have sent the entire strategy in the wrong direction.

// 03
What Is Correlation?
// CONCEPT 01 — VARIABLES THAT MOVE TOGETHER

Correlation = How Strongly Two Variables Move Together

Simple Definition

Correlation measures how strongly two variables move together. It describes three things at once: the direction of the relationship (do they move the same way or opposite ways?), the strength (a little or a lot?), and the consistency (does the pattern hold across all data points or only sometimes?).

Two Quick Examples

Example 1: as study hours increase, exam marks also increase. These two variables show a positive correlation — they move in the same direction.

Example 2: as exercise increases, heart rate at rest usually decreases. These two variables show a negative correlation — they move in opposite directions.

⚠ THE CRUCIAL INSIGHT

Correlation only measures association — that two variables tend to move together. It does NOT prove cause-and-effect. Whether exercise causes heart rate to drop is a different question, requiring different evidence. We will understand this difference shortly.

Four Types of Relationships
// CONCEPT 02 — KNOW THESE BY SIGHT

Four Patterns Every Data Scientist Recognizes Instantly

// TYPE 01
Positive Correlation

↗ STUDY ↑ → MARKS ↑
// TYPE 02
Negative Correlation
As one variable increases, the other decreases. Points move in opposite directions.
↘ PRICE ↑ → DEMAND ↓
// TYPE 03
Zero Correlation
No consistent pattern. Changes in one variable don't predict changes in another.
SHOE SIZE ≠ IQ
// TYPE 04
Nonlinear (Curved)

STRESS vs PERFORMANCE

A. Positive Correlation — Moving Up Together

As one variable increases, the other also increases. Classic examples: study hours ↔ marks, height ↔ weight, work experience ↔ salary. Visually, the scatter plot points move upward together — a rising trend from left to right.

B. Negative Correlation — Moving in Opposite Directions

As one variable increases, the other decreases. Examples: exercise ↔ body fat (more exercise → less body fat), or price ↔ demand (higher price → lower demand). Visually, points slope downward from left to right.

C. Zero Correlation — No Relationship

No consistent pattern exists between the variables. Example: shoe size ↔ IQ — changes in one variable do not predict changes in the other. Scatter plot looks like a random cloud of points.

D. Nonlinear Relationships

Sometimes variables ARE related, but not linearly. A famous example is stress vs. performance. Moderate stress improves performance — but too much stress reduces it. This creates a curved relationship (an inverted-U shape).

⚠ WHY NONLINEAR MATTERS

Standard correlation measures (especially Pearson) measure only linear relationships. They can return a near-zero number even when a strong nonlinear relationship exists. Always plot your data first — never trust a correlation number alone.

// 05
Scatter Plots — The Foundation of Correlation
// CONCEPT 03 — ALWAYS PLOT FIRST

The Single Most Important Chart for Relationships

Before calculating any correlation number, the rule is simple: visualize the relationship first. The tool for that is the scatter plot. It places one variable on the X-axis, another variable on the Y-axis, and plots each observation as a single point. The cloud of points reveals the entire structure of the relationship.

Why Scatter Plots Matter So Much

Scatter plots immediately reveal:

  • Direction — do the points trend up, down, or have no trend?
  • Strength — are the points tightly packed along a line, or scattered loosely?
  • Outliers — single points sitting far away from the main mass.
  • Clusters — separate groups of points suggesting subgroups.
  • Nonlinear behavior — curves, U-shapes, plateaus that no single number can capture.
✓ THE GOLDEN RULE

Never trust correlation numbers without visualization. A famous demonstration called Anscombe's Quartet shows four completely different scatter plots — a linear trend, a parabola, an outlier-driven line, and a vertical cluster with one rogue point — that all have identical correlation values. Plot. First. Always.

// 07
Covariance — The Starting Point
// CONCEPT 04 — THE RAW BUILDING BLOCK

Covariance — The Foundation Correlation Is Built On

What Is Covariance?

Correlation does not appear from nowhere. It begins with a more basic measure called covariance. Covariance measures whether two variables tend to move together — but unlike correlation, the magnitude isn't on a clean 0–1 scale.

  • Positive covariance: the two variables tend to increase together.
  • Negative covariance: when one variable increases, the other tends to decrease.
  • Zero covariance: no linear pattern of joint movement.
// FORMULA — COVARIANCE
Cov(X,Y) = ( 1/n ) · Σ(xᵢ − x̄)(yᵢ − ȳ)

For each pair of observations, look at how X deviates from its mean and how Y deviates from its mean. Multiply these two deviations. Average across all observations. Positive products dominate when X and Y move together; negative products dominate when they move opposite.

x̄, ȳmeans of X and Y
xᵢ, yᵢindividual pair (i-th observation)
nnumber of pairs
WORKED EXAMPLE — COMPUTE COVARIANCE BY HAND
Data: 3 students. Study hours (X): 1, 2, 3. Marks (Y): 40, 60, 80.
Step 1 — Find the means: Mean of X = (1+2+3)/3 = 2. Mean of Y = (40+60+80)/3 = 60.
Step 2 — For each student, compute (X − mean X) × (Y − mean Y):
Student 1: (1−2) × (40−60) = (−1) × (−20) = +20 ✔ Both below their means → positive
Student 2: (2−2) × (60−60) = (0) × (0) = 0 — Right at both means
Student 3: (3−2) × (80−60) = (+1) × (+20) = +20 ✔ Both above their means → positive
Step 3 — Sum the products: 20 + 0 + 20 = 40
Step 4 — Divide by n: 40 / 3 ≈ 13.3
→ Covariance = +13.3. Positive number → when study hours go up, marks go up too. They move together! Notice how every product was positive or zero — that is what "moving together" looks like in the math.

Covariance has one big problem: its magnitude depends on the units and scale. If height is measured in centimeters vs meters, or weight in kilograms vs grams, the covariance number changes dramatically — even though the underlying relationship hasn't changed at all. This makes covariance hard to interpret universally. Is a covariance of 200 "big" or "small"? Depends entirely on the data scale.

This is why we normalize covariance — divide by the standard deviations of both variables. That normalized version is the Pearson correlation coefficient, which is unit-free and always between −1 and +1.

// 08
Pearson Correlation Coefficient
// CONCEPT 05 — THE WORLD-FAMOUS r

Pearson Correlation — The Most-Used Number in All of Statistics

What Is Pearson Correlation?

Pearson correlation measures the strength of a linear relationship between two variables. It is the most widely used correlation measure in the world. Its symbol is the lowercase letter r, and it always falls between −1 and +1.

Interpreting the Value of r

r ValueMeaning
+1Perfect positive linear relationship — every increase in X gives a proportional increase in Y
0No linear relationship
−1Perfect negative linear relationship — every increase in X gives a proportional decrease in Y

Example Intuition

If a student's study time increases and their marks increase consistently — almost in a straight line — the Pearson correlation becomes strongly positive, close to +1. If marks bounce around randomly regardless of study time, r will be near zero.

Magnitude Interpretation — How Strong Is "Strong"?

Practitioners use rough guidelines for the strength of a Pearson correlation:

0.0 – 0.2
VERY WEAK
0.2 – 0.4
WEAK
0.4 – 0.6
MODERATE
0.6 – 0.8
STRONG
0.8 – 1.0
VERY STRONG
✓ IMPORTANT CLARIFICATION

These interpretations depend on domain, context, and noise levels. In real-world systems — especially in social sciences or AI feature engineering — even a correlation of 0.3 can be important. In tightly controlled physics experiments, 0.99 might be the threshold of interest. The labels above are guidelines, not rules.

// FORMULA — PEARSON CORRELATION COEFFICIENT
r = Cov(X, Y) / ( σ_X · σ_Y )

Take the covariance between X and Y, then divide by the product of the standard deviations of both variables. This normalization strips away the units and scales — the result is a clean, unit-free number between −1 and +1.

Cov(X,Y)joint movement of X and Y
σ_Xstandard deviation of X
σ_Ystandard deviation of Y
Rangealways between −1 and +1

Pearson's Intuition — What Question Does It Really Ask?

Pearson compares joint movement against individual variability. It is asking: "When X changes, does Y consistently change in a proportional linear way?" If yes, r is close to ±1. If Y moves randomly when X changes, r is near zero. The standardization (dividing by standard deviations) ensures the answer is comparable across datasets regardless of units.

QUICK NUMERIC EXAMPLE
Step 1 — Data: 5 students. Study hours X: 1, 2, 3, 4, 5. Marks Y: 50, 60, 65, 75, 85.
Step 2 — Mean X = 3, Mean Y = 67.
Step 3 — Deviations and products: (X−3)(Y−67) pairs: (−2)(−17)=34, (−1)(−7)=7, (0)(−2)=0, (1)(8)=8, (2)(18)=36. Sum = 85.
Step 4 — Cov(X,Y) = 85/5 = 17. σ_X ≈ 1.41, σ_Y ≈ 11.66.
Step 5 — r = 17 / (1.41 × 11.66) ≈ +0.99
→ Almost perfect linear positive relationship between study hours and marks.

Properties of Pearson Correlation

  • A. Unit-free. Independent of whether values are measured in meters, kilograms, or dollars.
  • B. Symmetric. Correlation(X, Y) = Correlation(Y, X). The order doesn't matter.
  • C. Measures only LINEAR relationships. This is critical. Pearson can completely miss strong curved relationships.
  • D. Sensitive to outliers. Extreme values can drastically distort r. Without an outlier you may see strong positive correlation; one extreme point may flip it to negative.

When Pearson Correlation Works Best

Use Pearson when:

  • The relationship is linear.
  • Both variables are continuous numerical.
  • The data has few or no extreme outliers.
  • The distributions of X and Y are approximately normal (bell-shaped).
// 09
Spearman Correlation — The Rank-Based Alternative
// CONCEPT 06 — WHEN PEARSON ISN'T ENOUGH

Spearman Correlation — Built for Nonlinear, Rank-Based Data

Why Spearman Exists

Pearson measures linear relationships. But many real-world relationships are nonlinear, rank-based, or monotonic (consistently going up or consistently going down — but not in a straight line). For these situations, Spearman correlation handles the data much better than Pearson.

What Is Spearman Correlation?

Spearman correlation measures the relationship between the ranks of observations — not the raw values. Instead of using x = 90, 80, 70, it converts those into ranks 1, 2, 3, then computes the Pearson correlation on the ranks.

Rank Intuition — A Simple Example

Suppose three test scores are 90, 80, 70. Converting to ranks: 90 is highest (rank 1), 80 is second (rank 2), 70 is third (rank 3). Spearman then studies whether the ranks of X line up consistently with the ranks of Y. If both rankings agree perfectly, Spearman = +1. If they are completely opposite, Spearman = −1.

What Spearman Detects — Monotonic Relationships

Spearman captures monotonic relationships: as one variable increases, the other generally either increases (or generally decreases) — even if not in a straight line. The relationship can curve, plateau, accelerate — as long as it consistently goes one direction.

Example — Exercise and Fitness

Suppose as exercise hours increase, fitness improves rapidly at first, then more slowly. The relationship is nonlinear but monotonic (still consistently going up). Pearson may report a moderate correlation because the relationship isn't a perfect line. Spearman, however, captures this beautifully — it's near +1, because the rankings agree all the way.

WORKED EXAMPLE — COMPUTE SPEARMAN STEP BY STEP
Data: 4 students. Study hours (X): 1, 3, 2, 5. Exam marks (Y): 40, 70, 55, 90.
Step 1 — Convert X to ranks: 1 is smallest → rank 1. 2 → rank 2. 3 → rank 3. 5 is largest → rank 4. So X ranks: 1, 3, 2, 4
Step 2 — Convert Y to ranks: 40 → rank 1. 55 → rank 2. 70 → rank 3. 90 → rank 4. So Y ranks: 1, 3, 2, 4
Step 3 — Compare the ranks: X ranks = (1, 3, 2, 4). Y ranks = (1, 3, 2, 4). They match perfectly!
Step 4 — Compute Pearson correlation on the ranks: Since the ranks are identical, the correlation is +1.0
→ Spearman ρ = +1.0. Perfect agreement in rankings. Every time X increases in rank, Y increases in rank too — even though the raw numbers are not in a perfect straight line.

When to Use Spearman

Choose Spearman when:

  • The data contains outliers that would distort Pearson.
  • The relationship is monotonic but nonlinear.
  • Your variables are ordinal (e.g. ranks, satisfaction levels, education levels).
  • You care about ranking consistency more than magnitude of change.
✓ PRACTICAL TIP

If you don't know which to use, compute both. If Pearson and Spearman are very close, the relationship is essentially linear. If Spearman is much larger than Pearson, the relationship is monotonic but curved — and Spearman is the better metric to report.

// 12
Correlation Does NOT Mean Causation
// THE BIGGEST LESSON IN ALL STATISTICS

The Single Most Important Sentence in This Course

Just because two variables move together, that does NOT mean one causes the other. This is the single most important idea in all of statistics, and one of the most violated rules in business, news headlines, and even scientific papers.

The Classic Example — Ice Cream and Drowning

Every summer, ice cream sales rise and drowning deaths rise. The Pearson correlation between these two variables is strongly positive. Does ice cream cause drowning? Of course not. The hidden cause of both is summer / hot weather — more people buy ice cream because it's hot, and more people swim (and drown) because it's hot. Neither variable causes the other. They share a common cause.

⚠ JUST BECAUSE B FOLLOWS A DOES NOT MEAN A CAUSED B

This is one of the most common mistakes in all of reasoning. Two things happen together — or one follows the other — and we immediately assume one caused the other. Banning ice cream sales would not reduce drownings. The hidden third variable (hot weather) is everything.

This fallacy has a Latin name: "Post hoc, ergo propter hoc" — meaning "after this, therefore because of this." You may see this phrase in textbooks.

The General Lesson

When you see two variables moving together, there are always at least three possible explanations:

  • A causes B. (study time → marks: yes, plausible cause)
  • B causes A. (the relationship is reversed)
  • A third variable C causes both. (ice cream and drowning both caused by summer heat) — this is called confounding.

Correlation alone cannot tell these three apart.

// 13
Confounding Variables
// CONCEPT 07 — THE HIDDEN PUPPETEER

Confounders — The Hidden Variable Pulling the Strings

What Is a Confounder?

A confounder is a hidden factor that influences both variables you are studying. It creates a false sense of cause-and-effect between two things that are actually both being driven by the third variable.

A Famous Example — Coffee and Lung Cancer

Suppose data shows that coffee drinking correlates with lung cancer. Does coffee cause cancer? Probably not. The hidden variable is likely smoking. Smokers tend to drink more coffee (cigarettes pair with coffee culturally), and smokers have much higher cancer risk. Without accounting for smoking, the data misleads us into blaming coffee.

This is why epidemiologists, doctors, and data scientists carefully control for confounders in their studies. Failing to do so produces dangerous false conclusions that may harm public health or business strategy.

Common confounders include:

  • Age — confounds health and almost every behavior.
  • Income — confounds education, health, lifestyle.
  • Geography — confounds weather, culture, economy.
  • Time of year — confounds seasonal behaviors (the ice cream / drowning case).

// 15
Spurious Correlations
// CONCEPT 08 — MEANINGLESS COINCIDENCES

Spurious Correlations — The Nonsense Patterns That Fool Everyone

What Is a Spurious Correlation?

A spurious correlation is a meaningless statistical association. Two variables appear related — sometimes very strongly — but the relationship is purely by coincidence, driven by random chance, or by some hidden variable that doesn't actually link them.

Famous Spurious Examples

  • Number of pirates ↔ global warming. Both have been changing over centuries, but no causal connection exists.
  • Per-capita cheese consumption ↔ deaths from being tangled in bedsheets. Yes, this is a real famous example. Strong correlation, zero meaning.
  • US spending on science ↔ suicides by hanging. Both happened to trend over time. No relationship.

Why Spurious Correlations Happen

In large datasets, random patterns naturally appear. If you test enough pairs of unrelated variables, a few will look correlated just by accident — especially over time-series data that all trend together. This is why "data mining at scale" is dangerous if you skip causal reasoning.

⚠ IMPORTANT ML/AI INSIGHT

Large-scale data mining can produce fake discoveries. This is why modern ML pipelines need careful validation, causal reasoning, and domain knowledge — not just blind correlation hunting. An algorithm that finds a correlation between cheese consumption and bedsheet deaths is not insightful; it's noise.

// 16
Correlation Matrices
// CONCEPT 09 — MANY VARIABLES AT ONCE

Correlation Matrix — All Relationships in One View

What Is a Correlation Matrix?

A correlation matrix shows the correlations between many variables simultaneously. It is a square table where each row and column represents a variable, and each cell shows the correlation between two specific variables.

Structure

  • Rows and columns — list the variables in the dataset.
  • Diagonal — always 1.0 (every variable is perfectly correlated with itself).
  • Off-diagonal cells — show pairwise correlation values, usually between −1 and +1.
  • Symmetric — the entry at row i, column j equals the entry at row j, column i.

Example Correlation Matrix

HEIGHTWEIGHTAGE
HEIGHT1.000.700.10
WEIGHT0.701.000.20
AGE0.100.201.00

Reading this matrix: height and weight are strongly correlated (r = 0.70). Age has only weak relationships with the other two (0.10 and 0.20). The diagonal is always 1.00 by definition.

✓ HOW TO READ A CORRELATION MATRIX — 3 EASY STEPS

Step 1 — Ignore the diagonal. Every variable is perfectly correlated with itself (r = 1.0). Those cells tell you nothing useful.

Step 2 — Scan for the biggest numbers (ignoring the sign). Values close to +1 or −1 mean strong relationships. In the table above, the biggest off-diagonal value is 0.70 (Height ↔ Weight) — that is the strongest relationship.

Step 3 — Look for values near zero. Values close to 0 mean no relationship. Age ↔ Height = 0.10 means age and height are basically unrelated in this dataset.

Why Correlation Matrices Matter

Correlation matrices help data scientists in four big ways:

  • Identify relationships across many variables at once.
  • Detect redundancy — if two variables are very highly correlated, they carry almost the same information.
  • Understand feature interactions in machine learning datasets.
  • Guide feature selection — drop redundant variables, keep informative ones.
// 17
Correlation in Machine Learning
// MULTICOLLINEARITY & FEATURE WORK

Why Every ML Engineer Computes Correlations Before Modeling

Correlation is heavily used in machine learning for:

  • Feature engineering — discovering which features carry information about the target.
  • Dimensionality reduction — methods like PCA depend on covariance and correlation.
  • Multicollinearity detection — finding redundant features that can break models.
  • Recommendation systems — correlation between users and items drives suggestions.

Multicollinearity — The Hidden Model Killer

If two predictor features are highly correlated, the regression model becomes unstable. The coefficients become unreliable and may swing wildly with tiny data changes. The model still works, but the coefficients lose interpretability.

A Concrete Example

Suppose you build a regression model that predicts ice cream sales from two features: Celsius temperature AND Fahrenheit temperature. Both features are essentially the same information measured in different units. The correlation between them is 1.0. Including both creates a textbook multicollinearity problem — the model becomes unstable, even though the prediction quality may seem fine. The fix: drop one feature.

// 18
Hidden Dangers of Correlation
// WHEN CORRELATION LIES

Four Ways Correlation Can Fool You

A. Outliers

One extreme value can fake a strong correlation or destroy a real one. Always plot the data and check for influential points.

B. Nonlinear Patterns

Pearson can completely miss important relationships that are curved. The correlation number may be near zero while a strong U-shape or curve actually exists.

C. Aggregation Bias (Simpson's Paradox)

When you combine data across subgroups, the overall correlation can be the opposite direction from the correlation within each subgroup. This is the famous Simpson's Paradox — a single dataset can show a positive correlation overall but a negative one for every subgroup, or vice versa.

D. Reverse Causality

Does stress cause poor sleep? Or does poor sleep cause stress? Correlation alone cannot tell. Often both run together, and the direction of causality remains unclear without proper experiments.

// 19
Establishing Real Causation
// WHAT REAL CAUSATION REQUIRES

If Correlation Alone Isn't Enough — What Is?

Proving causation requires much stronger evidence than correlation. Four main approaches are used:

  • Controlled experiments — change one variable while keeping all else constant; observe the effect on the other.
  • Randomized trials (RCTs) — randomly assign subjects to treatment and control groups. The gold standard.
  • Causal inference methods — modern statistical tools like instrumental variables, propensity matching, and Pearl's do-calculus.
  • Domain knowledge — understanding the underlying physical, biological, or economic mechanism through which one variable can actually cause the other.

Correlation alone — without one of these — is never enough to claim causation.

Causal Thinking in AI & Data Science

Modern AI increasingly needs causal reasoning — not just pattern recognition. Why? Because predictive models built only on correlation can break when the environment changes. A model trained that "people on the website 30 minutes are likely to buy" may completely fail when the company forces users to spend 30 minutes through gimmicks.

Correlation-based models break under distribution shifts. Causal understanding improves robustness, interpretability, and generalization. This is one of the most active research areas in modern AI.

Common Beginner Mistakes
// MISTAKE 01
"Believing high correlation proves causation"
This is THE most common error. A high correlation only tells you variables move together — never that one causes the other.
FIX: Before claiming causation, ask: could a third variable be driving both? Could the direction be reversed? Is there an experiment confirming it?
// MISTAKE 02
"Ignoring scatter plots"
A correlation coefficient is just a single number. It can hide outliers, curves, clusters, and broken assumptions.
FIX: Plot the scatter first. Always. The chart tells you what r cannot.
// MISTAKE 03
"Using Pearson on nonlinear data"
Pearson can return ≈0 even when a strong curved relationship exists. The conclusion "no relationship" would be completely wrong.
FIX: Visually check for curves. If the relationship looks monotonic but curved, use Spearman instead.
// MISTAKE 04
"Ignoring outliers"
A single extreme point can take a true correlation of 0 and inflate it to 0.9, or take a true 0.8 and crush it to 0.
FIX: Investigate every outlier. Recompute correlation with and without it. Report both numbers if it matters.
// MISTAKE 05
"Assuming zero r means no relationship"
Pearson r near zero only means no linear relationship. Strong nonlinear relationships can still hide there.
FIX: Always pair r with a scatter plot. Try Spearman too. Look for U-shapes, plateaus, and curves.
// MISTAKE 06
"Forgetting confounding variables"
Without thinking about possible third variables, every observed correlation might be lying to you.
FIX: Ask yourself: "What hidden variable could explain both X and Y?" — and either measure it or design an experiment.

This module gave you the tools to study relationships between variables — and the wisdom to never confuse "moving together" with "causing." You learned that correlation measures association in three dimensions: direction, strength, and consistency. You met the four classic relationship types — positive, negative, zero, nonlinear — and learned that the scatter plot is the foundation of all relationship analysis. You mastered Pearson r as the world's most-used measure of linear association (between −1 and +1) and Spearman ρ as the rank-based champion for monotonic, nonlinear, or outlier-heavy data. You internalized the single most important lesson in all of statistics: correlation ≠ causation. You learned about confounders (hidden variables driving both X and Y), spurious correlations (meaningless coincidences from noise), multicollinearity (the redundant-feature trap in ML), and correlation matrices (the foundation of feature engineering). Real causation requires controlled experiments, randomized trials, or careful causal inference — never correlation alone.
KEY // 01
Correlation = Association
Measures how variables move together. Direction + strength + consistency.
KEY // 02
Pearson r ∈ [−1, +1]
Measures LINEAR relationships. Unit-free. Sensitive to outliers and curves.
KEY // 03
Spearman ρ
Rank-based. Captures monotonic relationships. Robust to outliers.
KEY // 04
Correlation ≠ Causation
The single most important rule. Always ask about confounders and reverse causality.
KEY // 05
Confounders
Hidden variables that drive both X and Y. Create fake-looking relationships.
KEY // 06
Correlation Matrices
All pairwise correlations at once. Foundation of feature engineering in ML.
// FORMULA RECAP
COVARIANCE:    Cov(X,Y) = (1/n) · Σ(xᵢ − x̄)(yᵢ − ȳ)
PEARSON r:    r = Cov(X,Y) / (σ_X · σ_Y)
SPEARMAN ρ:    Pearson r applied to RANKS
RANGE:    both r and ρ ∈ [−1, +1]
// FINAL INTUITION
Correlation answers: "Do these variables move together?"
Causation answers: "Does one variable actually produce changes in the other?"
Those are NOT the same question.
