Cumulative Distribution Function (CDF)
What a CDF is

The Universal Probability Function

The cumulative distribution function, or CDF, written F(x) or Fₓ(x), is the most powerful single description of any random variable. It answers a single question — but a question rich enough to encode every probabilistic fact about X:

F(x) = P(X ≤ x) — "the probability that X is at most x"

That's it. For every possible value x, F(x) tells you how much probability has "accumulated" from −∞ up to x. As x increases, F(x) gathers more and more probability. When x reaches +∞, F(x) reaches 1 — all probability is accumulated.

The CDF is the rare object in probability that works identically for discrete and continuous random variables. Whether X is a coin flip or a normal random variable, F(x) = P(X ≤ x) is well-defined and has the same meaning.

⟁ Cumulative Distribution Function — Universal Definition
F(x) = P(X ≤ x)

A CDF satisfies four universal properties — exact rules that hold for every random variable, discrete or continuous:

①   0 ≤ F(x) ≤ 1   for every x
②   F is non-decreasing:   if a ≤ b then F(a) ≤ F(b)
③   lim ₓ→ −∞ F(x) = 0     lim ₓ→ +∞ F(x) = 1
④   F is right-continuous:   F(x) = limₕ→0⁺ F(x + h)

Why each property? ① F is a probability, bounded in [0,1]. ② As you allow more values, the accumulated probability can only grow or stay flat — never shrink. ③ Nothing is below −∞; everything is at or below +∞. ④ A subtle property ensuring jumps at discrete values are counted "from the right" — formally distinguishing P(X=x) from P(X<x).

CDF for discrete variables

How CDFs Look for Discrete RVs — Staircase Functions

For a discrete random variable, F(x) is built by summing the PMF up to and including x:

⟁ CDF from PMF (Discrete)
F(x) = Σ pₓ(k)   for all k ≤ x

Start at the smallest value. F is flat at 0 until you reach the first value of the RV. There, F jumps up by p(x₁). It stays flat until the next value, then jumps by p(x₂). And so on. The result is a staircase function — a sequence of horizontal steps connected by vertical jumps.

Critical property: The jump height at each value x equals P(X = x) = the PMF at that point. So you can read off the PMF directly from the size of each step. PMF and CDF carry the same information — just expressed differently.

CDF for continuous variables

How CDFs Look for Continuous RVs — Smooth Curves

For a continuous random variable, F(x) is built by integrating the PDF up to x:

⟁ CDF from PDF (Continuous)
F(x) = ∫₋∞^x f(t) dt

As x grows, more area under the PDF accumulates. The CDF is the running total area. It starts at 0, increases smoothly (no jumps for continuous RVs), and approaches 1 as x → ∞.

Inverse relation — derivative gives back the PDF:

f(x) = dF(x) / dx

This is the fundamental theorem of calculus applied to probability. The PDF is the derivative of the CDF. Wherever the CDF is rising steeply, the density is high. Wherever the CDF is flat, the density is near zero. Reading these two together is the deepest visual skill in probability.

Why CDFs unify everything

The Unification Power — One Tool, Both Worlds

Here is what makes the CDF profound: every random variable has a CDF. There are random variables that have neither a PMF (because they're not discrete) nor a PDF (because their distribution has gaps). But they still have a CDF — and probabilities can still be computed from it.

Three universal properties make the CDF the workhorse of probability:

  • Universal definition. F(x) = P(X ≤ x) makes sense for any RV — discrete, continuous, mixed, or even pathological.
  • Universal access. Once you know F, you can compute P(X ≤ a), P(X > a), P(a < X ≤ b), and any other probability — all by simple arithmetic on F.
  • Universal inversion. The quantile function Q(p) = F⁻¹(p) gives you the value of X corresponding to probability p — the basis of statistics, simulation, and Monte Carlo methods.
⟁ Computing Probabilities From the CDF
P(X ≤ a) = F(a)
P(X > a) = 1 − F(a)
P(a < X ≤ b) = F(b) − F(a)
P(X = a) = F(a) − F(a⁻)   (jump height; 0 for continuous)

All four lines work for every random variable. Once you have F, you have the entire probabilistic description of X. This is why textbooks define distributions like the Normal, Beta, or Gamma directly through their CDFs — they're the most general representation.

The quantile / inverse CDF

The Inverse CDF — Going Backwards From Probabilities

If F maps "value of X" to "cumulative probability," its inverse F⁻¹ (called the quantile function) maps "cumulative probability" to "value of X":

⟁ Quantile Function (Inverse CDF)
Q(p) = F⁻¹(p) = smallest x such that F(x) ≥ p

Practical meaning: Q(0.5) is the median. Q(0.95) is the 95th percentile. Q(0.99) is the 99th percentile (Value-at-Risk in finance). Quantiles are how the real world communicates distributions — "the 95th-percentile latency," "the 90th-percentile salary," "the 5% tail risk."

Critical use — Inverse Transform Sampling: If U is uniform on [0,1], then X = F⁻¹(U) has CDF F. This is how computers generate random samples from any distribution — the foundation of every Monte Carlo simulation ever run.

When to use the CDF

Decision Guide — Reach for the CDF When…

  • You need a single uniform tool that handles discrete, continuous, or mixed RVs.
  • The question is "what's the probability of being at most / at least / between?"
  • You need to find percentiles, quantiles, or risk thresholds (financial VaR, latency percentiles, etc.).
  • You want to generate random samples from a distribution computationally (inverse-transform sampling).
  • You're working with a distribution whose CDF has a closed form but PDF doesn't (some Beta variants, Cauchy tails, etc.).

Reach instead for the PMF or PDF when:

  • You need to visualize the shape of the distribution (peaks, modes, multi-modality).
  • You need the relative likelihood at specific values for Bayesian inference or maximum likelihood estimation.
  • You're working with calculus operations on density directly (entropy ∫f log f, divergence, etc.).

Real-World Applications

Finance — Value at Risk FIN

VaR is literally a CDF inverse: "the loss exceeded only 5% of the time." Q(0.05) on the loss distribution defines bank capital requirements worldwide.

SLO & Latency Tracking OPS

"P99 latency" = Q(0.99) of response-time CDF. Google, Netflix, Amazon all use CDFs to monitor service quality. CDFs directly drive SLA design.

ML Classification ML

The softmax output IS a PMF over classes. Argmax = the mode. ROC curves are CDF-based: each point is a (false-positive rate, true-positive rate) pair indexed by threshold.

A/B Testing BIZ

P-values, confidence intervals, and test statistics all come from CDFs of test distributions (t, F, chi-squared). Statistical inference is CDF computation.

Monte Carlo Simulation SIM

Inverse-transform sampling — X = F⁻¹(U) — is how computers generate random variables. Every physics simulation, RL training, and game AI uses this.

Quality Control MFG

Cp / Cpk indices use the CDF of part measurements to quantify how often tolerances are violated. Six Sigma is literally about controlling tail probabilities.

Insurance Pricing INS

Claim-size distributions are described by CDFs. Reinsurance kicks in at high quantiles; pricing models compute expected payouts via integrating the CDF tail.

Reliability Engineering ENG

The "lifetime CDF" F(t) tells you the probability of failure by time t. The survival function S(t) = 1 − F(t) tells you reliability — a $billion industry built on CDFs.

Bayesian Posteriors AI

Credible intervals are quantiles of the posterior CDF. "There is a 95% credible interval [a, b]" means F_posterior(b) − F_posterior(a) = 0.95.

Common Mistakes (CDF Edition)
01
Confusing F(x) with f(x). The CDF F is always between 0 and 1. The PDF f can exceed 1. They are NOT the same function — F is cumulative, f is local.
02
Forgetting the staircase property for discrete RVs. The discrete CDF is flat between values and jumps at each possible value of X. It's not smooth — it has corners and jumps.
03
Using P(a < X ≤ b) = F(a) − F(b). Wrong order! The correct formula is F(b) − F(a). Always subtract the smaller-x value from the larger.
04
Thinking F(x) can decrease. Impossible. The CDF is monotonically non-decreasing — once accumulated, probability never un-accumulates. If your computed CDF goes down somewhere, you made an arithmetic error.
05
Inverting the wrong way. F maps value → probability; F⁻¹ maps probability → value. Don't flip them. "What value has F = 0.95?" is a quantile question = F⁻¹(0.95), not F(0.95).
06
Forgetting limits. Many students write F(x) without specifying its behavior at ±∞. Always remember: F(−∞) = 0, F(+∞) = 1. These limits define a valid CDF.
07
For continuous RVs, mixing up F(x) and P(X < x). For continuous RVs they're the same since P(X = x) = 0, but for discrete RVs F(x) = P(X ≤ x) includes the value x while P(X < x) does not. Be careful.
08
Trying to compute PMFs from CDFs by differentiation. Only works for continuous CDFs. For discrete, you compute P(X=k) = F(k) − F(k⁻) — the jump height.
