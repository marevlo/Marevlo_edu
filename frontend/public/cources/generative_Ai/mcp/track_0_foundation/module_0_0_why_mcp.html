01  The blind spot

A language model is a pure function.

At inference time, a model is the map tokens_in → tokens_out. It owns no process, no file descriptor, no socket, no clock. Every claim it makes about the world is either (a) compressed into its weights during training, or (b) supplied in the prompt at inference time. There is no third source.

This sounds obvious, but the consequences are sharp. Out of the box, a model cannot:

  • Read a row from your Postgres database
  • List the files in your home directory
  • Tell you whether a Slack message arrived 30 seconds ago
  • Know today's exchange rate, today's weather, or today's date in your timezone

Even Retrieval-Augmented Generation does not change this. RAG is a polite name for "something outside the model fetches text and pastes it into the prompt." The model itself stays inert. Whatever the model is going to do in the world — something else has to actually do the doing. That something else is what MCP standardises.

02  The N × M problem

Why every team kept rebuilding the same glue.

Suppose you have M = 5 AI clients (Claude Desktop, Cursor, an in-house agent, a copilot in Slack, an automation worker) and N = 20 backends you'd like all of them to reach (GitHub, Jira, Postgres, S3, an internal wiki, a payments API, …).

Without a protocol you write M × N = 100 bespoke integrations. Each has its own auth, its own tool-schema convention, its own error shape, and its own way of rotting when an upstream API changes. Worse, the glue isn't portable: the connector you wrote for one client cannot be lifted into another without a rewrite.

The MCP move

Each client speaks the wire protocol once. Each backend is wrapped in an MCP server once. The count collapses from M × N to M + N. Capability negotiation at connect-time means clients discover tools dynamically — they don't have to be recompiled to learn new ones.

The phrase you'll hear is "USB-C for AI." It's marketing, but the analogy is structurally accurate: a single physical/logical interface that any device can speak, with a negotiated capability layer on top. The interesting work moves up the stack — to what tools should exist, who's allowed to call them, and what happens when they fail.

03  Vocabulary

Three nouns you will use a thousand times.

The spec is careful about these. Mixing them up is the #1 source of confusion in MCP threads — get the distinction now and the rest of the course reads cleanly.

Host
The user-facing app. Owns the LLM connection, the UI, and the permission surface. Examples: Claude Desktop, Cursor, your custom agent.
Client
Lives inside the host. The host spawns one client per connected server. Clients speak JSON-RPC on the host's behalf — one TCP-like connection each.
Server
The program that exposes tools, resources, and prompts. Local (stdio) or remote (HTTP). Servers do not call the LLM — they only expose capabilities.

And three primitives the protocol cares about most:

  • Tools — model-invoked actions (search_issues, create_pr). The LLM picks when.
  • Resources — application-controlled context (files, query rows). The host decides what to surface.
  • Prompts — user-invoked templates, typically slash-commands. A human triggers them.

Who is in the driver's seat — model, host, or user — is the axis these three primitives separate on. Track 2 treats each one in its own module.

04  Timeline

From announcement to stateless core in eighteen months.

MCP moves quickly. The dates aren't trivia — knowing which spec version introduced what saves you from copy-pasting examples that no longer match the wire.

Nov 25 · 2024
Anthropic announces MCP. Initial spec, Python + TypeScript SDKs, reference servers for GitHub, Slack, Postgres, filesystem, Puppeteer. stdio is the only transport.
Mar · 2025
Spec revision adds OAuth 2.1 as the standard auth for remote servers. Streamable HTTP is introduced; legacy HTTP+SSE enters deprecation.
Mid · 2025
OpenAI, Microsoft, and Google ship first-class MCP support in their agent stacks. The protocol is no longer an Anthropic-only thing.
Late · 2025
Governance shifts toward a neutral foundation. Reverse-DNS extension IDs and a formal SEP process (Spec Enhancement Proposals) land.
2026 · "stateless era"
Stateless core: servers stop holding per-session state by default — state lives in explicit handles the client passes on each request. Two new primitives ship: MCP Apps (server-rendered HTML in a sandboxed iframe) and Tasks (long-running ops with progress, cancel, polling).
