02 · The Problem
Why RAG Exists

LLMs are powerful — but they are frozen, forgetful, and hallucination-prone. RAG fixes the three fatal flaws that make raw LLMs unusable in production.

01

Frozen Knowledge

LLMs are trained on data up to a fixed date. They cannot know about yesterday's news, your company's product launch this morning, or your private documentation.

→ FIX: Retrieve fresh data at query time
02

Hallucinations

When the model doesn't know, it invents answers — confidently. This makes raw LLMs dangerous for legal, medical, financial, and enterprise applications.

→ FIX: Ground responses in retrieved evidence
03

No Private Data

Your company's wikis, contracts, codebases, customer data — none of it exists in the LLM's brain. Fine-tuning is slow and expensive. RAG is instant.

→ FIX: Index private knowledge in a vector store
03 · Mental Model
Closed-Book VS Open-Book

The single most important mental model in this course

Mode A

Raw LLM

Like a student taking a closed-book exam. They answer from memory — and if they don't know, they guess.

  • Knowledge frozen at training time
  • Cannot cite sources
  • Hallucinates when uncertain
  • No access to private data
Mode B — RAG

LLM + Retrieval

Like a student taking an open-book exam. Before answering, they look up the relevant pages — and cite them.

  • Fresh, queryable knowledge
  • Every answer is citable
  • Refuses gracefully when no evidence
  • Works on YOUR data

05 · Code Preview
A Naive RAG in 30 Lines

You'll build this in Mini-Capstone 0. Don't worry if it looks foreign — by Module 0.6 every line will make sense.

Naive RAG SetupPython
# Naive RAG — the seed of every production system
from openai import OpenAI
from qdrant_client import QdrantClient

client = OpenAI()
qdrant = QdrantClient("localhost", port=6333)

def embed(text):
    return client.embeddings.create(model="text-embedding-3-small", input=text).data[0].embedding

def retrieve(query, k=5):
    hits = qdrant.search("docs", embed(query), limit=k)
    return [h.payload["text"] for h in hits]

def ask(question):
    context = "\n\n".join(retrieve(question))
    prompt = f"Answer using ONLY this context:\n{context}\n\nQ: {question}"
    return client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}]
    ).choices[0].message.content

print(ask("What is our refund policy?"))

⚡ Engineering reality: This naive version works for a demo. It fails in production. The next 38 modules teach you why — and how to fix every failure mode that makes real RAG systems hard.
