02 · Step 1 · Setup
Install Python 3.11+ & uv

We need Python 3.11 or newer for modern type-hints, performance gains, and library compatibility. For dependency management we use uv — Astral's blazing Rust-based replacement for pip, virtualenv, and pyenv combined. It's the same tool used by teams at Hugging Face, Modal, and major AI startups in 2026.

Why uv instead of pip? uv resolves and installs a typical RAG stack (langchain, qdrant-client, openai, cohere, sentence-transformers) in ~3 seconds vs pip's ~45 seconds. It also handles virtual environments, Python versions, and lockfiles in one tool — replacing pip + venv + pip-tools + pyenv. You'll thank yourself in Module 2 when you're installing 30+ packages.

cadet@macbook ~ • zsh
# 1. Install Homebrew if you don't have it (skip if installed)
❯ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 2. Install Python 3.12 via Homebrew
❯ brew install python@3.12

# 3. Install uv (the fast package manager we'll use everywhere)
❯ curl -LsSf https://astral.sh/uv/install.sh | sh

# 4. Verify
❯ python3 --version
Python 3.12.7
❯ uv --version
uv 0.5.4
cadet@ubuntu ~ • bash
# Ubuntu / Debian (22.04+)
$ sudo apt update
$ sudo apt install -y python3.12 python3.12-venv python3-pip

# Install uv
$ curl -LsSf https://astral.sh/uv/install.sh | sh
$ source $HOME/.local/bin/env

# Fedora / RHEL / CentOS
$ sudo dnf install -y python3.12 python3-pip

# Verify
$ python3.12 --version && uv --version

cadet@windows • PowerShell
# 1. Install Python via winget (Windows 10/11)
PS> winget install Python.Python.3.12

# 2. Install uv
PS> powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# 3. CLOSE and REOPEN PowerShell so PATH refreshes, then verify

PS> uv --version

⚡ Windows tip: Strongly recommend WSL2 (Windows Subsystem for Linux) for AI development. Many libraries (FAISS, sentence-transformers, llama-cpp) work more reliably on Linux. Install with: wsl --install — then follow the Linux instructions inside it.

03 · Step 2 · Isolation
Create Your Project Environment

A virtual environment is a sealed bubble of Python + dependencies for one project. Why does this matter? Because your RAG project will pin specific versions of langchain, openai, qdrant-client, etc. Other projects on your machine will pin different versions. Without isolation, they break each other. This is the #1 source of "it worked yesterday" bugs in AI engineering.

Project Setup

~/projects • initializing rag-academy
# Create the project directory you'll use for the entire course

# Initialize a uv project (creates pyproject.toml + .python-version)
❯ uv init --python 3.12

# Create the virtual environment

# Activate it

❯ .venv\Scripts\activate # Windows

🚨 Real-world horror story: In 2024 alone, GitGuardian detected over 23 million leaked secrets on public GitHub. OpenAI keys are stolen by bots within ~60 seconds of being pushed publicly. Always commit .gitignore before creating .env.

Loading .env in Python

In every script and notebook, this two-line pattern reads your keys safely:

from dotenv import load_dotenv
import os

load_dotenv()  # reads .env into environment variables

# Now your keys are available via os.environ
openai_key = os.environ["OPENAI_API_KEY"]

# SDKs auto-detect environment variables too:
from openai import OpenAI
client = OpenAI()  # picks up OPENAI_API_KEY automatically
06 · Step 5 · Vector DB
Launch Qdrant Locally

Qdrant is our course-default vector database — open-source, blazing fast (Rust-based), production-proven at companies like Bayer, Bosch, and Disney, and supports everything we need: HNSW, hybrid search, payload filtering, and quantization. We run it locally via Docker so you can iterate offline and own your data.

Why local Qdrant for learning? Managed vector DBs (Pinecone, Weaviate Cloud) charge per query and require network round-trips. For a course where you'll re-index a 500-doc corpus dozens of times across mini-capstones, that means cost and latency. Local Qdrant is free, instant, and behaves identically to its managed counterpart — same client, same API. We graduate to managed deployments in Track 7.

07 · Step 6 · Validation
The "Hello Embedding" Verification

Time to confirm everything works end-to-end. This script does three things in 20 lines: (1) loads your API keys, (2) calls OpenAI to embed two sentences, and (3) stores them in Qdrant and runs a similarity search. If all three succeed — your environment is production-ready.

src/m0_foundations/hello_embedding.py 
# hello_embedding.py — verifies your RAG environment
from dotenv import load_dotenv
from openai import OpenAI
from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams, PointStruct

load_dotenv()
client = OpenAI()
qdrant = QdrantClient(url="http://localhost:6333")

# 1. Create a collection (1536d = OpenAI embedding-3-small dim)
qdrant.recreate_collection(
    collection_name="hello",
    vectors_config=VectorParams(size=1536, distance=Distance.COSINE),
)

# 2. Embed two sentences
sentences = [
    "RAG gives LLMs access to private knowledge.",
    "Cats are small carnivorous mammals.",
]
embeddings = client.embeddings.create(
    model="text-embedding-3-small",
    input=sentences,
).data

# 3. Upsert into Qdrant
qdrant.upsert(
    collection_name="hello",
    points=[PointStruct(id=i, vector=e.embedding, payload={"text": sentences[i]})
            for i, e in enumerate(embeddings)],
)

# 4. Query: which sentence is closest to "What is retrieval-augmented generation?"
query_emb = client.embeddings.create(
    model="text-embedding-3-small",
    input="What is retrieval-augmented generation?",
).data[0].embedding

hits = qdrant.search(collection_name="hello", query_vector=query_emb, limit=2)
for h in hits:
    print(f"score={h.score:.3f}  →  {h.payload['text']}")

Run It

running verification
(rag-academy) ❯ uv run python src/m0_foundations/hello_embedding.py

score=0.842 → RAG gives LLMs access to private knowledge.
score=0.231 → Cats are small carnivorous mammals.

(rag-academy) ❯

If you see the RAG sentence win with a high score (~0.84) and the cat sentence score very low, congratulations! You have successfully created your vector store, retrieved embeddings from OpenAI, stored them in Qdrant, and performed a cosine similarity query.

08 · Troubleshooting
Troubleshooting Console

Stuck on configuration? Check these standard solutions for the most common environment issues.

'uv' is not recognized as an internal or external command

The shell needs to be refreshed to pick up PATH changes. Close all terminals, open a fresh command prompt/powershell, and test again. On Windows, verify that %USERPROFILE%\.local\bin is in your User Environment Variables.

python-dotenv loads empty/None keys

Double check that the file is literally named .env, NOT env.txt or .env.txt. Windows sometimes hides extensions. Run ls -la in bash or Get-ChildItem -Force in PowerShell to confirm the dotfile is clean.

Docker port collision (6333 already in use)

You probably have an old Qdrant container running. Shut it down and remove it first:

❯ docker stop qdrant && docker rm qdrant
OpenAI API RateLimitError or InsufficientQuotaError

Free-tier trials expire after 90 days. Check your usage page on platform.openai.com. You may need to add a credit card to billing (even a $5 credit will fix this instantly).

Windows execution policy blocks virtual environment activation

PowerShell prevents scripts from running by default. Enable it for your current user:

PS❯ Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
.env keys not loading in Jupyter notebook

Jupyter runs from the notebook's directory by default — not your project root. Force the path:

from dotenv import load_dotenv, find_dotenv
load_dotenv(find_dotenv())  # walks up the tree until it finds .env
