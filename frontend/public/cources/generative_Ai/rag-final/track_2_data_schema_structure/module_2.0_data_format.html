JSON JavaScript Object Notation

The universal exchange format. Strict syntax, ubiquitous tooling, native to every language and every API on Earth.

✓Universal language support
✓Strict, deterministic parsing
✗No comments (without extensions)
✗Verbose for config files
~Trailing commas forbidden
⚡ Use it for: API request/response bodies, LLM tool outputs, anything moving over the wire. 
# rag_config.yaml — human-friendly, comments welcome
project: customer-support-rag
version: "1.0.0"  # quoted to stay a string

embedding:
  provider: openai
  model: text-embedding-3-small
  dimensions: 1536
  batch_size: 100

retrieval:
  top_k: 5
  hybrid: true
  alpha: 0.6
  rerank: false

sources:
  - ./docs/handbook
  - ./docs/faq
  - ./docs/tickets
YAML YAML Ain't Markup Language

Indentation-driven, human-first. The default config language of Kubernetes, GitHub Actions, Ansible, and most ML training scripts.

✓Readable, comments allowed
✓Less visual noise
✓Anchors & aliases for DRY
✗Whitespace-sensitive (footgun)
✗Implicit typing surprises
⚡ Use it for: Pipeline configs, CI/CD workflows, Kubernetes manifests, ML hyperparameters. 
# rag_config.toml — obvious to humans, unambiguous for machines
project = "customer-support-rag"
version = "1.0.0"

[embedding]
provider = "openai"
model = "text-embedding-3-small"
dimensions = 1536
batch_size = 100

[retrieval]
top_k = 5
hybrid = true
alpha = 0.6
rerank = false

sources = [
  "./docs/handbook",
  "./docs/faq",
  "./docs/tickets",
]
TOML Tom's Obvious Minimal Language

The format that powers pyproject.toml and Rust's Cargo. Explicit, type-safe, and immune to YAML's footguns.

✓Unambiguous types
✓Comments natively supported
✓Stdlib parser in Python 3.11+
✗Deeply nested structures get clunky
✗Smaller ecosystem than YAML
⚡ Use it for: Project metadata, package configs, app settings — anything you'd put in pyproject.toml. 
# chunks.jsonl — one JSON object per line, streamable, append-only
{"id":"chunk_001","text":"Refund policy: returns within 30 days...","source":"faq.md"}
{"id":"chunk_002","text":"Shipping: orders ship within 2 business days...","source":"faq.md"}
{"id":"chunk_003","text":"Warranty terms: 1-year limited warranty on...","source":"warranty.md"}
{"id":"chunk_004","text":"Contact: support@example.com 24/7 chat...","source":"contact.md"}
{"id":"chunk_005","text":"Account: reset password via email link...","source":"account.md"}
# ... 2.4 million more lines
JSONL JSON Lines / NDJSON

One JSON object per line. The undisputed king of data corpora, OpenAI fine-tuning files, and ML training pipelines.

✓Streamable line-by-line
✓Append-only friendly
✓One corrupt line ≠ whole file lost
✗Not for nested config
✗Easy to break with newlines in values
⚡ Use it for: RAG chunks, embedding corpora, fine-tuning data, event logs, anything streamed.

Choose Wisely

The Decision Tree

When you're staring at a new file and don't know which format to use — answer these in order.

Question 01

Is this data moving over a network?

REST APIs, webhooks, LLM tool calls, function I/O — anything serialized between processes.

→ YES: JSON always
Question 02

Is this a corpus of many records?

RAG chunks, embedding training data, fine-tuning files, event logs, anything that grows row by row.

→ YES: JSONL stream it
Question 03

Is this Python project metadata?

Dependencies, build config, tool settings — the file at the root of your repo.

→ YES: TOML pyproject.toml
Question 04

Is this a pipeline / workflow config?

CI/CD, Kubernetes, Docker Compose, ML training hyperparameters, Airflow DAGs.

→ YES: YAML — but be careful
⚡ When in doubt: reach for JSON. It's never the most elegant choice, but it's never the wrong choice. Every team, every language, every parser supports it. The boring answer is the safe answer.
The Footguns

Common Pitfalls

Every format has traps. These are the ones that have shipped bugs to production for someone, somewhere — and might do the same to you.

YAML The Norway Problem

In YAML 1.1, the country code NO parses as the boolean false. Same trap with YES, ON, OFF.

country: NO        # → False (boolean!)
country: "NO"      # → "NO" (correct)

Fix: Always quote string-typed values that could look like keywords. YAML 1.2 is stricter, but most parsers default to 1.1.

JSON Trailing Commas Forbidden

JSON forbids a trailing comma after the last element. Most languages allow it in source code — JSON does not.

{
  "a": 1,
  "b": 2,    // ← syntax error
}

Fix: Use a linter, or switch to JSON5/JSONC if you absolutely need flexibility.

YAML Tabs vs Spaces

YAML forbids tabs for indentation. One stray tab character — invisible in most editors — and your whole file is invalid.

embedding:
→provider: openai  # tab = error
  provider: openai  # spaces = ok

Fix: Configure your editor to insert 2 spaces on Tab, always.

JSONL Newlines Inside Values

JSONL relies on \n as the record boundary. A literal newline inside a string field shatters the format.

{"text": "line one
line two"}  # BROKEN
{"text": "line one\nline two"}  # ok

Fix: Always use json.dumps() (Python) or equivalent — never hand-build JSONL.

YAML Octal Numbers

A version like 012 gets parsed as octal — meaning 10. Common landmine in port numbers and ZIP codes.

zip: 02134      # → 1116 (octal!)
zip: "02134"    # → "02134" (string)

Fix: Quote any number with leading zeros, or upgrade to YAML 1.2.

JSON No Native Date Type

JSON has no date type — dates are strings by convention. Different consumers parse them differently. Always specify the format you expect.

{
  "created": "2026-05-17T09:30:00Z"
  // ISO 8601 — the only sane choice
}

Fix: Standardize on ISO 8601 with UTC timezone. Document it.

TOML Deep Nesting Gets Ugly

TOML's table syntax is elegant for 2-3 levels — but stops working for deeply nested structures common in RAG configs.

[stages.retrieval.hybrid.weights]
bm25 = 0.4
dense = 0.6
# readable until you hit 5+ levels

Fix: If you're 4+ levels deep, switch to YAML or JSON.

JSONL Encoding Mismatches

RAG corpora often contain emoji, accents, CJK characters. Open a JSONL file as latin-1 instead of utf-8 and your data corrupts silently.

open("corpus.jsonl", encoding="utf-8")
# always — never assume the default

Fix: Always pass encoding="utf-8" explicitly on open.

Hands-On

Read & Write All Four

The exact Python snippets you'll reach for hundreds of times. Bookmark this section.

JSON Read & Write 
import json

# Read
with open("config.json", encoding="utf-8") as f:
    config = json.load(f)

# Write (pretty)
with open("config.json", "w", encoding="utf-8") as f:
    json.dump(config, f, indent=2, ensure_ascii=False)
YAML Read & Write 
import yaml  # pip install pyyaml

# Read — ALWAYS use safe_load
with open("config.yaml", encoding="utf-8") as f:
    config = yaml.safe_load(f)

# Write
with open("config.yaml", "w", encoding="utf-8") as f:
    yaml.safe_dump(config, f, sort_keys=False)
⚠ Security: Never use yaml.load() on untrusted input — it can execute arbitrary code. safe_load always.
TOML Read & Write 
import tomllib      # Python 3.11+ stdlib (read only)
import tomli_w      # for writing: pip install tomli-w

# Read — note: binary mode
with open("pyproject.toml", "rb") as f:
    config = tomllib.load(f)

# Write
with open("out.toml", "wb") as f:
    tomli_w.dump(config, f)
JSONL Stream & Append 
import json

# Stream read — never load whole file
with open("corpus.jsonl", encoding="utf-8") as f:
    for line in f:
        record = json.loads(line)
        process(record)

# Append a new chunk
with open("corpus.jsonl", "a", encoding="utf-8") as f:
    f.write(json.dumps(chunk, ensure_ascii=False) + "\n")
⚡ One more pro tip: Always pass ensure_ascii=False when writing — without it, your emoji and Unicode get mangled into \u003c-style escapes that are technically valid but unreadable.
Self-Check

Three Quick Questions

Test your instincts before moving on.

QUESTION 01

You need to ship 2 million RAG chunks from a preprocessing job to a vector DB loader. Best format?

Correct. JSONL lets you stream chunks line-by-line — no need to load 2M records into memory, append-friendly, and one corrupt line doesn't kill the whole file.
QUESTION 02

Your YAML config has country: NO. What happens?

Correct — the Norway Problem. YAML 1.1 treats NO, YES, ON, OFF as booleans. Always quote string values that look like keywords.
QUESTION 03

An LLM tool call needs to return structured data to your application. Best format?

Correct. Anything moving over a network or between processes wants JSON. Every LLM provider, every web API, every SDK speaks JSON natively.
