Mission Objectives

What You'll Build

A production-grade ingestion pipeline — idempotent, resumable, observable, and fast enough for millions of documents.

The 6 Stages

Discover → Load → Clean → Dedupe → Enrich → Persist. Each stage is its own concern.

Source Loaders

PDF, HTML, Markdown, Notion, S3, Confluence, GitHub — one loader per source type.

Metadata Discipline

Capture source, url, timestamp, author — citations depend on it.

Idempotent & Resumable

Re-runs don't duplicate. Failures don't restart from zero. Content hashing is your friend.

Stage 2 · Load

Source Loaders

Each source format needs a loader that knows its quirks. Pick the right one — or pay in bad chunks downstream.

PDF

Use pypdf, pdfplumber, or unstructured.

⚠ OCR for scanned PDFs.

HTML

trafilatura for content extraction, BeautifulSoup for surgical parsing.

⚠ Strip nav, ads, footers.

Markdown

markdown-it-py or mistune. Headers carry structure.

✓ Best for chunking by header.

DOCX / PPTX

python-docx, python-pptx. Preserve heading hierarchy.

⚠ Tables need special handling.

S3 / GCS / Azure

boto3, google-cloud-storage. Stream in, don't download first.

✓ Use fsspec for uniformity.

Notion

Official API + notion-client. Recurse into child pages.

⚠ Rate limits — backoff.

GitHub

Clone or use the API. Filter by .md, .rst, code files.

✓ Use .gitignore patterns.

Confluence / Jira

Atlassian REST APIs. Map projects/spaces to namespaces.

⚠ Permission propagation matters.

Whatever the source: produce ONE shape

Every loader normalizes its output into a single Document Pydantic model. From this point on, the pipeline doesn't care where the data came from.

from pydantic import BaseModel, Field
from datetime import datetime

class Document(BaseModel):
    """Canonical post-load shape — same for every source."""
    doc_id:      str                           # stable id (content hash)
    source:      str                           # "notion", "s3", "github"
    uri:         str                           # original location
    text:        str                           # cleaned plain text
    title:       str | None = None
    author:      str | None = None
    created_at:  datetime | None = None
    updated_at:  datetime | None = None
    tags:        list[str] = Field(default_factory=list)
    extra:       dict = Field(default_factory=dict)  # source-specific blob
Stages 3-4 · Clean & Dedupe

Cleaning & Deduplication

Bad data in, bad answers out. Cleaning is unglamorous, but it's where you save the most production pain.

1

Normalize encoding

UTF-8

Force everything to UTF-8. Decode latin-1 fallbacks. Replace smart quotes, em-dashes, NBSPs with their ASCII equivalents — embedding models are funky about Unicode noise.

2

Strip boilerplate

BOILERPLATE

Headers, footers, navigation, "click here to subscribe" — content the user didn't ask about. trafilatura for HTML; regex / custom rules for PDFs.

3

Collapse whitespace

WHITESPACE

Runs of 5+ blank lines → one. Tabs → spaces. Lines that are just punctuation → drop. Each kept character should earn its embedding cost.

4

Hash for identity

SHA-256

Compute a content hash after cleaning. This is the doc_id — stable across re-runs. If the hash is unchanged, the document doesn't need to be re-embedded.

5

Exact-dedupe

EXACT

Drop documents with duplicate content hashes. Two PDFs of the same memo? One copy in the index. Cuts embedding cost & retrieval noise.

6

Near-dedupe (optional)

MINHASH

datasketch MinHash + LSH catches "same doc with different timestamp/footer" cases. Threshold ≈ 0.85 Jaccard. Optional, but powerful at scale.

Clean & Hash — All In One 

import re, hashlib, unicodedata

def normalize(text: str) -> str:
    # 1. unicode normalize
    text = unicodedata.normalize("NFKC", text)
    # 2. smart quotes → ascii
    text = text.translate(str.maketrans("\u201c\u201d\u2018\u2019\u2014", '""\'\'-'))
    # 3. collapse runs of whitespace, strip lines
    text = re.sub(r"[ \t]+", " ", text)
    text = re.sub(r"\n{3,}", "\n\n", text)
    return text.strip()

def content_hash(text: str) -> str:
    return hashlib.sha256(text.encode("utf-8")).hexdigest()[:16]

def prepare(raw: str, source: str, uri: str) -> Document:
    text   = normalize(raw)
    doc_id = content_hash(text)
    return Document(doc_id=doc_id, source=source, uri=uri, text=text)
Stage 5 · Enrich

Metadata is Half the System

Every chunk that lands in your vector DB needs companion metadata. Citations, filters, freshness checks, access control — they all live here.

Field Type Why it matters
source str "notion", "s3", "github". Lets you filter by origin.
uri str The URL or path. Required for clickable citations.
doc_id str Content hash. Stable across re-runs → idempotency.
chunk_id str {doc_id}:{seq}. Powers chunk-level citations.
title str | None For human-readable citation labels.
created_at / updated_at datetime Freshness filters. "Last 30 days only" needs this.
author str | None Attribution; sometimes access control.
tags / namespace list[str] Hard filters at query time — public vs internal, team scoping.
acl list[str] Who can see this. Filter at retrieval, not generation.
page / section str | None Deep-link citations: "answer cites page 14".
⚡ Filter at retrieval, not generation. Drop chunks the user shouldn't see before they reach the LLM context window. Filtering after generation is a leak.
⚡ Cheap to capture, expensive to add later. Add every metadata field during ingestion — even ones you don't currently use. Backfilling means re-indexing.
⚡ Time-travel queries: Vector DBs let you filter by metadata. With updated_at indexed, "what changed last week" becomes a 5ms query.

Glue & Plumbing

Orchestration — How It All Runs

Three strategies, three sizes. Pick the smallest one that meets your reliability bar.

# ingest.py — simplest possible pipeline
from pathlib import Path
import json

def run(input_dir: Path, output: Path):
    seen = set()
    with open(output, "w", encoding="utf-8") as out:
        for path in input_dir.rglob("*"):
            if not path.is_file():
                continue
            raw = load(path)                  # dispatch by extension
            doc = prepare(raw, "local", str(path))
            if doc.doc_id in seen:        # exact dedupe
                continue
            seen.add(doc.doc_id)
            for chunk in chunk_doc(doc):
                out.write(chunk.model_dump_json() + "\n")

if __name__ == "__main__":
    run(Path("./corpus"), Path("./out/chunks.jsonl"))

When to use

  • < 100k documents
  • One source, one machine
  • Re-run is acceptable on failure
⚡ Don't underestimate this tier. A well-written script + JSONL output is usable in production for a long, long time. 
# Tier 2 — work queue, parallel workers, resumable
from concurrent.futures import ProcessPoolExecutor
import redis, json

r = redis.Redis()

def enqueue_all(paths):
    for p in paths:
        r.sadd("ingest:pending", str(p))

def worker():
    while path := r.spop("ingest:pending"):
        try:
            doc = prepare(load(path), "local", path)
            if r.sismember("ingest:done", doc.doc_id):
                continue
            for chunk in chunk_doc(doc):
                index(chunk)
            r.sadd("ingest:done", doc.doc_id)
        except Exception as e:
            r.sadd("ingest:failed", json.dumps({"path": path, "err": str(e)}))

# Resumable: re-run just retries pending + failed

When to use

  • Millions of documents
  • Need parallelism & resumability
  • Want per-doc retry logic
⚡ Pro tip: Redis sets are great for the pending/done/failed pattern. Celery, RQ, and SQS all support it too. 
# Tier 3 — Prefect DAG (Dagster / Airflow work the same way)
from prefect import flow, task

@task(retries=3, retry_delay_seconds=10)
def discover(source) -> list[str]:
    return list_uris(source)

@task(retries=3)
def load_and_clean(uri) -> Document:
    return prepare(load(uri), "s3", uri)

@task
def chunk_and_index(doc: Document) -> int:
    for chunk in chunk_doc(doc):
        index(chunk)
    return 1

@flow(name="rag-ingestion")
def ingest_flow(source):
    uris     = discover(source)
    docs     = load_and_clean.map(uris)
    counts   = chunk_and_index.map(docs)
    return sum(counts)

When to use

  • Scheduled / recurring runs
  • Multiple sources, multiple stages
  • Need observability dashboards
⚡ Don't start here. Most teams that pick a DAG framework on day one regret it. Start with Tier 1 → graduate when you actually feel the pain.
What Will Go Wrong

Common Failure Modes

Every one of these has shipped to production for someone. Build for them before they bite.

Silent encoding corruption

A latin-1 file opened as UTF-8 produces "Iâ€™m fine" everywhere. The pipeline succeeds, retrieval returns garbage. Fix: always specify encoding="utf-8" + log byte counts as a sanity check.

Near-duplicate explosion

"v1", "v2", "v3" of the same PDF all get indexed → retrieval returns three nearly-identical chunks for every query, crowding out diverse context. Fix: MinHash + LSH near-dedupe with a sensible threshold.

Stale data, no updated_at

A policy from 2019 is the top hit because nobody filters by freshness. Fix: capture created_at & updated_at at load time and filter at retrieval.

Non-idempotent ingestion

Re-running the script doubles every document in the vector DB. Fix: upsert by stable chunk_id = doc_id:seq; never blind-insert.

Privilege leak via retrieval

Internal HR doc retrieved for an external customer's question. Fix: index ACL metadata and filter at retrieval time. Never rely on the LLM to "decide" what's safe to mention.

Loader-specific text loss

PDF tables become unreadable strings. Slide titles vanish. Code blocks lose indentation. Fix: use format-aware loaders (unstructured, pdfplumber) and audit a sample of outputs.

Hash-before-clean mistake

Hashing the raw bytes makes "same content with extra whitespace" look different. Fix: always hash the cleaned text — that's your stable identity.

No observability

A loader silently drops 30% of inputs and no one notices for weeks. Fix: log per-stage counts. If loaded ≠ cleaned + dropped, alert.

Self-Check

Three Quick Questions

Lock in the pipeline before moving to chunking.

QUESTION 01

When should you compute the content hash that serves as your stable doc_id?

Correct. Hashing after cleaning gives you stable identity across whitespace, encoding, and boilerplate noise. Two PDFs of the same memo end up with the same hash and dedupe correctly.
QUESTION 02

An internal HR document accidentally got retrieved for an external user. Where do you fix this?

Correct. Filter at retrieval, not generation. Capture ACL metadata during ingestion, index it, and use the vector DB's filter to drop forbidden chunks before they reach the LLM.
QUESTION 03

You're standing up RAG for a 30,000-document internal knowledge base. Best initial orchestration tier?

Correct. 30k documents fits comfortably in Tier 1. A clean script writing JSONL handles it in minutes. Upgrade to Tier 2 when you feel real pain — usually around millions of docs, multiple sources, or scheduled re-runs.
