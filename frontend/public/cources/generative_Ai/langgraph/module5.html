02 · User Experience
Streaming — Real-Time User Experience

Nobody wants to stare at a loading spinner for 30 seconds. Streaming delivers output as it is generated, creating the ChatGPT-like experience where text appears word by word.

Mode 1

values

User sees full state after each node.

Example: [waiting...] → FULL STATE 1 → [waiting...] → FULL STATE 2 → DONE

Best for: debugging dashboards and admin tools.

Mode 2

updates

User sees only changes per node.

Example: [waiting...] → {intent: "order"} → [waiting...] → {response: "..."} → DONE

Best for: efficient real-time UIs and progress indicators.

Mode 3

messages

Token-by-token streaming.

Example: H-e-l-l-o-,- -I- -c-a-n- -h-e-l-p...

Best for: chat UIs and ChatGPT-style experiences.

Mode 4

custom

User sees your own events.

Example: "Searching database..." → "Analyzing results..." → "Writing response..."

Best for: custom progress indicators.

04 · Observability
Observability with LangSmith

You cannot fix what you cannot see. LangSmith traces every LLM call, tool execution, state change, and routing decision. For production agents, the source says this is not optional; it is essential.

Enable LangSmithEnvironment variables
# Enable LangSmith — just set environment variables
import os
os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_API_KEY"] = "ls-..."  # From smith.langchain.com
os.environ["LANGCHAIN_PROJECT"] = "aria-production"

# That's it! All invocations are now automatically traced.

06 · Deployment
FastAPI Deployment — Complete Example

This is the production-ready deployment example from the source.

FastAPI DeploymentStandard + streaming endpoints
from fastapi import FastAPI, HTTPException
from fastapi.responses import StreamingResponse
from pydantic import BaseModel
import json, asyncio

app = FastAPI(title="ARIA API", version="5.0")

class ChatRequest(BaseModel):
    message: str
    thread_id: str

@app.post("/chat")
async def chat(req: ChatRequest):
    """Standard request-response endpoint."""
    try:
        config = {"configurable": {"thread_id": req.thread_id}}
        result = await aria.ainvoke(
            {"messages": [HumanMessage(content=req.message)]},
            config
        )
        return {"response": result["messages"][-1].content}
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

@app.post("/chat/stream")
async def chat_stream(req: ChatRequest):
    """Server-Sent Events streaming endpoint."""
    async def generate():
        config = {"configurable": {"thread_id": req.thread_id}}
        async for event in aria.astream(
            {"messages": [HumanMessage(content=req.message)]},
            config, stream_mode="messages"
        ):
            if isinstance(event, tuple):
                chunk = event[0].content
                if chunk:
                    yield f"data: {json.dumps({'token': chunk})}\n\n"
        yield f"data: {json.dumps({'done': True})}\n\n"
    return StreamingResponse(generate(), media_type="text/event-stream")

# Run: uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4

08 · Operations
Production Best Practices Checklist
  • PostgreSQL checkpointer, not MemorySaver or SQLite
  • Connection pooling, for example max_size=20 for medium traffic
  • Circuit breakers on all cycles, such as max_iterations in state
  • LangSmith tracing enabled
  • Error handling at node, graph, and app levels
  • Token-level cost tracking per conversation
  • Health check endpoint at /health
  • Async everywhere: ainvoke, astream
  • Rate limiting on API endpoints
  • Input validation, including max message length and sanitization
