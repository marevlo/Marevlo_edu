GPT-3.
In-context
learning.

GPT-2 showed that next-token prediction produces emergent capabilities. GPT-3 (Brown et al., 2020) asked: what if we just make it bigger? At 175 billion parameters — 1,000× GPT-2 Small — something unexpected happened. The model could learn new tasks from examples in the prompt, with no gradient updates, no fine-tuning, no weight changes. Show it three examples of sentiment classification, and it classifies the fourth. This is in-context learning, and it redefined what a language model is: not just a text generator, but a general-purpose few-shot learner.

Scaling laws — loss is predictable, capabilities are not.

Kaplan et al. (2020) discovered that language model loss follows a power law in three variables: parameter count N, dataset size D, and compute budget C:

L(N) ≈ (Nc / N)αN    where αN ≈ 0.076 (1.1)

Double the parameters → loss decreases by a fixed, predictable amount. This relationship holds over seven orders of magnitude — from 1K to 10B parameters. No phase transitions, no plateaus, no surprises in the loss curve itself.

But here's the twist: while loss is smooth and predictable, capabilities are not. At 1.3B parameters, GPT-3 can barely do arithmetic. At 175B, it can do multi-digit addition, translate between languages, and write code — capabilities that appear suddenly at a specific scale, not gradually. These are emergent abilities.

GPT-3's scale — the numbers that changed AI
Parameters: 175B (175,000,000,000). That's 1,000× GPT-2 XL (1.5B) and 1,400× BERT-base (110M).96 layers, d=12288, 96 heads, d_ff=49152. Context: 2048 tokens.
Training data: 300B tokens from a filtered web crawl (CommonCrawl, WebText2, Books, Wikipedia).~570 GB of text. Each token seen ~0.5 times on average — much less repetition than BERT's 40 epochs.
Training compute: ~3.14 × 10²³ FLOPS. At A100 speeds: ~355 GPU-years. At 2020 V100 costs: ~$4.6MChinchilla (2022) later showed this budget was under-allocated: 175B params should have been trained on ~3.5T tokens, not 300B. GPT-3 was parameter-rich but data-poor.
Inference cost per token: ~0.6ms on 8× A100. 1000-token generation: ~600ms.KV cache memory: 96 layers × 2 × 2048 × 12288 × 2 bytes ≈ 9.4 GB per sequence. Batch size is memory-limited.
Scaling laws. Loss (cyan) follows a smooth power law — predictable, no surprises. Capabilities (pink) emerge suddenly at a threshold scale. GPT-3's 175B parameters crossed the threshold where in-context learning, arithmetic, and translation appeared. The pulsing dot marks GPT-3's position.
ScalingQuestion 01 / 07
Chinchilla (2022) showed GPT-3 was under-trained. What does this mean?
AGPT-3's architecture was wrong.
BFor 175B parameters, the optimal training data should have been ~3.5 trillion tokens (Chinchilla's "compute-optimal" ratio of ~20 tokens per parameter). GPT-3 was trained on only 300B tokens — about 10× less than optimal. A 70B model trained on 1.4T tokens (Chinchilla) matched GPT-3's quality at 2.5× fewer parameters. The lesson: both parameters and data must scale together. Spending the compute budget on a smaller model with more data is more efficient than a larger model with less data.
CGPT-3 needed more layers.
DGPT-3's learning rate was too high.

In-context learning — the key emergent ability.

In-context learning (ICL) is GPT-3's signature capability: the model performs a task by conditioning on examples in the prompt, with no gradient updates. Three modes:

ModePrompt structureGPT-3 accuracy on SST-2
Zero-shotTask description only. No examples.76.4%
One-shotTask description + 1 example.84.6%
Few-shotTask description + 10-50 examples.95.3%
Fine-tuned BERTGradient updates on labeled data93.5%

Notice: GPT-3's few-shot (no training, just examples in the prompt) outperforms fine-tuned BERT on sentiment classification. The model "learned" the task from 50 examples in the context window — no weight changes.

# Few-shot prompting for CineMatch review classification
prompt = """Classify each CineMatch review as positive or negative.

Review: "An absolute masterpiece of storytelling and visual art"
Sentiment: positive

Review: "Terrible acting, predictable plot, waste of time"
Sentiment: negative

Review: "Surprisingly thoughtful with great performances throughout"
Sentiment: positive

Review: "Started strong but fell apart in the second half"
Sentiment:"""

# GPT-3 completes: " negative"
# No fine-tuning. No gradient updates. No labeled training data.
# The model inferred the task from the 3 examples in the prompt.
What in-context learning is NOT

ICL is not training. The model's weights are frozen. No gradients flow. No parameters update. The model performs the task by pattern matching over the examples in its context window — recognizing the format (review → sentiment) and completing the pattern for the new input. This is fundamentally different from fine-tuning, where the model's weights change to better fit the task. ICL is inference-time adaptation via the context window.

ICLQuestion 02 / 07
GPT-3 can do few-shot sentiment classification at 95.3%. Does it "understand" sentiment, or is it just pattern matching?
AIt understands — 95.3% proves comprehension.
BPure pattern matching — it just copies the format.
CNeither — the model is just memorizing the examples.
DThe distinction is less clear than it seems. The model learned, during pretraining, that certain words correlate with positive/negative contexts — that's a form of "understanding" sentiment. In-context, it recognizes the input-output format and applies this prelearned knowledge. Whether this constitutes "understanding" or "sophisticated pattern matching" is a philosophical question. What matters practically: it produces correct outputs 95.3% of the time without task-specific training — a capability that's useful regardless of the underlying mechanism.
03 / 08
How in-context learning works — the current best theory.

The mechanism behind ICL is an active research area. The leading theories:

1. Implicit Bayesian inference (Xie et al., 2021): During pretraining, the model learns a distribution over tasks. The few-shot examples in the prompt act as evidence that narrows down which task the model should perform. The model doesn't learn a new task — it identifies which of its prelearned tasks best matches the examples.

2. Induction heads (Olsson et al., 2022): Specific attention heads learn a simple circuit: "find a pattern earlier in the context that matches the current input, then copy the token that followed that pattern." For few-shot classification, this means: "find the example whose review looks most like the new review, then copy the sentiment label that followed."

3. Implicit gradient descent (von Oswald et al., 2023): Transformers can implement gradient descent in their forward pass — the attention layers implicitly compute weight updates that would optimize a linear model on the in-context examples. The model is running a mini-optimization loop inside its forward pass, without actually updating any stored weights.

In-context learning: information budget per example
GPT-3's context: 2048 tokens. System prompt + task description: ~50 tokens. Each example: ~30 tokens (review + label).Available for examples: 2048 − 50 = ~1998 tokens ÷ 30 = ~66 examples maximum
Each additional example provides ~0.5-1.0 accuracy points (on SST-2, saturating around 30-50 examples)Diminishing returns: examples 1-5 add ~2 points each. Examples 30-50 add ~0.1 points each.
The sweet spot: 5-15 examples for most classification tasks. Beyond 15, marginal gain is small.For generation tasks (translation, summarization): 3-5 examples are usually sufficient — more examples waste context on demonstrations rather than the actual input.
MechanismQuestion 03 / 07
You give GPT-3 few-shot examples for CineMatch genre classification. The first example has the wrong label (you accidentally labeled a comedy as "horror"). What happens?
AThe model might follow the wrong label for similar inputs. ICL is pattern matching — the model treats the examples as ground truth. If the wrong-labeled example is similar to the test input, the induction heads will match against it and copy the wrong label. This is both a strength (no training needed) and a weakness (no error correction). Fix: validate your few-shot examples carefully. A single bad example can corrupt nearby predictions.
BThe model self-corrects — it "knows" the correct genre from pretraining.
CThe model ignores wrong examples and uses only correct ones.
DThe model always outputs "unknown" when examples conflict.
04 / 08
Prompt engineering — programming with natural language.

With ICL, the prompt is the program. The way you structure the prompt determines the model's behavior — this is prompt engineering.

Key prompt engineering principles.

1. Be explicit about the task format. Don't just say "classify this review." Show the exact input-output format you want: "Review: ... → Sentiment: positive/negative".

2. Choose diverse, representative examples. Include examples from each class. If you have 3 sentiment categories, include at least one of each. Avoid examples that are all easy — include borderline cases.

3. Order matters. Recent examples have more influence than early ones (recency bias in attention). Put the most representative examples last, closest to the test input.

4. Use chain-of-thought for reasoning. Instead of "Q: ... A: 42", use "Q: ... Let's think step by step: First... Then... Therefore, the answer is 42." This activates the model's reasoning capabilities for math, logic, and multi-step problems.

The majority label bias

If 8 of your 10 examples are "positive" and 2 are "negative", the model will be biased toward "positive."

ICL picks up on label distribution in the examples. If one label is over-represented, the model assumes that label is more likely for new inputs. Always balance your examples: equal numbers per class. If you have 3 categories and 9 example slots, use exactly 3 per category. This bias is one of the most common ICL failure modes and is easy to prevent.

CineMatch Integration

Few-shot prompting for rapid CineMatch feature prototyping.

The use case: CineMatch's product team wants to test a new review classification category — "spoiler detection." Before building a labeled dataset and fine-tuning DeBERTa, they prototype with GPT-3 few-shot prompting: 10 examples of spoiler/non-spoiler reviews in the prompt, then classify 1000 test reviews. If few-shot accuracy exceeds 85%, the feature is worth building properly.

The workflow: Prototype with few-shot (2 hours) → Evaluate → If promising, collect labeled data (2 weeks) → Fine-tune DeBERTa (2 days) → Deploy fine-tuned model (fast, cheap). Few-shot ICL is the rapid prototype; fine-tuned encoders are the production system.

GPT-3's architecture — same blocks, different scale.

GPT-3 uses the same decoder block as GPT-2 — pre-norm, causal self-attention, FFN — just bigger. The architecture innovations are minimal: alternating dense and locally-banded sparse attention layers (in the original paper, though most implementations use dense throughout).

GPT-3 175B parameter breakdown
Token embedding: 50,257 × 12,288 = 617.6M. Position embedding: 2,048 × 12,288 = 25.2MCombined embeddings: 642.8M (only 0.4% of total — at this scale, embeddings are negligible)
Per block — MHA: 4 × 12,288² = 604.0M. FFN: 2 × 12,288 × 49,152 + biases = 1,208.1M. LN: ~25KPer block total: 1,812.1M ≈ 1.81B
96 blocks: 96 × 1.81B = 173.8B
Total: 173.8B + 0.6B + final LN = ~174.6B ≈ 175B99.6% of parameters are in the 96 transformer blocks. The embedding table is a rounding error at this scale.
ScaleQuestion 05 / 07
GPT-3's per-block FFN has 1.21B parameters — more than the entire GPT-2 Small model (117M). What does this imply about the FFN's role?
AThe FFN is too large and should be reduced.
BMost FFN parameters are wasted.
CThe FFN should be replaced with attention.
DThe FFN is the model's primary knowledge store. At 1.21B parameters per layer, the FFN has enough capacity to store millions of factual associations — "Paris is the capital of France," "Inception was directed by Christopher Nolan," "NaCl dissolves in water." The attention layers route information; the FFN layers store and retrieve it. This is why larger models know more facts — they have more FFN capacity for knowledge storage.

The limitations that led to GPT-3.5 and GPT-4.

GPT-3 was a breakthrough, but it had critical problems that limited its usefulness as a product:

ProblemExampleRoot cause
Unhelpful"What's the capital of France?" → continues writing a quiz instead of answeringTrained to predict next token, not to be helpful
HallucinationConfidently states false factsNo training signal for factual accuracy — just plausibility
HarmfulGenerates toxic, biased, or dangerous content when promptedTrained on unfiltered web text containing all of the above
Not steerableIgnores instructions like "answer in one sentence"Never trained to follow instructions — just to continue text

These are not bugs in the architecture — they're consequences of the training objective. "Predict the next token" optimizes for text that looks like the internet. The internet contains helpful answers, but also quizzes, hallucinations, toxicity, and verbosity. The model learned all of it equally.

The alignment gap

A more capable model is not automatically a more useful model.

GPT-3 is massively more capable than GPT-2 — it can do arithmetic, translation, summarization, and code. But it's not more helpful in a product context, because it doesn't know that it should answer questions directly instead of continuing to write quiz questions, or that it should decline harmful requests instead of completing them. Capability without alignment is dangerous and commercially useless. This gap between capability and helpfulness is the alignment problem, and solving it is Module 2.3's subject.

LimitationsQuestion 06 / 07
You deploy raw GPT-3 as CineMatch's chatbot. A user asks: "What are some good thriller movies?" GPT-3 responds with: "What are some good thriller movies? This is a common question that..." and continues for 500 tokens without ever listing a movie. Why?
AGPT-3 doesn't know any thriller movies.
BGPT-3 was trained to predict the most likely next token given web text. The most likely continuation of "What are some good thriller movies?" in web text is not a helpful answer — it's an SEO blog post, a Quora re-statement, or an essay introduction. The model is doing exactly what it was trained to do: produce text that looks like the internet. To get helpful answers, you need instruction-tuning (Module 2.3), which explicitly trains the model to respond helpfully to questions.
CThe temperature is wrong.
DThe context window is too small.
07 / 08
The GPT-3 legacy — what scale taught us.
Paper breakdown

"Language Models are Few-Shot Learners" (Brown et al., May 2020)

GPT-3's paper demonstrated three things that reshaped the field. First, scaling works: 1000× more parameters than GPT-2 produced qualitatively new capabilities, not just quantitatively better text. Second, in-context learning is real: a frozen model can learn tasks from examples in the prompt, rivaling fine-tuned models on many benchmarks. Third, the gap between capability and alignment is the next frontier — GPT-3 could do impressive things but couldn't reliably be helpful.

The paper's legacy is a research program: scale the model (GPT-4), align the model (InstructGPT), and eventually do both simultaneously (RLHF + scale). Module 2.3 covers the alignment breakthrough; Module 2.4 covers the scaling frontier.
