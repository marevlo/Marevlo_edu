ViT & DeiT.
Patches are
all you need.

Every module so far processed text tokens. The Vision Transformer (Dosovitskiy et al., 2021) asked: what if we treat image patches as tokens? Split a 224×224 image into 196 patches of 16×16, flatten each into a vector, add position embeddings, and feed to a standard transformer encoder. No convolutions. No pooling. No image-specific inductive biases. The result: ViT matches or beats the best CNNs on ImageNet — proving that transformers are a universal architecture for any sequential data, not just text. DeiT (Touvron et al., 2021) made this practical with knowledge distillation from a CNN teacher.

01 / 08
Patch embedding — images as token sequences.

A 224×224 RGB image has 224 × 224 × 3 = 150,528 values. Treating each pixel as a token (150K tokens) is computationally infeasible. ViT's solution: split the image into non-overlapping patches of size P×P (typically 16×16), then project each patch into a d-dimensional embedding.

Patch embedding arithmetic
Image: 224×224 pixels, 3 channels (RGB). Patch size P=16.Number of patches: (224/16)² = 14 × 14 = 196 patches
Each patch: 16 × 16 × 3 = 768 values. Projection: Linear(768, d).For ViT-Base (d=768): the projection is 768→768. Coincidence — d happens to equal P²×C for base config.
Result: 196 patch tokens + 1 [CLS] token = 197 tokens of dimension 768.This is identical in shape to BERT processing 197 text tokens. The transformer encoder doesn't know it's processing an image.
Position embedding: learned, (197, 768). Each patch gets a position vector.Unlike text, patches have 2D spatial positions. But ViT uses 1D position indices — and it works. The model learns spatial relationships from data.
Attention cost: 197² = 38,809 attention scores per head. Trivial — much less than text at 512².This is why ViT doesn't need efficient attention: 196 patches = short sequence. Even ViT-Huge (14×14=196 patches at 16×16, or 16×16=256 patches at 14×14) is manageable.
PatchesQuestion 01 / 07
ViT uses 1D position embeddings for 2D image patches. Why doesn't it use 2D position encoding?
A2D encoding is too complex to implement.
BIt doesn't need to. The learned 1D position embeddings implicitly capture 2D spatial relationships. After training, visualizing the position embeddings reveals a 2D grid structure: nearby patches (in 2D space) have similar position embeddings. The model learns spatial layout from data without being told about 2D structure. Ablations show that 2D-aware position encodings improve results by only ~0.3% — not worth the added complexity.
CImages don't have spatial structure that matters.
DRoPE is used instead.

The architecture is a standard transformer encoder — identical to BERT. The only difference: the input embedding is a patch projection instead of a token embedding. LayerNorm, multi-head attention, FFN, residual connections — all identical. The [CLS] token representation is fed to a classification head, just like BERT.

ViT needs massive data — or a teacher

ViT-Base trained on ImageNet-1K alone (1.3M images) underperforms a ResNet-50. It needs ImageNet-21K (14M) or JFT-300M to match CNNs.

CNNs have built-in inductive biases: translation equivariance (a cat is a cat regardless of position) and locality (nearby pixels are related). ViT has none — it must learn these from data. With 1.3M images, there's not enough data to learn spatial structure from scratch. With 14M+, ViT learns spatial relationships and surpasses CNNs. DeiT solves this by distilling CNN knowledge into ViT, achieving strong results even with only ImageNet-1K.

DataQuestion 02 / 07
DeiT matches ViT quality using only ImageNet-1K (1.3M images) instead of JFT-300M. How?
ADeiT uses a different architecture than ViT.
BDeiT trains for more epochs on the same data.
CKnowledge distillation from a CNN teacher. DeiT adds a "distillation token" (alongside [CLS]) that learns to match a RegNet CNN teacher's predictions. The CNN teacher provides the spatial inductive biases that ViT lacks — locality, translation equivariance — as soft labels. The student ViT learns from both the ground truth labels (via [CLS]) and the teacher's predictions (via the distillation token). This transfers the CNN's spatial priors into the ViT without modifying the architecture.
DDeiT uses a more aggressive data augmentation pipeline.
03 / 08
ViT as a vision encoder for CineMatch.
CineMatch Integration

ViT powers CineMatch's visual content pipeline.

Poster classification: ViT-Base (86M params) classifies movie posters into 20 visual style categories (minimalist, action-packed, dark/moody, animated, etc.). Fine-tuned on 50K CineMatch poster images. Accuracy: 91.3%. Used for visual recommendation: "you liked the dark, moody poster style of Zodiac — here are similar-looking films."

Thumbnail quality scoring: ViT-Small (22M params) rates user-uploaded thumbnails on a 1-5 quality scale. Low-quality uploads get automated improvement suggestions. Inference: ~5ms per image on A10G.

Why ViT over CNN: ViT integrates with CineMatch's transformer-based text pipeline. The same attention mechanism, same embedding space, same serving infrastructure. This unification matters in Phase 6.3 (CLIP) and 6.4 (multimodal LLMs) where vision and language models must share representations.

ApplicationQuestion 03 / 07
CineMatch's poster classifier uses ViT-Base (86M, 196 patches). A user uploads a 4K poster (3840×2160). How should it be processed?
AProcess at full 4K resolution — more pixels = more information.
BUse larger patches (64×64) to keep 196 patches at higher resolution.
CSplit into multiple 224×224 crops and classify each separately.
DResize to 224×224. ViT-Base was trained with 224×224 input and learned position embeddings for a 14×14 patch grid. Using a different resolution changes the patch count, breaking the position embeddings. While fine-tuning at higher resolution is possible (interpolate position embeddings), it requires retraining. For a production classifier, resize to the training resolution — poster style classification doesn't need 4K detail, and the 5ms inference time stays constant.
04 / 08
ViT vs CNN — when to use which.
ViT vs CNN decision framework in 2026
ViT wins when: large dataset (>1M images OR distillation available), need to integrate with text transformers (multimodal), high-resolution with patch scaling, pretrained models available (ViT-B/L on ImageNet-21K)CineMatch: poster classification (50K images + DeiT distillation), visual-language integration.
CNN wins when: small dataset (<100K images, no distillation), real-time edge inference (mobile, embedded), detection/segmentation requiring multi-scale features, no transformer infrastructureCineMatch: on-device video frame analysis in the mobile app (MobileNet, not ViT).
The 2026 reality: ViT (and its descendants: Swin, DINOv2, EVA) has largely replaced CNNs in cloud/server vision. CNNs remain dominant on edge/mobile. The dividing line is hardware: GPU → ViT, mobile CPU → CNN.ConvNeXt (2022) showed CNNs can match ViT with ViT-style training recipes — the architecture gap has narrowed.
ViT vs CNNQuestion 04 / 07
ViT-Base (86M) and ResNet-152 (60M) achieve similar ImageNet accuracy (~82%). ViT has more parameters but uses self-attention. Which is faster at inference?
AViT is always faster due to parallelism.
BDepends on hardware. On GPU (high parallelism): ViT is competitive — self-attention over 196 tokens parallelizes well. On CPU (low parallelism): ResNet is 2-3× faster — convolutions are heavily optimized for CPU inference (im2col, Winograd). On mobile NPU: ResNet (or MobileNet) is much faster — mobile accelerators are designed for convolution, not attention. The winner depends on deployment hardware, not architecture quality.
CResNet is always faster due to fewer parameters.
DThey're identical in speed.
05 / 08
Attention visualization — what ViT learns to see.

Visualizing ViT's attention reveals that different heads learn different visual patterns: some attend to local texture, others to global structure, others to object boundaries. By layer 8+, attention heads clearly focus on semantically meaningful regions — the dog's face, the car's outline, the sky vs ground boundary. This is remarkable because ViT has no built-in knowledge of objects or boundaries — it discovers them from data.

Self-supervised ViT (DINO) produces even better features

DINO (Caron et al., 2021) trains ViT without any labels using self-distillation. The resulting attention maps are even more object-focused: the [CLS] attention in a DINO-trained ViT segments objects from background with no segmentation supervision. DINOv2 (2023) extends this to produce universal visual features that transfer to any downstream task — classification, segmentation, depth estimation — without fine-tuning. DINOv2 features are to vision what BERT embeddings are to text: general-purpose representations.

RepresentationQuestion 05 / 07
ViT's [CLS] token attends to all 196 patches. After training, the [CLS] representation is used for classification. Why does a single [CLS] vector capture the entire image?
AIt doesn't — the classification head uses all 196 patch embeddings.
B[CLS] only captures the most salient patch.
CThrough 12 layers of self-attention, the [CLS] token learns to aggregate information from all patches with learned, task-relevant weighting. In early layers, [CLS] attends broadly (gathering general image statistics). In later layers, it attends selectively (focusing on discriminative regions — the dog's face for dog-vs-cat, the text for document classification). By layer 12, the [CLS] vector is a compressed representation of the entire image, weighted by task relevance. Same mechanism as BERT's [CLS] for text classification.
DThe classification head averages all 196 patches, not just [CLS].
06 / 08
Patch size trade-offs — resolution vs compute.
Patch size affects everything
P=16 (standard ViT): 224/16 = 14×14 = 196 patches. Attention: 196² ≈ 38K. Good balance.Default for classification. Sufficient resolution for most tasks.
P=32 (coarser): 224/32 = 7×7 = 49 patches. Attention: 49² ≈ 2.4K. 16× cheaper.Faster but loses fine detail. Each patch covers 32×32 = 1024 pixels — small objects become invisible.
P=8 (finer): 224/8 = 28×28 = 784 patches. Attention: 784² ≈ 615K. 16× more expensive.Better fine detail but 16× the compute. Needed for dense tasks (segmentation, detection).
The trade-off: smaller patches = more tokens = better resolution = more compute. This is the vision analog of the text context length problem.Swin Transformer (Module 6.2) solves this with hierarchical local attention — getting fine patches without quadratic cost.
Patch sizeQuestion 06 / 07
CineMatch wants to detect small text in movie posters (titles, credits). ViT-Base with P=16 misses small text. What's the best fix?
AIncrease input resolution to 384×384 or 512×512 (keeping P=16). This gives 24×24=576 or 32×32=1024 patches — more tokens per image, capturing finer detail. The attention cost goes from 196² to 576² or 1024² (9× or 27× more), but FlashAttention handles this easily. Fine-tune with interpolated position embeddings at the new resolution. This is the standard approach for tasks requiring fine detail.
BUse P=4 patches — smallest possible.
CAdd a CNN text detector before ViT.
DViT can't detect small text — use OCR instead.

ViT's legacy — transformers are universal.
LegacyQuestion 07 / 07
What is ViT's most important contribution?
AA new image classification architecture.
BReplacing CNNs in all vision tasks.
CThe patch embedding technique.
DProving that transformers are a universal architecture. Before ViT, transformers were "the NLP architecture" and CNNs were "the vision architecture." ViT showed that the same transformer — same attention, same FFN, same LayerNorm — processes images as well as text. This opened the door to multimodal models (CLIP, GPT-4V) where a single architecture processes text, images, audio, and video. ViT's legacy is architectural unification: one architecture for all modalities.
