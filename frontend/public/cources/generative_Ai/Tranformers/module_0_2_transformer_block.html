The Transformer Block.
Two sub-layers, stacked forever.

Module 0.1 gave us the attention operation. This module wraps it in the infrastructure that makes it trainable at depth: residual connections, layer normalization, and a feed-forward network. Together, these form the transformer block — the unit that gets stacked 12, 24, 80, or 128 times to build every model in this track. We build the encoder block, the decoder block, compare pre-norm vs post-norm, and derive the full parameter count. By the end, you can build any transformer variant from these pieces.

The encoder block — attention + FFN + the glue.

An encoder block has exactly two sub-layers, each wrapped in a residual connection and layer normalization:

Sub-layer 1:   x' = LayerNorm(x + MultiHeadAttn(x, x, x))
Sub-layer 2:   out = LayerNorm(x' + FFN(x')) (1.1)

That's the entire encoder block. Multi-head attention mixes information across positions (each position queries all others). The FFN transforms each position independently (same weights, applied position-by-position). Residual connections ensure gradients survive deep stacking. LayerNorm keeps activations stable.

Encoder block (post-norm). Input x passes through MHA (cross-position mixing) and FFN (per-position transformation), each with a residual skip (green dashed) and LayerNorm. Shape is preserved: (B, T, d) in → (B, T, d) out. Stack N of these → the full encoder.
Tensor shapes through one encoder block (d=768, h=12, d_ff=3072, B=32, T=128)
Input x: (32, 128, 768)32 sequences, 128 tokens each, 768-dim embeddings
MHA projections: Q = x·W_Q → (32, 128, 768). Same for K, V. Reshape to (32, 12, 128, 64) per head.Params: 3 × 768² = 1,769,472 for Q/K/V projections
Attention scores: Q·K^T/√64 → (32, 12, 128, 128). Softmax → (32, 12, 128, 128). Scores·V → (32, 12, 128, 64).128×128 = 16,384 attention weights per head. Memory: 32×12×128×128×2 bytes ≈ 12.6 MB in BF16

Residual + LN: LN(x' + ffn_out) → (32, 128, 768). Total block params: 2,359,296 + 4,722,432 + 3,072 = 7,084,800~7.1M per block. For BERT-base (12 blocks): 12 × 7.1M = 85.0M encoder params
Encoder blockQuestion 01 / 07
In the encoder block, the FFN has 2× more parameters than the MHA (4.7M vs 2.4M at d=768). Why is the FFN so large?
AIt needs extra parameters to compensate for attention's limitations.
BThe FFN expansion factor (4×) is arbitrary and could be reduced.
CThe FFN processes images which need more parameters than text.
DThe FFN is a 2-layer MLP with inner dimension d_ff = 4d. Each of its two weight matrices is (768 × 3072) — larger than any single MHA projection (768 × 768). This 4× expansion was chosen empirically: it gives the per-position transformation enough capacity to learn complex feature interactions. Reducing to 2× (d_ff = 1536) saves 50% of FFN params but degrades quality measurably. Recent models like LLaMA use 8/3×d ≈ 2.67× with SwiGLU activation for better efficiency.
02 / 08
The feed-forward network — what it actually does.
FFN(x) = GELU(x · W₁ + b₁) · W₂ + b₂ (2.1)

The FFN is a 2-layer MLP applied to each position independently. Think of it as a 1×1 convolution over the sequence dimension — the same weights at every position. After attention has mixed information from all positions into each position's representation, the FFN transforms that enriched representation nonlinearly. What does it learn?

Knowledge storage. Recent interpretability work (Geva et al., 2021) shows that FFN layers act as key-value memories. The first layer (W₁) acts as a set of "keys" — patterns that activate on specific input features. The second layer (W₂) stores corresponding "values" — the information to inject when a key matches. In BERT, individual FFN neurons fire for concepts like "is_a_country", "past_tense_verb", or "negative_sentiment".

GELU vs ReLU — why the activation changed.

The original transformer (2017) used ReLU. BERT and all subsequent models use GELU (Gaussian Error Linear Unit):

GELU(x) = x · Φ(x) ≈ x · σ(1.702x) (2.2)

Where Φ is the standard normal CDF. Unlike ReLU (hard cutoff at 0), GELU provides a smooth transition around zero — small negative inputs get slightly passed through rather than completely zeroed. This produces smoother loss landscapes and slightly better optimization for transformers, though the difference is small (~0.1-0.3% on benchmarks). Modern models universally use GELU or SwiGLU.

import torch, torch.nn as nn

class TransformerFFN(nn.Module):
    """Position-wise feed-forward network."""
    def __init__(self, d=768, d_ff=3072, dropout=0.1):
        super().__init__()
        self.w1 = nn.Linear(d, d_ff)       # expand: 768 → 3072
        self.w2 = nn.Linear(d_ff, d)       # contract: 3072 → 768
        self.act = nn.GELU()
        self.drop = nn.Dropout(dropout)

    def forward(self, x):
        # x: (B, T, 768) — same shape in and out
        return self.drop(self.w2(self.act(self.w1(x))))

# SwiGLU variant (LLaMA, Mistral) — gated FFN:
class SwiGLUFFN(nn.Module):
    def __init__(self, d=768, d_ff=2048):  # note: 2/3 × 3072 ≈ 2048
        super().__init__()
        self.w1 = nn.Linear(d, d_ff, bias=False)
        self.w3 = nn.Linear(d, d_ff, bias=False)  # gate projection
        self.w2 = nn.Linear(d_ff, d, bias=False)

    def forward(self, x):
        return self.w2(nn.functional.silu(self.w1(x)) * self.w3(x))
FFNQuestion 02 / 07
The FFN applies the same weights at every position in the sequence. Why not use different weights per position?
APyTorch doesn't support per-position linear layers.
BPer-position weights would reduce the model's expressiveness.
CWeight sharing across positions (like convolution) gives a fixed parameter budget regardless of sequence length, enables variable-length inputs, and acts as regularization — the FFN learns transformations useful at any position. Attention handles position-specific cross-position mixing; the FFN handles position-generic feature transformation. Division of labor.
DPosition embeddings already encode all position-dependent information.
03 / 08
Residual connections — why depth works.

Without residual connections, stacking 12+ transformer blocks is nearly impossible to train. The gradient must flow backward through every block, and in a plain stack, the gradient is a product of many Jacobians — which tends to vanish or explode.

The residual connection adds the input directly to the output: y = F(x) + x. The gradient is:

∂y/∂x = ∂F/∂x + I (3.1)

That identity matrix I ensures gradients always have a non-vanishing path through the block, regardless of what ∂F/∂x does. Through L blocks, the gradient from the loss to the input is:

∂L/∂x₀ = ∂L/∂xL · (I + Σl ∂Fl/∂x₀) (3.2)

A sum, not a product. Sums cannot vanish the way products can (one small factor kills the entire product). This is why ResNet-152 and GPT-4 (rumored 120 layers) train at all.

ResidualQuestion 03 / 07
If you remove the residual connections from a 24-layer BERT-large, what happens?
ATraining is slightly slower but converges to the same quality.
BTraining fails catastrophically. Without residual connections, gradients must pass through 24 multiplicative Jacobians — the product either vanishes (gradients → 0, no learning in early layers) or explodes (loss → NaN). The model cannot be trained to any useful quality. This has been confirmed empirically — skip-connection ablation is one of the most reliable ways to destroy a transformer.
CThe model becomes more expressive because it's forced to learn F(x) directly.
DOnly the first few layers train; the rest have zero gradients but the model still works.
04 / 08
LayerNorm — keeping activations in range.

LayerNorm normalizes each example independently (unlike BatchNorm, which normalizes across the batch). For a vector x ∈ ℝd:

LayerNorm(x) = γ · (x − μ) / √(σ² + ε) + β    where μ = mean(x), σ² = var(x) (4.1)

γ (scale) and β (shift) are learned parameters — 2d total per LayerNorm. At d=768, that's 1,536 parameters — negligible compared to MHA and FFN.

Pre-norm vs post-norm.

The original transformer (2017) placed LayerNorm after the residual add (post-norm, equation 1.1). Most modern models place it before the sub-layer (pre-norm):

Post-norm:   x' = LayerNorm(x + F(x))
Pre-norm:   x' = x + F(LayerNorm(x)) (4.2)
Pre-norm vs post-norm — the gradient difference
Post-norm: ∂x'/∂x = ∂LN/∂(x+F(x)) · (I + ∂F/∂x). The LayerNorm Jacobian modulates the residual path.At 24 layers, the product of 24 LN Jacobians can cause subtle gradient scale drift
Pre-norm: ∂x'/∂x = I + ∂F/∂(LN(x)) · ∂LN/∂x. The identity flows through untouched — LN only affects the F branch.The residual stream is "clean" — gradients pass through I without any normalization distortion
Result: pre-norm trains more stably at depth. Post-norm sometimes achieves slightly better final quality (the LN regularizes both branches).BERT, GPT-2: post-norm. GPT-3, LLaMA, Mistral, most modern models: pre-norm
Common confusion

LayerNorm, not BatchNorm. Always.

Transformers never use BatchNorm. Two reasons: (1) BatchNorm normalizes across the batch, but sequence lengths vary — you'd be normalizing across padded and real tokens together. (2) BatchNorm stores running statistics from training, creating a train/eval discrepancy that interacts badly with attention's dynamic routing. LayerNorm normalizes each example independently — no batch dependency, no running statistics, no train/eval mismatch.

NormalizationQuestion 04 / 07
Why did modern transformers (GPT-3+) switch from post-norm to pre-norm?
APre-norm keeps the residual stream "clean" — the identity path I in ∂x'/∂x is unmodulated by LayerNorm. At 24+ layers, this makes gradient flow more stable. Post-norm applies LN to the residual path, subtly distorting gradient magnitudes at each layer — which compounds at depth. Pre-norm trains more reliably for deep models, though post-norm occasionally gives slightly better converged quality.
BPre-norm uses fewer parameters.
CPre-norm is faster at inference.
DPost-norm was a bug in the original implementation.
05 / 08
The decoder block — adding causal masking and cross-attention.

The decoder block has three sub-layers instead of two:

1. Masked Self-Attention:   Q, K, V all from decoder state + causal mask
2. Cross-Attention:   Q from decoder, K/V from encoder output
3. FFN:   same as encoder (5.1) 
class TransformerDecoderBlock(nn.Module):
    """Decoder block with masked self-attn + cross-attn + FFN (pre-norm)."""
    def __init__(self, d=768, h=12, d_ff=3072, drop=0.1):
        super().__init__()
        # Sub-layer 1: masked self-attention
        self.ln1 = nn.LayerNorm(d)
        self.self_attn = nn.MultiheadAttention(d, h, dropout=drop, batch_first=True)
        # Sub-layer 2: cross-attention (Q from decoder, K/V from encoder)
        self.ln2 = nn.LayerNorm(d)
        self.cross_attn = nn.MultiheadAttention(d, h, dropout=drop, batch_first=True)
        # Sub-layer 3: FFN
        self.ln3 = nn.LayerNorm(d)
        self.ffn = nn.Sequential(
            nn.Linear(d, d_ff), nn.GELU(), nn.Dropout(drop),
            nn.Linear(d_ff, d), nn.Dropout(drop),
        )

    def forward(self, x, enc_out, causal_mask=None):
        # Pre-norm style (modern)
        x = x + self.self_attn(*[self.ln1(x)]*3, attn_mask=causal_mask)[0]
        x = x + self.cross_attn(self.ln2(x), enc_out, enc_out)[0]
        x = x + self.ffn(self.ln3(x))
        return x

The causal mask is a lower-triangular matrix that sets future-position attention scores to −∞ before softmax, making their weights zero. This ensures position t only sees positions ≤ t — matching inference, where future tokens don't exist yet.

Decoder-only models skip cross-attention

GPT, LLaMA, and Mistral use the decoder block without sub-layer 2.

The full encoder-decoder architecture (T5, BART) uses all three sub-layers. Decoder-only models (GPT, LLaMA, Mistral) remove cross-attention entirely — there's no encoder to cross-attend to. Their block has just masked self-attention + FFN (2 sub-layers, like the encoder block but with a causal mask). This is the dominant architecture in 2026.

DecoderQuestion 05 / 07
A decoder-only model (GPT-style) has the same two sub-layers as the encoder block — MHA + FFN. What's the only structural difference?
AThe decoder uses a different activation function.
BThe decoder has more parameters per block.
CThe decoder uses cross-attention instead of self-attention.
DThe causal (triangular) attention mask. The encoder allows all-to-all attention. The decoder restricts position t to only attend to positions ≤ t. Same MHA operation, same FFN, same residuals, same LayerNorm — the only difference is a boolean mask applied to the attention scores before softmax. This single mask is what makes the model autoregressive.

CineMatch Integration

All three families in one product.

Encoder-only (BERT/DeBERTa): Powers CineMatch's semantic search — encodes reviews and movie descriptions into 768-d vectors for nearest-neighbor retrieval. Also classifies review sentiment and extracts movie entities (NER).

Decoder-only (LLaMA/Mistral): Powers CineMatch's conversational recommendation engine — "Hey CineMatch, I loved Inception but hated Interstellar. What should I watch?" The model generates personalized recommendations in natural language.

Encoder-decoder (T5): Powers CineMatch's review summarization — takes a long user review and generates a concise 2-sentence summary for display in the browse UI.
