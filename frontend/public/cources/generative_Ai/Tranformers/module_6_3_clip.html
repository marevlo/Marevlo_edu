CLIP.
Connecting
vision & language.

ViT processes images. BERT processes text. CLIP (Radford et al., 2021) connects them — training a vision encoder and a text encoder jointly so that matching image-text pairs have similar embeddings. The result: a model that can classify images zero-shot (no labeled training data) by comparing image embeddings to text descriptions. "A photo of a dog" vs "A photo of a cat" — whichever text is more similar to the image embedding wins. For CineMatch, CLIP enables zero-shot poster classification, image-text search, and the visual backbone for multimodal LLMs.

Contrastive learning — aligning vision and language.

CLIP trains two encoders simultaneously: a ViT (image encoder) and a transformer (text encoder). For each batch of N (image, text) pairs, the training objective maximizes similarity between matching pairs and minimizes similarity between non-matching pairs:

L = −(1/N) Σi [log (exp(sim(Ii, Ti)/τ) / Σj exp(sim(Ii, Tj)/τ))] (1.1)

This is the InfoNCE contrastive loss. For each image Ii, the correct text Ti should have higher cosine similarity than all other texts Tj in the batch. With batch size N=32,768, each image is compared against 32,768 text descriptions — the correct one should score highest.

CLIP training scale — 400M image-text pairs from the internet
Dataset: 400 million (image, text) pairs scraped from the internet. Image alt-text, captions, titles.Not curated, not labeled, not clean. The internet is the teacher — noisy but massive.
Batch size: 32,768. Each image contrasts against 32,767 negatives.Large batches are essential for contrastive learning — more negatives = harder task = better representations.
Image encoder: ViT-L/14 (307M params). Text encoder: Transformer (63M params). Total: ~370M trainable params.The image encoder is 5× larger than the text encoder — visual representation is harder than text representation.
Training: 12.8B image-text pairs seen (32 epochs over 400M). ~18K GPU-hours on V100s.Expensive pretraining, but the result is a reusable visual foundation model.
ContrastiveQuestion 01 / 07
CLIP's batch size is 32,768 — each image is compared against 32,767 negatives. Why so large?
ALarge batches are always better for training.
BGPU memory requires large batches.
CContrastive learning quality scales with the number of negatives. With 100 negatives, distinguishing a dog photo from "a photo of a dog" vs 99 random texts is easy — many negatives are obviously wrong ("a recipe for chocolate cake"). With 32,767 negatives, the task includes hard negatives ("a photo of a wolf," "a puppy playing") that force fine-grained visual-linguistic alignment. More negatives = harder contrastive task = sharper embedding space.
D32,768 is a power of 2 — required by GPU architecture.
02 / 08
Zero-shot classification — no training data needed.

CLIP's zero-shot classification works by treating class labels as text prompts:

# CLIP zero-shot genre classification for CineMatch posters
import clip, torch
from PIL import Image

model, preprocess = clip.load("ViT-L/14")

# The poster image
image = preprocess(Image.open("inception_poster.jpg")).unsqueeze(0)

# Class labels as text descriptions
genres = ["a movie poster for an action film",
          "a movie poster for a comedy",
          "a movie poster for a horror movie",
          "a movie poster for a sci-fi thriller",
          "a movie poster for a romantic drama"]

text_tokens = clip.tokenize(genres)

# Compute similarities
with torch.no_grad():
    image_features = model.encode_image(image)           # (1, 768)
    text_features = model.encode_text(text_tokens)        # (5, 768)
    image_features /= image_features.norm(dim=-1, keepdim=True)
    text_features /= text_features.norm(dim=-1, keepdim=True)
    similarity = (image_features @ text_features.T) * 100  # scaled cosine sim

probs = similarity.softmax(dim=-1)
# Result: "sci-fi thriller" → 78%, "action" → 14%, "horror" → 5%, ...

No fine-tuning. No labeled poster dataset. No genre-specific training. CLIP classifies by comparing the image embedding to text descriptions of each genre. The genre whose text embedding is most similar to the image embedding wins.

Zero-shotQuestion 02 / 07
CLIP zero-shot achieves ~76% on ImageNet classification (1000 classes) — without seeing any ImageNet images during training. How is this possible?
ACLIP memorized ImageNet from the internet.
B76% is random chance on 1000 classes.
CCLIP only works on easy examples.
DCLIP learned the visual concept of each object from 400M internet image-text pairs. It knows what a "golden retriever" looks like because it's seen millions of images captioned with similar words. At test time, it compares the test image's embedding to the embedding of "a photo of a golden retriever" — if the image looks like what it learned from internet captions, the similarity is high. Zero-shot classification works because CLIP's pretraining data implicitly covers most visual concepts through natural internet captions.
03 / 08
Prompt engineering for CLIP — text templates matter.

CLIP's zero-shot accuracy is highly sensitive to how you phrase the text prompts. "dog" performs worse than "a photo of a dog" which performs worse than "a photo of a dog, a type of pet":

CLIP prompt format can swing accuracy by 10+ points

"cat" → 63% accuracy. "a photo of a cat" → 73%. "a photo of a cat, a type of pet" → 76%.

CLIP was trained on internet image-text pairs, where captions are full sentences ("a golden retriever playing in the park"), not single words ("dog"). Single-word prompts are out-of-distribution for the text encoder. Using "a photo of a {class}" matches the caption distribution better. Adding context ("a type of pet", "a type of vehicle") further disambiguates — "jaguar" could be a car or an animal, but "a jaguar, a type of big cat" is unambiguous. OpenAI's paper uses 80 hand-crafted templates per class and ensembles them for best accuracy.

04 / 08
CLIP as a visual foundation model.

CLIP's image encoder (ViT-L/14) produces general-purpose visual representations that transfer to many downstream tasks — often better than ImageNet-supervised ViT, because CLIP's training data is more diverse (400M internet images vs 1.2M curated ImageNet images).

CineMatch Integration

CLIP is CineMatch's visual Swiss Army knife.

Zero-shot genre classification: No labeled posters needed. Define genres as text prompts, classify by similarity. Add new genres (e.g., "K-drama," "anime") instantly by adding new text prompts — no retraining.

Image-text search: Users type "movies with rainy city scenes" → CLIP encodes the query, compares to pre-computed poster/frame embeddings, returns visually matching results. This is semantic visual search — not keyword matching, not metadata lookup.

Content moderation: "a photo containing violence," "an image with explicit content" — CLIP similarity scores against moderation templates flag problematic uploaded images zero-shot.

Multimodal LLM backbone: CLIP's ViT encoder is the standard image encoder for multimodal LLMs (Module 6.4). CineMatch's chatbot can "see" movie posters and frames because CLIP's visual representations are aligned with language.

05 / 08
CLIP's limitations and when to fine-tune.
CLIP zero-shot vs fine-tuned accuracy on domain tasks
Generic classification (ImageNet): CLIP zero-shot: 76.2%. Fine-tuned ViT: 87.8%. Gap: 11.6 points.CLIP's zero-shot is impressive but not competitive with supervised training on abundant labeled data.
Specialized domain (CineMatch poster genre): CLIP zero-shot: ~68%. Fine-tuned ViT (5K labels): ~78%. Fine-tuned CLIP (5K labels): ~82%.Fine-tuning CLIP is better than fine-tuning ViT from scratch because CLIP's pretrained features are already language-aligned.
When to fine-tune: If CLIP zero-shot accuracy is below the task requirement AND labeled data is available, fine-tune CLIP's image encoder (freeze text encoder). This gives the best of both: CLIP's pretrained representations + domain-specific fine-tuning.Common approach: linear probe (train only a classification head on CLIP features) achieves 80% of full fine-tuning benefit at 1% of the cost.
LimitationsQuestion 05 / 07
CLIP zero-shot struggles with counting ("a photo of 3 dogs" vs "a photo of 5 dogs") and spatial relationships ("a cat on a table" vs "a table on a cat"). Why?
ACLIP's image encoder can't process numbers.
BCLIP wasn't trained on counting tasks.
CContrastive learning optimizes for bag-of-concepts similarity, not compositional understanding. "3 dogs on a table" and "5 dogs under a table" have similar contrastive features (dogs, table) even though the count and spatial relationship differ. CLIP's similarity is dominated by which objects/concepts are present, not their precise arrangement or quantity. This is because internet captions rarely distinguish "3 dogs" from "5 dogs" — the contrastive signal for counting is weak in the training data. Compositional understanding requires architectural changes (Module 6.4's multimodal LLMs).
DThe text encoder can't understand spatial prepositions.
06 / 08
SigLIP and CLIP variants — the 2026 landscape.

CLIP's contrastive approach spawned a family of improved models. SigLIP (Zhai et al., 2023) replaces InfoNCE with a simpler sigmoid loss (each pair independently classified as matching/non-matching), enabling smaller batch sizes without quality loss. OpenCLIP provides open-source reproductions with various ViT sizes trained on LAION datasets. EVA-CLIP scales to ViT-E (4.4B params) for maximum representation quality.

LandscapeQuestion 06 / 07
For CineMatch's visual search feature (users describe what they want, system returns matching movies), should they use CLIP or a supervised image classifier?
ASupervised classifier — higher accuracy on known genres.
BSupervised classifier — works with any query.
CCLIP — but only for image classification, not search.
DCLIP — visual search requires open-vocabulary matching. A supervised classifier only knows its fixed set of labels (20 genres). It can't handle "movies with rainy city scenes" or "posters with retro 80s aesthetic" — these aren't in its label set. CLIP matches arbitrary text queries to images in a shared embedding space. Any text description becomes a search query. This open-vocabulary capability is what makes CLIP uniquely suited for visual search — no predefined categories, no retraining for new queries.

CLIP's legacy — the bridge to multimodal.
LegacyQuestion 07 / 07
What is CLIP's most important contribution to the field?
AThe highest image classification accuracy.
BProving that vision and language can share an embedding space, learned from internet-scale data without manual labels. This single insight enables: zero-shot classification (any label via text), visual search (any query via text), image generation guidance (DALL-E, Stable Diffusion use CLIP to steer generation), and multimodal LLMs (CLIP's image encoder feeds visual tokens into language models). CLIP is less a model and more a paradigm: align modalities through contrastive learning, then compose them in downstream applications.
CA faster version of ViT.
DThe first model to process images with a transformer.
