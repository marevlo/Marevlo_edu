Swin
Transformer.
Hierarchical vision.

ViT processes all patches at one scale with global attention. Swin Transformer (Liu et al., 2021) introduces two ideas from CNNs: hierarchical feature maps (progressively lower resolution, like a CNN's pooling layers) and local windowed attention (each patch attends only within its window, like a convolution's receptive field). The result: a vision transformer that produces multi-scale features for detection and segmentation while maintaining linear complexity in image size. Swin became the default vision backbone in 2022-2024, replacing both ViT and CNN backbones for dense prediction tasks.

Shifted windows — local attention for images.

ViT's global attention at T=196 is cheap. But for object detection at 1024×1024 (T=4096 patches), global attention costs T² = 16.8M per head — expensive for processing many images. Swin's solution: partition the image into non-overlapping windows (typically 7×7 patches = 49 tokens per window) and compute attention only within each window.

Swin's windowed attention cost
Image: 56×56 patches (after 4× downsampling of 224×224 at patch size 4). Window: 7×7 = 49 patches.Number of windows: (56/7)² = 64 windows.
Attention per window: 49² = 2,401 scores. Total: 64 × 2,401 = 153,664 scores.Compare to global ViT: 3,136² = 9.8M scores. Swin is 64× cheaper.
But windows don't interact! Patch (6,6) in window A can't attend to patch (7,7) in window B — they're in adjacent windows.Solution: shift the window partition by half a window in alternating layers.
Shifted windows: Layer L uses window grid at position (0,0). Layer L+1 shifts the grid by (3,3), so window boundaries of layer L+1 fall in the middle of layer L's windows. Patches that were separated by window boundaries in layer L are now in the same window in layer L+1.After 2 layers: every patch has attended to every patch within a 14×14 region. Like Longformer's sliding window propagation, but in 2D.
Shifted windowsQuestion 01 / 07
Swin uses 7×7 windows. After 2 layers (regular + shifted), what's the effective receptive field?
A7×7 — same as the window size.
B49×49 — windows squared.
C~14×14. The shift is half a window (3.5 patches). Layer 1: each patch sees its 7×7 window. Layer 2 (shifted): the window now overlaps with the adjacent windows from layer 1, connecting patches that were previously separated. After 2 layers, the effective receptive field doubles to ~14×14. With S stages of 2 layers each, the receptive field grows to cover the full feature map — similar to how CNN receptive fields grow through stacked convolution layers.
DThe entire image — shifting gives global attention.
02 / 06
Hierarchical feature maps — the CNN-like structure.

CNNs produce features at multiple scales: early layers have high resolution (many positions, few channels), later layers have low resolution (few positions, many channels). This feature pyramid is essential for object detection (small objects need high-res features, large objects need low-res features). ViT produces single-scale features. Swin produces multi-scale features through patch merging.

StageResolutionChannelsPatchesWindow attention
Stage 156 × 5696313649 tokens × 64 windows
Stage 2 (merge 2×2)28 × 2819278449 tokens × 16 windows
Stage 3 (merge 2×2)14 × 1438419649 tokens × 4 windows
Stage 4 (merge 2×2)7 × 77684949 tokens × 1 window (global!)

Patch merging concatenates 2×2 adjacent patches into one, then projects to 2× channels. This halves spatial resolution and doubles channels — exactly like a CNN's strided convolution or max-pooling. The final stage (7×7 = 49 patches in 1 window) is effectively global attention, but at very low computational cost.

HierarchicalQuestion 02 / 07
Swin's Stage 4 has only 49 patches in a single 7×7 window. This means Stage 4 has global attention. Why not use global attention throughout?
AGlobal attention is always worse than windowed.
BCost. Stage 1 has 3,136 patches. Global attention: 3136² = 9.8M scores. Windowed: 64 windows × 49² = 154K scores — 64× cheaper. By Stage 4 (49 patches), global and windowed are identical. The hierarchical design uses cheap local attention at high resolution (where most computation happens) and global attention at low resolution (where it's cheap). This mirrors CNNs: local processing early (convolutions), global processing late (fully-connected or global average pooling).
CWindowed attention produces better features.
DStage 4 global attention is a bug in the design.
03 / 06
Swin for object detection and segmentation.
CineMatch Integration

Swin as CineMatch's visual backbone for scene analysis.

Scene understanding: CineMatch analyzes movie frames to automatically tag scene types (indoor, outdoor, action, dialogue, chase, etc.). Swin-Tiny (29M params) with a Mask R-CNN detection head identifies and segments objects in each frame: characters, vehicles, weapons, animals. This feeds the automated content tagging pipeline — "this movie contains car chases (72% of action scenes), firearms (3 scenes), and animals (2 scenes)."

Why Swin over ViT: Scene analysis requires detecting objects of different sizes in high-resolution frames (1280×720). Swin's multi-scale features handle both large objects (buildings, landscapes) and small objects (faces, weapons) simultaneously. ViT's single-scale output would need separate processing for each scale.

Why Swin over CNN: Swin-Tiny (29M) matches ResNet-50's detection performance with fewer parameters and better scalability. Swin-Large matches the best CNN detection backbones while being simpler to scale (just add more layers and channels).

DetectionQuestion 03 / 07
For CineMatch's scene analysis, Swin produces features at 4 scales (56×56, 28×28, 14×14, 7×7). How does the detection head use all 4 scales?
AFeature Pyramid Network (FPN). The detection head connects all 4 scales with top-down and lateral connections, creating a feature pyramid where: 56×56 features detect small objects (faces, props), 28×28 features detect medium objects (people, furniture), 14×14 features detect large objects (vehicles, buildings), and 7×7 features capture global scene layout. Each object is detected at the scale matching its size. This multi-scale approach is why Swin excels at detection — and why ViT (single scale) struggles.
BOnly the largest scale (7×7) is used for detection.
CAll scales are averaged together.
DThe detection head processes each scale separately and merges results.

SelectionQuestion 04 / 07
For CineMatch's multimodal chatbot (Module 6.4), should the image encoder be Swin or ViT?
ASwin — always the better vision model.
BSwin — multi-scale features improve LLM understanding.
CNeither — use a CNN for image encoding.
DViT. For multimodal LLMs, the image encoder outputs a flat sequence of patch tokens that gets concatenated with text tokens and fed to the decoder. ViT's flat architecture naturally produces this: 196 patch tokens, each a d-dimensional vector. Swin's hierarchical output (4 different scales) would need to be flattened and projected, adding complexity with no proven benefit for language-conditioned understanding. All major multimodal LLMs (GPT-4V, LLaVA, Gemini) use ViT-style encoders, not Swin.

Swin's legacy — the vision backbone convergence.
LegacyQuestion 05 / 07
Swin combines windowed attention (from Longformer) with hierarchical features (from CNNs). What does this hybrid design tell us about transformer architecture design?
APure transformers are better than hybrids.
BCNNs are still superior to transformers.
CThe best architectures borrow good ideas regardless of their origin. Swin takes CNN insights (local processing early, global late; multi-scale features; translation equivariance through shifted windows) and implements them with transformer primitives (attention instead of convolution). It's not "transformer vs CNN" — it's "attention mechanism + the structural priors that actually help." Pure ViT proves transformers work without priors. Swin proves they work better with the right priors.
DHybrid designs are always optimal.
ProductionQuestion 06 / 07
CineMatch needs to detect small text overlays in movie posters (watermarks, age ratings, studio logos). Which model handles this best?
AViT-Base — global attention sees everything.
BSwin-Tiny with FPN. Small text overlays are tiny objects (10-50 pixels in a 1024×768 poster). Swin's high-resolution Stage 1 features (256×192 at initial patch size 4) preserve fine details that ViT's 14×14 patch grid (at patch size 16) would merge into larger patches, losing the text. FPN connects high-res features to the detection head, enabling precise localization of small overlays.
CDeBERTa — it's a text model and text overlays are text.
DAny model works equally well for small object detection.
SynthesisQuestion 07 / 07
In 2026, what is the standard vision backbone for new projects?
AIt depends on the task. For classification and multimodal: ViT (flat, simple, CLIP-compatible). For detection and segmentation: Swin (multi-scale, FPN-compatible). For mobile deployment: EfficientNet or MobileNet CNN (optimized for inference speed). The "standard" is task-specific. The general trend: ViT for representation learning, Swin for dense prediction, with both outperforming CNNs at comparable compute budgets.
BAlways Swin — it's the most recent and capable.
CAlways ResNet — proven and reliable.
DAlways ViT — the simplest architecture.
