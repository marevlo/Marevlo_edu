02 / Project Brief
What You're Building
A research-summarization engine. Single function in, structured report out. Production-shaped but compact enough to fit in one module.
Project Module · Deliverable

DELIVERABLE: a Python module exposing research(topic) → ResearchReport. Internally it plans, fetches, summarizes in parallel, and synthesizes with citations. No framework agents (yet) — just LCEL chains composed cleanly.

INPUT
SIGNATURE
research(topic: str, depth: int = 6) → ResearchReport
EXAMPLES
  • "the rise of small language models in 2026"
  • "comparative carbon profile of EV batteries"
  • "state of AI agent benchmarks (τ-bench, AgentBench)"
DEPTH

Number of sub-questions to generate. Controls breadth × cost.

OUTPUT
TYPE
ResearchReport (Pydantic)
FIELDS
  • title: str
  • summary: str (executive)
  • themes: list[Theme]
  • sources: list[Source] (numbered)
  • open_questions: list[str]
CITATION DISCIPLINE

Every claim in themes[].content carries a [N] referencing sources[N].

CONSTRAINTS & SCOPE
✓ Async + gated parallelism (Module 2.2 pattern).
✓ LCEL chains for every stage — no agent frameworks yet.
✓ Pydantic schemas at every contract boundary.
✓ Citation indices must round-trip — no orphan refs.
✗ Source retrieval engine is out of scope — RAG course.
✗ No LangGraph yet (state machine arrives in 2.4).
03 / System Architecture
A Five-Stage Pipeline
A topic flows through five stages. Each is a small LCEL chain — composed at the orchestration layer with async fan-out for parallelism.
Topic → Report — data flow
Diagram
PLAN
N sub-questions → FETCH
parallel retrieval → MAP
summarize each → REDUCE
cluster + cite → REPORT
typed Pydantic
STAGE 01
PLAN — decompose topic into N sub-questions

LCEL chain: ChatPromptTemplate | model | PydanticOutputParser<SubQueries>. Module 2.1 prompting + Module 1.0 schema.

STAGE 02
FETCH — retrieve sources for each sub-question

Per sub-question, call a Retriever tool. Concurrent — gated. The retrieval implementation is a black box here (returns to Module 2.5).

STAGE 03
MAP — summarize each source into a SourceNote

Direct application of Module 2.2's map. abatch with max-concurrency, returns list[SourceNote].

STAGE 04
REDUCE — cluster notes by theme + attach citations

Anchoring reduce verbs from Module 2.2 (§7): "RANK, DEDUP, SYNTHESIZE" — never "summarize". Outputs list[Theme].

STAGE 05
REPORT — assemble the final ResearchReport

Pure Python — no LLM call. Validates citation round-trip, builds final typed object. Cheap, deterministic, testable.

SHAPE 1: MAP

Stage 1 maps topic → N sub-questions. The first fan-out the LLM controls.

SHAPE 2: MAP-REDUCE

Stage 3+4 is a canonical map-reduce — sources mapped to notes, notes reduced to themes.

SHAPE 3: DETERMINISTIC FINAL

Stage 5 has no LLM. Assembly stays cheap, reproducible, unit-testable.

04 / The Four Schemas
Type-Safe Boundaries
Four Pydantic models = four contracts. Each stage produces one. The reduce stage must validate the citation round-trip.
schemas.py — the typed contract surface
Pydantic v2
from pydantic import BaseModel, Field, field_validator
from typing import Literal

# [01] STAGE 1 OUTPUT — sub-questions plan
class SubQuery(BaseModel):
    question: str = Field(..., description="A single concrete sub-question.")
    rationale: str = Field(..., max_length=200)

class SubQueries(BaseModel):
    """LLM-decomposed sub-questions for the topic."""
    queries: list[SubQuery] = Field(..., min_length=3, max_length=10)

# [02] STAGE 3 OUTPUT (per source) — one summarized source
class SourceNote(BaseModel):
    source_id: int                       # 0..N-1, position in sources list
    sub_query: str
    key_points: list[str] = Field(..., max_length=5)
    confidence: float = Field(..., ge=0, le=1)
    quote: str = Field(..., max_length=280)   # representative excerpt

# [03] STAGE 4 OUTPUT — clustered themes with citations
class Theme(BaseModel):
    name: str
    content: str                        # prose claims; must contain [N] tokens
    cited_sources: list[int]            # source_ids referenced in content

# [04] STAGE 5 OUTPUT — the final report
class Source(BaseModel):
    id: int; url: str; title: str; published: str | None = None

class ResearchReport(BaseModel):
    title: str
    summary: str = Field(..., max_length=600)
    themes: list[Theme]
    sources: list[Source]
    open_questions: list[str] = []

    @field_validator("themes")
    def _citations_resolve(cls, themes, info):
        """Reject reports with orphan citation indices."""
        valid_ids = {s.id for s in info.data.get("sources", [])}
        for t in themes:
            unknown = set(t.cited_sources) - valid_ids
            if unknown:
                raise ValueError(f"theme '{t.name}' cites unknown sources: {unknown}")
        return themes

SubQueries

Output of PLAN. Bounded count (3–10) prevents runaway fan-out.

SourceNote

Output of MAP. source_id threads citations across the pipeline.

Theme

Output of REDUCE. Holds prose claims and the source IDs they cite.

ResearchReport

Final assembled output. Field validator catches orphan citations.

stages/plan.py — topic → SubQueriesLCEL CHAIN · STAGE 01
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import PydanticOutputParser
from schemas import SubQueries

parser = PydanticOutputParser(pydantic_object=SubQueries)

prompt = ChatPromptTemplate.from_messages([
    ("system",
     "You are a research planner. Decompose the user's topic into "
     "3–{max_q} CONCRETE sub-questions. Each must be independently answerable.\n\n"
     "CONSTRAINTS:\n"
     "- No question may repeat or paraphrase another.\n"
     "- Cover different angles: definition / mechanism / evidence / impact / limits.\n\n"
     "{format}"),
    ("human", "Topic: {topic}"),
]).partial(format=parser.get_format_instructions())

model = ChatOpenAI(model="gpt-4o-mini", temperature=0.2)

plan_chain = prompt | model | parser
stages/fetch.py — retrieval-as-a-toolSTAGE 02 · OUT OF SCOPE INTERNALS
from typing import Protocol
from schemas import Source

class Retriever(Protocol):
    """A tool that turns a sub-query into a list of Source objects.
       Internals (web search, vector store, hybrid) live in the RAG course."""
    async def __call__(self, query: str, k: int = 5) -> list[Source]: ...

# For this module we treat retrieval as a black-box callable.
# In Module 2.5 we will replace this with retrieval-as-a-tool the AGENT decides to call.

async def fetch_sources(
    retriever: Retriever,
    sub_queries: list[str],
    *, k: int = 5, max_concurrency: int = 6,
) -> list[Source]:
    """Parallel retrieval, gated, deduped by URL, ID-assigned 0..N-1."""
    sem = asyncio.Semaphore(max_concurrency)
    async def _one(q):
        async with sem:
            return await retriever(q, k=k)

    nested = await asyncio.gather(*[_one(q) for q in sub_queries])
    seen, sources = set(), []
    for batch in nested:
        for s in batch:
            if s.url in seen: continue
            seen.add(s.url)
            s.id = len(sources)
            sources.append(s)
    return sources
stages/map.py — source → SourceNoteSTAGE 03 · PER-SOURCE MAP
from schemas import SourceNote, Source

note_parser = PydanticOutputParser(pydantic_object=SourceNote)

map_prompt = ChatPromptTemplate.from_messages([
    ("system",
     "Extract a focused note from ONE source for ONE sub-question. "
     "Stay faithful to the source — no facts not present.\n{format}"),
    ("human",
     "SUB-QUERY: {sub_query}\n\nSOURCE [{source_id}] (\"{title}\"):\n{snippet}"),
]).partial(format=note_parser.get_format_instructions())

map_chain = map_prompt | model | note_parser

async def map_sources(
    pairs: list[tuple[Source, str]],   # (source, sub_query)
    *, max_concurrency: int = 12,
) -> list[SourceNote]:
    return await map_chain.abatch(
        [{"source_id": s.id, "title": s.title,
          "snippet": s.snippet, "sub_query": q} for s, q in pairs],
        config={"max_concurrency": max_concurrency},
    )
stages/reduce.py — notes → Themes (cited)STAGE 04 · ANCHORING REDUCE
from schemas import Theme
from pydantic import BaseModel
from typing import List

class ThemeBundle(BaseModel):
    themes: list[Theme]
    open_questions: list[str] = []

bundle_parser = PydanticOutputParser(pydantic_object=ThemeBundle)

# Anchoring verbs from Module 2.2 §7 — RANK / DEDUP / SYNTHESIZE, not "summarize"
reduce_prompt = ChatPromptTemplate.from_messages([
    ("system",
     "You are a synthesis analyst. Given source notes:\n"
     "1) CLUSTER notes into 3–6 coherent themes\n"
     "2) RANK themes by evidence strength + source count\n"
     "3) For each theme, write prose claims with INLINE [N] citations "
     "   referencing source_id values seen in the notes\n"
     "4) Never invent a source_id. If unsure, omit the claim.\n"
     "5) End with open_questions the sources did NOT resolve.\n\n{format}"),
    ("human", "TOPIC: {topic}\n\nNOTES:\n{notes}"),
]).partial(format=bundle_parser.get_format_instructions())

reduce_chain = reduce_prompt | model | bundle_parser
engine.py — the public entrypointSTAGE 05 · ORCHESTRATION
from schemas import ResearchReport
from stages.plan   import plan_chain
from stages.fetch  import fetch_sources, Retriever
from stages.map    import map_sources
from stages.reduce import reduce_chain

async def research(topic: str, retriever: Retriever, depth: int = 6) -> ResearchReport:
    # [01] PLAN
    plan = await plan_chain.ainvoke({"topic": topic, "max_q": depth})
    sub_qs = [q.question for q in plan.queries]

    # [02] FETCH — parallel retrieval
    sources = await fetch_sources(retriever, sub_qs, k=5)

    # Pair every source with its triggering sub-query for MAP
    pairs = [(s, sub_qs[min(i, len(sub_qs)-1)]) for i, s in enumerate(sources)]

    # [03] MAP — async fan-out per source
    notes = await map_sources(pairs)

    # [04] REDUCE — anchoring synthesis with citations
    bundle = await reduce_chain.ainvoke({
        "topic": topic,
        "notes": [n.model_dump() for n in notes],
    })

    # [05] REPORT — deterministic assembly, field validator catches orphan refs
    return ResearchReport(
        title=f"Research: {topic}",
        summary=bundle.themes[0].content[:600] if bundle.themes else "",
        themes=bundle.themes,
        sources=sources,
        open_questions=bundle.open_questions,
    )

07 / Failure Modes
Where Research Agents Break
Five issues every research agent eventually hits — and the structural fix that closes each one.
Five failure modes & structural fixes
Expandable
F1 · Hallucinated Citations
F2 · Source Drift
F3 · Theme Collapse
F4 · Source-Quality Spam
F5 · Rate-Limit Cliff

F1 — Hallucinated Citations

Model writes [7] but source 7 never existed.

Why: reduce prompt didn't constrain to seen source_ids. Fix: our ResearchReport._citations_resolve validator rejects orphan refs at model load time. The reduce step retries (next step's observation) with a clean error message.

F2 — Source Drift

Map summary contains facts not in the source — the LLM "fills in the blanks".

Why: model fills gaps from priors when the source is thin. Fix: the quote field on SourceNote forces the model to point at a passage. Optional second-pass faithfulness check (Track 3 territory).

F3 — Theme Collapse

Reduce produces one giant theme instead of 3–6 distinct ones.

Why: reduce prompt said "summarize" instead of "cluster, rank, synthesize". Fix: anchoring verbs (Module 2.2 §7). Plus a constraint 3 ≤ len(themes) ≤ 6 on the bundle parser.

F4 — Source-Quality Spam

Retriever returns SEO content / press releases — themes inherit the bias.

Why: garbage in, garbage out. The retriever is the upstream quality gate. Fix: retriever-side concerns — domain allow-listing, recency filters, deduping. Out of scope here, fully covered in the RAG course.

F5 — Rate-Limit Cliff

High-depth topics blow past provider TPM and the run aborts halfway.

Why: uncapped fan-out at MAP stage. Fix: max_concurrency in abatch + asyncio.Semaphore (Module 1.0 + 2.2 pattern). For very long runs, retry-with-exponential-backoff on 429s — Module 0.2 error-recovery pattern.

The Pattern Behind the Fixes

Four of the five fixes are structural — schemas, validators, anchoring verbs, gated concurrency. "Better prompting" is rarely the answer in a research pipeline.
