02 / Core Primitives
The 8 Building Blocks
Everything an agent in Track 2 is made of. If you can name and place these eight, every codebase becomes legible.
TOOL M 0.2 · 1.0

A typed function the LLM may call. Pydantic args, load-bearing description.

@tool decorator + args_schema
PROMPT M 2.1

Parameterized message templates. Persona + instruction + variables.

ChatPromptTemplate.from_messages
STATE M 2.4 · 2.8

TypedDict with reducers. Public + private (subgraph) layers.

Annotated[list, add_messages]
NODE M 2.4

Pure function: state → state update. Where work happens.

def node(state) → dict
EDGE M 2.4

Routing between nodes. Fixed or conditional on state.

add_conditional_edges(...)
CHECKPOINTER M 2.4 · 2.7

State persistence per thread. Enables resume + HITL.

PostgresSaver / RedisSaver
SUBGRAPH M 2.8

Compiled graph used as a node. Multi-agent building block.

builder.compile() as node
INTERRUPT M 2.4 · 2.11

Pause for human input or external decision. HITL primitive.

interrupt(payload)
03 / Patterns Index
12 Patterns at a Glance
Each pattern: one-line definition, when to reach for it, one-line warning. Click the source tag for the full module.

SINGLE-AGENT

REACT LOOP M 2.4 · 2.9

Think→Act→Observe. Plan-as-you-go. The default.

⚠ drifts on 10+ step tasks
PLAN-AND-EXECUTE M 2.9

Plan once with strong model, execute steps with cheap model, replan on drift.

⚠ stale plans on dependent steps
REFLEXION M 2.9

Draft → critique → revise. Best for writing, code, schema-validated output.

⚠ cap at 3 attempts
TREE OF THOUGHTS M 2.9

Expand N branches, score, prune, expand best. 10-20× cost vs ReAct.

⚠ rarely worth it online
MAP-REDUCE M 2.2 · 2.3

Fan out parallel work over inputs, then reduce into one synthesis.

⚠ bound concurrency
RETRIEVAL-AS-TOOL M 2.5

RAG retriever wrapped as a tool the LLM picks. Multi-source via parallel/cascade.

⚠ tune description for under/over-retrieve

MULTI-AGENT

SUPERVISOR M 2.8

One router agent dispatches to specialist workers. The 90% pattern.

⚠ supervisor becomes bottleneck
HAND-OFF M 2.8

Hand-off as a Command-returning tool. No central router. Stage-based flows.

⚠ cap handoff_count
HIERARCHICAL M 2.8

Supervisor of supervisors. Org-shaped teams of specialists.

⚠ cost compounds 2×

PRODUCTION

CRASH-ONLY DESIGN M 2.7

Let processes die. Checkpointers resume from last state. Simpler than retry trees.

→ recursion_limit + outer timeout
3-LAYER MEMORY M 2.11

Short-term (subgraph) · Working (checkpointed thread) · Long-term (namespaced store).

⚠ never global memory
DEFENSE IN DEPTH M 2.11

5 layers: Edge → Input → Prompt → Tool → Output. Assume any single layer fails.

⚠ tool layer is load-bearing
04 / Decision Cards
7 Pickers for the Common Questions
Each card answers a single "which one?" question that comes up while building. Read it. Decide. Move on.

Q1 · Which planning pattern?

M 2.9
→ Single attempt usually works → ReAct
→ Long, sequential, predictable → Plan-and-Execute
→ Output quality needs critique → Reflexion
→ Hard search problem, offline budget → Tree of Thoughts

Q2 · Which multi-agent topology?

M 2.8
× Tools confuse the LLM? Add tools or filter intent first.
→ Clear specialists, central routing → Supervisor
→ Stage-based workflow, no router → Hand-Off
→ 6+ specialists, org-shaped → Hierarchical

Q3 · Which checkpointer?

M 2.7
→ Dev / tests / CLI → MemorySaver
→ Single-process desktop → SqliteSaver
→ Production multi-user → PostgresSaver
→ High-frequency short TTL → RedisSaver

Q4 · Which stream mode?

M 2.7
→ Debugging dashboard → values (full state per step)
→ Progress UI → updates (deltas per node)
→ End-user typing effect → messages (token-by-token)
→ Need two at once → pass a list, get (mode, chunk) tuples

Q5 · RAG or long context or fine-tune?

M 2.5
→ Small static corpus, queries touch most of it → Long context
→ Large / changing / multi-source / cited → RAG-as-tool
→ Narrow specialist, high-volume, format strict → Fine-tune
→ Most production agents combine all three.

Q6 · Which guardrail layer for this risk?

M 2.11
→ Rate / auth / spam → Edge (L1)
→ Direct injection · off-topic → Input (L2)
→ Tool misuse · destructive actions → Tool (L4) ← load-bearing
→ PII / uncited claims → Output (L5)

Q7 · Which framework for this project?

M 2.10
→ Production w/ HITL, resume, multi-step state → LangGraph
→ Demo-quality multi-agent crew → CrewAI
→ Typed Python service, single-agent → Pydantic AI
→ Pure OpenAI shop, fast TTL → OpenAI Agents SDK

Q-Bonus · Single agent or split?

M 2.8
★ Default: single agent + tools
→ Split when: distinct personas, 10+ confusing tools, independent failure domains, model tiering, or true parallel work
× Don't split because: "it sounds cool", org chart fit
05 / Hard Rules
20 Non-Negotiables
If you violate these, the agent will surface a problem within weeks of production. Keep this list above your monitor.
Rule Why Source
1. Pydantic on every tool boundaryUntyped tool args = invalid actions, ambiguous calls, security gapM 1.0 · 2.11
2. Tool description is load-bearingLLM picks tools from the description, not the name. Write it like docs.M 2.5
3. Tool errors return as ToolMessage, not raiseLets the agent recover. Raised exceptions kill the trajectory.M 2.7
4. Always set recursion_limit + outer timeoutOtherwise one loop bug = open-ended cost + frozen requestM 2.4 · 2.7
5. Model has retry + fallback configured429s and 500s happen. Without fallback the request just fails.M 2.7
6. Crash-only design over elaborate retry treesCheckpointers resume. Let processes die.M 2.7
7. Namespace every thread_id by tenantCross-tenant collision = data leak. No exceptions.M 2.7
8. Set TTLs on every persistent storeCheckpoints + memory grow unboundedly without them.M 2.7 · 2.11
9. Public state is append-only; private fields have single writersConcurrent writes without reducers = silent data loss.M 2.8
10. Stream first tokens within TTFB budgetSilent pauses kill perceived responsiveness regardless of total time.M 2.7
11. Default to single agent until 10+ tools confuse itMulti-agent costs 2-3× per request. Earn the price.M 2.8
12. Reach for ReAct first; escalate only when it visibly failsMost production agents don't need fancier planning.M 2.9
13. Cap Reflexion attempts at 3Beyond that, critic yes-mans and quality regresses.M 2.9
14. Cap multi-agent handoff_count (≤ 3)Prevents A↔B ping-pong from running forever.M 2.8
15. Tier models — small for routing, strong for synthesis10-100× cost gap, often invisible quality differenceM 2.7 · 2.9
16. Treat every byte the LLM reads as adversarialUser input, RAG docs, tool outputs, MCP results — all are attack surfacesM 2.11
17. Tool guards must hold even if injection succeedsAllowlist + schema + rate + blast radius are non-negotiableM 2.11
18. HITL on every irreversible actionPayments, deletes, public posts — interrupt before executeM 2.11
19. Append-only audit log on every tool side-effectIf you can't reconstruct what the agent did, you can't defend itM 2.11
20. Never log secrets, API keys, or raw PIIRedact before structlog. Use trace IDs, not full transcriptsM 2.7 · 2.11
06 / Paste-Ready Snippets
Six Snippets You'll Reach For
Minimal, copy-pasteable. Each ports to your project with one or two renames. Compose freely.
1 · Typed tool with Pydantic schema
M 1.0
from langchain_core.tools import tool
from pydantic import BaseModel, Field

class SearchArgs(BaseModel):
    query: str = Field(..., description="What to search for")
    top_k: int = Field(4, ge=1, le=10)

@tool(args_schema=SearchArgs)
def search_docs(query, top_k=4):
    """Search the product docs. Use for HOW-TO questions."""
    return retriever.invoke(query)[:top_k]
2 · Minimal ReAct agent
M 2.4
from langgraph.prebuilt import create_react_agent
from langchain_openai import ChatOpenAI

agent = create_react_agent(
    model=ChatOpenAI(model="gpt-4o-mini"),
    tools=[search_docs, lookup_order],
    prompt="You are a support assistant. Use tools.",
)
out = agent.invoke({"messages": [("user", "where is order #7711?")]})
3 · Multi-tenant Postgres checkpointer
M 2.7
from langgraph.checkpoint.postgres import PostgresSaver

checkpointer = PostgresSaver.from_conn_string(os.environ["DB"])
checkpointer.setup()
graph = builder.compile(checkpointer=checkpointer)

config = {"configurable": {
    "thread_id": f"{tenant}:{conv_id}",
    "checkpoint_ns": tenant,
}, "recursion_limit": 25}
4 · Supervisor with structured Route
M 2.8
class Route(BaseModel):
    next: Literal["billing", "refund", "FINISH"]

supervisor = prompt | ChatOpenAI("gpt-4o-mini").with_structured_output(Route)

def supervisor_node(state):
    return {"next": supervisor.invoke(state).next}

builder.add_conditional_edges("supervisor", lambda s: s["next"],
    {"billing": "billing", "refund": "refund", "FINISH": END})
5 · Token-stream over SSE
M 2.7
@app.post("/chat/stream")
async def stream(req):
    async def events():
        async for tok, _ in graph.astream(
            req.input, config=cfg, stream_mode="messages"
        ):
            if tok.content:
                yield f"data: {tok.content}\n\n"
    return StreamingResponse(events(), media_type="text/event-stream")
6 · HITL interrupt before irreversible
M 2.11
from langgraph.types import interrupt

@tool
def issue_refund(order_id, amount_cents):
    decision = interrupt({
        "kind": "approve_refund",
        "order_id": order_id,
        "amount_cents": amount_cents,
    })
    if not decision["approved"]:
        return {"status": "declined"}

07 / Ship Checklist
15 Gates Before Day 1
Combines Module 2.7's 12 production gates with the safety gates from Module 2.11. If you can tick all 15, you're ready for real users.
✓
[STATE] Durable checkpointer wired (Postgres / Redis)
✓
[STATE] thread_id namespaced by tenant; cross-tenant test passes
✓
[STATE] TTL / GC policy documented and scheduled
✓
[ERR] Tool errors returned as ToolMessage, not raised
✓
[ERR] Model retry + fallback configured
✓
[ERR] recursion_limit + outer timeout + safe default
✓
[STREAM] Token stream wired; TTFB < 800ms p95
✓
[STREAM] Progress updates on tool calls (no silent pauses)
✓
[OBS] Tracing enabled (LangSmith / OTel); request_id propagated
✓
[OBS] Metrics on cost/latency/error_rate exported & alerted
✓
[COST] Per-tenant budget cap enforced at edge
✓
[COST] Prompt cache enabled for system prompt
✓
[SAFE] Input guards (heuristic + classifier) at front door
✓
[SAFE] Tool allowlist + schema + rate + blast-radius per tool
✓
[SAFE] HITL + append-only audit on every irreversible tool
