AXIS 1 — PLAN DEPTH

Does the agent commit to a plan upfront, or decide step-by-step? Upfront helps multi-step tasks stay coherent; reactive handles uncertainty better.

AXIS 2 — SELF-CRITIQUE

Does the agent explicitly evaluate its own work? None = trust the first attempt. Explicit = score / critique / revise. Trades cost for quality.

The Reframe

Four patterns aren't memorized. They are the four corners of how-much-plan × how-much-critique.

03 / ReAct Reframed
Plan As You Go
You built ReAct in Module 2.4. Here we look at it through the planning lens: it's a pattern that refuses to plan ahead. Each step is decided fresh, given the observation just received.
ReAct REACTIVE · NO CRITIQUE
THINK → ACT → OBSERVE ↩

Think → Act → Observe → repeat. No global plan exists at any moment. The agent reasons about the next single step, given everything seen so far.

SWEET: Tool-use, lookups, customer support — most production agents
BREAKS: 10+ step tasks · coherence drift · "myopic" decisions

Why Start Here

ReAct is the cheapest, fastest, most LLM-native planning pattern. For ~80% of production agents it's the right answer — adding more planning machinery is over-engineering.

  • ✓ 1 LLM call per step, no extra coordination cost
  • ✓ Handles uncertainty — replans every turn implicitly
  • ✓ Built-in pattern in Module 2.4 — easy debugging
  • ✗ Loses the plot on multi-step tasks without a guiding plan
The Rule

Reach for another pattern only when ReAct visibly fails — not when it sounds "too simple".

04 / Plan-and-Execute
Commit to a Plan First
Generate a step-by-step plan once with a strong model. Execute each step with a cheap model. Optionally replan when reality diverges. Cheaper than ReAct for long sequences; harder when the world keeps changing.

plan_and_execute.py — LangGraph skeleton
Planner + Executor nodes
from typing import Annotated, TypedDict
from pydantic import BaseModel, Field
from operator import add

class Plan(BaseModel):
    steps: list[str] = Field(..., min_length=1, max_length=10)

class State(TypedDict):
    input: str
    plan: list[str]
    past_steps: Annotated[list[tuple[str, str]], add]   # (step, result)
    response: str

# [PLANNER] strong model, structured output, called once
planner = planner_prompt | ChatOpenAI(model="gpt-4o").with_structured_output(Plan)

def plan_node(state: State):
    plan = planner.invoke({"input": state["input"]})
    return {"plan": plan.steps}

# [EXECUTOR] cheap ReAct agent runs one step at a time
executor = create_react_agent(
    model=ChatOpenAI(model="gpt-4o-mini"), tools=TOOLS
)

def execute_step(state: State):
    step = state["plan"][0]
    result = executor.invoke({"messages": [("user", step)]})
    return {
        "past_steps": [(step, result["messages"][-1].content)],
        "plan": state["plan"][1:],          # pop the executed step
    }

# [REPLAN] strong model again — given past_steps, decide:
#   1) "we're done, respond" OR  2) "replan: new steps"
# A conditional edge routes by the replanner's output type.

Use When

  • The task has clear sequential structure (research → analyze → write)
  • You want to show the plan to the user before executing
  • You need to tier models: strong for plan, cheap for exec

Avoid When

  • Steps depend heavily on results of earlier steps (the plan is stale by step 2)
  • Tool calls might fail in surprising ways → constant replanning kills the savings
  • The task is short (1-3 steps) — overhead exceeds benefit
05 / Reflexion
Generate · Critique · Revise
An agent does its first attempt. A critic (often the same model with a different prompt) scores it. The agent revises. Loop until the score crosses a threshold — or you run out of budget.
Reflexion Loop

↑ if no → back to DRAFT with critique appended (attempt N+1)
reflexion.py — generate / critique / revise
Structured critique output
class Critique(BaseModel):
    score: float = Field(..., ge=0, le=1)
    issues: list[str] = Field(..., max_length=5)
    must_fix: bool

class State(TypedDict):
    task: str
    draft: str
    critiques: list[Critique]
    attempt: int

# DRAFT — first attempt + revisions consume prior critiques.
draft_chain  = draft_prompt  | model
critic_chain = critic_prompt | model.with_structured_output(Critique)

def draft_node(s: State):
    msg = {"task": s["task"], "critiques": s["critiques"]}
    return {"draft": draft_chain.invoke(msg).content,
            "attempt": s["attempt"] + 1}

def critic_node(s: State):
    crit = critic_chain.invoke({"task": s["task"], "draft": s["draft"]})
    return {"critiques": s["critiques"] + [crit]}

def route(s: State):
    latest = s["critiques"][-1]
    if latest.score >= 0.85:  return "__end__"      # quality bar
    if s["attempt"] >= 3:    return "__end__"      # budget cap
    return "draft"                                  # revise
WRITING

Drafts → critique on style/clarity/structure → revise. Often the difference between B+ and A− output.

CODE

Generate → run tests → revise on failures. Self-correcting code agents lean heavily on this pattern.

STRUCTURED OUTPUT

Schema validation IS the critique. Pydantic fails → feed error back as the critique → revise. Cheaper than a critic LLM.

Diminishing Returns

Attempt 1 → 2 helps a lot. 2 → 3 helps less. 3 → 4 usually doesn't help — and may regress because the critic becomes a yes-man on the third pass. Cap at 3 attempts. If you can't reach quality in 3, the prompt needs rework — not more loops.

06 / Tree of Thoughts
Many Branches, One Winner
Explore N candidate next-thoughts in parallel. Score each. Expand only the most promising. Prune the rest. Best-First Search with the LLM as both generator and scorer.

EXPAND

Generate K candidate next-thoughts from current node.

SCORE

LLM scores each by promise — heuristic, value-fn, or success probability.

PRUNE + EXPAND

Keep top-B (beam). Recurse on each. Stop at depth D or solution found.

Worth the Cost

  • Math / logic puzzles · Game of 24 · constraint satisfaction
  • Hard code synthesis where first attempt is usually wrong
  • Creative ideation where exploring diverse directions is the goal
  • Offline evaluation runs (where cost is amortized)

The Bill

  • K × D nodes total — 5 × 4 = 20 LLM calls minimum
  • Plus a scoring call per node — typically 10-20× ReAct
  • Latency = D × max-step-time — kills real-time UX
  • Rarely worth it for online customer-facing agents
Verdict

Tree of Thoughts is a research tool, not a default. Reach for it only when a single attempt demonstrably fails.

07 / Pick the Pattern
A 4-Question Decision Framework
Walk the four questions top-down. The first "yes" tells you which pattern fits. Most production agents stop on Q1.
Question ReAct P&E Reflexion ToT
Q1. Will a single attempt usually work? ★ yes — — —
Q2. Long task with clear sequential structure? drifts ★ yes — —
Q3. Output quality benefits from explicit critique? no no ★ yes —
Q4. Hard search problem · first attempt usually wrong? no replan helps ★ yes
PATTERNS COMPOSE

Plan-and-Execute where each step is a ReAct sub-agent. Or Reflexion wrapping the final output of a Plan-and-Execute. They aren't exclusive — they're layers.

PRODUCTION DEFAULT

ReAct + structured-output critique is a strong default. You get the simplicity of ReAct with a one-pass quality gate via Pydantic validation. Add full Reflexion only if eval shows it earns the cost.

08 / Planning Pitfalls
Where Planning Backfires
Five failure modes that show up the first time teams reach past ReAct. Recognize them early.
Five failure modes · click to expand the fix
Interactive
P1 · Planning Theatre
P2 · Stale Plans
P3 · Over-Reflection
P4 · Branch Explosion
P5 · Self-Critic Yes-Manning
P1

Planning Theatre

A 5-step plan for a task ReAct would have done in 2 turns.

Fix: bench ReAct first. Only escalate if ReAct's failure rate or step count is measurably bad. Don't pick a heavier pattern because it sounds more "serious".

P2

Stale Plans

Plan made at step 0 doesn't fit reality by step 3 — executor blindly follows it.

Fix: wire a real replanner — not just an early-exit. The replanner sees past_steps and produces either "respond now" or a fresh plan from this point.

P3

Over-Reflection

Reflexion runs 5 attempts. Attempt 5 is worse than attempt 2.

Fix: hard-cap at 3 attempts. Track the score per attempt and stop if it doesn't improve from N → N+1. Keep best-so-far, not the last attempt.

P4

Branch Explosion

ToT with K=8 candidates × D=4 depth = 4096+ LLM calls for one query.

Fix: beam width B = 1 or 2. Aggressive pruning. Hard budget on total nodes per query. Or skip ToT entirely and use Reflexion + ReAct instead.

P5

Self-Critic Yes-Manning

Same model as critic — it rubber-stamps its own output by attempt 2.

Fix: use a different model as critic, or a deterministic check (tests, schema, retriever-grounded faithfulness). Adversarial prompting helps too — "find at least 3 issues" forces the critic to look.

The Through-Line

Planning patterns aren't about more thinking. They're about the right amount of thinking — measured by cost, latency, and eval.
