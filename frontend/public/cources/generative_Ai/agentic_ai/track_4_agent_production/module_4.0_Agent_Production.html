03 / Five Paradigm Shifts
Five Shifts in Mental Model
RAG-trained instincts will mislead you in agent production. Internalize these five shifts before touching infrastructure.
SHIFT 01

Stateless → Stateful

A RAG call is independent. An agent run accumulates state across N steps — and may pause, resume, even restart on a different node.

→ checkpointers, durable state, session affinity
SHIFT 02

Short → Long-running

RAG p99 ≈ 2-5s. Agent p99 can be 60s, 5 min, an hour. Connection lifetimes, timeouts, and load-balancer assumptions all change.

→ streaming, WebSockets, background workers
SHIFT 03

Cheap → Expensive

RAG ≈ $0.005/query. Agent p50 ≈ $0.04, p99 can be $2+. A bad release doesn't crash — it bills you.

→ percentile cost SLOs, circuit breakers
SHIFT 04

Idempotent → Side-effectful

RAG retries are free. Agent retries can re-issue refunds, re-send emails, re-trigger workflows. Retries become a correctness question.

→ idempotency keys, approval gates, audit logs
SHIFT 05

Call → Trajectory

In RAG you trace one LLM call. In agents you trace N steps with branching, retries, and tool calls. Single-span thinking breaks.

→ trajectory-aware tracing & dashboards
CONSEQUENCE

Operations > software

Agent prod is closer to running a distributed workflow engine than a stateless API. Borrow patterns from Temporal, Airflow — not from REST microservices.

→ embrace durable execution, replay, runbooks
04 / Side-by-Side
RAG Prod vs. Agent Prod
A row-by-row map of where assumptions break. Reference this any time you're tempted to reuse a RAG pattern unchanged.
Dimension RAG Production Agent Production
Request shape Single request → single response Goal → multi-step trajectory → result
Latency (p99) ~ 2–5 seconds 10s to minutes; sometimes hours
Cost (p99) ~ $0.005 / query ~ $0.50 – $5 / trajectory
State Stateless Stateful · checkpointed · resumable
Idempotency Idempotent by default (retry-safe) Side-effectful (refunds, emails, writes)
Tracing unit One LLM call span Trajectory tree · tool spans · retries
Failure modes Timeout · 5xx · bad retrieval Loops · tool errors · context bloat · runaway cost
Scaling unit Concurrent requests Concurrent trajectories · long-lived sessions
Auth surface User auth on API User auth + per-tool scoped credentials
Observability Logs · metrics · 1-span traces Trajectory replay · state transitions · per-step cost
Rollback Redeploy old image · done Redeploy + drain in-flight runs + checkpoint migration
Blast radius Bad answer to one user Bad action affecting DB / external system / wallet
The Blast Radius Warning

The single biggest difference: an agent in production touches the real world. A buggy RAG answer is embarrassing. A buggy agent issues 5,000 refunds in 10 minutes. Treat agent production with the operational discipline you'd give a payment system, not a chatbot.

05 / Seven New Concerns
Seven Agent-Specific Production Concerns
RAG production didn't make you face these. Track 4 will hand you the toolbox for each.
CONCERN 01

State persistence

Where do checkpoints live? How long? Who owns the schema when the agent graph changes?

→ Module 4.1 (LangGraph Platform)
CONCERN 02

Streaming & long connections

WebSockets, SSE, partial state delivery. UX assumes the agent is "thinking", not "broken".

→ Module 4.1 (streaming patterns)
CONCERN 03

Trajectory observability

Per-step traces, replay debugging, multi-agent correlation. Logs aren't enough.

→ Module 4.2 (observability)
CONCERN 04

Long-tail cost control

Cascade routing, circuit breakers, batch APIs, prompt compression. Mean cost is fine; tails kill margins.

→ Module 4.3 (cost engineering)
CONCERN 05

Long-term memory

Episodic, semantic, procedural memory; eviction; GDPR. Agents that "remember" need governance.

→ Module 4.4 (memory architectures)
CONCERN 06

Sandboxing actions

Code execution, file system, network policy, per-tool scopes. Containment isn't optional for agents.

→ Module 4.6 (security & sandboxing)
CONCERN 07

Human-in-the-loop governance

Approvals, audit trails, SOC2 / HIPAA / EU AI Act compliance. Agents that act need accountability chains.

→ Module 4.7 (HITL & governance)
CROSS-CUTTING

Versioning & rollout

Prompts, graphs, tools all change. A/B testing, shadow mode, statistical significance — across versions and traffic.

→ Modules 4.5 + 4.8 (versioning + launch audit)
FINAL GATE

Production readiness

50-point launch audit, SLOs/SLIs, runbooks, decommissioning. The capstone checklist for Track 4.

→ Module 4.8 (readiness checklist)

08 / Recap
Key Takeaways
TAKEAWAY 01

RAG production primitives — Docker, K8s, FastAPI, observability, caching, auth — all carry forward. Track 4 layers on what's new, not what's repeated.

TAKEAWAY 02

Five paradigm shifts: stateless → stateful, short → long-running, cheap → expensive, idempotent → side-effectful, call → trajectory.

TAKEAWAY 03

Borrow ops patterns from durable execution engines (Temporal, Airflow), not stateless microservices. Replay, retries, and idempotency are first-class.

TAKEAWAY 04

Agent blast radius is real-world. A bug doesn't 500 — it acts. Use payment-system discipline, not chatbot discipline.

TAKEAWAY 05

Seven new concerns: state, streaming, observability, long-tail cost, memory, sandboxing, governance. Each gets a dedicated module.

TAKEAWAY 06

Track cost in percentiles, not means. A p99 cost spike is the agent-equivalent of a 5xx storm.

TAKEAWAY 07

The agent prod lifecycle has eight stages — deploy, run, observe, optimize, remember, evolve, secure, govern — and each maps to a Track 4 module.

TAKEAWAY 08

The Track ends at Module 4.8: a 50-point readiness checklist. Treat it as the launch gate — not a doc you write after the incident.
