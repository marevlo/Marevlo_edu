Caching → RAG Track 7
From Course 01 you already know how to cache. Here's the quick recall — then we'll see why agent cost needs more than caching.

From Course 01 you already know how to cache. Here's the quick recall — then we'll see why agent cost needs more than caching.

exact-match cache semantic cache (embedding) provider prompt caching retrieval cache (vector store) TTL & invalidation hit-rate / savings
PORTABLE

Still applies verbatim

Caching tool results, prompt-prefix caching for static system prompts, and embedding caches for repeated lookups all carry forward into agents unchanged.

RE-FRAMED

Cost grows with steps

RAG cost = (input + output) × 1 call. Agent cost = Σ(input_k + output_k) × N steps. Cache hit-rate doesn't tell the whole story — step count does.

NEW

Per-trajectory accounting

You now need to attribute cost to which step, which sub-agent, and which tool call. Caching alone won't get you there.

SOURCE 01

Step count

Each thought-action-observe loop = a full LLM round-trip. Loops are linear in step count.

SOURCE 02

Context bloat

Every step appends prior observations. By step 8, the context can be 10× step 1.

SOURCE 03

Long tool outputs

A web_search that returns 5K tokens of HTML gets baked into the next prompt. Truncate at the wire.

SOURCE 04

Reasoning verbosity

CoT-style thoughts produce long output tokens. Output tokens cost 3-5× input on most providers.

# Declare budgets up front. Treat them as a contract.

from dataclasses import dataclass

@dataclass
class TokenBudget:
    system_prompt:   int = 600
    context_window:  int = 4000
    tool_results:    int = 1500
    llm_thoughts:    int = 2000
    final_answer:    int = 500
    max_steps:       int = 7

    @property
    def total(self):
        return (self.system_prompt + self.context_window
                + self.tool_results + self.llm_thoughts
                + self.final_answer)

BUDGET = TokenBudget()                  # 8,600 tok / trajectory
# Enforce at the wire — never let oversized content reach the LLM.

def truncate_observation(obs, budget):
    tokens = count_tokens(obs)
    if tokens <= budget:
        return obs
    # Strategy: keep head + tail, drop middle, mark elision.
    head = slice_tokens(obs, 0, budget * 0.6)
    tail = slice_tokens(obs, -budget * 0.3, None)
    return f"{head}\n[...{tokens - budget} tokens elided...]\n{tail}"

def enforce_step_budget(trajectory, budget):
    if len(trajectory.steps) >= budget.max_steps:
        raise StepBudgetExceeded(
            f"Hit {budget.max_steps} steps without final answer")
# Emit segment-level telemetry on every trajectory.

def record_cost(trajectory, prices):
    by_segment = defaultdict(lambda: {"tok_in": 0, "tok_out": 0, "usd": 0.0})
    for step in trajectory.steps:
        seg  = step.segment            # router | llm | tool | synth
        cost = (step.tok_in  * prices[step.model]["in"] +
                step.tok_out * prices[step.model]["out"])
        by_segment[seg]["tok_in"]  += step.tok_in
        by_segment[seg]["tok_out"] += step.tok_out
        by_segment[seg]["usd"]     += cost
    emit("trajectory.cost", by_segment, trace_id=trajectory.id)
    return by_segment
06 / The 8 Levers
8 Levers for Agent Cost Reduction
Ordered by typical impact in production. Combine 3-4 and a 65% cost cut without quality loss is common.
LEVER 01 HIGH

Model Routing

Use a cheap model for routing and tool args; reserve frontier model for hard reasoning. Often 60% savings alone.

→ Haiku/4-mini for ⅔ of calls, premium for the rest
LEVER 02 HIGH

Prompt Caching

Provider-side caching of static system prompts + tool registry. Typically 80-90% off cached tokens.

→ Pin system prompt + tool defs at the head
LEVER 03 HIGH

Context Pruning

Summarize prior observations once they age out of the active window. Don't carry every step verbatim to step 9.

→ Rolling summary + last-k full observations
LEVER 04 HIGH

Early Exit

Confidence-gated termination. If the model is confident after step 3, don't burn 5 more loops "verifying".

→ Self-confidence ≥ threshold → final_answer
LEVER 05 MID

Parallel Tool Calls

When tools are independent, fan them out in a single step. Cuts latency, not cost — but unlocks early exit later.

→ tool_choice: parallel
LEVER 06 MID

Async Batching

For background workloads, use the provider's batch API — typically 50% off list price. Latency: hours, not seconds.

→ batch_create() for offline pipelines
LEVER 07 MID

Distillation

Fine-tune a small open-weight model on traces from your frontier agent. 10-20× cheaper at inference on hot paths.

→ Llama-class for the router or tool-arg model
LEVER 08 LOW

Structured Output

JSON-mode / Pydantic-as-schema cuts CoT verbosity. Smaller outputs = lower output-token bill.

→ output_schema with no "explain why"

// THE p95 WARNING

At p50 the unit economics are spectacular. At p95 the trajectory is unprofitable. Either price for the tail, cap the tail (max-steps + escalation), or accept that 5% of traffic is loss-leader. The mean alone would have hidden this.

METRIC 01

Cost per resolved task

Total spend ÷ tasks where success = 1. Failed trajectories still cost — don't divide by total attempts.

METRIC 02

Gross margin per trajectory

Revenue minus full cost stack — LLM, tools, safety, infra, escalation. Report p50 and p95.

METRIC 03

Break-even price

Min price per task at which p95 trajectory still clears margin. The number to quote in commercial negotiations.

09 / Recap
Key Takeaways
TAKEAWAY 01

RAG caching primitives carry forward — but they're necessary, not sufficient. Agent cost grows with step count, context bloat, and tool-output size.

TAKEAWAY 02

Profile per trajectory — segment cost into router · llm · tool · synth · overhead. The biggest segment is the optimization target.

TAKEAWAY 03

Report p50 / p95 / p99. Never mean. The top 5% of runs typically drive ~50% of the bill.

TAKEAWAY 04

Declare token budgets per segment and enforce in code. Truncate at retrieval time, not generation time.

TAKEAWAY 05

The big four levers — routing, prompt caching, context pruning, early exit — typically deliver 60-65% cost cuts together.

TAKEAWAY 06

Reach for parallel calls, batching, distillation, structured output as second-pass levers — each adds 5-15% on top.

TAKEAWAY 07

Track gross margin per trajectory at both p50 and p95. The p95 line is what tells you if the unit economics actually work.

TAKEAWAY 08

Cost belongs in CI. Assert percentile ceilings + regression-vs-main + margin floors. A 2× cost regression should block a merge.
