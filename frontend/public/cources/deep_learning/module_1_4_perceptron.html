The perceptron. What a single neuron cannot do.

Before logistic regression, before TensorFlow, before backpropagation — there was the perceptron, the original learning machine of 1958. It worked. Then in 1969 a tiny mathematical proof showed it could never solve a problem as simple as XOR, and the field collapsed. This module covers what the perceptron is, why it failed, and why understanding its failure is the entire motivation for everything that comes next.

01 — Origin
A neuron, simulated.

The story starts with biology. By the late 1940s, neuroscientists understood that brain cells communicate via electrical impulses: dendrites collect signals from neighbors, the cell body sums them, and if the sum exceeds a threshold, the neuron fires — sends a pulse down its axon to the next cell. Otherwise, silence. Binary. On or off.

In 1957, the psychologist Frank Rosenblatt — funded by the US Navy — proposed that this all-or-nothing firing logic could be modeled as a simple machine. He called it the perceptron. Its rule was extraordinarily compact:

output = 1   if   wTx + b > 0 0   otherwise (1.1)

That's it. Take a weighted sum of the inputs. If positive, output 1 (fire). If not, output 0 (silent). The function that does this — outputting 1 above zero, 0 below — is called the step function or Heaviside step. It's the simplest activation function imaginable.

A historical moment

The 1958 New York Times.

Rosenblatt demonstrated the Mark I perceptron — a 400-cell physical machine — and the press lost its mind. July 1958: the New York Times reported that the perceptron was the embryo of an electronic computer that would be able to walk, talk, see, write, reproduce itself, and be conscious of its existence. Rosenblatt, asked when this would happen, said within a year. It did not. The perceptron is to modern AI roughly what the Wright Flyer is to a 787 — undeniably foundational, recognizably the same lineage, and embarrassingly limited.

Comparing the perceptron to logistic regression.

The two models are structurally the same — a single neuron computing wTx + b, then applying an activation. They differ only in the choice of activation (step vs. sigmoid) and how they're trained. Modern logistic regression is, in essence, the perceptron with a smooth activation that lets us use calculus.

02 — Learning rule
The perceptron learning rule.

How did Rosenblatt train the perceptron? He invented an update rule that predates gradient descent for neural networks. The intuition is simple: when the perceptron predicts wrong, nudge the weights in a direction that would have made it more likely to be right. When it predicts correctly, leave the weights alone.

Concretely, for each training example with input x and true label y ∈ {0, 1}:

w ← w + η · (y − ŷ) · x (2.1)

If ŷ matches y, the update is zero — no change. If we predicted 0 but the truth was 1 (we missed a positive), the difference (y − ŷ) = 1, so we add ηx to w — pushing wTx larger so next time we're more likely to fire. Symmetrically for false positives.

The convergence theorem

If the data is linearly separable, the perceptron will find a separator.

In 1962, Rosenblatt and others proved a clean theorem: if there exists any hyperplane that separates the two classes, the perceptron's update rule is guaranteed to find some such hyperplane in a finite number of steps. This is a strong guarantee — no learning rate tuning needed, no convergence concerns. It made the perceptron seem like a near-magical solution. Then came 1969.

import numpy as np

def perceptron_train(X, y, lr=0.1, epochs=100):
    """Rosenblatt's original perceptron learning algorithm."""
    n, d = X.shape
    w = np.zeros(d)
    b = 0.0
    for epoch in range(epochs):
        errors = 0
        for xi, yi in zip(X, y):
            # Forward: step function
            y_pred = 1 if (xi @ w + b) > 0 else 0
            # Update: nudge if wrong
            update = lr * (yi - y_pred)
            w += update * xi
            b += update
            errors += int(update != 0)
        if errors == 0:
            print(f"converged at epoch {epoch+1}")
            break
    return w, b

Six lines of meaningful code. Notice what's not here: no gradient computation, no autograd, no loss function in the modern sense. The learning rule is direct — a behavioral nudge based on whether the prediction was right or wrong. This algorithm could be (and was) implemented in hardware with potentiometers and motors.

Worked example — two perceptron update steps
Initial weights: w = [0, 0], b = 0, η = 1. Two features: (attention_span, runtime_norm)
Example 1: x = [0.8, 0.3], y = 1. Score = 0·0.8 + 0·0.3 + 0 = 0. ŷ = 0 (not > 0). Wrong!Update: w += 1·(1−0)·[0.8, 0.3] = [0.8, 0.3]. b += 1·1 = 1.
Example 2: x = [0.2, 0.9], y = 0. Score = 0.8·0.2 + 0.3·0.9 + 1 = 0.16 + 0.27 + 1 = 1.43. ŷ = 1 (> 0). Wrong!Update: w += 1·(0−1)·[0.2, 0.9] = [0.8−0.2, 0.3−0.9] = [0.6, −0.6]. b += −1 = 0.
Verify example 1 again: 0.6·0.8 + (−0.6)·0.3 + 0 = 0.48 − 0.18 = 0.30 > 0 → ŷ = 1. Correct now!Two updates rotated the decision boundary. If data is linearly separable, convergence is guaranteed.

03 — XOR
XOR — the problem that broke the perceptron.

In 1969, Marvin Minsky and Seymour Papert — both at MIT, both critics of Rosenblatt's hype — published a slim book called Perceptrons. In it, they proved a result that ended an era. They showed there exists a logical function as simple as can be that no perceptron can learn: XOR, exclusive-or.

XOR takes two binary inputs and outputs 1 only when exactly one of them is 1. Here's its truth table:

x₁x₂XOR(x₁, x₂)
000
011
101
110

Plot these four points. The two "outputs of 1" — at (0,1) and (1,0) — sit on opposite corners of a unit square. The two "outputs of 0" — at (0,0) and (1,1) — sit on the other opposite corners. There is no straight line — none whatsoever — that puts the two 1s on one side and the two 0s on the other.

Figure 1. Both AND and XOR have four data points on the unit square. AND is linearly separable — one line works. XOR has its 1s on one diagonal and its 0s on the other — no line can separate them.
The 1969 verdict

Minsky and Papert proved formally that XOR — and an entire class of similar problems — are impossible for any single-layer perceptron, regardless of training. They wrote that adding hidden layers could solve XOR, but they (incorrectly, it turned out) suggested no good way existed to train such multi-layer networks. The book was widely read and widely cited as evidence that neural networks were a dead end. Funding evaporated. The first AI winter began.

Why does XOR break a single neuron?

Every linear classifier — perceptron, logistic regression, linear SVM — divides the input space with a single hyperplane. Problems whose decision regions are not connected by a single hyperplane are fundamentally outside the hypothesis class. No amount of training, no clever loss function, no different optimizer fixes this. The model literally cannot represent the function.

A useful generalization: any problem whose solution requires the model to learn an interaction between features (rather than weighted sums of features individually) is non-linearly separable. XOR is the canonical example because it requires "1 if x₁ and x₂ disagree, 0 if they agree" — a relation between the inputs, not a sum of them.

04 — The fix
The fix — and why it took 17 years.

Minsky and Papert themselves noted that XOR could be solved if you allowed two layers — first compute some intermediate features, then combine them. Visually:

Figure 2. A 2-layer network solving XOR. The hidden layer learns two intermediate features (OR and NAND); the output layer combines them with AND. XOR = (x₁ OR x₂) AND NOT(x₁ AND x₂). Three perceptrons, two layers — and the impossible becomes routine.

The hidden layer is where the magic happens. By allowing the network to compute intermediate values that are themselves functions of the inputs, and then combining those, we get expressive power well beyond any single linear classifier. The hidden values can encode interactions, conjunctions, and other non-linear relationships.

Why 17 years?

The problem wasn't representation — it was training.

Multi-layer networks were obvious in principle from the day Minsky and Papert published. The hard part was: how do you train them? The perceptron rule worked because the output layer's error could be directly attributed to its weights. With hidden layers, what's the gradient with respect to a hidden unit's weights? Unclear — the unit doesn't have a target value of its own. 1986: David Rumelhart, Geoffrey Hinton, and Ronald Williams published the answer — backpropagation. Same chain rule from Module 0.3, applied recursively through layers. It made training arbitrarily-deep networks tractable. And the field came back to life.

05 — Thesis
The Phase 1 thesis, in one sentence.

Everything we've built — linear regression, logistic regression, the perceptron — is a single neuron. One weighted sum, one activation, one prediction. Each variant differs only in its choice of activation and loss, and each is trained by some form of gradient-style update.

This single-neuron family is impressively useful. Linear regression handles continuous prediction. Logistic regression handles binary classification. Softmax classification handles multi-class. With well-engineered features, this family solved most of the predictive modeling industry needed before the 2010s. Modern deep learning still uses these models all the time — every output layer of every neural network is one of these three.

But each is fundamentally limited in the same way. They can only represent linear functions of their inputs (after the activation). When the relationship between features and target involves interactions, ratios, distances, or other non-linear combinations, the single neuron is helpless. Real-world problems — recommendation, vision, language — are full of such structure.
