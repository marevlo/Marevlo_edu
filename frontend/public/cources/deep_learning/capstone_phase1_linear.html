Phase 1 Capstone. CineMatch linear baselines.

Before building anything deep, you need a baseline that's fast to train, easy to debug, and sets the floor for every subsequent model. This capstone builds two: a linear regression model predicting watch-time and a logistic regression model predicting "will rate 4+ stars?" Both use the 4 CineMatch features from Module 0.4, both train in seconds, and both establish the numbers that Phase 2's MLP must beat.

01 — The two baselines
The two baselines.

Baseline 1 — regression. Predict continuous watch-time (minutes) from the 4 CineMatch features: user_avg_rating, movie_popularity, genre_match_score, time_since_last_watch. Loss: MSE. Metric: RMSE on the test set.

Baseline 2 — classification. Predict binary "will rate 4+ stars?" from the same 4 features. Loss: binary cross-entropy. Metrics: accuracy, AUC-ROC on the test set.

# cinematch/models/linear.py
import torch
import torch.nn as nn

class LinearRegression(nn.Module):
    """Watch-time prediction: y = Xw + b"""
    def __init__(self, in_features=4):
        super().__init__()
        self.linear = nn.Linear(in_features, 1)

    def forward(self, x):
        return self.linear(x).squeeze(-1)


class LogisticRegression(nn.Module):
    """Rate 4+? prediction: p = σ(Xw + b)"""
    def __init__(self, in_features=4):
        super().__init__()
        self.linear = nn.Linear(in_features, 1)

    def forward(self, x):
        return self.linear(x).squeeze(-1)  # logits; use BCEWithLogitsLoss

Both models have exactly 5 parameters (4 weights + 1 bias). Training is convex — every run finds the same optimum regardless of initialization (Module 1.1, 1.3).

02 — Training & evaluation
Training loop and evaluation.
# Train both baselines
def train_baseline(model, X_train, y_train, X_val, y_val,
                   loss_fn, lr=0.01, epochs=100):
    opt = torch.optim.SGD(model.parameters(), lr=lr)
    for epoch in range(epochs):
        model.train()
        pred = model(X_train)
        loss = loss_fn(pred, y_train)
        opt.zero_grad(); loss.backward(); opt.step()

        if (epoch + 1) % 20 == 0:
            model.eval()
            with torch.no_grad():
                val_pred = model(X_val)
                val_loss = loss_fn(val_pred, y_val)
            print(f"epoch {epoch+1:3d} | train {loss:.4f} | val {val_loss:.4f}")
    return model

# Regression baseline
reg_model = train_baseline(

    loss_fn=nn.MSELoss(), lr=0.01, epochs=200

CineMatch Integration

These numbers are the bar.

Every subsequent model's value is measured relative to these baselines. The Phase 2 MLP reaches 93.5% accuracy / 0.967 AUC. The Phase 3 LSTM gets Hits@10 = 26%. The Phase 4 transformer gets Hits@10 = 38%. Each improvement is meaningful because we know the linear floor. Without baselines, you can't tell if a complex model is adding value or just adding complexity.

What the trained logistic regression actually computes
Trained weights: w = [0.42, 1.15, −0.38, 0.67], b = −0.894 weights + 1 bias = 5 parameters total. Trained in < 1 second.
New user-movie pair: x = [0.7, 0.85, 0.2, 0.6] (avg_rating=0.7, popularity=0.85, genre_match=0.2, recency=0.6)
Score: z = 0.42·0.7 + 1.15·0.85 + (−0.38)·0.2 + 0.67·0.6 + (−0.89) = 0.294 + 0.978 − 0.076 + 0.402 − 0.89 = 0.708
Probability: σ(0.708) = 1/(1+e^{−0.708}) = 0.670 — model predicts 67% chance the user rates 4+Above 0.5 threshold → predict "will rate 4+". But only 67% confident — near the boundary.
Interpretation: popularity (w=1.15) dominates. A popular movie with moderate genre match gets recommended. The model can't capture "this user only likes obscure horror" — that requires nonlinear interactions that Phase 2's MLP will learn.This is exactly the ceiling the linear model hits at 72.4% accuracy.
BaselinesQuestion 01 / 06
Why build a linear baseline before training a deep model?
ALinear models are always better than deep models.
BPyTorch requires a linear model to be trained first.
CThe baseline sets the floor. If a 5-parameter linear model reaches 72% accuracy, a 2,433-parameter MLP that reaches 73% isn't worth the complexity. The baseline quantifies the marginal value of every subsequent addition — without it, you can't distinguish genuine improvement from noise.
DTo initialize the deep model's weights.
03 — Limitations
What the linear model can't do.

Linear models can only learn linear decision boundaries. If the relationship between genre_match_score and watch-time is non-linear (e.g., users who are moderate genre-matchers watch longest, while perfect-matchers and zero-matchers both watch less), linear regression can't capture that U-shape. The logistic regression can only draw a straight hyperplane through 4D feature space — no curves, no interactions.

This is exactly what motivates Phase 2's MLP: a single hidden layer with ReLU activations can learn any continuous function (the UAT from Module 2.1), including the non-linear interactions that the linear baseline misses. The gap between 72.4% (linear) and 93.5% (MLP) is the value of non-linearity.

Linear limitationsQuestion 02 / 06
The logistic regression baseline has a 72.4%→93.5% gap to the Phase 2 MLP. What causes this gap?

The hidden layer's ReLU activations create piecewise-linear decision boundaries that approximate any non-linear relationship.
CThe MLP uses a different loss function.

ConvexityQuestion 03 / 06
You train the logistic regression baseline from 10 different random initializations. All 10 reach the same final accuracy. Why?
ABCE loss of a logistic regression is convex — every local minimum is the global minimum. Regardless of initialization, gradient descent finds the same optimum.
BThe model has only 5 parameters, so there's only one possible solution.
CPyTorch uses the same random seed by default.
DSGD always converges to the same point regardless of the loss surface.
04 — Production role
The baseline's role in the CineMatch pipeline.

These linear baselines aren't just training exercises — they persist in the production stack as sanity checks and fallbacks. If the transformer recommender's validation loss suddenly degrades (bad deploy, data pipeline bug), CineMatch falls back to the logistic regression baseline, which is immune to the kinds of failures that affect complex models. The baseline also serves as the "null hypothesis" for every A/B test: a new model must beat logistic regression by a statistically significant margin to justify deployment.

Production roleQuestion 04 / 06

ALogistic regression produces the same quality recommendations.
BLogistic regression doesn't require a GPU.
CLogistic regression never makes mistakes.
DThe logistic regression is simple (5 parameters, no GPU needed, sub-millisecond inference), reliable (convex loss = deterministic training), and still provides reasonable recommendations (72% accuracy vs 50% random). It's much better than showing nothing or random results while the transformer is being fixed.
EvaluationQuestion 05 / 06
Your logistic regression has 72.4% accuracy and 0.791 AUC. A colleague's MLP has 73.1% accuracy and 0.802 AUC. Is the MLP worth deploying?
AYes — higher is always better.
BNo — the MLP is worse because it has more parameters.
CProbably not. The 0.7% accuracy gain and 0.011 AUC gain are within noise for most dataset sizes. An MLP with 2,400+ parameters that barely beats a 5-parameter baseline likely isn't learning meaningful non-linear patterns — it's just slightly overfitting. Increase MLP capacity, add better features, or use a fundamentally different approach (sequences, attention). The MLP must substantially beat the baseline to justify its complexity.
DNeed more epochs to decide.
SynthesisQuestion 06 / 06
Across the entire DL track, CineMatch's classification accuracy progressed: 50% (random) → 72% (logistic) → 93.5% (MLP) → transformer-based Hits@10 from 14% to 38%. What's the single biggest jump and what caused it?
AMLP → transformer (+24% Hits@10) — attention is the biggest factor.
BLogistic → MLP (+21.1% accuracy) — the introduction of non-linearity. Going from a linear decision boundary to piecewise-linear (ReLU hidden layer) captures feature interactions that account for the single largest accuracy gain in the entire track. Every subsequent improvement (CNN features, LSTM sequences, attention, pretraining) adds less individually than this first step from linear to non-linear.
CRandom → logistic (+22% accuracy) — the introduction of features.
