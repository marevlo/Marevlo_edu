D1 / DERIVATION
IPS — why inverse propensity weighting is unbiased
↳ corresponds to M30 S3: IPS debiasing
Real-life analogy

Imagine surveying restaurant satisfaction, but you only survey people who walked past the restaurant (your "exposure" policy). Busy streets get more surveys than quiet streets. A restaurant on a busy street might get 100 surveys averaging 4.2 stars, while an equally good restaurant on a quiet street gets 3 surveys averaging 4.5 stars. IPS corrects this: weight each survey by 1/(probability of walking past). The busy-street surveys get low weight (1/100); the quiet-street surveys get high weight (1/3). The weighted average reflects true quality, not foot traffic.

IPS estimator and unbiasedness proof
$$\hat{R}_{\text{IPS}}(\pi) = \frac{1}{N}\sum_{n=1}^N \frac{r_n}{\hat{p}_n} \cdot \mathbb{1}[o_n = 1]$$
$r_n$Reward (click/purchase) for interaction $n$
$\hat{p}_n$Propensity: estimated probability that item $n$ was shown to the user by the logging policy
$o_n$Observation indicator: 1 if item was shown, 0 if not
Unbiased$\mathbb{E}[\hat{R}_{\text{IPS}}] = \mathbb{E}\left[\frac{r_n \cdot \mathbb{1}[o_n=1]}{p_n}\right] = \frac{\mathbb{E}[r_n] \cdot p_n}{p_n} = \mathbb{E}[r_n]$. The propensity in the denominator cancels the selection bias in the numerator. This works for ANY logging policy, as long as $p_n > 0$ for all items (every item has some chance of being shown)
SNIPS — Self-Normalized IPS
$$\hat{R}_{\text{SNIPS}}(\pi) = \frac{\sum_n r_n / \hat{p}_n}{\sum_n 1 / \hat{p}_n}$$
Why SNIPS?Standard IPS has high variance when propensities are small (rare items get weight 1/0.001 = 1000). SNIPS normalizes by the sum of weights, which bounds the estimator to the range of observed rewards. Slightly biased but much lower variance — usually a better bias-variance tradeoff in practice
D2 / FORMALIZATION
Thompson sampling — Beta-Bernoulli model
↳ corresponds to M30 S5: Thompson sampling
Thompson sampling for item exploration
$$\text{Prior: } \text{CTR}_i \sim \text{Beta}(\alpha_i, \beta_i) \quad \text{initialized } \alpha_i = \beta_i = 1$$ $$\text{After } s_i \text{ successes and } f_i \text{ failures: } \text{CTR}_i \sim \text{Beta}(\alpha_i + s_i, \beta_i + f_i)$$ $$\text{At each step: sample } \tilde{\theta}_i \sim \text{Beta}(\alpha_i + s_i, \beta_i + f_i) \text{ for each item}$$ $$\text{Recommend: } i^* = \arg\max_i \tilde{\theta}_i$$
$\alpha, \beta$Beta distribution parameters. $\alpha - 1$ = number of clicks, $\beta - 1$ = number of non-clicks. The Beta distribution naturally captures "how uncertain are we about this item's true CTR?"
New item$\text{Beta}(1, 1) = \text{Uniform}(0, 1)$. Maximum uncertainty. Samples are uniformly random → high exploration probability
1000-click item$\text{Beta}(51, 951)$ (5% CTR). Very peaked around 0.05. Samples barely vary → nearly deterministic → pure exploitation
Thompson sampling: 3 items, 5 rounds
Every step
SETUP

3 items. True CTRs: A=0.08, B=0.05, C=0.12 (unknown to the algorithm). All start at Beta(1,1).

ROUND 1
$$\text{Sample: } \tilde{\theta}_A = 0.72, \tilde{\theta}_B = 0.31, \tilde{\theta}_C = 0.85$$ $$\text{Show C (highest sample). User clicks → C: Beta(2, 1)}$$
ROUND 2
$$\text{Sample: } \tilde{\theta}_A = 0.45, \tilde{\theta}_B = 0.68, \tilde{\theta}_C = 0.71$$ $$\text{Show C again. No click → C: Beta(2, 2)}$$
ROUND 3
$$\text{Sample: } \tilde{\theta}_A = 0.59, \tilde{\theta}_B = 0.22, \tilde{\theta}_C = 0.41$$ $$\text{Show A (C's samples are lower now — more data pulled it toward true CTR)}$$ $$\text{No click → A: Beta(1, 2)}$$
PATTERN

After many rounds, C (true CTR=0.12) dominates — it gets shown most because its Beta posterior concentrates at the highest value. But A and B still occasionally get sampled because their posteriors still have some spread. Thompson sampling naturally shifts from exploration to exploitation as data accumulates — no ε parameter to tune.

Problem: Gini coefficient for exposure monitoring
Interview
PROBLEM

Kartify has 480K items. In the last week, the top 1% of items (4,800) received 60% of all impressions. Compute the Gini coefficient and interpret it.

SOLUTION
$$G = \frac{\sum_{i=1}^n \sum_{j=1}^n |x_i - x_j|}{2n^2 \bar{x}}$$

For a quick approximation: if 1% of items get 60% of exposure, the Lorenz curve bows significantly. $G \approx 0.85$ (very concentrated). For reference: $G = 0$ means perfectly uniform exposure (every item gets the same impressions). $G = 1$ means one item gets all impressions.

Kartify's target: $G < 0.70$. At $G = 0.85$, the feedback loop is severe — the system is converging on a narrow set of popular items. Actions: increase Thompson sampling exploration rate, enforce minimum impression quotas for new items, or add an explicit diversity term to the ranking objective.

Problem: Position bias correction in training
Interview
PROBLEM

Position 1 has CTR=15%, position 10 has CTR=1.5%. You train a ranking model on raw click labels. How does position bias affect the model, and how do you correct it?

SOLUTION
$$P(\text{click} \mid u, i, p) = P(\text{examine} \mid p) \times P(\text{click} \mid u, i, \text{examined})$$

The examination hypothesis: a click requires (1) the user examines the position and (2) the item is relevant given examination. Position affects (1) but not (2). The model should learn (2) — item relevance — not (1).

Correction: Include position as a feature during training. The model learns: "this item had 15% CTR AND was in position 1" → it attributes part of the CTR to position, part to item quality. At serving time, set position = mean position (e.g., 5) for all items → the model predicts quality-only CTR without position boost. This is the standard approach used at Google, Meta, and most production systems.
