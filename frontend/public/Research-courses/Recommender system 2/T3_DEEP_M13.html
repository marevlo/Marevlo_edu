D1 / FORMALIZATION
Weighted blend — tuning α, β, γ
↳ corresponds to M13 S2: Three Hybrid Strategies
Weighted blend scoring
$$\text{score}(u, i) = \alpha \cdot s_{\text{pop}}(i) + \beta \cdot s_{\text{cbf}}(u, i) + \gamma \cdot s_{\text{cf}}(u, i)$$ $$\text{subject to: } \alpha + \beta + \gamma = 1, \quad \alpha, \beta, \gamma \geq 0$$
$s_{\text{pop}}(i)$Popularity score for item $i$ (Module 06). Same for all users — global relevance signal
$s_{\text{cbf}}(u,i)$Content-based score (Module 07). Cosine between user profile and item features
$s_{\text{cf}}(u,i)$Collaborative filtering score (Modules 09–11). Item-kNN aggregated similarity
$\alpha, \beta, \gamma$Blend weights. Tuned by grid search on validation nDCG@10. Kartify defaults: $\alpha=0.15, \beta=0.25, \gamma=0.60$ for users with 5+ interactions

Tuning procedure

Prerequisite: All three source scores must be on comparable scales. Normalize each to $[0, 1]$ range before blending: $s_{\text{norm}} = (s - s_{\text{min}}) / (s_{\text{max}} - s_{\text{min}})$.

Grid search for optimal weights
$$(\alpha^*, \beta^*, \gamma^*) = \arg\max_{\alpha+\beta+\gamma=1} \text{nDCG@10}_{\text{val}}(\alpha, \beta, \gamma)$$
Search grid$\alpha \in \{0, 0.05, 0.10, \ldots, 1.0\}$, $\beta \in \{0, 0.05, \ldots, 1-\alpha\}$, $\gamma = 1-\alpha-\beta$. Total: $\binom{21}{2} = 210$ combinations. Each evaluated on validation set — takes minutes
D2 / WORKED
SAR — matrix multiplication on a 3×4 example
↳ corresponds to M13 S3: SAR
Real-life analogy

SAR is like asking: "For each item Priya has interacted with (weighted by recency), how similar is every other item to it? Sum those up." It's a two-step lookup: (1) how strongly is Priya connected to each item she's seen? (affinity), (2) how similar is each candidate to Priya's seen items? (similarity). The product gives scores for all items.

SAR: 3 users × 4 items
Every multiplication
STEP 1: AFFINITY MATRIX A (with decay)

Binary interactions with time decay ($\lambda = 0.1$):

$$A = \begin{bmatrix} 0.95 & 0.50 & 0 & 0 \\ 0 & 0.90 & 0.74 & 0 \\ 0.82 & 0 & 0 & 0.95 \end{bmatrix}$$

User 1 clicked item A recently (0.95) and item B a while ago (0.50). User 3 clicked A (0.82) and D (0.95).

STEP 2: ITEM SIMILARITY S (Jaccard)
$$S = \begin{bmatrix} 1.0 & 0.33 & 0 & 0.50 \\ 0.33 & 1.0 & 0.50 & 0 \\ 0 & 0.50 & 1.0 & 0.33 \\ 0.50 & 0 & 0.33 & 1.0 \end{bmatrix}$$
STEP 3: SCORES = A × S

User 1's scores (row 1 of $A$ × columns of $S$):

$$\text{score}(1, A) = 0.95 \times 1.0 + 0.50 \times 0.33 = 0.95 + 0.165 = 1.115$$ $$\text{score}(1, B) = 0.95 \times 0.33 + 0.50 \times 1.0 = 0.314 + 0.50 = 0.814$$ $$\text{score}(1, C) = 0.95 \times 0 + 0.50 \times 0.50 = 0 + 0.25 = 0.250$$ $$\text{score}(1, D) = 0.95 \times 0.50 + 0.50 \times 0 = 0.475 + 0 = 0.475$$

Exclude seen items (A, B). Recommendations for User 1: D (0.475) > C (0.250).

D ranks higher because it's similar to item A (Jaccard 0.50), which User 1 interacted with recently (affinity 0.95). C is only similar to B (Jaccard 0.50), which User 1 interacted with less recently (affinity 0.50).

Problem 1: Design a hybrid for a cold-start heavy platform
System Design
PROBLEM

A classifieds platform (like OLX) has 80% anonymous visitors, 15% users with 1–5 interactions, and 5% power users with 50+ interactions. Design a hybrid strategy.

SOLUTION

Switching strategy (M13 Strategy 2):

• 80% anonymous: Category-specific popularity (M06, time-windowed within the category the user is browsing). Zero personalization, but category context narrows the relevance space.

• 15% thin users (1–5): CBF (M07) — even 1 click gives a content profile. Weight: 70% CBF + 30% category popularity. CBF's filter bubble risk is acceptable with so few interactions.

• 5% power users (50+): Full hybrid — 15% popularity + 25% CBF + 60% item-kNN (M09). CF signal is strong here. SAR (M13) could replace the weighted blend.

Key insight: the switching thresholds ({0, 1–5, 50+}) should be treated as confidence levels, not hard categories. A soft blend that continuously shifts weight from popularity → CBF → CF as interaction count grows is more robust than hard switches.

Problem 2: Diagnose a production issue
Debugging
PROBLEM

After deploying item-kNN on Kartify, the team notices: (a) recommendations are repetitive — the same 50 items appear for everyone, (b) new products uploaded yesterday have zero recommendations, (c) nDCG@10 improved by 20% offline but online CTR is flat. Diagnose each issue and propose fixes using concepts from M06–M13.

SOLUTION

(a) Repetitive recommendations: Missing popularity dampening (M11 D3). Popular items have high co-interactions with everything → high similarity to all items → appear in every user's list. Fix: add IDF dampening ($1/\log(1+|\mathcal{H}_u|)$) to the similarity computation, or apply diversity re-ranking (MMR) post-scoring.

(b) New products invisible: Item cold-start (M07/M08). Item-kNN requires co-interactions to compute similarity — new items have zero → zero similarity → never recommended. Fix: add CBF as a retrieval source (M07). New products can be matched to existing products by content features from day one. This is why M13's hybrid pipeline has both CBF and CF as candidate sources.

(c) Offline up, online flat: Offline-online gap (M05 D2). Three likely causes: (1) Selection bias — the test set only contains items shown by the old system; the new model recommends different items with no ground truth. (2) Position bias — the model learned position effects from training data. (3) The offline metric (nDCG) doesn't correlate with the online metric (CTR) — they measure different things. Fix: run a shadow deployment first, then A/B test with long-horizon metrics (retention, not just CTR).

Problem 3: The math behind the ceiling
Conceptual
PROBLEM

Prove mathematically why neighborhood methods have a "sparsity ceiling" that matrix factorization doesn't.

SOLUTION

Neighborhood methods compute similarity as a function of co-interactions: $\text{sim}(i,j) = f(\{u : r_{ui} > 0 \text{ AND } r_{uj} > 0\})$. If $\{u\} = \emptyset$ (no shared users), then $\text{sim}(i,j) = 0$ regardless of how related the items are.

At Kartify's sparsity (0.002% fill), the probability of two random items sharing even one user is ~0.06% (computed in DEEP M01 D5). This means 99.94% of item pairs have exactly zero collaborative similarity. The model is blind to relationships in that 99.94%.

Matrix factorization learns embeddings $\mathbf{v}_i, \mathbf{v}_j \in \mathbb{R}^k$. The predicted similarity is $\mathbf{v}_i^\top \mathbf{v}_j$ — a dot product in latent space. Even if items $i$ and $j$ have zero co-interactions, their embeddings are influenced by items they DO share users with. If $i$ is similar to item $m$, and $m$ shares users with $j$, then $\mathbf{v}_i$ and $\mathbf{v}_j$ will be pulled into similar regions of latent space — giving them non-zero predicted similarity without any direct co-interaction.

$$\text{Neighborhood: } \text{sim}(i,j) = 0 \text{ if } U_i \cap U_j = \emptyset$$ $$\text{MF: } \text{sim}(i,j) = \mathbf{v}_i^\top \mathbf{v}_j \neq 0 \text{ even if } U_i \cap U_j = \emptyset$$

This is the transitive generalization that MF provides and neighborhood methods cannot. It's why Module 14 — matrix factorization — is the most important algorithmic leap in the track.

D4 / ADDITIONAL INTERVIEW
Interview problems — hybrid systems
Real-life analogy — hybrid weighting

A hybrid recommender is like a doctor consulting multiple specialists. The cardiologist (CF model) has deep expertise in one area. The generalist (CBF model) has broad but shallow knowledge. The internist (popularity) knows what works for most people. A good doctor weights their opinions based on the patient: for a patient with extensive medical history (many interactions), trust the specialist. For a new patient (cold start), lean on the generalist. The hybrid blending weights are the doctor's judgment about which specialist to trust for THIS patient.

Problem: Hybrid for a two-sided marketplace
Interview
PROBLEM

Kartify has both buyers and sellers. Sellers want their items to be recommended. Buyers want relevant items. Design a hybrid that balances both sides. What models would you blend, and how would you weight them?

SOLUTION

Buyer side: Use item-kNN + BPR-MF for relevance (optimize for buyer satisfaction). These models predict what the buyer wants to see.

Seller side: Add a "seller boost" component that gives weight to items from sellers who are paying for promotion or who have low visibility (fairness). This is essentially a business rule layer.

Blending: $ ext{score} = lpha \cdot ext{CF\_score} + eta \cdot ext{CBF\_score} + \gamma \cdot ext{seller\_boost}$. Set $lpha > eta > \gamma$ to prioritize buyer relevance while giving sellers meaningful exposure. The weights should be tuned via A/B test — too much seller boost degrades buyer experience and hurts long-term retention.

Problem: When does a hybrid HURT performance?
Interview
PROBLEM

You blend item-kNN and CBF with equal weights ($lpha = eta = 0.5$). The hybrid performs WORSE than item-kNN alone. What went wrong?

SOLUTION

If CBF is significantly weaker than item-kNN on this dataset, blending with equal weights dilutes the strong signal with noise. The CBF scores may not be calibrated to the same scale as kNN scores — if CBF scores range [0, 0.3] while kNN scores range [0, 0.9], the CBF component barely affects the ranking, or worse, it pushes irrelevant items up.

Fixes: (1) Normalize scores to [0, 1] before blending. (2) Learn the weights via cross-validation instead of setting them manually. (3) Use rank-based blending (Borda count) instead of score-based — average the RANKS, not the scores. (4) Check if CBF is actually providing complementary signal — if it's just noise, set $eta = 0$ and use kNN alone. A hybrid is only worth the complexity if the components provide different, complementary information.
