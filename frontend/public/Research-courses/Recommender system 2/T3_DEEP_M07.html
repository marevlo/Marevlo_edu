D1 / DERIVATION
TF-IDF — from information theory
↳ corresponds to M07 S3: TF-IDF — Smart Weighting
Real-life analogy

Imagine reading a detective novel. The word "the" appears on every page — it tells you nothing about what makes this page special. The word "murder" appears on 3 pages — when you see it, you know something important is happening. The word "cyanide" appears on 1 page — it's extremely informative about that specific page. TF-IDF formalizes this: common words get low weight (they're uninformative), rare words get high weight (they're distinctive). For Kartify, replace "words" with "product tags."

Term Frequency (TF)
$$\text{TF}(t, d) = \frac{f_{t,d}}{\sum_{t' \in d} f_{t',d}}$$
$t$A term (tag). For Kartify: "ethnic", "cotton", "floral", "banarasi", etc.
$d$A document (product). Each Kartify product's tag list is a "document"
$f_{t,d}$How many times term $t$ appears in document $d$. For binary tags, this is 0 or 1
DenominatorTotal tag count for product $d$. Normalizes for products with many tags vs few
Inverse Document Frequency (IDF)
$$\text{IDF}(t) = \log\frac{N}{|\{d \in \mathcal{D} : t \in d\}|}$$
$N$Total number of documents (products). Kartify: $N = 480{,}000$
$|\{d : t \in d\}|$How many products have this tag. "clothing" appears in 456,000 products → IDF is tiny. "banarasi" appears in 1,440 products → IDF is large
$\log$Logarithm compresses the ratio. Without it, a tag in 1 product vs 480K products would have a 480,000:1 ratio. Log makes it ~13:0
TF-IDF combined
$$\text{TF-IDF}(t, d) = \text{TF}(t, d) \times \text{IDF}(t)$$

High TF-IDF: the tag appears in this product (high TF) AND is rare across the catalog (high IDF). This is a distinctive tag for this product. Low TF-IDF: the tag is common (low IDF) or absent from this product (TF = 0).

Information-theoretic interpretation

IDF is the surprisal (self-information) of encountering a tag in a random product. If tag $t$ appears in fraction $p_t$ of products, the surprisal is $-\log(p_t) = \log(1/p_t) = \log(N / |\{d : t \in d\}|)$. A common tag (high $p_t$) is unsurprising (low information). A rare tag (low $p_t$) is surprising (high information). TF-IDF weights features by their information content — exactly what we want for distinguishing between products.

D2 / PROOF
Cosine similarity — scale invariance proof
↳ corresponds to M07 S2: Three Steps (cosine in Step 3)
Real-life analogy

Imagine two arrows pointing in the same direction on a map. One arrow is 1cm long, the other is 100cm long. They point the same way — the angle between them is 0°. Cosine similarity measures the ANGLE, not the length. Two users who agree on everything but one clicked 10 items and the other clicked 1,000 have cosine ≈ 1, because they point in the same direction in taste space — one just has more data.

Proof: cosine is scale-invariant
$$\text{Claim: } \cos(\alpha \mathbf{a}, \beta \mathbf{b}) = \cos(\mathbf{a}, \mathbf{b}) \quad \forall \alpha, \beta > 0$$ $$\text{Proof: } \cos(\alpha \mathbf{a}, \beta \mathbf{b}) = \frac{(\alpha \mathbf{a}) \cdot (\beta \mathbf{b})}{\|\alpha \mathbf{a}\| \cdot \|\beta \mathbf{b}\|}$$ $$= \frac{\alpha \beta (\mathbf{a} \cdot \mathbf{b})}{|\alpha| \|\mathbf{a}\| \cdot |\beta| \|\mathbf{b}\|} = \frac{\alpha \beta (\mathbf{a} \cdot \mathbf{b})}{\alpha \beta \|\mathbf{a}\| \|\mathbf{b}\|} = \frac{\mathbf{a} \cdot \mathbf{b}}{\|\mathbf{a}\| \|\mathbf{b}\|} = \cos(\mathbf{a}, \mathbf{b}) \quad \blacksquare$$
$\alpha, \beta$Any positive scaling factors. E.g., $\alpha = 10$ means user A interacted 10× more than some baseline
Key step$\alpha$ and $\beta$ cancel in numerator and denominator. The scaling DISAPPEARS — only the direction (the angle) remains
Why this mattersA power user with 500 interactions and a new user with 5 interactions can have the same cosine similarity to an item — their taste DIRECTION is comparable even though their activity levels differ massively

SETUP — 5 PRODUCTS, 8 TAGS

Catalog = 5 products. Tags = {ethnic, kurta, cotton, floral, saree, silk, earrings, gold}.

Product A (Floral Kurta): {ethnic, kurta, cotton, floral}

Product B (Silk Saree): {ethnic, saree, silk}

Product C (Gold Earrings): {earrings, gold}

Product D (Cotton Kurti): {ethnic, kurta, cotton}

Product E (Floral Silk Saree): {ethnic, saree, silk, floral}

STEP 1 — COMPUTE IDF FOR EACH TAG

$N = 5$ products. Count how many products contain each tag:

$$\text{ethnic: 4 products} \Rightarrow \text{IDF} = \log(5/4) = \log(1.25) = 0.223$$ $$\text{kurta: 2 products} \Rightarrow \text{IDF} = \log(5/2) = \log(2.5) = 0.916$$ $$\text{cotton: 2 products} \Rightarrow \text{IDF} = \log(5/2) = 0.916$$ $$\text{floral: 2 products} \Rightarrow \text{IDF} = \log(5/2) = 0.916$$ $$\text{saree: 2 products} \Rightarrow \text{IDF} = \log(5/2) = 0.916$$ $$\text{silk: 2 products} \Rightarrow \text{IDF} = \log(5/2) = 0.916$$ $$\text{earrings: 1 product} \Rightarrow \text{IDF} = \log(5/1) = \log(5) = 1.609$$ $$\text{gold: 1 product} \Rightarrow \text{IDF} = \log(5/1) = 1.609$$

"ethnic" has the lowest IDF (0.223) because it appears in 4 of 5 products — it's not distinctive. "earrings" and "gold" have the highest IDF (1.609) because they appear in only 1 product — they're very distinctive.

STEP 2 — TF-IDF VECTORS

For binary TF (tag present = 1, absent = 0), TF-IDF = IDF when present, 0 when absent:

$$\mathbf{A} = [0.223, 0.916, 0.916, 0.916, 0, 0, 0, 0]$$ $$\mathbf{B} = [0.223, 0, 0, 0, 0.916, 0.916, 0, 0]$$ $$\mathbf{C} = [0, 0, 0, 0, 0, 0, 1.609, 1.609]$$ $$\mathbf{D} = [0.223, 0.916, 0.916, 0, 0, 0, 0, 0]$$ $$\mathbf{E} = [0.223, 0, 0, 0.916, 0.916, 0.916, 0, 0]$$
STEP 3 — COSINE SIMILARITY cos(A, D)

Products A (Floral Kurta) and D (Cotton Kurti) — both kurtas, share ethnic + kurta + cotton:

$$\mathbf{A} \cdot \mathbf{D} = (0.223)(0.223) + (0.916)(0.916) + (0.916)(0.916) + (0.916)(0) + \ldots$$ $$= 0.050 + 0.839 + 0.839 + 0 = 1.728$$ $$\|\mathbf{A}\| = \sqrt{0.223^2 + 0.916^2 + 0.916^2 + 0.916^2} = \sqrt{0.050 + 0.839 + 0.839 + 0.839} = \sqrt{2.567} = 1.602$$ $$\|\mathbf{D}\| = \sqrt{0.223^2 + 0.916^2 + 0.916^2} = \sqrt{0.050 + 0.839 + 0.839} = \sqrt{1.728} = 1.315$$ $$\cos(\mathbf{A}, \mathbf{D}) = \frac{1.728}{1.602 \times 1.315} = \frac{1.728}{2.107} = 0.820$$

High similarity (0.82) — they share 3 of 4 tags, and the shared tags (kurta, cotton) have high IDF. The "ethnic" tag contributes little because it's common (IDF = 0.223).

STEP 4 — COSINE cos(A, C)

Products A (Floral Kurta) and C (Gold Earrings) — zero tag overlap:

$$\mathbf{A} \cdot \mathbf{C} = 0 \quad \text{(no shared tags)}$$ $$\cos(\mathbf{A}, \mathbf{C}) = \frac{0}{\|\mathbf{A}\| \cdot \|\mathbf{C}\|} = 0.000$$

Zero similarity — CBF says kurtas and earrings are completely unrelated. This is CBF's blind spot from Module 07: in reality, saree buyers frequently buy earrings. CF (Module 08) can discover this pattern; CBF cannot.

D4 / FROM SCRATCH
Complete content-based recommender — no sklearn
↳ corresponds to M07 S2: Three Steps
# ═══ CONTENT-BASED FILTERING — from scratch, no sklearn ═══
import numpy as np
from collections import Counter

def build_vocabulary(catalog):
    """Build tag → index mapping from product catalog."""
    all_tags = set()
    for product in catalog:
        all_tags.update(product['tags'])
    return {tag: idx for idx, tag in enumerate(sorted(all_tags))}

def compute_idf(catalog, vocab):
    """IDF = log(N / doc_freq) for each tag."""
    N = len(catalog)
    doc_freq = Counter()
    for product in catalog:
        for tag in set(product['tags']):
            doc_freq[tag] += 1
    idf = np.zeros(len(vocab))
    for tag, idx in vocab.items():
        idf[idx] = np.log(N / max(doc_freq[tag], 1))
    return idf

def tfidf_vector(tags, vocab, idf):
    """Compute TF-IDF vector for one product."""
    vec = np.zeros(len(vocab))
    tag_counts = Counter(tags)
    total = sum(tag_counts.values())
    for tag, count in tag_counts.items():
        if tag in vocab:
            tf = count / total
            vec[vocab[tag]] = tf * idf[vocab[tag]]
    return vec

def cosine_sim(a, b):
    """Cosine similarity between two vectors."""
    dot = np.dot(a, b)
    norm = np.linalg.norm(a) * np.linalg.norm(b)
    return dot / max(norm, 1e-10)

def build_user_profile(liked_item_ids, item_vectors):
    """Average TF-IDF vectors of liked items = user taste vector."""
    vecs = [item_vectors[iid] for iid in liked_item_ids if iid in item_vectors]
    return np.mean(vecs, axis=0) if vecs else np.zeros(next(iter(item_vectors.values())).shape)

def recommend_cbf(user_profile, item_vectors, seen_items, top_k=10):
    """Score all unseen items by cosine similarity to user profile."""
    scores = {}
    for iid, vec in item_vectors.items():
        if iid not in seen_items:
            scores[iid] = cosine_sim(user_profile, vec)
    return sorted(scores, key=scores.get, reverse=True)[:top_k]

Problem 1: TF-IDF interpretation
Interview
PROBLEM

A product has tags {clothing, women, cotton, banarasi, handloom}. The catalog has 480K products. "clothing" appears in 456K products; "banarasi" appears in 1,440. Compute the IDF of each tag. Which tag contributes most to this product's TF-IDF vector?

SOLUTION
$$\text{IDF(clothing)} = \log(480{,}000 / 456{,}000) = \log(1.053) = 0.051$$ $$\text{IDF(banarasi)} = \log(480{,}000 / 1{,}440) = \log(333.3) = 5.810$$

"banarasi" (IDF = 5.81) contributes 114× more than "clothing" (IDF = 0.05). "clothing" is essentially noise — it appears in 95% of products and distinguishes nothing. "banarasi" is highly informative — it narrows the product space to 0.3% of the catalog. This is why TF-IDF works: it automatically discovers which features matter.

Problem 2: CBF limitation
Interview
PROBLEM

A user has browsed 20 ethnic kurtas on Kartify. CBF recommends 10 more ethnic kurtas. The user actually wants to discover jewellery that matches their kurta collection. Why does CBF fail, and what approach would succeed?

SOLUTION

CBF computes $\cos(\text{user\_profile}, \text{item\_vector})$. The user's profile vector is dominated by tags like {ethnic, kurta, cotton, floral}. Jewellery items have tags like {earrings, gold, silver, traditional}. Zero tag overlap → cosine = 0. CBF literally cannot recommend across categories with disjoint tag vocabularies.

CF succeeds because it finds users with similar kurta browsing patterns who ALSO bought jewellery. The cross-category pattern (kurtas → earrings) lives in user behavior, not in product features. This is the M07→M08 bridge: CBF sees item features; CF sees user behavior. Both are needed.
