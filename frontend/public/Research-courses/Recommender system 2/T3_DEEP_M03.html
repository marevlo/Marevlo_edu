The math behind
the lies in your data.

Module 03 explained that your data is biased, sparse, and misleading. This companion formalizes exactly HOW — with missing data theory, confidence weighting formulas, bias definitions, and the code to build a weighted interaction matrix.

📖 Narrative: M03 — Your Data Is Lying to You
D1 / FORMALIZATION
Missing data mechanisms — MCAR, MAR, MNAR
↳ corresponds to M03 S1: What Does "Empty" Mean?
Real-life analogy

Imagine a restaurant satisfaction survey. MCAR: surveys are randomly handed to 10% of diners — missing data is completely random. MAR: surveys are given to all dinner guests but not lunch guests — missing data depends on observable factors (meal type). MNAR: surveys are left on tables and only collected by diners who CHOOSE to fill them in — satisfied diners fill them in, unsatisfied diners leave. Missing data depends on the answer itself. Recsys data is ALWAYS MNAR — users only interact with items they expect to like.

Missing data mechanisms
$$\text{Let } O_{ui} \in \{0, 1\} \text{ indicate whether } R_{ui} \text{ is observed}$$ $$\text{MCAR: } P(O_{ui} = 1) = p \quad \text{(constant, independent of everything)}$$ $$\text{MAR: } P(O_{ui} = 1 \mid \mathbf{x}_u, \mathbf{x}_i) \quad \text{(depends on observable features)}$$ $$\text{MNAR: } P(O_{ui} = 1 \mid r_{ui}, \mathbf{x}_u, \mathbf{x}_i) \quad \text{(depends on the rating itself)}$$
$O_{ui}$Observation indicator — 1 if we see the interaction, 0 if we don't
$r_{ui}$The TRUE preference — what the user would rate if they saw the item. This is what we're trying to predict but can never directly observe for missing entries
$\mathbf{x}_u, \mathbf{x}_i$Observable user/item features (demographics, category, price, etc.)

In recsys, the data is always MNAR because: users choose to consume items they expect to like → ratings are biased positive → items with low expected ratings are rarely observed → the missing entries are systematically different from the observed ones.

Why this matters practically: if you train a model only on observed entries (ignoring missing ones), you're training on a biased sample of positive preferences. The model learns "everything is good" because it only sees things users chose to consume. This is why WRMF includes unobserved entries (with low confidence) and BPR creates negative samples from unobserved items.

D2 / DERIVATION
Confidence weighting — from intuition to formula
↳ corresponds to M03 S2: Four Types of Signal & S3: Confidence Spectrum
Real-life analogy

Imagine you're a detective gathering evidence. A single eyewitness (one click) gives weak evidence — maybe they're mistaken. Five eyewitnesses (five clicks) give stronger evidence. A signed confession (a purchase) is very strong. No evidence at all (no interaction) doesn't prove innocence — maybe you just haven't investigated yet. Confidence weighting assigns each piece of evidence a "reliability score."

Confidence weighting (Hu, Koren, Volinsky 2008)
$$c_{ui} = 1 + \alpha \cdot r_{ui}$$
$c_{ui}$Confidence for pair $(u, i)$ — how much we trust our belief about whether user $u$ likes item $i$
$r_{ui}$Raw interaction count. Clicks, purchase count, or play count. For Kartify: if Priya clicked a saree 3 times, $r_{ui} = 3$
$\alpha$Confidence scaling rate. Controls how quickly confidence grows with interactions. Typical: $\alpha = 40$. Higher $\alpha$ → more trust in observed interactions. Lower $\alpha$ → more conservative
$1$ (baseline)Even when $r_{ui} = 0$ (no interaction), confidence is 1, not 0. This means: "we have SOME belief that this user doesn't like this item, but we're not very confident." This is the key insight — absence is uncertainty, not rejection
Worked: Confidence values for Kartify interactions
Numerical
COMPUTE

Using $\alpha = 40$ (standard WRMF setting):

$$\text{Priya never saw Running Shoes: } c = 1 + 40 \times 0 = 1 \quad \text{(minimal confidence)}$$ $$\text{Priya viewed a Kurta once: } c = 1 + 40 \times 1 = 41$$ $$\text{Priya clicked a Saree 3 times: } c = 1 + 40 \times 3 = 121$$ $$\text{Priya purchased a Lehenga: } c = 1 + 40 \times 5 = 201 \quad \text{(purchase = 5 implicit units)}$$ $$\text{Priya bought + reviewed Earrings: } c = 1 + 40 \times 8 = 321 \quad \text{(buy + review = 8)}$$

The ratio: Priya purchasing earrings generates 321× more training signal than the absence of interaction with running shoes. The model is 321× more certain she likes earrings than it is that she doesn't like running shoes. This asymmetry is the mathematical encoding of "absence isn't rejection."

D3 / QUANTIFICATION
Signal hierarchy — Kartify's implicit weights
↳ corresponds to M03 S3: The Confidence Spectrum

Module 03's confidence spectrum was qualitative (low → high). Here's the quantitative version — how Kartify should weight each event type when building the interaction matrix:

Kartify event weighting scheme
$$r_{ui} = \sum_{e \in \text{events}(u,i)} w_e \cdot \text{decay}(t_e)$$ $$\text{where } w_e = \begin{cases} 0.5 & \text{page view} > 3\text{s} \\ 1.0 & \text{page view} > 10\text{s} \\ 2.0 & \text{add to wishlist} \\ 3.0 & \text{add to cart} \\ 5.0 & \text{purchase} \\ 8.0 & \text{repeat purchase} \\ -3.0 & \text{return/refund} \end{cases}$$ $$\text{decay}(t_e) = e^{-\lambda_t \cdot \text{age\_days}(t_e)}, \quad \lambda_t = 0.05$$
$w_e$Weight for event type $e$. Design decision — these numbers reflect Kartify's belief about which signals are strongest
$\text{decay}(t_e)$Time decay — recent events count more than old ones. $\lambda_t = 0.05$ gives half-life of ~14 days. A click from yesterday weighs $e^{-0.05} = 0.95$ of full value. A click from 30 days ago weighs $e^{-1.5} = 0.22$
$r_{ui}$The AGGREGATED interaction score for pair $(u,i)$ — the sum across all events. This is what enters $c_{ui} = 1 + \alpha \cdot r_{ui}$
D4 / FORMALIZATION
Three biases — formal definitions
↳ corresponds to M03 S4: Three Lies in Your Data

Popularity bias

Popularity bias
$$P(\text{interact} \mid u, i) = P(\text{like} \mid u, i) \times P(\text{exposed} \mid u, i)$$

We observe interactions, not preferences. Popular items have higher $P(\text{exposed})$ because previous recsys showed them more. If we naively use interaction count as a proxy for preference, popular items look "better" than they are — their high interaction count reflects high exposure, not necessarily high quality.

$P(\text{like} \mid u,i)$True preference — would user $u$ like item $i$ if they saw it? THIS is what we want to estimate
$P(\text{exposed} \mid u,i)$Exposure probability — did the system even show item $i$ to user $u$? This depends on the PREVIOUS recsys, creating a feedback loop

Position bias

Position bias (examination hypothesis)
$$P(\text{click} \mid u, i, \text{pos}_k) = P(\text{examine} \mid \text{pos}_k) \times P(\text{click} \mid \text{examined}, u, i)$$

Users examine items at higher positions more often. An item at position 1 might be examined 100% of the time; position 10 might be examined 20% of the time. If we train on clicks without adjusting for position, the model conflates "shown at the top" with "relevant."

$P(\text{examine} \mid \text{pos}_k)$Probability user looks at position $k$. Typically $\propto 1/k$ or $1/\log(k+1)$. Position 1: ~100%. Position 5: ~40%. Position 10: ~15%
$P(\text{click} \mid \text{examined})$Probability of clicking IF the user actually looks at the item. This is the TRUE relevance signal we want — independent of position

Selection bias (propensity)

Inverse Propensity Scoring (IPS) correction
$$\hat{\mathcal{L}}_{\text{IPS}} = \frac{1}{|\mathcal{U}|}\sum_{u}\sum_{i \in \mathcal{I}_u^+} \frac{\ell(f(u,i), r_{ui})}{P(O_{ui} = 1 \mid \mathbf{x}_u, \mathbf{x}_i)}$$
$P(O_{ui} = 1)$Propensity — the probability that pair $(u,i)$ was observed. Items shown to many users have high propensity; items never shown have low propensity. This is estimated from logging data
$1 / P(O_{ui} = 1)$Inverse propensity weight — upweights interactions with rarely-shown items (they're more informative) and downweights interactions with frequently-shown items (they're confounded by exposure)
The takeaway

All three biases have formal corrections. Popularity bias: IDF weighting on items. Position bias: examination models or position-debiased training. Selection bias: IPS weighting. These corrections are covered in practice in later modules, but the formal framework starts here — understanding WHAT is biased before learning HOW to fix it.

D5 / FROM SCRATCH
Building a weighted interaction matrix from Kartify logs
↳ corresponds to M03 S5: Kartify's Data Audit
# Build WEIGHTED interaction matrix with event types + time decay
import pandas as pd
import numpy as np
from scipy.sparse import csr_matrix
from datetime import datetime

# Event weights (design decision from D3)
EVENT_WEIGHTS = {
    'page_view_short': 0.5,
    'page_view_long': 1.0,
    'add_wishlist': 2.0,
    'add_cart': 3.0,
    'purchase': 5.0,
    'repeat_purchase': 8.0,
    'return': -3.0
}

def build_weighted_R(logs, now, decay_lambda=0.05):
    """Build weighted interaction matrix with time decay."""
    # Map IDs
    user_ids = logs.user_id.unique()
    item_ids = logs.item_id.unique()
    uid_map = {u: i for i, u in enumerate(user_ids)}
    iid_map = {it: i for i, it in enumerate(item_ids)}

    # Compute weighted scores per (user, item) pair
    scores = {}
    for _, row in logs.iterrows():
        key = (uid_map[row.user_id], iid_map[row.item_id])
        age = (now - row.timestamp).days
        weight = EVENT_WEIGHTS.get(row.event_type, 0)
        decay = np.exp(-decay_lambda * age)
        scores[key] = scores.get(key, 0) + weight * decay

    # Build sparse matrix
    rows, cols, vals = [], [], []
    for (u, i), v in scores.items():
        rows.append(u); cols.append(i); vals.append(max(v, 0))

    R = csr_matrix((vals, (rows, cols)),
                   shape=(len(user_ids), len(item_ids)))
    return R, uid_map, iid_map

# Confidence matrix for WRMF
def confidence_matrix(R, alpha=40):
    """C_ui = 1 + alpha * R_ui"""
    C = R.copy()
    C.data = 1 + alpha * C.data
    return C
D6 / INTERVIEW
Interview problems — data & bias
↳ tests M03 concepts
Problem 1: Confidence asymmetry
Interview
PROBLEM

In WRMF with $\alpha = 40$: a user purchased an item ($r_{ui} = 5$) and has no interaction with another item ($r_{uj} = 0$). How much more does the model "care" about getting the purchase prediction right vs the non-interaction prediction?

SOLUTION
$$c_{ui} = 1 + 40 \times 5 = 201 \quad (\text{purchase})$$ $$c_{uj} = 1 + 40 \times 0 = 1 \quad (\text{no interaction})$$ $$\text{Ratio} = \frac{c_{ui}}{c_{uj}} = \frac{201}{1} = 201\times$$

The model weighs the purchase prediction 201× more heavily. This is intentional: we're VERY confident she likes the purchased item, but only mildly confident she doesn't like the unseen item (she might love it but never saw it). The 201:1 ratio is the mathematical encoding of Module 03's "absence isn't rejection."

Problem 2: Position bias impact
Interview
PROBLEM

In Kartify's search results, position 1 gets clicked 40% of the time. Position 5 gets clicked 10%. If both positions show equally relevant items ($P(\text{click} \mid \text{examined}) = 0.5$), what are the examination probabilities? If you train on raw clicks, what does the model learn?

SOLUTION
$$P(\text{click} \mid \text{pos}_1) = P(\text{examine} \mid \text{pos}_1) \times 0.5 = 0.40$$ $$\Rightarrow P(\text{examine} \mid \text{pos}_1) = 0.80$$ $$P(\text{click} \mid \text{pos}_5) = P(\text{examine} \mid \text{pos}_5) \times 0.5 = 0.10$$ $$\Rightarrow P(\text{examine} \mid \text{pos}_5) = 0.20$$

Position 1 is examined 4× more often. A model trained on raw clicks learns "items at position 1 are 4× more relevant" — which is false. They're equally relevant; they just get seen more. The model will keep putting the same items at position 1, "confirming" they're best. This is the feedback loop from Module 03: position bias → biased training data → model reproduces position bias → cycle repeats.

Problem 3: IPS weighting
Interview
PROBLEM

Item A was shown to 100,000 users and clicked by 3,000 (CTR = 3%). Item B was shown to 500 users and clicked by 25 (CTR = 5%). If you use raw click count as the training signal, which item appears more popular? If you use IPS weighting with propensity $\propto$ exposure count, which item scores higher?

SOLUTION

Raw clicks: Item A = 3,000, Item B = 25. A looks 120× more "popular."

$$\text{IPS-weighted: Item A} = \frac{3{,}000}{100{,}000 / Z} = \frac{3{,}000 \cdot Z}{100{,}000}$$ $$\text{IPS-weighted: Item B} = \frac{25}{500 / Z} = \frac{25 \cdot Z}{500}$$ $$\text{Ratio} = \frac{3{,}000 / 100{,}000}{25 / 500} = \frac{0.03}{0.05} = 0.6$$

After IPS correction, Item B scores higher — its per-impression click rate is 5% vs A's 3%. IPS reveals that B is actually more relevant; A only looked popular because of higher exposure. This is the mathematical fix for popularity bias.
