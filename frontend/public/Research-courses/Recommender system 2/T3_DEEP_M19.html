Factorization Machines —
from formula to O(kn) trick.

Module 19 introduced features and FM intuitively. This companion derives the FM equation, shows the O(kn) computational trick that makes it practical, works through a prediction with 5 features by hand, and explains why FM reduces to MF when features are just IDs.

D1 / DERIVATION
The FM equation — three components
↳ corresponds to M19 S3: Factorization Machines
Real-life analogy

Imagine predicting house prices. A linear model uses features independently: $price = w_1 \times \text{sqft} + w_2 \times \text{bedrooms} + \ldots$ But some features interact: a large house (high sqft) in a good neighborhood (high zip_score) is worth MORE than the sum of their individual contributions. FM learns these interactions automatically by giving each feature an embedding and computing all pairwise dot products — "sqft × neighborhood" gets an interaction score even if that specific combination is rare in the data.

Factorization Machine (degree 2)
$$\hat{y}(\mathbf{x}) = \underbrace{w_0}_{\text{global bias}} + \underbrace{\sum_{i=1}^{n} w_i x_i}_{\text{linear terms}} + \underbrace{\sum_{i=1}^{n}\sum_{j=i+1}^{n} \langle \mathbf{v}_i, \mathbf{v}_j \rangle x_i x_j}_{\text{pairwise interactions}}$$
$\mathbf{x} \in \mathbb{R}^n$Input feature vector. For recsys: a one-hot/multi-hot encoding of all features. Example: $x = [0,\ldots,1,\ldots,0, | 0,\ldots,1,\ldots,0, | 0.8, 0.3, 1, 0]$ where the first block is one-hot user ID, second is one-hot item ID, rest are continuous/categorical features
$w_0$Global bias — like $\mu$ in biased MF. One scalar
$w_i$Per-feature weight — linear effect of feature $i$. Like $b_u$ and $b_i$ in biased MF
$\mathbf{v}_i \in \mathbb{R}^k$Embedding vector for feature $i$. Each feature value gets its own $k$-dimensional embedding. User #847291 gets $\mathbf{v}_{847291}$. Category "ethnic" gets $\mathbf{v}_\text{ethnic}$. Price bin "premium" gets $\mathbf{v}_\text{premium}$
$\langle \mathbf{v}_i, \mathbf{v}_j \rangle$Dot product between feature embeddings = learned interaction strength. "How much do features $i$ and $j$ interact?"
$x_i x_j$Feature activation product. For one-hot features: only 1 when BOTH features are active. This selects which interactions contribute

Why FM reduces to biased MF

When $\mathbf{x}$ contains only two one-hot features — user_id and item_id — the FM prediction becomes:

FM with user/item IDs only = biased MF
$$\hat{y} = w_0 + w_{u} \cdot 1 + w_{i} \cdot 1 + \langle \mathbf{v}_{u}, \mathbf{v}_{i} \rangle \cdot 1 \cdot 1 = w_0 + w_u + w_i + \mathbf{v}_u^\top \mathbf{v}_i$$

This is exactly $\mu + b_u + b_i + \mathbf{u}^\top \mathbf{v}$ from Module 15. FM with ID-only features IS biased MF. Every additional feature is bonus.

D2 / THE TRICK
The O(kn) computation trick
↳ corresponds to M19 S3

Naively computing all pairwise interactions costs $O(kn^2)$ — for each of $\binom{n}{2}$ pairs, compute a $k$-dimensional dot product. With $n = 1000$ features, that's 500K dot products. Too slow.

The trick: rewrite the pairwise sum using the identity $(a+b+c)^2 = a^2 + b^2 + c^2 + 2(ab + ac + bc)$:

O(kn) reformulation
$$\sum_{i=1}^{n}\sum_{j=i+1}^{n} \langle \mathbf{v}_i, \mathbf{v}_j \rangle x_i x_j = \frac{1}{2}\sum_{f=1}^{k}\left[\left(\sum_{i=1}^{n} v_{i,f} x_i\right)^2 - \sum_{i=1}^{n} v_{i,f}^2 x_i^2\right]$$
Left side$O(kn^2)$ — sum over all pairs, each pair needs a $k$-dim dot product
Right side$O(kn)$ — for each of $k$ dimensions: sum $n$ terms (square it), subtract $n$ squared terms. Total: $2kn$ operations
SpeedupFor $n=1000, k=128$: from $128 \times 500{,}000 = 64\text{M}$ ops to $2 \times 128 \times 1000 = 256\text{K}$ ops. 250× faster. This trick makes FM practical for high-dimensional feature vectors
D3 / WORKED
FM prediction with 5 features — by hand
↳ corresponds to M19 S3 & S4
Predict: Priya × Pashmina Shawl × Evening × Mobile
Step by step
SETUP (k=2 for simplicity)

Active features (all $x_i = 1$): user=Priya, item=Shawl, category=ethnic, time=evening, device=mobile.

$$w_0 = 0.5, \quad w_\text{Priya} = 0.3, \quad w_\text{Shawl} = 0.1, \quad w_\text{ethnic} = 0.2, \quad w_\text{eve} = 0.0, \quad w_\text{mobile} = -0.1$$ $$\mathbf{v}_\text{Priya} = [0.8, 0.3], \quad \mathbf{v}_\text{Shawl} = [0.6, 0.5], \quad \mathbf{v}_\text{ethnic} = [0.7, 0.4]$$ $$\mathbf{v}_\text{eve} = [0.2, 0.1], \quad \mathbf{v}_\text{mobile} = [0.1, -0.2]$$
LINEAR TERMS
$$w_0 + \sum w_i = 0.5 + 0.3 + 0.1 + 0.2 + 0.0 + (-0.1) = 1.0$$
INTERACTIONS (using O(kn) trick)

For $f=1$ (first embedding dimension):

$$S_1 = \sum v_{i,1} x_i = 0.8 + 0.6 + 0.7 + 0.2 + 0.1 = 2.4$$ $$Q_1 = \sum v_{i,1}^2 x_i^2 = 0.64 + 0.36 + 0.49 + 0.04 + 0.01 = 1.54$$ $$\text{Interaction}_1 = \frac{1}{2}(S_1^2 - Q_1) = \frac{1}{2}(5.76 - 1.54) = 2.11$$

For $f=2$:

$$S_2 = 0.3 + 0.5 + 0.4 + 0.1 + (-0.2) = 1.1$$ $$Q_2 = 0.09 + 0.25 + 0.16 + 0.01 + 0.04 = 0.55$$ $$\text{Interaction}_2 = \frac{1}{2}(1.21 - 0.55) = 0.33$$
$$\text{Total interactions} = 2.11 + 0.33 = 2.44$$
FINAL PREDICTION
$$\hat{y} = 1.0 + 2.44 = \boxed{3.44}$$

The interaction term (2.44) dominates — Priya's embedding aligns strongly with ethnic items in the evening. The linear terms (1.0) capture individual biases. Without FM, we'd have just the linear prediction (1.0) — missing the 2.44 from feature interactions entirely.

Problem 1: FM vs MF parameter count
Interview
PROBLEM

Kartify has 2.3M users, 480K items, 500 category values, 100 brand values, 24 hours, and 7 days. With $k=64$, how many parameters does FM have? Compare to MF.

SOLUTION
$$\text{MF parameters} = (2.3\text{M} + 480\text{K}) \times 64 = 178\text{M}$$ $$\text{FM parameters} = (2.3\text{M} + 480\text{K} + 500 + 100 + 24 + 7) \times (64 + 1)$$ $$= 2{,}780{,}631 \times 65 = 180.7\text{M}$$

FM adds only ~2.7M parameters over MF (<1.5% increase) for all the extra features. The user/item ID embeddings dominate. The feature embeddings (500 categories × 64 = 32K params, etc.) are tiny. Feature-rich models are essentially free in parameter count — the expressiveness gain is enormous relative to the parameter cost.
