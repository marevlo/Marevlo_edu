Part 02

Four Popularity Variants

From DEEP M06 — each variant uses a different scoring formula to rank items.

Raw interaction count is the simplest popularity signal, but it over-weights old items. The four variants progressively improve this by adding time-awareness and statistical regularization.

1. Raw Count
score(i) = |interactions(i)|
Total number of interactions. Fast, simple, biased toward old items.
2. Time-Windowed
score(i) = count(i, last 20%)
Only count recent interactions. Captures trending items.
3. Exponential Decay
score(i) = Σ exp(−λ·age)
Weight each interaction by recency. λ=0.1/day. Smooth decay.
4. Bayesian Average
score = (C·μ + Σr) / (C + n)
Shrinks low-count items toward global mean. C=50 is prior strength.
Lab_02 · exponential decay + Bayesian average
# Variant 3: Exponential decay (λ = 0.1 per day)
now  = train_df.timestamp.max()
lam  = 0.1 / 86400   # per second
decay_scores = defaultdict(float)
for _, row in train_df.iterrows():
    age = now - row.timestamp
    decay_scores[row.item_id] += np.exp(-lam * age)

# Variant 4: Bayesian average (regularizes low-count items)
C           = 50              # prior weight (strength of prior)
global_mean = train_df.rating.mean()
item_stats  = train_df.groupby('item_id').rating.agg(['sum', 'count'])

def bayesian_score(item_id):
    if item_id in item_stats.index:
        s, n = item_stats.loc[item_id, 'sum'], item_stats.loc[item_id, 'count']
    else:
        s, n = 0, 0
    return (C * global_mean + s) / (C + n)   # pulls toward μ when n is small

Part 03

TF-IDF Content-Based Filtering

From DEEP M07 — represent items as TF-IDF vectors, build user profiles, score by cosine similarity.

Content-based filtering recommends items similar to what a user has liked before. Each item is represented as a TF-IDF vector over its genre tags. The user's preference profile is the average of their liked items' vectors.

Lab_02 · TF-IDF item vectors
from collections import Counter

# Build vocabulary and IDF
vocab    = {tag: idx for idx, tag in enumerate(sorted(all_tags_set))}
doc_freq = Counter(tag for tags in item_tags.values() for tag in set(tags))
idf      = {tag: np.log(n_items / max(doc_freq[tag], 1)) for tag in vocab}

# Build normalized item vectors (binary TF × IDF)
item_vectors = np.zeros((n_items, len(vocab)))
for item_id, tags in item_tags.items():
    for tag in tags:
        item_vectors[item_id, vocab[tag]] = idf[tag]   # TF=1 * IDF

# L2-normalize for cosine similarity via dot product
norms = np.linalg.norm(item_vectors, axis=1, keepdims=True)
norms[norms == 0] = 1
item_vectors_normed = item_vectors / norms
Lab_02 · CBF recommend()
def cbf_recommend(user_id, k=10):
    # Step 1: find user's liked items (rating ≥ 3)
    liked = R_train[user_id].indices[R_train[user_id].data >= 3]
    if len(liked) == 0:
        return list(range(k))   # cold-start fallback

    # Step 2: user profile = mean of liked item vectors
    profile   = item_vectors_normed[liked].mean(axis=0)
    prof_norm = np.linalg.norm(profile)
    if prof_norm == 0: return list(range(k))
    profile   = profile / prof_norm

    # Step 3: cosine similarity = dot product (both are unit vectors)
    scores    = item_vectors_normed @ profile    # (n_items,)

    # Step 4: exclude seen items, return top-k
    seen = set(R_train[user_id].indices)
    for s in seen: scores[s] = -np.inf
    return np.argsort(scores)[-k:][::-1].tolist()
💡 Why CBF personalizes from day one

CBF only needs a user's interaction history — no other users required. This makes it uniquely good for new users who have rated even one or two items. The tradeoff: it can only recommend items similar to what the user already knows, creating a "filter bubble."
