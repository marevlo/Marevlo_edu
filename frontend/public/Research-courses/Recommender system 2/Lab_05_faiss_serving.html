Part 01
Generate Embeddings
Clustered synthetic embeddings scaled to Kartify-style catalogue size

We synthesise 480 K item embeddings (shrink to 100 K for development) drawn from 50 cluster centroids with Gaussian noise, then L2-normalise so that inner product equals cosine similarity. This mirrors real production embeddings from a trained BPR-MF or two-tower model.

Key insight

L2-normalising both item and user vectors converts the maximum inner product search (MIPS) problem into a maximum cosine similarity search — which FAISS's IndexFlatIP handles natively.

Lab_05_faiss_serving.py — generate_embeddings()
def generate_embeddings(n_items=480_000, n_users=10_000, k=128):
    n_clusters      = 50
    cluster_centers = np.random.randn(n_clusters, k).astype(np.float32)
    item_embeddings = np.zeros((n_items, k), dtype=np.float32)

    for i in range(n_items):
        cluster            = i % n_clusters
        item_embeddings[i] = cluster_centers[cluster] \
                             + np.random.randn(k).astype(np.float32) * 0.3

    # L2-normalise → inner product == cosine similarity
    norms           = np.linalg.norm(item_embeddings, axis=1, keepdims=True)
    item_embeddings = item_embeddings / np.maximum(norms, 1e-8)

    user_embeddings = np.random.randn(n_users, k).astype(np.float32)
    user_norms      = np.linalg.norm(user_embeddings, axis=1, keepdims=True)
    user_embeddings = user_embeddings / np.maximum(user_norms, 1e-8)

    return item_embeddings, user_embeddings
Part 02
Brute Force Baseline
Exact search — perfect recall, our latency ceiling to beat

The brute-force baseline computes every item score for a query and returns the exact top-500. This is the recall ceiling (1.000) and the latency ceiling we must beat with ANN. Any production system serving millions of queries per second cannot afford this approach at Kartify scale.

IndexFlatIP
Exact
Brute-force inner product scan. Guaranteed Recall@500 = 1.000. No training needed. Prohibitive at >500 K items.
IndexIVFPQ
Approx
Inverted file + Product Quantisation. Clusters search space, compresses vectors. ~100× speedup at 95%+ recall.
IndexHNSWFlat
Approx
Hierarchical navigable small-world graph. Excellent recall, low latency, no training phase, slightly higher memory.
Lab_05_faiss_serving.py — build_flat_index() + brute_force_search()
def build_flat_index(item_embeddings):
    k     = item_embeddings.shape[1]
    index = faiss.IndexFlatIP(k)   # exact inner product
    index.add(item_embeddings)
    return index


def brute_force_search(user_emb, item_embeddings, k=500):
    scores = item_embeddings @ user_emb
    top_k  = np.argpartition(scores, -k)[-k:]
    top_k  = top_k[np.argsort(scores[top_k])[::-1]]
    return top_k, scores[top_k]


def benchmark_latency(index, user_embeddings, k=500, n_queries=1000, nprobe=None):
    if nprobe is not None and hasattr(index, 'nprobe'):
        index.nprobe = nprobe
    latencies = []
    for i in range(min(n_queries, len(user_embeddings))):
        query = user_embeddings[i:i + 1]
        t0    = time.perf_counter()
        index.search(query, k)
        latencies.append((time.perf_counter() - t0) * 1000)
    latencies = np.array(latencies)
    return {"P50_ms": np.percentile(latencies, 50),
            "P99_ms": np.percentile(latencies, 99),
            "mean_ms": np.mean(latencies)}

Part 03
IVF-PQ Index
Inverted file + product quantisation — two speedups in one

IVF-PQ combines two orthogonal techniques: IVF partitions the vector space into nlist clusters so each query searches only nprobe clusters instead of all vectors; PQ compresses each vector from 128 × 4 bytes down to m bytes of quantised codes. The result is dramatic memory reduction and CPU cache efficiency.

nprobe trade-off

Higher nprobe = more clusters searched = higher recall, but slower. The nprobe sweep in Part 05 finds the minimum nprobe where P99 < 10 ms and Recall > 95%.

Lab_05_faiss_serving.py — build_ivf_pq_index()
def build_ivf_pq_index(item_embeddings, nlist=1024, m=32, nbits=8):
    """IVF-PQ: cluster search + vector compression — ~100× faster at 95%+ recall."""
    k         = item_embeddings.shape[1]          # embedding dim
    quantizer = faiss.IndexFlatIP(k)              # coarse quantiser
    index     = faiss.IndexIVFPQ(quantizer, k, nlist, m, nbits)
    #                                        ^     ^   ^  ^
    #                                       dim nlist  m  bits per sub-vector

    index.train(item_embeddings)   # train cluster centroids + PQ codebook
    index.add(item_embeddings)     # add all vectors
    return index

Part 04
HNSW Index
Graph-based ANN — no training, excellent recall by default

HNSW builds a layered navigable small-world graph during index construction. At query time it performs a beam search from an entry point through the graph layers. The M parameter controls graph connectivity: higher M means better recall but more memory and slower build time. HNSW typically achieves 95–99% recall with P99 well under 5 ms at 100 K scale.

Extension task

Try setting index.hnsw.efSearch = 64 before searching to trade slightly more latency for higher recall on harder queries.

Lab_05_faiss_serving.py — build_hnsw_index()
def build_hnsw_index(item_embeddings, M=32, ef_construction=200):
    """HNSW: hierarchical graph — excellent recall, memory-efficient."""
    k     = item_embeddings.shape[1]
    index = faiss.IndexHNSWFlat(k, M)
    index.hnsw.efConstruction = ef_construction   # more neighbours → better graph

    index.add(item_embeddings)    # no training step needed
    return index


def benchmark_recall(index, item_embeddings, user_embeddings,
                     k=500, n_queries=100, nprobe=None):
    if nprobe is not None and hasattr(index, 'nprobe'):
        index.nprobe = nprobe
    recalls = []
    for i in range(min(n_queries, len(user_embeddings))):
        query         = user_embeddings[i]
        gt_indices, _ = brute_force_search(query, item_embeddings, k)
        gt_set        = set(gt_indices)
        _, ann_idx    = index.search(user_embeddings[i:i + 1], k)
        ann_set       = set(ann_idx[0])
        recalls.append(len(gt_set & ann_set) / len(gt_set))
    return np.mean(recalls)
