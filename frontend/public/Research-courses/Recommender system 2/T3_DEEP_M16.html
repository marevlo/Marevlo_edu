D1 / DERIVATION
BPR loss — from Bayesian posterior to SGD gradient
↳ corresponds to M16 S2: BPR — The Tournament
Real-life analogy

Imagine judging a baking competition. You can't give exact scores to each cake (that's MSE). But you CAN say "cake A is better than cake B." From thousands of such pairwise comparisons, you can infer a global ranking of all cakes — even cakes that were never directly compared. BPR does exactly this: learn from "item A beats item B for user U" to infer a complete ranking.

Step 1: The Bayesian objective

We want to maximize the posterior probability that observed items are preferred over unobserved items:

BPR-OPT (maximize)
$$\text{BPR-OPT} = \sum_{(u,i,j) \in D_S} \ln \sigma(\hat{x}_{uij}) - \lambda_\Theta \|\Theta\|^2$$
$D_S$Training triples: $(u, i, j)$ where user $u$ interacted with item $i$ (positive) but not $j$ (negative, sampled). One triple per training step
$\hat{x}_{uij}$Score difference: $\hat{r}_{ui} - \hat{r}_{uj}$. Positive means the model correctly ranks $i$ above $j$ for user $u$
$\sigma(x)$Sigmoid: $\frac{1}{1+e^{-x}}$. Maps the score difference to a probability in $(0,1)$. When $\hat{x}_{uij} \gg 0$: $\sigma \approx 1$ (confident correct ranking). When $\hat{x}_{uij} \ll 0$: $\sigma \approx 0$ (wrong ranking)
$\ln \sigma(\hat{x}_{uij})$Log-likelihood of correct ranking. Maximizing this = maximizing the probability that positive items outscore negatives

Step 2: Expand for MF

Score difference for MF
$$\hat{x}_{uij} = \hat{r}_{ui} - \hat{r}_{uj} = \mathbf{u}_u^\top \mathbf{v}_i - \mathbf{u}_u^\top \mathbf{v}_j = \mathbf{u}_u^\top (\mathbf{v}_i - \mathbf{v}_j)$$

User biases cancel (they appear in both scores). Item biases are typically dropped in BPR-MF for simplicity.

Step 3: Compute gradients via chain rule

Chain rule: gradient of log-sigmoid
$$\frac{\partial}{\partial \Theta} \ln \sigma(\hat{x}_{uij}) = \frac{\sigma'(\hat{x}_{uij})}{\sigma(\hat{x}_{uij})} \cdot \frac{\partial \hat{x}_{uij}}{\partial \Theta}$$ $$\text{Using } \sigma'(x) = \sigma(x)(1 - \sigma(x)) \text{ and } 1 - \sigma(x) = \sigma(-x):$$ $$= (1 - \sigma(\hat{x}_{uij})) \cdot \frac{\partial \hat{x}_{uij}}{\partial \Theta} = \sigma(-\hat{x}_{uij}) \cdot \frac{\partial \hat{x}_{uij}}{\partial \Theta}$$
$\sigma(-\hat{x}_{uij})$The "surprise factor." When the model already ranks correctly ($\hat{x} \gg 0$): $\sigma(-\hat{x}) \approx 0$ → tiny gradient (don't waste updates on easy cases). When the model ranks WRONG ($\hat{x} < 0$): $\sigma(-\hat{x}) \approx 1$ → full gradient (fix this mistake!). This is BPR's built-in curriculum: it focuses learning on hard cases

Step 4: Per-parameter gradients and update rules

BPR-MF SGD updates
$$\mathbf{u}_u \leftarrow \mathbf{u}_u + \eta \left[\sigma(-\hat{x}_{uij}) \cdot (\mathbf{v}_i - \mathbf{v}_j) - \lambda \, \mathbf{u}_u\right]$$ $$\mathbf{v}_i \leftarrow \mathbf{v}_i + \eta \left[\sigma(-\hat{x}_{uij}) \cdot \mathbf{u}_u - \lambda \, \mathbf{v}_i\right]$$ $$\mathbf{v}_j \leftarrow \mathbf{v}_j + \eta \left[-\sigma(-\hat{x}_{uij}) \cdot \mathbf{u}_u - \lambda \, \mathbf{v}_j\right]$$
$\mathbf{v}_i - \mathbf{v}_j$Direction from negative to positive item. User embedding moves TOWARD the positive item and AWAY from the negative
$+\mathbf{u}_u$ for $\mathbf{v}_i$Positive item moves toward the user — "be more aligned with this user's taste"
$-\mathbf{u}_u$ for $\mathbf{v}_j$Negative item moves AWAY from the user — "be less aligned with this user"
D2 / WORKED
One BPR training step — every number
↳ corresponds to M16 S2
BPR step: (Priya, Saree, Running Shoes), k=2
Step by step
SETUP
$$\mathbf{u}_\text{Priya} = [0.5, 0.3], \quad \mathbf{v}_\text{Saree} = [0.8, 0.2], \quad \mathbf{v}_\text{Shoes} = [0.1, 0.6]$$ $$\eta = 0.05, \quad \lambda = 0.01$$
COMPUTE SCORES
$$\hat{r}_\text{Priya,Saree} = 0.5 \times 0.8 + 0.3 \times 0.2 = 0.40 + 0.06 = 0.46$$ $$\hat{r}_\text{Priya,Shoes} = 0.5 \times 0.1 + 0.3 \times 0.6 = 0.05 + 0.18 = 0.23$$ $$\hat{x}_{uij} = 0.46 - 0.23 = 0.23 \quad \text{(positive — correct ranking!)}$$
COMPUTE SIGMOID FACTOR
$$\sigma(-\hat{x}_{uij}) = \sigma(-0.23) = \frac{1}{1 + e^{0.23}} = \frac{1}{1 + 1.259} = \frac{1}{2.259} = 0.443$$

$\sigma(-0.23) = 0.443$ — not close to 0 or 1. The model is slightly correct but not confident. Moderate gradient — worthwhile update.

UPDATE ALL THREE EMBEDDINGS

User (Priya):

$$\mathbf{u} \leftarrow [0.5, 0.3] + 0.05 \times [0.443 \times ([0.8, 0.2] - [0.1, 0.6]) - 0.01 \times [0.5, 0.3]]$$ $$= [0.5, 0.3] + 0.05 \times [0.443 \times [0.7, -0.4] - [0.005, 0.003]]$$ $$= [0.5, 0.3] + 0.05 \times [0.310, -0.177] - [0.005, 0.003]]$$ $$= [0.5, 0.3] + 0.05 \times [0.305, -0.180] = [0.5, 0.3] + [0.0153, -0.0090]$$ $$= [0.515, 0.291]$$

Priya's embedding moved toward the Saree and away from the Shoes in latent space.

Positive item (Saree):

$$\mathbf{v}_\text{Saree} \leftarrow [0.8, 0.2] + 0.05 \times [0.443 \times [0.5, 0.3] - 0.01 \times [0.8, 0.2]]$$ $$= [0.8, 0.2] + 0.05 \times [0.222, 0.133] - [0.008, 0.002]]$$ $$= [0.8, 0.2] + [0.0107, 0.0066] = [0.811, 0.207]$$

Negative item (Shoes):

$$\mathbf{v}_\text{Shoes} \leftarrow [0.1, 0.6] + 0.05 \times [-0.443 \times [0.5, 0.3] - 0.01 \times [0.1, 0.6]]$$ $$= [0.1, 0.6] + 0.05 \times [-0.222, -0.133] - [0.001, 0.006]]$$ $$= [0.1, 0.6] + [-0.0111, -0.0070] = [0.089, 0.593]$$

Shoes moved AWAY from Priya — correctly, since she didn't click them.

VERIFY: NEW SCORE DIFFERENCE
$$\hat{r}_\text{new,Saree} = 0.515 \times 0.811 + 0.291 \times 0.207 = 0.418 + 0.060 = 0.478$$ $$\hat{r}_\text{new,Shoes} = 0.515 \times 0.089 + 0.291 \times 0.593 = 0.046 + 0.173 = 0.219$$ $$\hat{x}_\text{new} = 0.478 - 0.219 = 0.259 \quad (\text{was } 0.23 \text{ → increased})$$

Score gap increased from 0.23 to 0.259. One BPR step made the model more confident in the correct ranking. After millions of steps, the gap becomes large and consistent across all users.

BPR-MF — complete PyTorch implementation

# ═══ BPR-MF — complete, production-structured ═══
import torch, torch.nn as nn, numpy as np

class BPRMF(nn.Module):
    def __init__(self, n_users, n_items, k=128):
        super().__init__()
        self.user_emb = nn.Embedding(n_users, k)
        self.item_emb = nn.Embedding(n_items, k)
        nn.init.normal_(self.user_emb.weight, std=0.01)
        nn.init.normal_(self.item_emb.weight, std=0.01)

    def forward(self, users, pos_items, neg_items):
        u = self.user_emb(users)              # (batch, k)
        p = self.item_emb(pos_items)           # (batch, k)
        n = self.item_emb(neg_items)           # (batch, k)
        pos_score = (u * p).sum(dim=1)        # (batch,)
        neg_score = (u * n).sum(dim=1)        # (batch,)
        return pos_score, neg_score

def bpr_loss(pos_score, neg_score):
    """BPR loss: -log σ(pos - neg)"""
    return -torch.log(torch.sigmoid(pos_score - neg_score) + 1e-10).mean()

def sample_negatives(user_items, n_items, method='uniform'):
    """Sample negative item for each user (not in their history)."""
    neg = np.random.randint(0, n_items, len(user_items))
    # Rejection: resample if negative is in user's history
    for idx, (user, pos) in enumerate(user_items):
        while neg[idx] in user_histories[user]:
            neg[idx] = np.random.randint(0, n_items)

Interview problems — BPR

Problem 1: When does BPR NOT learn?
Interview
PROBLEM

If $\hat{x}_{uij} = 5.0$ (model very confidently ranks positive above negative), what is $\sigma(-5.0)$? What is the gradient magnitude? What does this mean for training efficiency?

SOLUTION
$$\sigma(-5.0) = \frac{1}{1+e^{5}} = \frac{1}{1+148.4} = 0.0067$$

The gradient is multiplied by 0.0067 — essentially zero. The model is already correct and confident, so the update is negligible. This is BPR's built-in curriculum: it automatically focuses on the cases where the model is wrong or uncertain ($\sigma(-\hat{x}) \approx 0.5$) and ignores easy cases ($\sigma(-\hat{x}) \approx 0$). This is also why uniform negative sampling is wasteful — most random negatives are "easy" (large $\hat{x}$, near-zero gradient).

Problem 2: BPR vs MSE on implicit data
Interview
PROBLEM

Two models are trained on Kartify's 18M implicit interactions. Model A uses MSE loss (target=1 for observed, 0 for sampled negatives). Model B uses BPR loss. Both achieve similar training loss. Which likely has better nDCG@10? Why?

SOLUTION

Model B (BPR) wins on nDCG. MSE trains the model to predict $\hat{r} \approx 1$ for positives and $\hat{r} \approx 0$ for negatives — it learns absolute scores. But nDCG only cares about relative ordering. MSE might assign scores [0.8, 0.7, 0.6] to three positives and [0.2, 0.3, 0.4] to negatives — good separation but NO learning about which positive is best.

BPR directly optimizes relative ordering. It doesn't care about absolute scores — only that positives beat negatives. Over training, it also implicitly learns to rank positives against each other (through the transitive structure of the embeddings). This alignment between training objective and evaluation metric is why BPR consistently outperforms MSE on ranking metrics.
