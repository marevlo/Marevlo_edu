D1 / DERIVATION
Biased MF loss — every variable, every gradient
↳ corresponds to M15 S2: Biased MF
Real-life analogy

Imagine predicting exam scores. The final score has four components: (1) the average score across ALL students and ALL exams (μ ≈ 70). (2) Student bias — Alice always scores +10 above average. (3) Exam difficulty — this exam is 5 points harder than average. (4) Personal fit — Alice is specifically good at this type of exam. Biased MF separates these four effects, so each can be learned independently.

Biased MF prediction
$$\hat{r}_{ui} = \mu + b_u + b_i + \mathbf{u}_u^\top \mathbf{v}_i$$
$\mu$Global mean rating — computed once from training data, fixed during training. Kartify: $\mu = 3.8$
$b_u$User bias — scalar per user. Captures "Priya rates 0.3 above average on everything." Initialized to 0, learned by SGD
$b_i$Item bias — scalar per item. Captures "this saree is rated 0.5 above average by everyone." Initialized to 0, learned by SGD
$\mathbf{u}_u^\top \mathbf{v}_i$Latent interaction — the PERSONAL match between Priya's taste and this saree's character. This is the only personalized component. Without biases, the dot product would also have to capture $\mu$, $b_u$, $b_i$ — wasting capacity
Loss function (regularized MSE)
$$\mathcal{L} = \sum_{(u,i) \in \Omega} \underbrace{\left(r_{ui} - \mu - b_u - b_i - \mathbf{u}_u^\top \mathbf{v}_i\right)^2}_{\text{prediction error}} + \lambda \underbrace{\left(b_u^2 + b_i^2 + \|\mathbf{u}_u\|^2 + \|\mathbf{v}_i\|^2\right)}_{\text{regularization}}$$

All five SGD update rules

Let $e_{ui} = r_{ui} - \hat{r}_{ui}$ (prediction error). Each parameter moves in the direction that reduces the loss:

SGD updates (one observed rating at a time)
$$b_u \leftarrow b_u + \eta \left(e_{ui} - \lambda \cdot b_u\right)$$ $$b_i \leftarrow b_i + \eta \left(e_{ui} - \lambda \cdot b_i\right)$$ $$\mathbf{u}_u \leftarrow \mathbf{u}_u + \eta \left(e_{ui} \cdot \mathbf{v}_i - \lambda \cdot \mathbf{u}_u\right)$$ $$\mathbf{v}_i \leftarrow \mathbf{v}_i + \eta \left(e_{ui} \cdot \mathbf{u}_u - \lambda \cdot \mathbf{v}_i\right)$$
$e_{ui}$Positive error → prediction too low → all parameters increase. Negative error → prediction too high → all parameters decrease
$e_{ui} \cdot \mathbf{v}_i$Gradient for user embedding: if error is positive (under-predicted), move $\mathbf{u}_u$ toward $\mathbf{v}_i$. This increases the dot product → higher prediction next time
$-\lambda \cdot (\cdot)$Regularization pull: always shrinks parameters toward zero. Prevents any single bias or embedding dimension from dominating

SETUP
$$R = \begin{bmatrix} 5 & ? & 3 \\ 4 & 1 & ? \end{bmatrix} \quad \mu = \frac{5+3+4+1}{4} = 3.25$$

$k=2$, $\eta=0.01$, $\lambda=0.02$. Initialize:

$$b_\text{Alice}=0, \; b_\text{Bob}=0, \; b_A=0, \; b_B=0, \; b_C=0$$ $$\mathbf{u}_\text{Alice}=[0.1, 0.2], \; \mathbf{u}_\text{Bob}=[-0.1, 0.3]$$ $$\mathbf{v}_A=[0.2, -0.1], \; \mathbf{v}_B=[0.1, 0.4], \; \mathbf{v}_C=[-0.2, 0.1]$$
STEP 1: (ALICE, MOVIE A, r=5)

Predict:

$$\hat{r} = 3.25 + 0 + 0 + (0.1)(0.2) + (0.2)(-0.1) = 3.25 + 0.02 - 0.02 = 3.25$$

Error: $e = 5 - 3.25 = 1.75$

Update biases:

$$b_\text{Alice} = 0 + 0.01(1.75 - 0.02 \times 0) = 0.0175$$ $$b_A = 0 + 0.01(1.75 - 0.02 \times 0) = 0.0175$$

Update embeddings:

$$\mathbf{u}_\text{Alice} = [0.1, 0.2] + 0.01(1.75 \times [0.2, -0.1] - 0.02 \times [0.1, 0.2])$$ $$= [0.1, 0.2] + 0.01([0.35, -0.175] - [0.002, 0.004])$$ $$= [0.1, 0.2] + [0.00348, -0.00179] = [0.10348, 0.19821]$$
$$\mathbf{v}_A = [0.2, -0.1] + 0.01(1.75 \times [0.1, 0.2] - 0.02 \times [0.2, -0.1])$$ $$= [0.2, -0.1] + 0.01([0.175, 0.35] - [0.004, -0.002])$$ $$= [0.2, -0.1] + [0.00171, 0.00352] = [0.20171, -0.09648]$$

After step 1: biases and embeddings have moved to reduce the error. Next prediction for (Alice, A) will be higher than 3.25.

STEP 2: (ALICE, MOVIE C, r=3)
$$\hat{r} = 3.25 + 0.0175 + 0 + (0.10348)(-0.2) + (0.19821)(0.1)$$ $$= 3.2675 + (-0.02070) + 0.01982 = 3.2666$$ $$e = 3 - 3.2666 = -0.2666$$

Negative error — prediction was too high! Updates will slightly reduce biases and push embeddings apart:

$$b_\text{Alice} = 0.0175 + 0.01(-0.2666 - 0.02 \times 0.0175) = 0.0175 - 0.00267 = 0.01483$$ $$b_C = 0 + 0.01(-0.2666) = -0.00267$$

Alice's bias decreased slightly (she over-predicted C), and Movie C got a negative bias (it's below average).

AFTER FULL EPOCH (ALL 4 OBSERVATIONS)

After processing (Alice,A)=5, (Alice,C)=3, (Bob,A)=4, (Bob,B)=1, the model has learned initial values for all biases and embeddings. Key observations:

• $b_A > 0$ (Movie A is above average — both users rated it high)

• $b_B < 0$ (Movie B is below average — Bob gave it 1 star)

• $b_\text{Alice} > b_\text{Bob}$ (Alice rates higher on average: mean 4.0 vs 2.5)

After 50+ epochs, the model converges: predictions closely match observed ratings AND produce reasonable values for the missing entries (Alice-B and Bob-C). The missing entry predictions are the recommendations.

D3 / FROM SCRATCH
Biased MF — complete PyTorch implementation
↳ corresponds to M15 S3 & S5
# ═══ BIASED MF — PyTorch, production-ready structure ═══
import torch
import torch.nn as nn

class BiasedMF(nn.Module):
    def __init__(self, n_users, n_items, k=128, global_mean=0.0):
        super().__init__()
        self.mu = nn.Parameter(torch.tensor(global_mean))
        self.user_bias = nn.Embedding(n_users, 1)
        self.item_bias = nn.Embedding(n_items, 1)
        self.user_emb = nn.Embedding(n_users, k)
        self.item_emb = nn.Embedding(n_items, k)
        # Small random init — critical for convergence
        nn.init.normal_(self.user_emb.weight, std=0.01)
        nn.init.normal_(self.item_emb.weight, std=0.01)
        nn.init.zeros_(self.user_bias.weight)
        nn.init.zeros_(self.item_bias.weight)

    def forward(self, user_ids, item_ids):
        u = self.user_emb(user_ids)            # (batch, k)
        v = self.item_emb(item_ids)            # (batch, k)
        dot = (u * v).sum(dim=1)               # (batch,)
        bu = self.user_bias(user_ids).squeeze() # (batch,)
        bi = self.item_bias(item_ids).squeeze() # (batch,)
        return self.mu + bu + bi + dot

# ═══ TRAINING LOOP ═══
def train_biased_mf(model, train_data, epochs=20, lr=0.005, reg=0.02):
    optimizer = torch.optim.SGD(model.parameters(), lr=lr, weight_decay=reg)
    for epoch in range(epochs):
        total_loss = 0
        for users, items, ratings in train_data:  # batched DataLoader
            preds = model(users, items)
            loss = ((ratings - preds) ** 2).mean()
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
            total_loss += loss.item()
        print(f"Epoch {epoch+1}: loss={total_loss:.4f}")

# ═══ RECOMMEND ═══
def recommend(model, user_id, seen_items, top_k=10):
    with torch.no_grad():
        all_items = torch.arange(model.item_emb.num_embeddings)
        user = torch.full_like(all_items, user_id)
        scores = model(user, all_items)
        scores[seen_items] = float('-inf')
        return torch.topk(scores, top_k).indices.tolist()

Problem 1: Bias decomposition
Interview
PROBLEM

A movie gets predicted rating 4.5 for a user. The global mean is 3.3, the user bias is +0.5, and the item bias is +0.9. What is the latent dot product contribution? What happens if you remove biases from the model entirely?

SOLUTION
$$\mathbf{u}^\top \mathbf{v} = 4.5 - 3.3 - 0.5 - 0.9 = -0.2$$

The latent dot product is actually negative (−0.2). The high prediction comes entirely from biases — the user rates everything high (+0.5), the movie is universally liked (+0.9). The dot product says "this user is actually slightly LESS aligned with this movie than average, but the biases still make it a 4.5."

Without biases: the dot product alone would need to produce 4.5. But it would also need to produce high values for ALL movies this generous user rated — wasting latent capacity on capturing $\mu$ and $b_u$ instead of taste nuances. Biases free the dot product to focus on personalization.

Problem 2: Regularization trade-off
Interview
PROBLEM

You train biased MF with $\lambda = 0$ (no regularization) on Kartify's 18M interactions. Training RMSE = 0.01 (nearly perfect). Test RMSE = 1.8. Diagnose and fix.

SOLUTION

Classic overfitting. With $\lambda = 0$, the model memorizes every observed rating perfectly (train RMSE ≈ 0) but learns nothing generalizable. The embeddings grow unboundedly large — a user with 5 interactions has their embedding fitted to exactly 5 points, capturing noise, not signal.

Fix: Add regularization. Start with $\lambda = 0.02$. Cross-validate on held-out nDCG@10. Typical search: $\lambda \in \{0.001, 0.005, 0.01, 0.02, 0.05, 0.1\}$. At Kartify's sparsity (7.8 interactions/user), stronger regularization ($\lambda \approx 0.05$) is usually needed — the fewer observations per user, the more the model must rely on the prior (embeddings near zero) rather than the data.
