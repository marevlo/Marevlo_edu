D1 / DERIVATION
Sample size formula — derived from scratch
↳ corresponds to M29 S3: Power analysis
Real-life analogy

Imagine you suspect a coin is slightly biased — heads comes up 52% instead of 50%. To prove this, you could flip it 10 times (too noisy — you might see 6 heads by chance even with a fair coin) or 10,000 times (clearly enough — a fair coin almost never produces 5,200 heads out of 10,000). The sample size formula tells you the MINIMUM number of flips needed to distinguish a 52% coin from a 50% coin with confidence. In A/B testing, the "coin" is whether a user converts, and the "bias" is the model's true effect.

Two-sample z-test for means
$$n = \frac{(z_{\alpha/2} + z_\beta)^2 \cdot 2\sigma^2}{\delta^2}$$
$n$Required sample size PER GROUP (control and treatment each need $n$ users)
$z_{\alpha/2}$Critical value for significance level. $\alpha = 0.05 \implies z_{0.025} = 1.96$. This controls the false positive rate: how often you'd declare a winner when there's no real difference
$z_\beta$Critical value for power. Power $= 1 - \beta = 0.80 \implies z_{0.20} = 0.84$. This controls the false negative rate: how often you'd miss a real effect
$\sigma^2$Variance of the metric (e.g., variance of GMV per user). Higher variance → need more users to see through the noise
$\delta$Minimum Detectable Effect (MDE): the smallest improvement you want to detect. $\delta = \text{baseline} \times \text{relative\_lift}$. Smaller effects need more users
D2 / WORKED
Kartify power analysis — every number
↳ corresponds to M29 S3
How many users to detect 2% GMV lift?
Every step
INPUTS
$$\text{Baseline GMV/user/week: } \mu = ₹350$$ $$\text{Standard deviation: } \sigma = ₹500 \text{ (GMV is heavy-tailed)}$$ $$\text{Desired MDE: } 2\% \text{ relative lift} \implies \delta = 0.02 \times 350 = ₹7$$ $$\alpha = 0.05, \quad \text{Power} = 0.80$$
CALCULATION
$$n = \frac{(1.96 + 0.84)^2 \times 2 \times 500^2}{7^2} = \frac{(2.80)^2 \times 500{,}000}{49}$$ $$= \frac{7.84 \times 500{,}000}{49} = \frac{3{,}920{,}000}{49} = 80{,}000 \text{ users per group}$$
IMPLICATIONS

Total users needed: 160,000 (80K control + 80K treatment).

At 10M DAU: Assign 1.6% of traffic to the experiment. Duration: 1 week minimum (capture weekly patterns).

What if you want 1% MDE instead of 2%? $\delta = ₹3.5$, so $n = 80{,}000 \times (7/3.5)^2 = 80{,}000 \times 4 = 320{,}000$ per group. 4× more users needed to detect a 2× smaller effect. This is why tiny improvements are hard to validate online.

Why interleaving needs 10× fewer users
$$n_{\text{interleaving}} = \frac{(z_{\alpha/2} + z_\beta)^2 \cdot 2\sigma^2_{\text{within}}}{\delta^2_{\text{pref}}}$$
$\sigma^2_{\text{within}}$Within-user variance of preference signals. Much smaller than between-user variance $\sigma^2$ because each user serves as their own control. Typical: $\sigma_{\text{within}} \approx 0.3 \times \sigma$
Ratio$n_{\text{AB}} / n_{\text{interleaving}} \approx (\sigma / \sigma_{\text{within}})^2 \approx (1/0.3)^2 \approx 11\times$. Interleaving needs ~10× fewer users because it eliminates the between-user noise that dominates A/B test variance

Interview problems — experimentation

Problem: The variance reduction trick
Interview
PROBLEM

GMV variance is $\sigma = ₹500$, requiring 80K users. A colleague suggests: "use GMV minus the user's historical average as the metric instead of raw GMV." Why does this help, and by how much?

SOLUTION

This is CUPED (Controlled-experiment Using Pre-Experiment Data). The adjusted metric is: $Y_i^{\text{adj}} = Y_i - \theta \hat{Y}_i^{\text{pre}}$, where $\hat{Y}_i^{\text{pre}}$ is the user's pre-experiment average and $\theta$ is the correlation coefficient.

If the user's historical GMV explains 50% of their current GMV ($R^2 = 0.5$), the variance of the adjusted metric is: $\sigma^2_{\text{adj}} = \sigma^2 (1 - R^2) = 500^2 \times 0.5 = 125{,}000$. The required sample size: $n = 80{,}000 \times 0.5 = 40{,}000$. 50% variance reduction → 50% sample size reduction. This is why every mature experimentation platform uses CUPED — it makes tests 2× faster for free.

Problem: Multiple metric testing
Interview
PROBLEM

Your A/B test checks 5 metrics: GMV, CTR, CVR, D7 retention, and basket size. You use p < 0.05 for each. What's the probability of at least one false positive if there's actually no real effect?

SOLUTION
$$P(\text{at least 1 false positive}) = 1 - (1 - 0.05)^5 = 1 - 0.95^5 = 1 - 0.774 = 0.226$$

22.6% chance of a false positive — not 5%. This is the multiple comparisons problem. Fix: use Bonferroni correction ($\alpha_{\text{adjusted}} = 0.05/5 = 0.01$ per metric) or designate ONE primary metric for the ship decision and treat others as directional only. Kartify uses GMV as the sole primary metric (p < 0.05) and all others as secondary (directional, no p-value threshold for ship decision).
