Every zero is
a question mark.

BPR samples random negatives and trains on pairs. WRMF takes a radically different approach: train on EVERY user-item pair — all 1.1 trillion of them — but weight them by confidence. Observed interactions get high confidence. Unobserved pairs get low confidence. The absence of a click isn't evidence of dislike — it's uncertainty. This module introduces the second major implicit-feedback loss and the Alternating Least Squares solver that makes it tractable.

01 / THE GAP

Spotify's "Discover Weekly" playlist — the feature that arguably saved the company — was originally powered by WRMF (called "implicit ALS" internally). The key insight that made it work: a user listening to a song 50 times is a much stronger signal than listening once, but not 50× stronger. WRMF's confidence weighting — $c_{ui} = 1 + \alpha \cdot r_{ui}$ — captures exactly this: each additional play adds evidence, but with diminishing marginal confidence. The same principle applies to Kartify: a user viewing an item 5 times is more interested than someone who viewed it once, but the relationship is sublinear.

What BPR doesn't use — the zeros
BPR learns from positive-vs-negative pairs. It ignores the vast sea of zeros entirely. What if the zeros contain signal?

BPR's training data is (user, positive_item, negative_item). The negative is sampled randomly from the unobserved set — it represents "we assume you don't like this." But Module 03 taught us that an unobserved interaction could mean many things: never saw it, saw it and wasn't interested, saw it and bookmarked it for later, or genuinely disliked it. BPR treats all of these the same — as negatives.

WRMF (Hu, Koren, Volinsky 2008) takes the opposite approach: don't sample negatives. Use ALL unobserved pairs as training data — but with low confidence. Every pair in the 2.3M × 480K matrix becomes a training example. The observed interactions say "target = 1, high confidence." The unobserved pairs say "target = 0, low confidence — we're not sure."

Pause — recall Module 03

In DEEP M03, we derived the confidence formula: c_ui = 1 + α · r_ui. If Priya clicked a saree 3 times (r = 3) and α = 40, what's the confidence? What about an item she never saw?

Saree: $c = 1 + 40 \times 3 = 121$. Unseen item: $c = 1 + 40 \times 0 = 1$. The model is 121× more certain about the saree than about the unseen item. WRMF encodes this asymmetry directly into the loss function — the gradient from the saree pair is 121× stronger than the gradient from the unseen pair.
02 / THE APPROACH
Train on everything — weight by confidence
The WRMF loss: same MF architecture, different training philosophy.
# WRMF in one equation:
# minimize Σ c_ui · (p_ui − u·v)² + λ(‖U‖² + ‖V‖²)
#   over ALL (u, i) pairs — observed AND unobserved
#
# p_ui = 1 if observed, 0 if unobserved    (binary preference)
# c_ui = 1 + α · count                     (confidence)
#
# The key insight: unobserved pairs have target=0 with confidence=1
# They WEAKLY pull the prediction toward 0
# Observed pairs STRONGLY pull the prediction toward 1

The beauty of this formulation: it respects the asymmetry of implicit data. A click is a strong positive signal — the user chose to interact. But an absence of click is a weak negative signal — maybe they never saw the item. WRMF encodes this directly: high-confidence positives and low-confidence negatives. The model learns to distinguish "probably likes" from "uncertain" rather than "likes" from "dislikes."

The full-matrix view

BPR sees the matrix as "observed entries + random negatives." WRMF sees the ENTIRE matrix — every one of the 1.1 trillion cells — as a training signal, just with different confidence levels. This complete view gives WRMF access to structural patterns that random sampling misses: if a user has zero interactions with an entire category, WRMF's many low-confidence zeros in that category push the user's embedding away from it. BPR wouldn't discover this unless it happened to sample negatives from that category.

03 / THE SOLVER
ALS — why WRMF can't use SGD
Training on all 1.1 trillion pairs with SGD would take forever. ALS solves each user's embeddings in closed form.

BPR trains with SGD — sample a triple, compute gradient, update. This works because BPR only needs one sampled negative per step. WRMF needs ALL negatives per step — the loss sums over every unobserved pair. With 480K items per user, that's 480K terms in every user's gradient. SGD on one triple at a time would take billions of steps.

Alternating Least Squares (ALS) solves this by alternating between two closed-form updates:

Step 1: Fix items. For each user $u$, solve for the optimal embedding $\mathbf{u}_u$ that minimizes the WRMF loss — given fixed item embeddings. This has a closed-form solution (it's a weighted least squares problem). Solve for all 2.3M users in parallel.

Step 2: Fix users. For each item $i$, solve for the optimal embedding $\mathbf{v}_i$ — given fixed user embeddings. Same closed-form. Solve for all 480K items in parallel.

Repeat for 15–20 iterations. Each iteration is guaranteed to reduce the loss (it's solving an exact subproblem). Convergence is fast — typically 10–15 iterations suffice.

Spotify's scale

Spotify's music recommendation was powered by implicit ALS (WRMF) running on Apache Spark. With 100M+ users and 80M+ tracks, the ALS solver distributed the per-user updates across thousands of cores. Each user's update is independent — embarrassingly parallel. This is the architectural reason ALS dominates for large-scale implicit CF: the per-user solve is independent, making it trivially distributable.

04 / THE CHOICE
BPR vs WRMF — when to use which
Two algorithms for the same problem. Different strengths.

Choose BPR when: you care most about ranking quality (nDCG, MAP). You have a GPU. You want embeddings that work well with ANN retrieval. You want a simple training loop. BPR is the modern default for most recsys teams.

Choose WRMF when: you need the full matrix completion view (predicting scores for every pair, not just top-K). You have a large CPU cluster (Spark). You want deterministic convergence (ALS doesn't need learning rate tuning). You need confidence weighting for multi-signal implicit data (clicks + purchases with different weights).

In practice: many teams run both. BPR-MF for the primary retrieval channel (better nDCG). WRMF for secondary channels like "complete your look" or "items similar to X" (better at reconstructing the full interaction pattern).

Kartify decision

Primary retrieval: BPR-MF (M16) — optimized for ranking, GPU-trainable, ANN-serveable.
"Complete your look" module: WRMF — multi-signal confidence (click=1, cart=3, purchase=5), better at predicting cross-category co-purchases because the full-matrix view captures category-level patterns from the mass of zeros.

After this module, you understand both implicit-feedback MF approaches:

WRMF: Train on ALL pairs (observed + unobserved) with confidence weighting. c_ui = 1 + α·r_ui. Respects the "zero ≠ dislike" asymmetry from Module 03.
ALS solver: Alternating Least Squares — fix items, solve users in closed form, then vice versa. Embarrassingly parallel. No learning rate tuning.
BPR vs WRMF: BPR wins on ranking (nDCG). WRMF wins on full-matrix completion and multi-signal confidence weighting. Use both.
Kartify uses both: BPR-MF for homepage retrieval. WRMF for "Complete Your Look" cross-category recommendations.
