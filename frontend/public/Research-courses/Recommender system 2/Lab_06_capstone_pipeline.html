Part 01
Data & Evaluation
MovieLens 100K — temporal split — nDCG@10 harness

We use the MovieLens 100K dataset with a temporal 80/20 train/test split on implicit positive interactions (ratings ≥ 4.0). The evaluation harness wraps any recommend_fn(user_id) → [item_ids] function and returns nDCG@10, Precision@10, and Recall@10 averaged over up to 500 test users.

Why temporal split

A random split leaks future data into training. A temporal split simulates the real deployment scenario: the model trains on past interactions and is evaluated on future ones — so it cannot cheat by seeing items the user rated after the split point.

def evaluate(recommend_fn, test_dict, k=10, n_eval=500, label=""):
    ndcgs, precs, recalls = [], [], []
    for u in list(test_dict.keys())[:n_eval]:
        recs      = recommend_fn(u)[:k]
        rel       = test_dict[u]
        relevance = [1.0 if i in rel else 0.0 for i in recs]
        dcg       = sum(r / np.log2(p + 2) for p, r in enumerate(relevance))
        ideal     = sum(1.0 / np.log2(p + 2) for p in range(min(k, len(rel))))
        ndcgs.append(dcg / ideal if ideal > 0 else 0)
        precs.append(sum(relevance) / k)
        recalls.append(sum(relevance) / len(rel) if rel else 0)
    return {"nDCG@10": np.mean(ndcgs), "P@10": np.mean(precs), "R@10": np.mean(recalls)}
Part 02
Stage 1 · Retrieval
Three candidate generators — Popularity, Item-kNN, BPR-MF

Retrieval is a scalability-first stage: it must score all N items and return a manageable candidate set (top-100) quickly. We evaluate three strategies from this course before wiring the best one into the full pipeline.

Lab_06_capstone_pipeline.py — Stage 1 models
def popularity_model(R_train):
    pop = np.array(R_train.sum(axis=0)).flatten()
    def recommend(u, k=100):
        scores = pop.copy()
        seen   = set(R_train[u].nonzero()[1])
        for s in seen: scores[s] = -np.inf
        return np.argsort(scores)[-k:][::-1]
    return recommend


def item_knn_model(R_train, shrinkage=100, n_neighbors=50):
    # cosine similarity with shrinkage from Lab 03
    RtR    = (R_train.T @ R_train).toarray()
    norms  = np.sqrt(np.array(R_train.power(2).sum(axis=0)).flatten())
    cosine = RtR / np.outer(np.maximum(norms, 1e-8), np.maximum(norms, 1e-8))
    sim    = cosine * (RtR / (RtR + shrinkage))   # shrinkage-damped
    np.fill_diagonal(sim, 0)
    def recommend(u, k=100):
        interacted         = np.where(R_train[u].toarray().flatten() > 0)[0]
        scores             = sim[interacted].sum(axis=0)
        scores[interacted] = -np.inf
        return np.argsort(scores)[-k:][::-1]
    return recommend


# BPR-MF trained for 15 epochs (see Lab 04 for full training code)
def bpr_fn(u, k=100):
    scores = bpr_model.predict_all(u)
    seen   = set(R_train[u].nonzero()[1])
    for s in seen: scores[s] = -np.inf
    return np.argsort(scores)[-k:][::-1]

Part 03
Stage 2 · Feature Reranking
Blend retrieval scores with side signals to sharpen the top-10

The ranker receives the top-100 candidates from BPR-MF and reorders them using a weighted combination of three signals: the BPR retrieval score (normalised), item popularity, and a position-decay term. This is a simplified version of the linear rankers used in production before gradient-boosted or neural point-wise rankers.

score(u, i) = 0.6 · bpr_norm(u,i) + 0.25 · pop_norm(i) + 0.15 · pos_decay(rank)
where bpr_norm and pop_norm are min-max scaled to [0, 1] over the candidate set
Lab_06_capstone_pipeline.py — build_ranker()
def build_ranker(R_train, model):
    item_pop = np.array(R_train.sum(axis=0)).flatten().astype(float)
    item_pop = item_pop / item_pop.max()   # normalise to [0, 1]

    def rerank(user_id, candidates, k=20):
        # retrieval score for each candidate
        ret_scores = model.predict_all(user_id)[candidates]
        ret_norm   = (ret_scores - ret_scores.min()) \
                   / (ret_scores.max() - ret_scores.min() + 1e-10)

        pop_scores  = item_pop[candidates]
        rank_scores = np.linspace(1.0, 0.0, len(candidates))   # position decay

        combined   = 0.6 * ret_norm + 0.25 * pop_scores + 0.15 * rank_scores
        top_k_idx  = np.argsort(combined)[-k:][::-1]
        return candidates[top_k_idx]

    return rerank


# wire into eval:
def ranked_fn(u, k=10):
    candidates = bpr_fn(u, k=100)
    return ranker(u, candidates, k=k)

Part 04
Stage 3 · MMR Diversity Reranking
Maximal Marginal Relevance — Module 27 in production

After ranking, the top-30 candidates can still be highly redundant — five nearly identical action movies, for instance. Maximal Marginal Relevance (MMR) iteratively selects the item that best balances relevance against similarity to already-selected items. The parameter λ controls the trade-off: λ = 1.0 means pure relevance, λ = 0.0 means pure diversity.

MMR(i) = λ · relevance(i) − (1 − λ) · max_{s ∈ S} sim(i, s)
S = already selected items · sim = cosine similarity in embedding space · λ = 0.7
Lab_06_capstone_pipeline.py — mmr_rerank()
def mmr_rerank(items, item_embeddings, k=10, lam=0.7):
    if len(items) <= k:
        return items
    selected  = [items[0]]         # always keep the top-ranked item
    remaining = list(items[1:])
    while len(selected) < k and remaining:
        best_score, best_idx = -np.inf, 0
        for idx, item in enumerate(remaining):
            relevance = 1.0 - (idx / len(remaining))   # position-based proxy
            item_vec  = item_embeddings[item]
            max_sim   = max(
                np.dot(item_vec, item_embeddings[s])
                / (np.linalg.norm(item_vec) * np.linalg.norm(item_embeddings[s]) + 1e-10)
                for s in selected
            )
            mmr = lam * relevance - (1 - lam) * max_sim
            if mmr > best_score:
                best_score, best_idx = mmr, idx
        selected.append(remaining.pop(best_idx))
    return np.array(selected)
λ intuition

At λ = 0.7 we keep 70% weight on relevance — diversity is a nudge, not a takeover. Users notice more variety without feeling their preferences are being ignored. Lower λ suits editorial "explore" shelves; higher λ suits "because you watched X" shelves.

Part 05
Full Pipeline
Three stages wired into a single recommend function

The full pipeline is remarkably concise once each stage is implemented: retrieve 100 candidates, rank down to 30, then MMR-rerank to the final top-10. The entire function is 4 lines. Real production systems wrap these stages in async services with caching, but the logic is identical.

Lab_06_capstone_pipeline.py — full_pipeline()
def full_pipeline(user_id, retrieval_fn, ranker_fn, item_embeddings, k=10):
    candidates = retrieval_fn(user_id, k=100)    # Stage 1: retrieve top-100
    ranked     = ranker_fn(user_id, candidates, k=30)  # Stage 2: rerank to top-30
    return mmr_rerank(ranked, item_embeddings, k=k, lam=0.7)  # Stage 3: diversity
