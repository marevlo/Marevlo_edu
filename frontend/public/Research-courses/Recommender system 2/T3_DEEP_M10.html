D1 / DERIVATION
Four similarity metrics — derived and compared
↳ corresponds to M10 S2: Three Metrics, Three Intuitions
1. Cosine similarity
$$\cos(\mathbf{a}, \mathbf{b}) = \frac{\mathbf{a} \cdot \mathbf{b}}{\|\mathbf{a}\| \cdot \|\mathbf{b}\|} = \frac{\sum_{u} a_u \cdot b_u}{\sqrt{\sum_u a_u^2} \cdot \sqrt{\sum_u b_u^2}}$$
Range$[-1, 1]$ for general vectors. $[0, 1]$ for non-negative vectors (implicit data)
NormalizesBy vector MAGNITUDE — a popular item (big norm) and a niche item (small norm) can still have high cosine
Best forBinary implicit data. The default choice for modern recsys
2. Jaccard similarity
$$J(\mathbf{a}, \mathbf{b}) = \frac{|U_a \cap U_b|}{|U_a \cup U_b|} = \frac{\text{users who bought BOTH}}{\text{users who bought EITHER}}$$
Range$[0, 1]$. 0 = no overlap, 1 = identical user sets
NormalizesBy SET UNION — penalizes pairs where one item has many more users than the other
Key diff vs cosineJaccard ignores frequency — if a user bought item A 10 times, Jaccard still counts them once. Cosine would weight them more
3. Pearson correlation
$$\rho(\mathbf{a}, \mathbf{b}) = \frac{\sum_{u \in U_{ab}}(a_u - \bar{a})(b_u - \bar{b})}{\sqrt{\sum_{u \in U_{ab}}(a_u - \bar{a})^2} \cdot \sqrt{\sum_{u \in U_{ab}}(b_u - \bar{b})^2}}$$
$\bar{a}, \bar{b}$Mean rating of items $a$ and $b$ (across co-rating users only)
$a_u - \bar{a}$Deviation from ITEM mean. "Did this user rate item $a$ above or below its average?"
$U_{ab}$Users who rated BOTH items. Pearson only uses co-rated pairs — empty cells are ignored
Best forExplicit ratings with item-level bias. Adjusts for "easy" vs "hard" items
4. Adjusted cosine
$$\text{adj\_cos}(\mathbf{a}, \mathbf{b}) = \frac{\sum_{u \in U_{ab}}(r_{ua} - \bar{r}_u)(r_{ub} - \bar{r}_u)}{\sqrt{\sum_u(r_{ua} - \bar{r}_u)^2} \cdot \sqrt{\sum_u(r_{ub} - \bar{r}_u)^2}}$$
$\bar{r}_u$Mean rating of USER $u$ (across all items they rated). Key difference from Pearson: Pearson subtracts the ITEM mean; adjusted cosine subtracts the USER mean
Why USER mean?Adjusts for generous vs harsh raters. If Alice rates everything 4+, her "3" is a strong negative signal. Subtracting her mean (4.2) makes the "3" become -1.2, revealing the true sentiment
Best forItem-item similarity on explicit ratings. The standard for item-based CF with ratings data
D2 / WORKED
All four metrics on the same 5×4 matrix
↳ corresponds to M10 S3: See the Difference Live
Compute cosine, Jaccard, Pearson, adjusted cosine for pair (A, B)
Side by side
MATRIX (EXPLICIT RATINGS)
$$R = \begin{bmatrix} 5 & 3 & ? & ? \\ 4 & ? & 2 & ? \\ ? & 4 & 5 & 3 \\ 3 & 5 & ? & 4 \\ ? & ? & 4 & 2 \end{bmatrix}$$

Items A, B, C, D. Users 1–5. "?" = unrated.

Focus: sim(A, B). Co-raters of A and B: Users 1 and 4.

COSINE(A, B)
$$\mathbf{a} = (5, 4, ?, 3, ?)^\top, \quad \mathbf{b} = (3, ?, 4, 5, ?)^\top$$ $$\text{Co-rated only: } \mathbf{a}' = (5, 3), \quad \mathbf{b}' = (3, 5)$$ $$\cos = \frac{5 \times 3 + 3 \times 5}{\sqrt{25+9} \times \sqrt{9+25}} = \frac{15 + 15}{\sqrt{34} \times \sqrt{34}} = \frac{30}{34} = 0.882$$
JACCARD(A, B)
$$U_A = \{1, 2, 4\}, \quad U_B = \{1, 3, 4\}$$ $$|U_A \cap U_B| = |\{1, 4\}| = 2, \quad |U_A \cup U_B| = |\{1, 2, 3, 4\}| = 4$$ $$J(A, B) = \frac{2}{4} = 0.500$$

Lower than cosine because Jaccard penalizes the non-shared users (2 and 3).

PEARSON(A, B)

Item means (over co-raters 1 and 4 only): $\bar{a} = (5+3)/2 = 4.0$, $\bar{b} = (3+5)/2 = 4.0$.

$$\rho = \frac{(5-4)(3-4) + (3-4)(5-4)}{\sqrt{(5-4)^2+(3-4)^2} \times \sqrt{(3-4)^2+(5-4)^2}}$$ $$= \frac{(1)(-1) + (-1)(1)}{\sqrt{1+1} \times \sqrt{1+1}} = \frac{-1 + (-1)}{\sqrt{2} \times \sqrt{2}} = \frac{-2}{2} = -1.000$$

Pearson = −1.0! User 1 rates A high and B low; User 4 rates A low and B high. They have OPPOSITE rating patterns → perfectly negative correlation. This is information cosine misses entirely (cosine saw 0.882).

ADJUSTED COSINE(A, B)

User means: $\bar{r}_1 = (5+3)/2 = 4.0$ (over items 1 rated). $\bar{r}_4 = (3+5+4)/3 = 4.0$ (over all items 4 rated).

$$\text{adj\_cos} = \frac{(5-4)(3-4) + (3-4)(5-4)}{\sqrt{(5-4)^2+(3-4)^2} \times \sqrt{(3-4)^2+(5-4)^2}} = \frac{-2}{2} = -1.000$$

Same result as Pearson here because user means happen to equal item means. They diverge when users have different baseline generosity levels (shown in D3).

SUMMARY
$$\text{Cosine} = 0.882 \quad \text{Jaccard} = 0.500 \quad \text{Pearson} = -1.000 \quad \text{Adj. Cosine} = -1.000$$

Cosine says A and B are highly similar (0.88). Pearson says they're perfectly ANTI-correlated (−1.0). Who's right? Pearson — the co-raters give them opposite ratings. Cosine is misled because it doesn't account for the direction of deviations. This is why Pearson/adjusted cosine is necessary for explicit ratings.

D3 / COMPARISON
Pearson vs adjusted cosine — when they diverge
↳ corresponds to M10 S4: Decision Framework

Pearson subtracts the item mean: "is this rating above or below this item's average?" It corrects for easy vs hard items.

Adjusted cosine subtracts the user mean: "is this rating above or below this user's personal baseline?" It corrects for generous vs harsh raters.

They give the same result when all users have the same mean rating. They diverge when users have different baselines — which is the common case with explicit ratings.

The decision rule

User-based CF → Pearson (compares users, subtract item means to remove item difficulty).
Item-based CF → Adjusted cosine (compares items, subtract user means to remove user generosity).
Implicit binary data → Plain cosine (no means to subtract — everything is 0 or 1).

D4 / INTERVIEW
Interview problems — similarity metrics
↳ tests M10 concepts
Problem 1: Metric mismatch
Interview
PROBLEM

An engineer uses cosine similarity for item-based CF on a movie rating dataset (1–5 stars). Two movies are rated [5, 1] and [1, 5] by two users. Cosine gives sim = 0.385. Is this a good similarity estimate? What metric should be used instead?

SOLUTION
$$\cos = \frac{5 \times 1 + 1 \times 5}{\sqrt{26} \times \sqrt{26}} = \frac{10}{26} = 0.385$$

Cosine says 0.385 — moderate similarity. But these movies have OPPOSITE ratings from both users! User 1 loves movie A and hates movie B; User 2 loves B and hates A. They should be negatively correlated.

$$\text{Pearson} = \frac{(5-3)(1-3) + (1-3)(5-3)}{2 \times 2} = \frac{-4 + (-4)}{4} = -2.0 / (\ldots) = -1.0$$

Pearson correctly identifies them as perfectly anti-correlated (−1.0). Use adjusted cosine or Pearson for explicit ratings. Cosine on raw ratings is misleading because it treats all non-zero values as "positive" — a 1-star rating and a 5-star rating both contribute positively to the dot product.

Problem 2: Jaccard vs cosine for "also bought"
Interview
PROBLEM

Item X has 10,000 purchases. Item Y has 50 purchases. 40 users bought both. Compute Jaccard and cosine. Which is more appropriate for an "also bought" widget?

SOLUTION
$$\text{Jaccard} = \frac{40}{10{,}000 + 50 - 40} = \frac{40}{10{,}010} = 0.004$$ $$\text{Cosine} = \frac{40}{\sqrt{10{,}000} \times \sqrt{50}} = \frac{40}{100 \times 7.07} = \frac{40}{707} = 0.057$$

Both are low, but cosine is 14× higher than Jaccard. Why? Jaccard's denominator is the union (10,010 users) — it heavily penalizes the popularity mismatch. Cosine's denominator uses geometric mean of norms — less sensitive to size differences.

For "also bought": conditional probability is better than both. $P(Y|X) = 40/10{,}000 = 0.4\%$ (given X was bought, Y was also bought 0.4% of the time). $P(X|Y) = 40/50 = 80\%$ (given Y was bought, X was also bought 80% of the time). The direction matters — showing X alongside Y makes more sense than showing Y alongside X.
