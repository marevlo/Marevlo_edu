LightGCN —
graph propagation computed by hand.

Module 23 introduced graph-based recommendation. This companion derives LightGCN's propagation rule, constructs the normalized adjacency matrix, works through 2 layers of propagation on a tiny 3-user × 3-item graph showing every multiplication, and provides a from-scratch PyTorch implementation.

D1 / DERIVATION
LightGCN propagation rule — derived
↳ corresponds to M23 S3: LightGCN
Real-life analogy

Imagine a room of people. Each person holds a card with a number (their initial "embedding"). In round 1, everyone replaces their card with the average of their neighbors' cards. In round 2, they do it again — but now each neighbor's card already absorbed THEIR neighbors' info from round 1. After 3 rounds, everyone's card reflects information from 3 hops away. LightGCN does exactly this on the user-item graph, then combines the cards from all rounds for the final representation.

LightGCN layer propagation
$$\mathbf{e}_u^{(l+1)} = \sum_{i \in \mathcal{N}_u} \frac{1}{\sqrt{|\mathcal{N}_u|}\sqrt{|\mathcal{N}_i|}} \mathbf{e}_i^{(l)}$$ $$\mathbf{e}_i^{(l+1)} = \sum_{u \in \mathcal{N}_i} \frac{1}{\sqrt{|\mathcal{N}_i|}\sqrt{|\mathcal{N}_u|}} \mathbf{e}_u^{(l)}$$
$\mathbf{e}_u^{(l)}$User $u$'s embedding at layer $l$. Layer 0 = initialized embeddings (same as MF). Layer $l$ = embeddings after $l$ rounds of propagation
$\mathcal{N}_u$Items that user $u$ interacted with (neighbors in the graph)
$\frac{1}{\sqrt{|\mathcal{N}_u|}\sqrt{|\mathcal{N}_i|}}$Symmetric normalization — prevents high-degree nodes (popular items, active users) from dominating. The factor $1/\sqrt{d_u \cdot d_i}$ ensures the aggregated signal has similar magnitude regardless of node degree
Matrix form
$$E^{(l+1)} = \tilde{A} \, E^{(l)} \quad \text{where} \quad \tilde{A} = D^{-1/2} A \, D^{-1/2}$$
$A$Adjacency matrix of the bipartite graph: $A = \begin{bmatrix} 0 & R \\ R^\top & 0 \end{bmatrix}$ where $R$ is the interaction matrix. Size: $(M+N) \times (M+N)$
$D$Degree matrix: diagonal, $D_{ii} = $ number of edges connected to node $i$
$\tilde{A}$Normalized adjacency. The symmetric normalization $D^{-1/2} A D^{-1/2}$ is the same normalization used in spectral graph theory — it ensures the propagation is a proper graph filter
Final embedding: multi-scale combination
$$\mathbf{e}_u = \frac{1}{K+1}\sum_{l=0}^{K} \mathbf{e}_u^{(l)}$$
$K$Number of propagation layers. Typical: $K = 3$. More layers = more hops, but also more over-smoothing risk (all embeddings converge to the same value)
Why average?Layer 0 captures the raw ID signal (same as MF). Layer 1 captures 1-hop neighborhood. Layer 2 captures 2-hop structure. Averaging all layers gives a multi-scale representation — both local identity and global graph position. This is what gives LightGCN its edge over MF
D2 / WORKED
2 layers of propagation on a 3×3 graph — every multiplication
↳ corresponds to M23 S2 & S3
LightGCN on 3 users × 3 items, K=2, k=2
Every step
THE GRAPH
$$R = \begin{bmatrix} 1 & 1 & 0 \\ 0 & 1 & 1 \\ 1 & 0 & 1 \end{bmatrix} \quad \text{Alice→{A,B}, Bob→{B,C}, Carol→{A,C}}$$

Degrees: Alice=2, Bob=2, Carol=2, A=2, B=2, C=2. (All degree 2 — symmetric graph.)

Normalization weight for any edge: $\frac{1}{\sqrt{2}\sqrt{2}} = \frac{1}{2} = 0.5$

LAYER 0 (INITIAL EMBEDDINGS)
$$\mathbf{e}^{(0)}_\text{Alice} = [1.0, 0.0], \; \mathbf{e}^{(0)}_\text{Bob} = [0.0, 1.0], \; \mathbf{e}^{(0)}_\text{Carol} = [0.5, 0.5]$$ $$\mathbf{e}^{(0)}_A = [0.8, 0.2], \; \mathbf{e}^{(0)}_B = [0.3, 0.7], \; \mathbf{e}^{(0)}_C = [0.6, 0.4]$$
LAYER 1 PROPAGATION

Users absorb from their items:

$$\mathbf{e}^{(1)}_\text{Alice} = 0.5 \times \mathbf{e}^{(0)}_A + 0.5 \times \mathbf{e}^{(0)}_B = 0.5[0.8, 0.2] + 0.5[0.3, 0.7] = [0.55, 0.45]$$ $$\mathbf{e}^{(1)}_\text{Bob} = 0.5 \times \mathbf{e}^{(0)}_B + 0.5 \times \mathbf{e}^{(0)}_C = 0.5[0.3, 0.7] + 0.5[0.6, 0.4] = [0.45, 0.55]$$ $$\mathbf{e}^{(1)}_\text{Carol} = 0.5 \times \mathbf{e}^{(0)}_A + 0.5 \times \mathbf{e}^{(0)}_C = 0.5[0.8, 0.2] + 0.5[0.6, 0.4] = [0.70, 0.30]$$

Items absorb from their users:

$$\mathbf{e}^{(1)}_A = 0.5 \times \mathbf{e}^{(0)}_\text{Alice} + 0.5 \times \mathbf{e}^{(0)}_\text{Carol} = 0.5[1.0, 0.0] + 0.5[0.5, 0.5] = [0.75, 0.25]$$ $$\mathbf{e}^{(1)}_B = 0.5 \times \mathbf{e}^{(0)}_\text{Alice} + 0.5 \times \mathbf{e}^{(0)}_\text{Bob} = 0.5[1.0, 0.0] + 0.5[0.0, 1.0] = [0.50, 0.50]$$ $$\mathbf{e}^{(1)}_C = 0.5 \times \mathbf{e}^{(0)}_\text{Bob} + 0.5 \times \mathbf{e}^{(0)}_\text{Carol} = 0.5[0.0, 1.0] + 0.5[0.5, 0.5] = [0.25, 0.75]$$
LAYER 2 PROPAGATION

Users absorb layer-1 item embeddings:

$$\mathbf{e}^{(2)}_\text{Alice} = 0.5 \times \mathbf{e}^{(1)}_A + 0.5 \times \mathbf{e}^{(1)}_B = 0.5[0.75, 0.25] + 0.5[0.50, 0.50] = [0.625, 0.375]$$

Alice's layer-2 embedding now contains 2-hop information: it reflects not just items A and B directly, but also the preferences of Carol (who shares item A) and Bob (who shares item B).

FINAL EMBEDDINGS (AVERAGE LAYERS 0-2)
$$\mathbf{e}_\text{Alice} = \frac{1}{3}([1.0, 0.0] + [0.55, 0.45] + [0.625, 0.375]) = \frac{1}{3}[2.175, 0.825] = [0.725, 0.275]$$

Predict Alice → item C:

$$\mathbf{e}_C = \frac{1}{3}([0.6, 0.4] + [0.25, 0.75] + \ldots) \approx [0.35, 0.60]$$ $$\text{score(Alice, C)} = [0.725, 0.275] \cdot [0.35, 0.60] = 0.254 + 0.165 = 0.419$$

Compared to MF (layer 0 only): $[1.0, 0.0] \cdot [0.6, 0.4] = 0.60$. The graph propagation changed the score because Alice's embedding now reflects her neighbors' preferences, and item C's embedding now reflects its users' tastes. The graph structure added information that raw MF embeddings didn't capture.

Problem 1: Over-smoothing
Interview
PROBLEM

What happens if you set K=20 (20 propagation layers)? Why does LightGCN typically use K=3?

SOLUTION

Over-smoothing: Each propagation layer averages a node's embedding with its neighbors. After many layers, all nodes' embeddings converge to the same value — the graph's "average signal." With K=20, every user embedding ≈ every other user embedding. The model loses the ability to distinguish individual users → nDCG drops to near-random.

Mathematically: $\tilde{A}^K$ converges to a rank-1 matrix as $K \to \infty$ (for connected graphs). All rows of $\tilde{A}^K E^{(0)}$ become identical → all embeddings collapse.

K=3 is the sweet spot because: K=1 adds 1-hop info (direct neighbors), K=2 adds 2-hop (friends-of-friends), K=3 adds 3-hop (structural communities). Beyond 3 hops, most useful collaborative structure has already been captured, and each additional layer adds more noise (over-smoothing) than signal.

Problem 2: LightGCN vs NGCF
Interview
PROBLEM

NGCF (the predecessor to LightGCN) uses trainable weight matrices $W_l$ and LeakyReLU activations at each layer. LightGCN removes both. Why does removing parameters IMPROVE performance?

SOLUTION

In node classification, each node has rich input features (text, images, attributes). Weight matrices transform these features into useful representations — essential for extracting signal from complex inputs. In recommendation, the only input per node is an ID embedding that's already learnable. Adding a weight transform on top of a learnable embedding is redundant — the model can simply adjust the embedding directly. The weight matrix adds parameters without adding representational power, increasing overfitting risk.

LeakyReLU is harmful because the embeddings need to combine additively across layers (the multi-scale average). Non-linear activations break this linear combination, making it harder for the model to benefit from the multi-layer averaging that gives LightGCN its power.

The lesson: more parameters ≠ better model. LightGCN's success is an example of "simplicity wins" — removing unnecessary complexity reveals the core mechanism that actually helps (graph smoothing).
