Part 01

Matrix Factorization Architecture

From M14 — decompose R ≈ U·V^T into k-dimensional user and item embeddings.

Matrix factorization learns a k-dimensional vector for each user and each item. The predicted rating (or relevance score) for user u and item i is their dot product: u⃗ · i⃗. During training, these vectors adjust so that pairs the user liked score higher than pairs they didn't.

Lab_04 · BPR_MF model
import torch.nn as nn

class BPR_MF(nn.Module):
    def __init__(self, n_users, n_items, k=64):
        super().__init__()
        self.user_emb = nn.Embedding(n_users, k)
        self.item_emb = nn.Embedding(n_items, k)
        # Small random init: prevents dead gradients early in training
        nn.init.normal_(self.user_emb.weight, std=0.01)
        nn.init.normal_(self.item_emb.weight, std=0.01)

    def forward(self, users, pos_items, neg_items):
        u     = self.user_emb(users)         # (batch, k)
        i_pos = self.item_emb(pos_items)     # (batch, k)
        i_neg = self.item_emb(neg_items)     # (batch, k)
        # x_uij = score(u,i+) - score(u,i-) = preference margin
        pos_scores = (u * i_pos).sum(dim=1)  # (batch,)
        neg_scores = (u * i_neg).sum(dim=1)  # (batch,)
        return pos_scores - neg_scores        # x_uij

    def predict(self, user_idx):
        """Score all items for a given user (inference)."""
        u      = self.user_emb.weight[user_idx]   # (k,)
        scores = self.item_emb.weight @ u          # (n_items,)
        return scores.detach().cpu().numpy()
Part 02

BPR Loss — Ranking Directly

From DEEP M16 — optimize P(user prefers i over j) instead of predicting ratings.

Instead of predicting exact ratings, BPR trains the model to rank liked items above unseen items. For each observed (user, item+) interaction, a random negative item is sampled, and the model is penalized if it doesn't score item+ higher.

BPR OBJECTIVE (from DEEP M16)
P(u prefers i over j) = σ(x_uij)
where   x_uij = u·i⁺ − u·i⁻   (preference margin)
Loss = −Σ log σ(x_uij) + λ‖Θ‖²
Lab_04 · BPR loss computation
x_uij = model(batch_users, batch_pos, batch_neg)
# x_uij > 0: model correctly ranks positive above negative
# x_uij < 0: model is wrong — gradient pushes it to improve

loss = -torch.log(torch.sigmoid(x_uij) + 1e-10).mean()
# = -log σ(x_uij) averaged over batch
# Minimizing this pushes σ(x_uij) → 1, i.e. x_uij → +∞
Part 03

Training Loop with Negative Sampling

Mini-batch SGD with Adam. For each positive, sample one random negative.
Lab_04 · training loop (simplified)
optimizer = torch.optim.Adam(model.parameters(), lr=lr, weight_decay=reg)

for epoch in range(epochs):
    model.train()
    perm = np.random.permutation(len(train_interactions))

    for start in range(0, len(perm), batch_size):
        batch_idx = perm[start:start + batch_size]
        batch_users = torch.LongTensor(users_arr[batch_idx])
        batch_pos   = torch.LongTensor(items_arr[batch_idx])

        # Sample uniform negatives (items NOT in user's history)
        neg_items = []
        for idx in batch_idx:
            u = users_arr[idx]
            neg = np.random.randint(n_items)
            while neg in user_items[u]:     # rejection sampling
                neg = np.random.randint(n_items)
            neg_items.append(neg)
        batch_neg = torch.LongTensor(neg_items)

        x_uij = model(batch_users, batch_pos, batch_neg)
        loss   = -torch.log(torch.sigmoid(x_uij) + 1e-10).mean()

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
💡 Why rejection sampling for negatives?

Sampling a positive item as a negative would give the model contradictory gradients: "rank item A above item A." The while loop ensures negatives are genuinely unobserved items for that user.

Hyperparameter Search

Larger k (more dimensions) gives the model more expressiveness but risks overfitting. Higher reg shrinks embeddings toward zero, acting as L2 regularization. Lower lr is safer but trains slower. Expect k=128 with moderate reg to win on ML-100K.
